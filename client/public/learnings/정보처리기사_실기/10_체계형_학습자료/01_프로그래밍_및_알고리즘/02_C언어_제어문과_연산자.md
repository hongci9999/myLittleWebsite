# C언어 제어문과 연산자

## 핵심 요약

실기 C언어의 절반은 "코드가 어떤 순서로 실행되는가"를 묻는 문제다. 이때 핵심은 연산자 우선순위, 전치/후치 증감, 조건식 판별, 반복문의 종료 시점을 정확히 잡는 것이다.

## 왜 중요한가

- `if`, `for`, `while` 자체는 쉽지만, 증감 연산과 함께 나오면 오답이 많다.
- 문제는 대개 "몇 번 반복되는가", "최종 출력은 무엇인가", "변수 값은 어떻게 변하는가"를 묻는다.
- 연산자 우선순위를 잘못 해석하면 한 줄 전체를 틀릴 수 있다.

## 1. 연산자 우선순위 핵심

대략적인 흐름은 다음 순서로 이해하면 된다.

1. 괄호 `()`
2. 단항 연산자 `++`, `--`, `!`
3. 산술 연산 `*`, `/`, `%`, `+`, `-`
4. 관계 연산 `<`, `>`, `<=`, `>=`
5. 동등 비교 `==`, `!=`
6. 논리 연산 `&&`, `||`
7. 대입 연산 `=`, `+=`

## 2. 전치/후치 증감 연산

| 형태 | 의미 |
|---|---|
| `++a` | 먼저 증가, 그다음 사용 |
| `a++` | 먼저 사용, 그다음 증가 |
| `--a` | 먼저 감소, 그다음 사용 |
| `a--` | 먼저 사용, 그다음 감소 |

```c
int a = 5;
int b = ++a;  // a=6, b=6
int c = a++;  // c=6, a=7
```

## 3. 조건문

### 3.1 `if`

```c
if (score >= 60) {
    printf("PASS");
} else {
    printf("FAIL");
}
```

### 3.2 `switch`

- 조건식 결과가 정수형/문자형일 때 자주 사용된다.
- `break`가 없으면 아래 `case`로 계속 내려간다.

```c
switch (a) {
    case 1: printf("one"); break;
    case 2: printf("two"); break;
    default: printf("other");
}
```

## 4. 반복문

### 4.1 `for`

```c
for (i = 0; i < 3; i++) {
    printf("%d ", i);
}
```

- 초기식: 한 번 실행
- 조건식: 반복할지 검사
- 증감식: 본문 실행 후 수행

### 4.2 `while`

```c
while (i < 3) {
    printf("%d ", i);
    i++;
}
```

### 4.3 `do while`

- 조건을 나중에 검사하므로 최소 1번은 실행된다.

## 5. `break`와 `continue`

- `break` : 반복문 자체 종료
- `continue` : 이번 회차만 건너뛰고 다음 반복 진행

```c
for (i = 1; i <= 5; i++) {
    if (i == 3) continue;
    if (i == 5) break;
    printf("%d ", i);
}
```

- 출력: `1 2 4`

## 6. 코드 추적 예제

### 예제 1: 전치/후치와 산술 결합

```c
int a = 2, b = 3;
int c = a++ + ++b;
printf("%d %d %d\n", a, b, c);
```

- `a++` -> 먼저 2를 사용, 이후 `a`는 3
- `++b` -> 먼저 4로 증가, 그 값을 사용
- `c = 2 + 4 = 6`
- 최종 값: `a=3, b=4, c=6`

### 예제 2: 반복문 추적

```c
int i, sum = 0;
for (i = 1; i <= 4; i++) {
    if (i % 2 == 0) sum += i;
}
printf("%d\n", sum);
```

- `i=1` -> 홀수, 더하지 않음
- `i=2` -> 짝수, `sum=2`
- `i=3` -> 홀수, 그대로
- `i=4` -> 짝수, `sum=6`
- 출력: `6`

### 예제 3: `switch` 함정

```c
int a = 2;
switch (a) {
    case 1: printf("A");
    case 2: printf("B");
    case 3: printf("C");
    default: printf("D");
}
```

- `case 2`부터 시작
- `break`가 없으므로 아래로 계속 실행
- 출력: `BCD`

## 7. 자주 틀리는 함정

### 7.1 `=` 와 `==`

- `=`는 대입, `==`는 비교다.
- 시험에서는 눈으로 빨리 읽다가 헷갈리기 쉽다.

### 7.2 반복 횟수 착각

- `for (i = 0; i < 5; i++)` 는 0,1,2,3,4 총 5번 반복이다.
- `<` 와 `<=` 를 구분해야 한다.

### 7.3 `continue` 후 나머지 코드 미실행

- `continue`를 만나면 그 아래 문장은 그 회차에서 실행되지 않는다.

### 7.4 `do while` 최소 1회 실행

- 조건이 처음부터 거짓이어도 본문이 한 번 실행된다.

## 8. 시험장에서 푸는 순서

1. 변수 초기값을 적는다.
2. 조건문의 참/거짓을 먼저 판별한다.
3. 반복문은 회차별로 표를 만들어 추적한다.
4. 증감 연산은 "사용 시점"과 "변경 시점"을 분리해서 본다.
5. `break`, `continue`, `switch`의 `break` 유무를 마지막에 다시 확인한다.

## 한 페이지 요약

- 연산자 우선순위와 증감 연산은 항상 같이 점검한다.
- 반복문 문제는 실행 순서를 표로 쓰면 실수가 줄어든다.
- `switch`에서 `break`가 없으면 다음 `case`로 계속 진행한다.
- `continue`는 이번 회차만 건너뛴다.
- `do while`은 최소 1회 실행된다는 점을 기억한다.
