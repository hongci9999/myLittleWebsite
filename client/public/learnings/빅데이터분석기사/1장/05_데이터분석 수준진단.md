# 데이터 분석 수준진단 (Data Analysis Maturity & Readiness Diagnosis)

## 📌 핵심 요약
조직의 현재 데이터 분석 역량을 객관적으로 파악하기 위해 **분석 준비도(Readiness)**와 **분석 성숙도(Maturity)**를 진단하고, 그 결과에 따라 미래의 목표 수준을 정의합니다. 6개 영역의 준비도 요소와 4단계의 성숙도 모델을 결합한 사분면 분석을 통해 조직의 분석 유형을 분류하고 맞춤형 전략을 수립합니다.

---

## 📝 상세 정리

### 1. 데이터 분석 수준진단 개요
- **목적:** 조직이 수행하고 있는 데이터 분석 수준을 이해하고, 진단 결과를 바탕으로 미래의 목표 수준을 정의함.
- **방법:** 분석 수준진단 프레임워크를 이용해 **6개 영역의 분석 준비도**와 **3개 영역의 분석 성숙도**를 동시에 평가함.

### 2. 분석 준비도 (Analysis Readiness)
조직의 데이터 분석 활용 준비가 어느 정도 되어 있는지 측정합니다. 일정 수준 충족 시 업무에 도입하며, 미달 시에는 데이터 분석 환경을 우선적으로 조성해야 합니다.

| 진단 영역 | 세부 항목 (Key Indicators) |
| :--- | :--- |
| **분석 업무 (Analysis Task)** | 발생한 사실 분석 업무, 예측 분석 업무, 시뮬레이션 분석 업무, 최적화 분석 업무, 분석 업무의 정기적 개선 |
| **인력 및 조직 (Personnel & Organization)** | 분석 전문가 존재, 전문가 교육 훈련 프로그램, 관리자들의 기본적인 분석 능력, 전사 분석 업무 총괄 조직(CoE) 존재, 경영진의 분석 업무 이해 능력 |
| **분석 기법 (Analysis Technique)** | 업무별 적합한 분석 기법 사용, 분석 업무 도입 방법론, 분석 기법 라이브러리(Library), 분석 기법 효과성 평가, 분석 기법의 정기적 개선 |
| **분석 데이터 (Analysis Data)** | 분석 업무를 위한 데이터 충분성·신뢰성·적시성, 비구조적 데이터 관리, 외부 데이터 활용 체계, 마스터 데이터 관리(MDM) |
| **분석 문화 (Analysis Culture)** | 사실(Fact)에 근거한 의사결정, 관리자의 데이터 중시, 회의 등에서의 데이터 활용, 경영진의 데이터 중시, 데이터 공유 및 협업 문화 |
| **분석 인프라 (Analysis Infrastructure)** | 운영 시스템 데이터 통합, 데이터 유통 체계, 분석 전용 서버 및 스토리지, 빅데이터 분석 환경, 통계 분석 환경, 비주얼 분석(Visual Analytics) 환경 |

### 3. 분석 성숙도 (Analysis Maturity)
조직의 데이터 분석 역량이 얼마나 체계적으로 발전했는지 단계별로 점검합니다. 데이터 수집, 분석, 소비 능력이 성숙도를 결정합니다.

| 단계 | 특징 및 정의 | 비즈니스 부문 (Business) | 조직·역량 부문 (Organization) | IT 부문 (IT) |
| :--- | :--- | :--- | :--- | :--- |
| **1단계: 도입 (Introduction)** | 환경 및 시스템 구축 시작 | 실적 분석 통계, 정기 보고 수행, 운영 데이터 기반 | 일부 부서 및 담당자 의존 | DW(Data Warehouse), DM(Data Mart), ETL, EAI |
| **2단계: 활용 (Utilization)** | 분석 결과를 실제 업무에 적용 | 미래 결과 예측, 시뮬레이션, 운영 데이터 기반 | 전문 담당 부서 존재, 분석 기법 도입, 관리자가 직접 수행 | 실시간 대시보드, 통계 분석 환경 |
| **3단계: 확산 (Diffusion)** | 전사 차원에서 분석 관리 및 공유 | 전사 성과 실시간 분석, 프로세스 혁신, 분석 규칙 관리, 이벤트 관리 | 모든 부서 수행, 분석 CoE(Center of Excellence) 조직, 데이터 사이언티스트 확보 | 빅데이터 관리 환경, 최적화, 비주얼 분석, 분석 전용 서버 |
| **4단계: 최적화 (Optimization)** | 혁신 및 성과 향상에 직접 기여 | 외부 환경 분석 활용, 최적화 업무 적용, 실시간 분석, 비즈니스 모델 진화 | 데이터 사이언스 그룹 운영, 경영진의 분석 활용, 전략 연계 | 분석 협업 환경, 샌드박스(Sandbox), 프로세스 내재화, 빅데이터 분석 고도화 |

### 4. 사분면 분석 (Quadrant Analysis)
분석 **준비도(가로축)**와 **성숙도(세로축)**를 결합하여 조직의 상태를 4가지 유형으로 분류합니다.

- **준비형 (Ready):** 준비도는 높으나 성숙도가 낮은 상태. 인프라는 갖춰졌으나 실제 업무 활용도가 낮음.
- **정착형 (Settled):** 준비도는 낮으나 성숙도가 높은 상태. 인프라 등은 부족하지만 특정 부서 중심으로 분석을 활발히 수행함.
- **도입형 (Introductory):** 준비도와 성숙도 모두 낮은 상태. 분석을 위한 기초 역량 배양이 우선임.
- **확산형 (Diffused):** 준비도와 성숙도 모두 높은 상태. 전사적으로 데이터 분석이 잘 정착된 이상적인 형태.

> [!INFO] 💡 AI 보충 설명
> **1. CoE (Center of Excellence):** 조직 내에서 데이터 분석을 총괄하는 '분석 전문가 조직'을 의미합니다. 데이터 거버넌스를 수립하고 각 부서의 분석 업무를 지원하는 핵심 역할을 합니다.
> **2. MDM (Master Data Management):** 기업 내의 핵심 데이터(고객, 제품 등)를 단일한 마스터 리스트로 관리하여 데이터의 일관성과 정확성을 유지하는 체계입니다.
> **3. ETL / EAI:** > - **ETL (Extract, Transform, Load):** 데이터를 추출, 변형, 적재하는 공정입니다.
> - **EAI (Enterprise Application Integration):** 기업 내 상이한 애플리케이션 간의 데이터 연동 체계입니다.
> **4. 분석 샌드박스 (Sandbox):** 분석가들이 자유롭게 데이터를 실험하고 모델을 개발할 수 있도록 마련된 격리된 분석 환경을 의미합니다.
