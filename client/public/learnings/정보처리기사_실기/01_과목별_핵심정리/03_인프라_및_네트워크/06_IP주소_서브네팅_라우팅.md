# IP 주소, 서브네팅, 라우팅

## 핵심 요약

네트워크 계산형의 핵심은 서브네팅이다. 여기에 IPv4/IPv6 특징, 주소 클래스, NAT, CIDR, ARP, ICMP를 함께 정리하면 문제 대부분을 커버할 수 있다.

## 1. IP 관련 핵심 프로토콜

| 프로토콜 | 역할 |
|---|---|
| IP | 패킷 전달 |
| ARP | IP -> MAC 변환 |
| RARP | MAC -> IP 변환 |
| ICMP | 오류 보고, 상태 진단 |

## 2. IPv4와 IPv6

| 항목 | IPv4 | IPv6 |
|---|---|---|
| 주소 길이 | 32비트 | 128비트 |
| 표기 | 점으로 구분된 10진수 | 콜론으로 구분된 16진수 |
| 특징 | 주소 부족 문제 | 주소 공간 확장 |

## 3. 주소 부족 해결

- CIDR: 클래스 구분 없이 유연하게 주소 할당
- NAT: 사설 IP를 공인 IP로 변환
- DHCP: 자동으로 IP 할당

## 4. 서브네팅 기본

- 서브넷을 나누면 네트워크를 더 작은 단위로 분리할 수 있다.
- 사용 가능한 호스트 수 = `2^(호스트 비트 수) - 2`

## 5. 계산 핵심

### 예: `/27`

- 호스트 비트 수 = 5
- 총 주소 수 = `2^5 = 32`
- 사용 가능 호스트 수 = `32 - 2 = 30`

### 블록 크기

- 마스크 마지막 옥텟이 224라면
- `256 - 224 = 32`
- 즉 32개 단위로 주소 블록이 증가

## 6. FLSM과 VLSM

| 방식 | 설명 |
|---|---|
| FLSM | 동일 크기 서브넷 분할 |
| VLSM | 필요한 크기별로 가변 분할 |

VLSM은 큰 그룹부터 먼저 할당하는 것이 기본 원칙이다.

## 7. 라우팅 기본

- 라우팅은 패킷의 최적 경로를 결정하는 과정이다.
- 라우터는 네트워크 계층 장비다.
- TTL은 패킷이 네트워크에서 무한 루프를 돌지 않도록 수명을 제한한다.

## 8. 예제

### 예제 1: `/24`를 6개 서브넷으로 분할

- 6개 이상 필요 -> 3비트 빌림 (`2^3 = 8`)
- `/24` -> `/27`
- 서브넷당 사용 가능 호스트 수 = 30

### 예제 2: VLSM

- 60대 필요 -> `/26` (62호스트 가능)
- 30대 필요 -> `/27` (30호스트 가능)
- 큰 그룹부터 먼저 배치

## 9. 자주 틀리는 함정

- 호스트 수 공식에서 `-2` 를 빼지 않는다
- 16대 필요할 때 `/28` 로 착각한다
- `/27` 의 블록 크기를 틀린다
- ARP와 RARP 방향을 반대로 외운다

## 10. 시험장에서 푸는 순서

1. 주소/프로토콜/계산 문제 중 무엇인지 구분한다.
2. 서브네팅이면 필요한 호스트 수를 먼저 본다.
3. 호스트 비트 수를 정한 뒤 `/표기`를 계산한다.
4. `2^n - 2` 로 사용 가능한 호스트 수를 확인한다.
5. VLSM은 큰 네트워크부터 할당한다.

## 한 페이지 요약

- IPv4는 32비트, IPv6는 128비트다.
- ARP는 IP를 MAC으로 변환한다.
- 사용 가능한 호스트 수는 `2^n - 2`다.
- FLSM은 고정 길이, VLSM은 가변 길이 분할이다.
- `/27` 은 총 32개 주소, 사용 가능 30개다.
