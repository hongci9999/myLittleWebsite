# 자료구조 및 알고리즘

## 핵심 요약
**[[자료구조]]**는 데이터의 효율적인 접근과 수정을 가능하게 하는 조직화 방법이며, **[[알고리즘]]**은 문제를 해결하기 위한 절차이다. 선형 구조([[스택]], [[큐]], [[데크]], 리스트)와 비선형 구조([[트리]], [[그래프]])의 특징 및 구현 방식, 그리고 효율적인 설계를 위한 알고리즘 기법을 학습함.

---

## 1. 자료구조 개요

### 1) 정의 및 목적
* 논리적으로 설계된 데이터 구조 및 관계.
* 시스템의 **성능**에 많은 영향을 미침.
* **목적**:
    * 데이터 용량과 실행 시간을 최소화.
    * 데이터의 추가, 삭제, 탐색을 효율적으로 수행.

### 2) 자료구조의 분류
1.  **단순 구조**: 정수, 실수, 문자, 문자열 등.
2.  **[[선형 구조]] (Linear)**: 데이터 간의 대응 관계가 **1:1**.
    * **종류**: [[스택]], [[큐]], [[데크]], 선형 리스트(배열), 연결 리스트.
    * **특징**: 순차 구조(배열)는 탐색 속도 우선, 연결 구조는 이동(삽입/삭제) 속도 우선.
3.  **[[비선형 구조]] (Non-Linear)**: 데이터 간의 관계가 **1:多** 또는 **多:多**.
    * **종류**: [[트리]], [[그래프]].
4.  **파일 구조**: 실제로 데이터가 디스크에 기록될 때 활용 (순차 파일, 색인 파일 등).

---

## 2. [[알고리즘]] (Algorithm)

### 1) 개요
* 문제를 해결하기 위한 효율적인 해법(절차).
* **표현 방법**: [[순서도]](도형과 기호), [[의사코드]](Pseudo-code, 일반적인 언어).

### 2) 알고리즘의 5가지 요건 (원칙)
1.  **입력 (Input)**: 외부에서 제공되는 데이터가 0개 이상.
2.  **출력 (Output)**: 반드시 **1개 이상**의 결과를 생성.
3.  **명확성 (Definiteness)**: 각 단계는 모호하지 않고 명확해야 함.
4.  **유한성 (Finiteness)**: 지정된 횟수만큼 반복 후 종료되어야 함.
5.  **유효성 (Effectiveness)**: 모든 연산은 실행 가능해야 함. (사용자 노트의 '희이 환벽한 구렁'을 '유한성, 유효성'으로 교정)

### 3) 성능 판단 기준
* **정확성**: 올바른 결과를 내는가.
* **명확성**: 이해하기 쉬운가.
* **수행량**: 연산의 수행 횟수 ([[시간 복잡도]]).
* **메모리 사용량**: 사용된 메모리 공간 ([[공간 복잡도]]).

### 4) 주요 설계 기법
* **[[동적 계획법]] (Dynamic Programming)**: 문제를 더 작은 문제의 연장선으로 생각하여 해결 (Bottom-up).
* **[[탐욕적 알고리즘]] (Greedy)**: 각 분기마다 그 순간에 최적이라고 생각되는 해를 선택. (종합적인 최적해를 보장하진 않음).
* **재귀적 알고리즘 (Recursive)**: 같은 풀이 과정을 반복하며 자기 자신을 호출.
* **근사 알고리즘 (Approximation)**: 복잡한 문제의 해를 빠른 시간 내에 근사하게 구함.
* **[[분할 정복법]] (Divide and Conquer)**: 큰 문제를 효율적으로 풀 수 있는 단위로 나누어 해결 후 결과를 합침 (예: 퀵 정렬, 병합 정렬).
* **[[백트래킹]] (Backtracking, 퇴각검색법)**:
    * 분기 구조 탐색 중 실패 시 이전 분기로 되돌아감.
    * 새로운 탐색이 가능한 분기까지 되돌림.
    * **[[DFS]] (Depth First Search, 깊이 우선 탐색)** 기반. (사용자 노트의 'dps' 교정)

## 3. 선형 구조 상세

### 1) [[스택]] (Stack)
* **특징**: 입출력이 **한쪽 끝(Top)**에서만 일어남.
* **구조**: **[[LIFO]] (Last In First Out, 후입선출)**.
* **용어**:
    * **Top**: 마지막에 삽입된 데이터의 위치 포인터.
    * **Push**: 데이터 삽입 (꽉 차면 **Overflow** 발생).
    * **Pop**: 데이터 삭제 (비어있으면 **Underflow** 발생).
* **활용**: 깊이 우선 탐색(DFS), 함수 호출, 재귀 호출, 후위 표기법(Postfix) 연산.

### 2) 수식 표기법 `매우 중요`
피연산자(숫자)와 연산자(+, - 등)를 배치하는 방법.

1.  **전위 표기법 (Prefix)**: 연산자가 앞(왼쪽)에 위치. (예: `+AB`) - Polish Notation.
2.  **중위 표기법 (Infix)**: 연산자가 중간에 위치. (예: `A+B`) - 사람이 사용하는 방식.
3.  **후위 표기법 (Postfix)**: 연산자가 뒤(오른쪽)에 위치. (예: `AB+`) - 역폴란드 표기법.

> [!INFO] 💡 AI 보충 설명: 수식 표기법 변환 예시
> **예제 수식**: $A * (B + C)$
>
> 1. **중위(Infix) → 후위(Postfix)**
>    - 연산 우선순위에 따라 괄호를 칩니다: $(A * (B + C))$
>    - 연산자를 각 괄호의 **오른쪽(뒤)**으로 뺍니다: $(A (BC)+ )*$
>    - 괄호를 제거합니다: **$ABC+*$**
>
> 2. **중위(Infix) → 전위(Prefix)**
>    - 연산 우선순위에 따라 괄호를 칩니다: $(A * (B + C))$
>    - 연산자를 각 괄호의 **왼쪽(앞)**으로 뺍니다: $*(A +(BC) )$
>    - 괄호를 제거합니다: **$*A+BC$**
>
> **Tip**: 스택을 이용한 후위 표기법 계산은 실기 단골 문제입니다.

### 3) [[큐]] (Queue)
* **특징**: 데이터의 삽입은 뒤(Rear), 삭제는 앞(Front)에서 일어남 (반대쪽).
* **구조**: **[[FIFO]] (First In First Out, 선입선출)**.
* **포인터**:
    * **Rear**: 가장 마지막에 저장된 데이터 위치 (삽입 시 +1).
    * **Front**: 가장 처음에 저장된 데이터 위치 (삭제 시 +1).
    * 초기값은 모두 -1 (공백 상태).
* **활용**: 대기 행렬(운영체제 스케줄링).

### 4) [[데크]] (Deque; Double Ended Queue)
* **특징**: 삽입과 삭제가 **양쪽 끝** 모두에서 가능.
* **포인터**: Left, Right 포인터가 있으며 데이터 삽입/삭제에 따라 증감.
* **변형**:
    * **Scroll (입력 제한)**: 입력은 한쪽, 출력은 양쪽.
    * **Shelf (출력 제한)**: 입력은 양쪽, 출력은 한쪽.

### 5) 선형 리스트 (Linear List)
데이터를 연속된 메모리 공간에 저장.

#### ① [[배열]] (Array)
* **특징**: 같은 유형의 데이터들이 연속적으로 나열됨.
* **장점**: 인덱스(Index)를 통한 접근 속도가 빠름.
* **단점**: 삽입/삭제 시 시간이 오래 걸림 (데이터 이동 필요).
    * 논리적 순서와 물리적 순서가 동일.
    * 고정 크기 메모리 사용.
* **이동 횟수 (평균)**:
    * 삽입 시: $\frac{n+1}{2}$
    * 삭제 시: $\frac{n-1}{2}$
    * (삭제가 삽입보다 이동 횟수가 적음)

#### ② [[연결 리스트]] (Linked List)
* **특징**: 데이터(노드)와 링크(포인터)로 구성. 배열의 단점(삽입/삭제 비효율) 보완.
* **장점**: 삽입/삭제가 용이 (링크만 변경). 희소 행렬 표현에 유리.
* **단점**: 포인터를 위한 추가 기억공간 필요, 접근 속도가 배열보다 느림(순차 탐색).
* **종류**:
    * **단일 연결 리스트**: 포인터 1개(다음 노드). 마지막 노드는 `Null`.
    * **단일 원형 연결 리스트 (Circular)**: 마지막 노드의 포인터가 **첫 노드**를 가리킴. (반복 탐색 가능).
    * **이중 연결 리스트 (Doubly)**: 포인터 2개(이전, 다음). 양방향 탐색 및 복구 가능.
    * **이중 원형 연결 리스트**.

## 4. 비선형 구조 상세

### 1) [[트리]] (Tree)
데이터를 1:多의 계층 구조로 표현. (사이클이 없는 그래프의 일종)



#### ① 주요 용어
* **Node (노드)**: 트리의 기본 요소.
* **Root (근 노드)**: 최상위 노드.
* **Leaf (단말 노드)**: 자식이 없는 최하단 노드. (Terminal Node)
* **Internal (내부 노드)**: 단말이 아닌 노드.
* **Sibling (형제 노드)**: 같은 부모를 가진 노드.
* **Degree (차수)**:
    * 노드의 차수: 그 노드의 자식 수.
    * 트리의 차수: 트리 내 노드 중 최대 차수.
* **Depth (깊이/레벨)**: 루트에서 해당 노드까지의 단계 (루트 = 1 or 0).
* **Height (높이)**: 트리의 최대 깊이.
* **공식**: $N$개의 노드를 가진 트리의 간선(Edge) 개수 = $N - 1$.

#### ② [[이진 트리]] (Binary Tree)
모든 노드의 자식이 최대 2개인 트리.
* **Full (정 이진 트리)**: 모든 노드가 0개 또는 2개의 자식을 가짐.
* **Complete (완전 이진 트리)**: 마지막 레벨을 제외하고 꽉 차 있으며, 마지막 레벨은 **왼쪽부터** 채워진 트리.
* **Perfect (포화 이진 트리)**: 모든 레벨이 꽉 찬 트리. (노드 개수 $2^k - 1$)

#### ③ 트리의 순회 (Traversal) `시험 빈출`
> [!INFO] 💡 AI 보충 설명: 순회 방법 및 예시
> 방문 순서는 **Root(부모)**를 언제 방문하느냐에 따라 결정됩니다.
>
> * **전위 순회 (Pre-order)**: **Root** → Left → Right
>     * 예: `A`가 부모, `B`가 좌, `C`가 우라면 → **A B C**
> * **중위 순회 (In-order)**: Left → **Root** → Right
>     * 예: **B A C** (수식 표기법과 유사)
> * **후위 순회 (Post-order)**: Left → Right → **Root**
>     * 예: **B C A** (디렉토리 용량 계산 등에 사용)

### 2) [[그래프]] (Graph)
데이터 간의 관계를 多:多로 표현. (네트워크 모델)

#### ① 특징
* 노드(Vertex)와 간선(Edge)으로 구성.
* 루트 노드나 부모-자식 관계가 없음.
* **사이클(Cycle)**이 존재할 수 있음.

#### ② 종류
* **무방향 그래프**: 간선에 방향이 없음. (양방향 통행)
* **방향 그래프**: 간선에 방향이 있음. (일방통행)
* **완전 그래프**: 모든 정점이 서로 연결된 그래프.
* **가중치 그래프**: 간선에 비용(Weight)이 할당된 그래프.

#### ③ 주요 용어
* **Vertex (정점)**: 트리의 노드와 같은 개념.
* **Edge (간선)**: 정점을 연결하는 선 (Branch).
* **Degree (차수)**:
    * 무방향: 정점에 연결된 간선의 수.
    * 방향: **In-degree(진입 차수)** + **Out-degree(진출 차수)**.
* **경로 길이**: 경로를 구성하는 데 사용된 간선의 수.
