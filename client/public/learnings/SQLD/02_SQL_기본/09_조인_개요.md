# 조인 (JOIN)

## 핵심 요약

**조인**은 **둘 이상의 테이블**을 연결해 **관련 데이터**를 **하나의 결과**로 조회하는 SQL 연산이다. **EQUI JOIN**: **`=`** 로 같은 값 연결. **NON EQUI JOIN**: **BETWEEN, >, <, !=** 등 **등호 외** 비교로 연결. **표준조인**: DBMS마다 다른 옛 방식을 **ANSI/ISO SQL**로 통일 — **`FROM … JOIN … ON`** 형태.

## 왜 중요한가

- 정규화로 나뉜 테이블을 **다시 엮어** 조회하는 핵심 기법이다.
- EQUI vs NON EQUI 구분은 **시험 빈출**.
- 관계대수 **⋈(조인)** 의 SQL 구현이다.

## 1. 조인이란

| 항목 | 내용 |
|------|------|
| **정의** | **2개 이상** 테이블을 **연결**하여, **관련 있는** 데이터를 **한 결과**로 보여 주는 SQL 연산 |
| **목적** | FK·공통 컬럼 등으로 **분리된** 정보를 **한 번에** 조회 |
| **관계대수** | **⋈ (조인)** — 공통 속성·조건으로 릴레이션 **병합** |

---

## 2. EQUI JOIN

**Equal(`=`)** 연산자로 **같은 값**일 때 행을 연결하는 조인.

```sql
SELECT 직원.이름, 부서.부서명
  FROM 직원
  JOIN 부서
    ON 직원.부서ID = 부서.부서ID;
```

### \<직원\> · \<부서\>

**직원**

| ID | 이름 | 부서ID |
|----|------|--------|
| 101 | 홍길동 | 10 |
| 102 | 이순신 | 20 |
| 103 | 임꺽정 | 10 |

**부서**

| 부서ID | 부서명 |
|--------|--------|
| 10 | 인사 |
| 20 | 개발 |

### 결과

| 이름 | 부서명 |
|------|--------|
| 홍길동 | 인사 |
| 이순신 | 개발 |
| 임꺽정 | 인사 |

→ `직원.부서ID = 부서.부서ID` 인 행만 **매칭**. **등호(=)** 조건이 핵심.

---

## 3. NON EQUI JOIN

**등호가 아닌** 비교 연산자(**BETWEEN**, **>**, **<**, **!=** …)로 연결하는 조인.

```sql
SELECT S.이름, S.점수, G.성적
  FROM 학생 S
  JOIN 성적기준 G
    ON S.점수 BETWEEN G.최소점수 AND G.최대점수;
```

### \<학생\> · \<성적기준\>

**학생**

| ID | 이름 | 점수 |
|----|------|------|
| 101 | 홍길동 | 85 |
| 102 | 이순신 | 73 |
| 103 | 임꺽정 | 59 |
| 104 | 장보고 | 92 |

**성적기준**

| 성적 | 최소점수 | 최대점수 |
|------|----------|----------|
| A | 90 | 100 |
| B | 80 | 89 |
| C | 70 | 79 |
| D | 60 | 69 |
| F | 0 | 59 |

### 결과

| 이름 | 점수 | 성적 |
|------|------|------|
| 홍길동 | 85 | B |
| 이순신 | 73 | C |
| 임꺽정 | 59 | F |
| 장보고 | 92 | A |

→ 점수가 **구간 [최소, 최대]** 에 들어가는 **성적기준 행**과 연결. **`=` 한 번**이 아니라 **범위 조건**이 조인 키.

| EQUI | NON EQUI |
|------|----------|
| **`=`** | **BETWEEN**, **>**, **<**, **!=** 등 |
| 부서ID = 부서ID | 점수 BETWEEN 최소 AND 최대 |

---

## 4. 표준조인 (Standard Join)

| 항목 | 내용 |
|------|------|
| **배경** | DBMS마다 **조인 문법·방식**이 달랐음 (Oracle `(+)`, 콤마+WHERE …) |
| **표준** | **ANSI/ISO SQL** 로 통일한 조인 |
| **형태** | **`FROM` 절에서 `JOIN` … `ON`** (또는 `USING`, `NATURAL` 등) |

```sql
-- 표준조인 예 (EQUI)
FROM 직원 JOIN 부서 ON 직원.부서ID = 부서.부서ID
```

> INNER / OUTER / CROSS 등 **조인 종류**는 다음 문서에서 확장.

---

## 5. EQUI vs NON EQUI vs 표준조인

| 구분 | 조건 연산 | 문법 예 |
|------|-----------|---------|
| **EQUI JOIN** | **`=`** | `ON A.id = B.id` |
| **NON EQUI JOIN** | **BETWEEN, >, <, !=** | `ON 점수 BETWEEN 최소 AND 최대` |
| **표준조인** | (위 조건을 **`JOIN … ON`** 으로 표기) | ANSI/ISO **`JOIN` 키워드** |

→ **표준조인**은 **문법 체계** 이름, **EQUI/NON EQUI**는 **조인 조건의 종류**이다. **표준조인 안에서** EQUI·NON EQUI 모두 가능.

---

## 6. 시험 포인트 / 함정

| 구분 | 내용 |
|------|------|
| 조인 | **2개 이상** 테이블, **관련 데이터** 통합 |
| EQUI | **`=`** 로 같은 값 매칭 |
| NON EQUI | **등호 외** — **BETWEEN** 구간 매칭 대표 예 |
| 표준조인 | **ANSI/ISO**, **`FROM … JOIN … ON`** |
| 관계대수 | **⋈** ↔ SQL **JOIN** |
| 함정 | NON EQUI를 **반드시 `!=`만**이라고 생각 → **BETWEEN도 NON EQUI** |
| 함정 | EQUI = **INNER JOIN만**이라고 단정 → 문맥상 **등호 조건** 의미 |
