# 데이터베이스 물리 속성 및 보안 설계

## 📌 핵심 요약
성능 최적화를 위한 테이블 구조 설계([[Partitioning]], 클러스터링)와 인덱스 설계 절차를 다룹니다. 또한 데이터 무결성 보장을 위한 4가지 요소, 분산 데이터베이스의 5가지 투명성, 그리고 기밀성과 무결성을 위한 보안 모델([[DAC]], [[MAC]], [[RBAC]])을 상세히 정리합니다.

---

## 📝 상세 정리

## 1. 저장 레코드 형식 설계

### 1.1 저장 테이블 유형
- **일반 유형 테이블:** 저장 위치는 테이블 밑에 붙는 방식(Heap 조직).
- **[[클러스터 인덱스]] 테이블 (Clustered Index Table):** - 기본키, 인덱스 등의 순서를 기반으로 데이터가 물리적으로 정렬되어 저장됨.
    - 빠르게 액세스 가능하나, 입력마다 위치 지정이 필요하여 유지비용이 상승함.
    - 테이블당 하나만 생성 가능함.
- **비 클러스터형 인덱스 (Non-Clustered Index):**
    - 데이터 페이지와 인덱스 페이지를 분리하여 관리함.
    - 인덱스에는 실제 데이터가 아닌 데이터가 저장된 위치인 **[[RID]](Row ID)**가 저장됨.
    - 검색은 느리지만 입력, 수정, 삭제 속도는 비교적 빠름.
    - 테이블별 여러 개 생성 가능함.



### 1.2 테이블 분할 (Partitioning)
- **수평 분할 (Horizontal Partitioning):** 릴레이션 스키마를 복제하여 **분할 키**를 기준으로 데이터 분산 저장. 테이블당 데이터와 인덱스 개수가 작아져 성능 향상.
- **수직 분할 (Vertical Partitioning):** 릴레이션 스키마를 분할하여 데이터 분산 저장. 사용 빈도가 적은 컬럼과 많은 컬럼을 분할함.
- **외부 테이블 (External Table):** 외부 파일(CSV 등)을 연결하여 사용하는 테이블. DBMS와 연결된 형태임.
- **임시 테이블 (Temporary Table):** 트랜잭션 및 세션별로 데이터를 저장하고 처리 가능. 휘발성 속성이며 타 세션과 공유할 수 없음.

### 1.3 컬럼 변환 및 최적화
- **물리적 순서 조정:** `NULL`이 없는 고정 길이 컬럼을 앞쪽에 배치. `NULL`이 많거나 가변 길이면 뒤로 배치함.
- **데이터 타입 및 길이 지정:** 고정 길이는 최소 길이, 가변 길이는 최대 길이로 지정.
- **문자열 비교:** 공백을 추가하여 길이를 같게 한 후 비교하거나, 공백 추가 없이 비교하는 방식 존재.
- **데이터 타입 변환 비교:** 문자열과 숫자 비교 시 숫자의 우선순위가 높기에 문자열을 숫자로 변환함.
- **테이블 스페이스:** 실제 데이터를 저장하는 물리적 공간. 용도로 구분하여 설계하며 효율적인 용량 설계가 필요함.

## 2. 스토리지 (Storage)

### 2.1 클러스터링 (Clustering)
- 데이터 액세스 효율 향상을 위해 동일한 성격의 데이터를 동일한 데이터 블록에 저장하는 물리적 저장 기법.
- **유형:** 처리 범위가 넓으면 단일 테이블 클러스터링, 조인이 많이 발생하면 다중 테이블 클러스터링.
- **주의:** 데이터 변경이나 탐색이 많으면 오히려 적용하지 않는 것이 좋음.

### 2.2 파티셔닝 (Partitioning) 유형
- **범위 분할 (Range):** 특정 컬럼 값의 범위를 기준으로 분할.
- **해시 분할 (Hash):** 해시 함수에 따라 데이터 배분.
- **조합 분할 (Composite):** 범위 분할 후 해시 분할 적용.
- **목록 분할 (List):** 관리자가 직접 값을 지정하여 분할.
- **효과:** 데이터 접근 범위 줄이기, 데이터 훼손 가능성 감소.

### 2.3 스토리지 연결 기술
- **[[DAS]] (Direct Attached Storage):** 서버에 직접 연결. 다른 서버와 파일 공유 불가능. 속도 빠름, 저렴하나 확장성 부족.
- **[[NAS]] (Network Attached Storage):** 네트워크를 통해 연결. 확장성 우수, 공유 가능하나 다중 접속 시 성능 저하.
- **[[SAN]] (Storage Area Network):** DAS와 NAS의 장점을 혼합. 비용이 많이 듦.

## 3. 물리 데이터베이스 설계서 작성

### 3.1 무결성 ([[Integrity]])
- **도메인 무결성:** 도메인 범위 안에서 표현됨. 데이터 형식, 타입, 길이, NULL 허용 여부 등의 제약으로 보장.
- **개체 무결성:** 중복 및 NULL에 대한 제한을 두어 식별자 역할을 할 수 있게 함.
- **참조 무결성:** 존재하지 않는 값을 참조할 수 없도록 일관성 보증.
- **사용자 정의 무결성:** 사용자가 직접 정의한 제약 조건.
- **강화 효과:** 데이터 품질에 직접적인 영향.

### 3.2 트랜잭션 설계 및 CRUD 분석
- **CRUD 매트릭스:** 프로세스와 테이블 간의 영향도를 테이블 형식으로 작성.

## 4. 인덱스 설계 (Index Design)

### 4.1 인덱스 종류
- **트리 기반 인덱스:** 트리 구조 활용.
    - **[[B-Tree]] 인덱스:** 모든 단말 노드(Leaf Node)의 레벨이 같음.
    - **[[B+Tree]] 인덱스:** 인덱스 세트와 순차 세트로 구성.
- **비트맵 인덱스 (Bitmap Index):** 비트로만 구성되어 다중 조건을 효율적으로 연산.
- **함수 기반 인덱스:** 컬럼 값을 특정 함수식에 적용하여 도출된 값을 사용.
- **도메인 인덱스:** 사용자가 직접 만들어 사용.

### 4.2 인덱스 설계 절차
1. 접근 경로 수집 → 2. 후보 컬럼 선정 → 3. 접근 경로 결정 → 4. 컬럼 조합 및 순서 결정 → 5. 적용 시험

## 5. 분산 설계 (Distributed Design)

### 5.1 분산 데이터베이스 관리 시스템 ([[DBMS]])
- **6가지 투명성 (Transparency):**
    - **분할 투명성:** 테이블의 분할 구조를 알 필요 없음.
    - **위치 투명성:** 물리적인 위치를 알 필요 없음.
    - **중복 투명성:** 데이터의 복제 및 저장 위치를 알 필요 없음.
    - **장애 투명성:** 장애 발생 시에도 무결성과 원자성 보장.
    - **병행 투명성:** 다수 트랜잭션 동시 수행 시 일관성 유지.

### 5.2 분산 데이터베이스 설계 기법
1. **테이블 위치 분산:** 중복되지 않게 분산.
2. **테이블 분할 (3대 원칙):**
    - **완전성:** 손실 없이 분할.
    - **재구성:** 원래 테이블로 재구성 가능해야 함.
    - **상호 중첩 배제:** 컬럼이 중복되지 않아야 함.
3. **테이블 할당:** 비중복 할당(단일 노드 존재) 또는 중복 할당(여러 노드 할당).

## 6. 보안 설계 및 접근 통제

### 6.1 접근 통제 기술
- **[[DAC]] (임의 접근 통제):** 사용자의 신원에 따라 권한 부여. 주체가 직접 권한 지정.
- **[[MAC]] (강제 접근 통제):** 주체와 객체의 등급 비교. 제3자(DBA)가 통제. 
    - 읽기(주체 $\ge$ 객체), 수정/등록(주체 $=$ 객체).
- **접근 통제 3요소:** 정책, 메커니즘, 보안 모델.
- **감사 추적:** 모든 활동 기록.

### 6.2 접근 통제 보안 모델
- **접근 통제 행렬 (Access Control Matrix):** 주체를 행, 객체를 열로 권한 표현.
- **기밀성 모델 ([[벨-라파둘라 모델]]):** 기밀성 중점.
    - **단순 보안 규칙:** 주체는 자신보다 높은 등급의 객체 읽기 불가.
    - **스타($*$) 보안 규칙:** 주체는 자신보다 낮은 등급의 객체에 정보 쓰기 불가.
    - **강한 스타 보안 규칙:** 주체는 자신과 등급이 다른 객체에 대해 읽기/쓰기 불가.
- **무결성 모델 ([[비바 모델]]):** 부당 변경 방지 중점.
    - **단순 무결성 규칙:** 자신보다 낮은 등급의 객체 읽기 불가.
    - **스타($*$) 무결성 규칙:** 주체는 자신보다 높은 등급의 객체에 정보 쓰기 불가.

### 6.3 접근 통제 정책 및 조건
- **신분 기반:** IBP(단일 사용자), GBP(그룹 기반).
- **규칙 기반:** MLP(기밀 분류 기반), CBP(집단별).
- **역할 기반 ([[RBAC]]):** 직무에 따라 부여.
- **추가 조건:** 값 종속 통제(저장 값 근거), 다중 사용자 통제(복수 사용자 요구), 컨텍스트 기반 통제(시간, 위치).

## 🎯 빈출 키워드
- **@([[Partitioning]]):** 범위(Range), 해시(Hash), 조합, 목록(List) 분할 유형.
- **@([[ACID]] 및 투명성):** 위치, 중복, 분할, 장애, 병행 투명성.
- **@([[B-Tree]] / [[B+Tree]]):** 인덱스 구조 및 검색 방식 차이.
- **@([[DAC]] / [[MAC]] / [[RBAC]]):** 신분, 강제, 역할 기반 접근 제어 정책.
- **@([[벨-라파둘라]] / [[비바 모델]]):** No Read Up, No Write Down 등 보안 규칙 차이.
