# 결함관리와 품질지표

## 핵심 요약

결함관리는 발견된 문제를 추적하고 해결 상태를 통제하는 활동이다. 시험에서는 결함 상태 흐름과 소프트웨어 품질 속성을 비교하는 문제가 자주 나온다.

## 1. 결함관리 개념

- 결함을 등록
- 원인 분석
- 수정
- 재검증
- 종료

## 2. 결함 상태 흐름

일반적으로 다음과 같이 이해한다.

- Open
- Assigned
- Resolved
- Closed

환경에 따라 Reopen, Deferred 같은 상태가 추가될 수 있다.

## 3. 결함관리 목적

- 추적성 확보
- 우선순위 관리
- 재발 방지

## 4. 품질 속성

| 품질 속성 | 의미 |
|---|---|
| 기능성 | 필요한 기능 제공 |
| 신뢰성 | 안정적으로 동작 |
| 사용성 | 사용하기 쉬움 |
| 효율성 | 자원 효율적 사용 |
| 유지보수성 | 수정과 개선이 쉬움 |
| 이식성 | 다른 환경으로 이전 쉬움 |

## 5. 예제

- 결함이 발견되어 개발자에게 배정 -> Assigned
- 수정 완료 후 테스트 대기 -> Resolved

## 6. 자주 틀리는 함정

- 결함관리와 형상관리를 같은 활동으로 본다
- 품질 속성을 성능 하나로만 좁게 이해한다

## 7. 시험장에서 푸는 순서

1. 문제에서 상태 흐름인지 품질 속성인지 구분한다.
2. 결함 상태는 시간 순서대로 정리한다.
3. 품질 속성은 기능/신뢰/사용/효율/유지/이식 기준으로 판단한다.

## 한 페이지 요약

- 결함관리는 등록부터 종료까지 상태를 추적한다.
- 결함 상태는 Open -> Assigned -> Resolved -> Closed 흐름으로 자주 정리한다.
- 품질 속성은 기능성, 신뢰성, 사용성, 효율성, 유지보수성, 이식성이 대표적이다.
