# 리눅스 명령어와 쉘 스크립트

## 핵심 요약

리눅스 문제는 실무 명령어 암기와 간단한 스크립트 해석이 핵심이다. 특히 `chmod` 권한 계산, 환경 변수, `if/for/while/case` 구조는 반복 출제 포인트다.

## 1. 커널과 쉘

- 커널: 하드웨어와 자원을 관리하는 핵심 부분
- 쉘: 사용자 명령을 해석해 커널에 전달하는 인터페이스

## 2. 자주 나오는 리눅스 명령어

| 명령어 | 의미 |
|---|---|
| `ls` | 목록 보기 |
| `pwd` | 현재 경로 출력 |
| `cd` | 디렉토리 이동 |
| `cp` | 복사 |
| `mv` | 이동/이름 변경 |
| `rm` | 삭제 |
| `grep` | 문자열 검색 |
| `chmod` | 권한 변경 |
| `chown` | 소유자 변경 |

## 3. 파일 권한과 chmod

권한 수치:

- `r` = 4
- `w` = 2
- `x` = 1

### 예: `chmod 754 file.txt`

- 7 = `rwx`
- 5 = `r-x`
- 4 = `r--`

즉 결과는 `rwxr-xr--`

## 4. 환경 변수

| 변수 | 의미 |
|---|---|
| `HOME` | 사용자 홈 경로 |
| `PATH` | 실행 파일 탐색 경로 |
| `PWD` | 현재 작업 디렉토리 |
| `SHELL` | 사용 중인 쉘 |

## 5. 쉘 스크립트 기본

```bash
#!/bin/bash
echo "hello"
```

### 입력/출력

- `read`
- `echo`

## 6. 조건문과 반복문

### if

```bash
if [ $num -gt 10 ]
then
  echo "big"
else
  echo "small"
fi
```

### case

```bash
case $answer in
  yes) echo "ok" ;;
  no) echo "cancel" ;;
esac
```

### for / while

```bash
for i in 1 2 3
do
  echo $i
done
```

## 7. 비교 연산자

| 연산자 | 의미 |
|---|---|
| `-eq` | 같다 |
| `-ne` | 다르다 |
| `-gt` | 크다 |
| `-ge` | 크거나 같다 |
| `-lt` | 작다 |
| `-le` | 작거나 같다 |

## 8. 예제

### 예제 1: chmod 계산

- `chmod 641 test.txt`
- 6 = `rw-`
- 4 = `r--`
- 1 = `--x`

### 예제 2: while 해석

```bash
i=1
while [ $i -le 3 ]
do
  echo $i
  i=`expr $i + 1`
done
```

출력:

- 1
- 2
- 3

## 9. 자주 틀리는 함정

- `fi`, `esac`, `done` 의 종료 키워드를 혼동한다
- `chmod` 숫자 계산을 반대로 읽는다
- `PATH` 와 `PWD` 의미를 혼동한다
- `until` 은 while과 반대라는 점을 놓친다

## 10. 시험장에서 푸는 순서

1. 명령어 의미 문제면 동사처럼 해석한다.
2. 권한 문제면 `r=4, w=2, x=1`부터 적는다.
3. 스크립트는 변수 초기값과 반복 조건을 먼저 본다.
4. 종료 키워드 `fi`, `esac`, `done` 을 체크한다.

## 한 페이지 요약

- 쉘은 명령 해석기, 커널은 자원 관리자다.
- `chmod` 는 `r=4, w=2, x=1` 계산으로 푼다.
- `HOME`, `PATH`, `PWD`, `SHELL` 은 자주 나오는 환경 변수다.
- 쉘 스크립트는 `if`, `case`, `for`, `while` 문법이 핵심이다.
