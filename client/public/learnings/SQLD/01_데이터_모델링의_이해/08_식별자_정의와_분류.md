# 식별자 정의와 분류

## 핵심 요약

**식별자**는 **인스턴스를 유일하게 구분**하는 **속성 또는 속성의 집합**이다. **주식별자(PK)** 는 **대표성·유일성·최소성·불변성·존재성(Not Null)** 을 만족하고, **보조식별자(AK)** 는 유일하지만 **대표성이 없다**. **외부식별자(FK)** 는 다른 엔터티에서 받은 식별자이며, **복합식별자**는 속성 2개 이상, **인조식별자**는 시스템이 부여한 번호(수강신청번호)이다.

## 왜 중요한가

- ERD에서 PK·FK·AK 표기와 논리→물리 키 설계의 핵심이다.
- 주식별자 5가지 성질은 객관식·단답 빈출이다.
- 수강신청·수강내역 예시는 [04](./04_엔터티_분류_유무형과_발생시점.md) 체인과 연결된다.

## 1. 식별자란

| 항목 | 내용 |
|------|------|
| **정의** | **인스턴스를 유일하게 구분**할 수 있는 **속성** 또는 **속성의 집합** |
| 논리 모델 | → **기본키(PK)**, **대체키(AK)**, **외래키(FK)** 로 구현 |

---

## 2. 식별자의 분류

### (1) 대표성에 따른 분류

| 분류 | 동의어 | 설명 |
|------|--------|------|
| **주식별자** | **PK**, 기본키 | 아래 **5가지 성질**을 모두 만족 |
| **보조식별자** | **AK**, 대체키 | 인스턴스는 **유일**하게 구분 가능하나 **대표성 없음** |

#### 주식별자(PK)의 5가지 성질

| 성질 | 의미 |
|------|------|
| **대표성** | 엔터티 인스턴스를 **대표**할 수 있음 |
| **유일성** | 값이 **중복되지 않음** |
| **최소성** | 인스턴스 구분에 **꼭 필요한** 최소 구성 |
| **불변성** | 식별 목적의 값이 **임의로 바뀌지 않음** |
| **존재성** | **Not Null** — 값이 반드시 존재 |

> 암기 힌트: **대유최불존** (대표 · 유일 · 최소 · 불변 · 존재)

#### 보조식별자(AK) 예

- **학생.주민등록번호**: 유일하지만 업무·개인정보 측면에서 **PK로 쓰지 않음** → AK

---

### (2) 생성 출처에 따른 분류

| 분류 | 설명 | 예 |
|------|------|-----|
| **내부식별자** | 해당 엔터티 **안에서** 정의·부여 | 학번, 과목명, 수강신청번호 |
| **외부식별자** | **다른 엔터티**에서 받은 식별자 = **FK** | 수강신청.**학번**, 수강내역.**과목명** |

---

### (3) 속성 개수에 따른 분류

| 분류 | 설명 | 예 |
|------|------|-----|
| **단일식별자** | 속성 **1개** | 학번, 과목명, 수강신청번호 |
| **복합식별자** | 속성 **2개 이상** 조합 | 수강내역: **(수강신청번호, 과목명)** |

---

### (4) 대체 가능성(인위성)에 따른 분류

| 분류 | 설명 | 예 |
|------|------|-----|
| **본질식별자** | 현실 업무에 **자연스럽게** 존재 | 학번, 과목명, 주민등록번호 |
| **인조식별자** | 시스템·설계상 **인위적으로** 부여 (Surrogate Key) | **수강신청번호** |

---

## 3. E-R 예: 학생 — 수강신청 — 수강내역 — 과목

| 엔터티 | 식별자·속성 | 분류 |
|--------|-------------|------|
| **학생** | **학번** | 주식별자(PK), 단일, 내부, 본질 |
| | 주민등록번호(AK) | **보조식별자(AK)** |
| | 이름 | 일반 속성 |
| **수강신청** | **수강신청번호** | 주식별자, **인조식별자** |
| | 학번 | **외부식별자(FK)** ← 학생 |
| | 수강신청일자 | 일반 |
| **수강내역** | **수강신청번호 + 과목명** | **복합식별자(PK)** |
| | 과목명 | PK 구성 + **FK** ← 과목 |
| | 학점, 출석횟수, 출석률 | 일반 |
| **과목** | **과목명** | 주식별자(PK), 단일, 본질 |

### 관계 (발생 시점 체인)

```mermaid
erDiagram
  STUDENT ||--o{ APPLY : ""
  APPLY ||--o{ DETAIL : ""
  SUBJECT ||--o{ DETAIL : ""
  STUDENT {
    string 학번 PK
    string 주민등록번호 AK
    string 이름
  }
  APPLY {
    string 수강신청번호 PK
    string 학번 FK
    date 수강신청일자
  }
  DETAIL {
    string 수강신청번호 PK_FK
    string 과목명 PK_FK
    int 학점
    int 출석횟수
    float 출석률
  }
  SUBJECT {
    string 과목명 PK
    string 강의실
    string 교수
  }
```

## 4. 용어 대응표

| 데이터 모델링 | 관계형 DB |
|---------------|-----------|
| 주식별자 | **PK** |
| 보조식별자 | **AK** (Alternate Key) |
| 외부식별자 | **FK** |
| 복합식별자 | **Composite PK** |
| 인조식별자 | **Surrogate Key** |

---

## 5. 시험 포인트 / 함정

| 구분 | 내용 |
|------|------|
| 정의 | 속성 **또는** 속성의 **집합** |
| PK 5성질 | **대유최불존** — 대표·유일·최소·불변·존재(Not Null) |
| AK | 유일 O, **대표성 X** — 주민등록번호 |
| FK | **외부식별자** = 다른 엔터티에서 온 식별자 |
| 복합 | 수강내역 **(수강신청번호, 과목명)** |
| 인조 | **수강신청번호** — 본질이 아닌 시스템 부여 |
| 함정 | AK를 PK로 쓰는 이유 = **대표성 부족** (유일성 부족이 아님) |
| 함정 | **내부식별자**와 **본질식별자** — 축이 다름 (출처 vs 자연/인조) |
