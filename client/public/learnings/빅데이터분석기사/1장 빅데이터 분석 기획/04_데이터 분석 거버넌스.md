# 데이터 분석 거버넌스 체계 및 데이터 거버넌스

## 📌 핵심 요약

기업 내 데이터 분석 업무를 문화로 정착시키고 지속적으로 고도화하기 위한 관리 체계인 **분석 거버넌스(Analysis Governance)**와 데이터의 품질, 표준화, 보안을 관리하는 **데이터 거버넌스(Data Governance)**의 구성 요소 및 프로세스를 학습합니다. 조직 구조, 인프라, 교육 체계 등 전사적 차원의 데이터 관리 전략 수립이 핵심입니다.

---

## 📝 상세 정리

### 1. 분석 거버넌스 (Analysis Governance)

- **정의:** 업무 프로세스, 정책 및 정보를 만들고 관리하는 지속적인 프로세스입니다.
- **목적:** 데이터 분석 업무를 기업 문화로 정착시키고 지속적으로 고도화하기 위해 필요합니다.
- **구성 요소:** \* **전략 (Strategy):** 분석 목적 및 방향 설정
  - **활동 (Activity):** 분석 과제 및 프로세스 관리
  - **조직 (Organization):** 분석 전담 조직 및 역할 정의
  - **기술 (Technology):** 분석 지원 인프라 및 플랫폼
  - **데이터 (Data):** 데이터 거버넌스 및 관리 체계

### 2. 분석 거버넌스 체계의 구성 요소

#### (1) 데이터 분석 관리 조직 (Organization)

데이터 분석 기획과 관리를 수행하는 조직으로, 전문 인력을 중심으로 구성됩니다.

- **집중형 (Centralized):** 전사 분석 업무를 전담 조직에서 전적으로 담당합니다.
- **기능형 (Functional):** 별도 조직 없이 각 현업 부서에서 직접 분석 업무를 수행합니다.
- **분산형 (Decentralized):** 분석 전문 인력을 현업 부서에 직접 배치하여 업무를 수행합니다.
  > [!INFO] 💡 AI 보충 설명: 분산형 조직 (Decentralized)
  > 분산형은 분석 인력이 현업 부서의 일원으로 활동하므로 **현업 밀착형 분석**이 가능하다는 장점이 있습니다. 부서 간의 우선순위에 따라 신속한 대응이 가능하지만, 전사 차원의 분석 표준화가 어렵고 인력 관리가 이원화될 수 있다는 단점이 있습니다.

#### (2) 분석 과제 기획 및 운영 프로세스 (Process)

- **목적:** 조직 내 데이터 분석 문화 경쟁력 확보, 시행착오 최소화, 효율적 진행
- **과제 발굴 단계:** 분석 아이디어 발굴 → 분석 과제 후보 제안 → 분석 과제 확정 → 팀 구성
- **과제 수행 및 모니터링 단계:** 분석 과제 실행 → 진행 관리 → 결과 공유 및 개선

#### (3) 데이터 분석 지원 인프라 (Infrastructure)

- **수립 원칙:** 마스터 플랜(Master Plan) 수립 단계에서부터 장기적, 지속적 활용이 가능하도록 고려해야 합니다.
- **구조의 변화:** 개별 시스템 단위보다는 **확장성**을 고려한 **플랫폼 구조** 도입이 필요합니다.
  - **개별 시스템:** 시스템 간 자체적인 데이터 교환, 독립적인 데이터 관리 체계.
  - **플랫폼 구조:** 분석 플랫폼 활용, 중앙집중적 데이터 관리, 시스템 간 인터페이스 최소화.
- **데이터 분석 플랫폼:** 프로그램이 실행될 수 있는 환경과 기초를 이루는 컴퓨터 시스템으로, 신규 니즈 발생 시 추가적인 서비스 제공 기능을 포함합니다.
  > [!INFO] 💡 AI 보충 설명: 광의 및 협의의 분석 플랫폼
  >
  > - **광의(Broad)의 분석 플랫폼:** 데이터 수집부터 저장, 분석, 시각화에 이르는 전 과정과 이를 지탱하는 하드웨어, 네트워크, 분석 도구 및 사용자 커뮤니티까지 포함하는 통합적 생태계를 의미합니다.
  > - **협의(Narrow)의 분석 플랫폼:** 데이터 분석가가 실제 모델을 개발하고 실행하기 위해 사용하는 소프트웨어 환경(예: Spark, R, Python 환경 등) 및 분석 라이브러리에 집중된 개념입니다.

#### (4) 데이터 분석 교육 및 마인드 육성 체계 (Culture)

- 분석 역량을 강화하기 위한 단계적 변화 관리: **준비기 → 도입기 → 안정 추진기** 순으로 진행됩니다.
- **주의 사항:** 도입기에서 과거의 방식으로 돌아가려는 경향이 있으나, 이 단계를 성공적으로 넘기면 분석 문화가 가속화됩니다.

### 3. 데이터 거버넌스 (Data Governance)

- **배경:** 데이터 중복, 비표준화에 따른 정합성 오류 등으로 데이터 활용도가 저하되는 것을 방지하기 위함입니다.
- **정의:** 표준화된 관리 체계를 수립하고 운영하기 위한 프레임워크와 저장소를 구축하는 것입니다.
- **특징:** 가용성, 유용성, 통합성, 보안성, 안정성을 확보하여 빅데이터 프로젝트를 성공으로 이끄는 기반이 됩니다. 전사 차원의 **IT 거버넌스**나 **EA(Enterprise Architecture)**의 구성 요소로서 역할을 수행합니다.

#### (1) 관리 대상

- **마스터 데이터 (Master Data):** 비즈니스 트랜잭션을 처리하기 위해 사용되는 기초 데이터 (예: 고객, 상품 정보).
- **메타 데이터 (Meta Data):** 데이터에 대한 구조화된 데이터로, 데이터의 속성과 정보를 설명합니다.
- **데이터 사전 (Data Dictionary):** 데이터의 이름, 표현 방식, 의미, 다른 자료와의 관계 등을 저장한 명세서입니다.

#### (2) 빅데이터 거버넌스 (Big Data Governance)

> [!INFO] 💡 AI 보충 설명: 빅데이터 거버넌스
> 기존의 데이터 거버넌스가 정형 데이터 위주였다면, 빅데이터 거버넌스는 **비정형/반정형 데이터**를 포함하며, 데이터의 양(Volume)과 속도(Velocity)를 감당하기 위한 실시간성 및 확장성이 추가된 개념입니다. 데이터 생명 주기 관리와 함께 데이터 프라이버시 및 윤리적 활용이 더욱 강조됩니다.

#### (3) 데이터 거버넌스 구성 요소

1.  **원칙 (Principle):** 데이터를 유지하고 관리하기 위한 지침 및 가이드.
2.  **조직 (Organization):** 데이터를 관리할 조직의 역할과 책임(R&R).
3.  **프로세스 (Process):** 관리를 위한 활동과 체계.

#### (4) 데이터 거버넌스 체계 상세

- **데이터 표준화:**
  - **데이터 표준 용어 설정:** 표준 단어 사전, 도메인 사전, 코드 등으로 구성하며 상호 검증 가능한 점검 프로세스를 구축합니다.
  - **명명 규칙(Naming Rule) 수립:** 기준 언어와의 매핑 상태를 유지합니다.
  - **메타 데이터 및 데이터 사전 구축:** 데이터 구조 체계를 마련하고 ERD(Entity Relationship Diagram)를 제공합니다.
- **데이터 관리 체계:**
  - 표준 데이터를 포함한 메타 데이터와 사전의 관리 원칙을 수립합니다.
  - 항목별 상세 프로세스를 수립하고 담당자(Data Steward 등)를 마련합니다.
  - **데이터 생명 주기 관리(Data Life Cycle Management) 방안**을 수립합니다.
- **데이터 저장소 관리 (Repository):**
  - 전사 차원의 저장소를 구성합니다.
  - 워크플로우 애플리케이션 지원 및 인터페이스를 통한 통제가 필수적입니다.
  - 구조 변경에 따른 **사전 영향 평가**를 수행합니다.
- **표준화 활동:**
  - 체계 구축 후 주기적으로 표준 준수 여부를 점검합니다.
  - 변화 관리 및 주기적인 교육을 진행합니다.
  - 표준화 개선 활동을 지속합니다.

### 4. 데이터 분석 교육 및 마인드 육성 체계 (Culture)

- **목적:** 단순한 기술 교육을 넘어 전사적 분석 문화를 조성하고, 데이터 중심의 의사결정(Data-Driven Decision Making)이 가능하도록 체계를 구축합니다.
- **진행 단계:**
  1. **준비기 (Preparation):** 분석의 필요성을 인식하고 교육 인프라를 구축하는 단계입니다.
  2. **도입기 (Introduction):** 실제 분석 교육을 시행하며 성공 사례를 창출하는 단계입니다.
     > [!CAUTION] ⚠️ 도입기 주의사항
     > 이 시기에는 기존의 익숙한 업무 방식으로 돌아가려는 '관성'이 강하게 작용합니다. 이 저항을 극복하고 성공 사례(Quick Win)를 만들어내면 변화가 가속화됩니다.
  3. **안정 추진기 (Settlement):** 분석 업무가 일상화되고, 전사적인 데이터 문화가 정착되어 고도화되는 단계입니다.

### 5. 데이터 품질 관리 (Data Quality Management)

데이터 거버넌스의 핵심 활동으로, 저품질 데이터는 잘못된 분석 결과와 의사결정으로 이어질 수 있어 체계적인 품질 관리가 필수입니다.

#### 5.1 데이터 품질 기준 (6대 차원)

데이터 품질을 평가하는 국제적으로 표준화된 6가지 차원입니다.

| 차원       | 영문          | 설명                                                                                                       |
| ---------- | ------------- | ---------------------------------------------------------------------------------------------------------- |
| **정확성** | Accuracy      | 데이터가 실제 사건이나 값을 올바르게 반영하는지. 신뢰할 수 있는 소스 기반의 정확도                         |
| **완전성** | Completeness  | 필요한 모든 데이터가 존재하고 누락값이 없는지. 누락 비율이 높으면 편향·오해의 소지                         |
| **일관성** | Consistency   | 서로 다른 데이터셋 간 레코드가 일치하는지. 조직 전체에서 통일적·표준화된 데이터                            |
| **적시성** | Timeliness    | 데이터가 예상 시간 내에 준비되고 필요 시점에 이용 가능한지. 최신 상태 유지                                 |
| **고유성** | Uniqueness    | 중복 데이터의 양. 각 개체가 고유하게 식별되고 중복 기록이 없는지                                           |
| **유효성** | Validity      | 비즈니스 규칙에 따른 형식·범위·패턴을 충족하는지. 메타데이터 기준 부합 여부                                |
| **접근성** | Accessibility | 사용자가 필요한 데이터에 쉽게 접근할 수 있는지, 데이터 조회 및 활용이 적시에 원활하게 이루어지는지 평가    |
| **보안성** | Security      | 데이터가 권한 있는 사용자만 접근 가능하도록 보호되고, 무단 접근·노출·위변조로부터 안전하게 관리되는지 평가 |

진단 절차 : 진단 대상 정의 -> 품질 진단 실시 -> 진단 결과 분석
개선 절차 : 개선 계획 수립 -> 개선 수행 -> 품질 통제

---

> [!INFO] 💡 AI 보충 설명: 분석 거버넌스의 성숙도 모델
> 거버넌스 체계는 단번에 완성되는 것이 아니라, 조직의 **준비도(Readiness)**와 **성숙도(Maturity)**에 따라 단계적으로 발전합니다. 빅데이터 분석기사 시험에서는 '분석 준비도'의 6개 영역(분석 업무, 인력 및 조직, 기법, IT 인프라, 데이터, 문화)과 '분석 성숙도(CMMI 모델 기반)'의 단계를 구분하는 문제가 자주 출제되니 함께 연계하여 학습하는 것이 좋습니다.
