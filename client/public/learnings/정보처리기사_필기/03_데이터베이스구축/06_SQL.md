# SQL 활용 및 데이터 정의/조작/제어 (1/2)

## 📌 핵심 요약
데이터베이스 구축을 위한 [[DDL]](정의어), [[DML]](조작어), [[DCL]](제어어)의 전반적인 문법과 활용 사례를 다룹니다. 제약 조건의 종류와 참조 무결성 옵션([[CASCADE]], [[RESTRICT]]), 그리고 효율적인 데이터 조회를 위한 다양한 [[SELECT]] 활용법을 상세히 정리합니다.

---

## 📝 상세 정리

## 1. 데이터 정의어 ([[DDL]], Data Definition Language)

### 1.1 [[CREATE]] (데이터베이스 객체 생성)
데이터베이스 객체를 생성하는 명령어입니다.
- **문법:** `CREATE <객체유형> <객체명>(<옵션>);`

> [!INFO] 💡 AI 보충 설명: CREATE TABLE 일반 문법 예시
> ```sql
> CREATE TABLE 사원 (
>     사원번호 INT NOT NULL,
>     성명 CHAR(10) UNIQUE,
>     부서코드 CHAR(10),
>     직급 CHAR(10) CHECK (직급 IN ('사원', '대리', '과장')),
>     PRIMARY KEY (사원번호),
>     FOREIGN KEY (부서코드) REFERENCES 부서(부서코드)
>         ON UPDATE CASCADE
>         ON DELETE SET NULL
> );
> ```

- **데이터 유형:**
    - `INT`: 정수형 숫자
    - `DOUBLE`: 실수형 숫자
    - `CHAR`: 고정 크기 문자열
    - `VARCHAR`: 가변 크기 문자열
    - `DATE`: 날짜 형식
- **필드 제약조건:**
    - `UNIQUE`: 중복된 값을 허용하지 않음
    - `NOT NULL`: NULL 값을 허용하지 않음
    - `CHECK`: 특정 범위나 조건에 맞는 값만 허용
- **테이블 제약조건:**
    - `PRIMARY KEY (<컬럼명>{, ...})`: 기본키를 정의 (중복 불가, NULL 불가)
    - `FOREIGN KEY`: 외래키 정의 (타 테이블의 기본키 참조)
        - **형식:** `FOREIGN KEY (내컬럼) REFERENCES 상대테이블(상대컬럼)`
    - `ON UPDATE`: 참조 데이터 갱신 시 처리 방식 지정
    - `ON DELETE`: 참조 데이터 삭제 시 처리 방식 지정
- **참조 무결성 처리 옵션:**
    - `NO ACTION`: 변화가 있어도 조치를 취하지 않음 (작업 중단)
    - `SET DEFAULT`: 관련 튜플을 기본값으로 지정
    - `[[CASCADE]]`: 관련 튜플을 함께 갱신하거나 삭제
    - `[[RESTRICT]]`: 참조하는 튜플이 없는 경우에만 삭제/갱신 허용
    - `NULLIFY`: 삭제 대신 해당 필드 값을 NULL로 변경
- **테이블 구조 복사:**
    - `SELECT`문에 의해 조회된 테이블 정보와 동일한 형식으로 테이블 생성 가능

### 1.2 [[ALTER]] (테이블 구조 변경)
- **컬럼 변경:**
    - 테이블의 마지막에 컬럼 추가: `ALTER TABLE 테이블명 ADD 컬럼명 데이터유형;`
    - `FIRST`: 테이블의 가장 앞에 컬럼 추가
    - `AFTER 컬럼명`: 특정 컬럼 뒤에 추가
    - **예시:** `ALTER TABLE 학생 ADD 주소 VARCHAR(50) AFTER 성명;`
- **컬럼 세부 변경:**
    - `MODIFY`: 데이터 유형 및 크기 변경. 예) `ALTER TABLE 학생 MODIFY 성명 VARCHAR(30);`
    - `RENAME COLUMN`: 컬럼명 변경. 예) `ALTER TABLE 학생 RENAME COLUMN 성명 TO 이름;`
    - `DROP`: 컬럼 삭제. 예) `ALTER TABLE 학생 DROP COLUMN 주소;`
- **제약조건 변경:**
    - `ADD CONSTRAINT`: 제약조건 추가. 예) `ALTER TABLE 학생 ADD CONSTRAINT 제약명 UNIQUE(학번);`
    - `ENABLE`: 제약조건 활성화
    - `DISABLE`: 제약조건 비활성화
    - `DROP`: 제약조건 삭제

### 1.3 [[DROP]] & [[TRUNCATE]] (객체 및 레코드 삭제)
- **DROP:** `DROP <객체유형> <객체명> [CASCADE | RESTRICT];` (객체 자체를 삭제)
- **TRUNCATE:** 테이블의 구조는 유지하면서 내부의 모든 레코드(데이터)만 삭제

## 2. 데이터 조작어 ([[DML]], Data Manipulation Language)

### 2.1 [[INSERT]] (데이터 삽입)
- **순서대로 모든 값 삽입:** `INSERT INTO 테이블명 VALUES (값1, 값2, ...);`
- **특정 컬럼 지정 삽입:** `INSERT INTO 테이블명 (컬럼1, 컬럼2) VALUES (값1, 값2);`
- **타 테이블 레코드 복사 삽입:** `INSERT INTO 테이블명 (필드...) SELECT 문;`

### 2.2 [[UPDATE]] (데이터 갱신)
- **문법:** `UPDATE 테이블명 SET 컬럼명 = 값, ... WHERE 조건식;`

### 2.3 [[DELETE]] (데이터 삭제)
- **문법:** `DELETE FROM 테이블명 WHERE 조건식;`
- **주의:** 조건식 생략 시 테이블 내 모든 레코드가 삭제됨

### 2.4 [[SELECT]] (데이터 조회)
- 결과를 릴레이션 형태로 출력함

> [!INFO] 💡 AI 보충 설명: SELECT 기본 구조 예시
> ```sql
> SELECT 이름, 학과 
> FROM 학생 
> WHERE 성적 >= 90 
> ORDER BY 이름 ASC;
> ```

- **옵션:**
    - `ALL`: 모든 레코드 검색 (기본값)
    - `DISTINCT`: 중복된 레코드는 하나만 출력
    - `GROUP BY`: 특정 컬럼 기준으로 튜플을 그룹화
    - `HAVING`: 그룹화된 결과에 대한 필터링 조건
    - `ORDER BY`: 결과 정렬 (`ASC`: 오름차순, `DESC`: 내림차순)

## 🎯 빈출 키워드
- **[[CASCADE]]**: 참조되는 테이블의 데이터가 삭제/수정될 때 참조하는 테이블도 연쇄적으로 반영되는 옵션
- **[[RESTRICT]]**: 참조 중인 데이터가 있으면 원본의 삭제/수정을 제한하는 옵션
- **[[DDL]]**: CREATE, ALTER, DROP, TRUNCATE 등 구조를 정의하는 언어
- **[[DML]]**: INSERT, UPDATE, DELETE, SELECT 등 데이터를 관리하는 언어

## 3. SELECT 활용

### 3.1 [[집계 함수]] 및 별칭([[AS]])
- **기능:** 여러 행의 값을 요약하여 하나의 값으로 반환
- **종류:**
    - `COUNT`: 레코드 개수
    - `SUM`: 합계 / `AVG`: 평균
    - `MAX`: 최대값 / `MIN`: 최소값
    - `STDDEV`: 표준편차
    - `VARIANCE`: 분산
- **[[AS]] 문 활용:**
    - **컬럼명 지정 예시:** `SELECT SUM(점수) AS 총점 FROM 성적;`
    - **일반 컬럼명 변경 예시:** `SELECT 이름 AS NAME FROM 학생;`

### 3.2 [[윈도우 함수]] (Window Function)
> [!INFO] 💡 AI 보충 설명: 윈도우 함수란?
> 행과 행 간의 관계를 정의하기 위해 사용하며, [[OLAP]](Online Analytical Processing) 연산 등에 활용됩니다. `OVER` 절과 함께 사용되는 것이 특징입니다.

- **순위 함수:**
    - `RANK`: 동일 순위 시 다음 순위를 건너뜀 (1, 2, 2, 4)
    - `DENSE_RANK`: 동일 순위가 있어도 다음 순위를 이어서 부여 (1, 2, 2, 3)
    - `ROW_NUMBER`: 동일 순위와 상관없이 고유한 일련번호 부여
- **행 순서 함수:**
    - `FIRST_VALUE`: 윈도우 내 첫 번째 값
    - `LAST_VALUE`: 윈도우 내 마지막 값
    - `LAG`: 현재 행 기준 이전 N번째 행의 값
    - `LEAD`: 현재 행 기준 이후 N번째 행의 값

### 3.3 조건식
- **[[AND]]**: 모든 조건을 만족해야 참
- **[[OR]]**: 하나라도 만족하면 참 (문법상 `IN` 연산자로 대체 표현 가능)
- **범위 비교**: `BETWEEN A AND B` ($A \le X \le B$)
- **[[IS NULL]]**: 데이터 값이 비어있는 경우를 검색
- **[[LIKE]] (패턴 검색):**
    - `%강`: '강'으로 끝나는 모든 문자열
    - `강__`: '강'으로 시작하는 세 글자 문자열 (언더바 하나당 한 글자)

### 3.4 하위 질의 ([[서브쿼리]], Subquery)
- **특징:** 메인쿼리 이전에 한 번만 실행되며 소괄호 `()`로 감싸야 함.
- **주의사항:** 메인쿼리와 칼럼 수/행 수가 일치해야 하며, 서브쿼리 내에서는 `ORDER BY`를 사용하지 않음.
- **단일 행 서브쿼리:** 결과가 1개 행. `=`, `<`, `>` 등 연산자 사용.
    - **예시:** `SELECT * FROM 학생 WHERE 성적 > (SELECT AVG(성적) FROM 학생);`
- **다중 행 서브쿼리:** 결과가 여러 행. `IN`, `ANY`, `SOME`, `ALL`, `EXISTS` 사용.
    - **예시:** `SELECT * FROM 과목 WHERE 과목번호 IN (SELECT 과목번호 FROM 수강 WHERE 학점='A');`
- **[[EXISTS]]:** 서브쿼리의 결과가 존재하는지 여부를 확인하여 메인쿼리 실행.
    - **예시:** `SELECT * FROM 사원 WHERE EXISTS (SELECT 1 FROM 급여 WHERE 사원번호 = 사원.사원번호);`

### 3.5 정렬과 그룹화
- **단일/다중 정렬:** `ORDER BY 컬럼1 [ASC|DESC], 컬럼2 [ASC|DESC]`
- **그룹 요약 함수:**
    - `GROUP BY`: 동일 데이터별로 그룹화. 예) `SELECT 성별, COUNT(*) FROM 학생 GROUP BY 성별;`
    - `ROLLUP`: 첫 번째 칼럼 기준 그룹별 합계와 전체 합계 산출
    - `CUBE`: 결합 가능한 모든 컬럼의 그룹별 합계 산출
    - `GROUPING SETS`: 각 그룹별 총 합계만 개별적으로 산출
    - `GROUPING`: 집계가 계산된 결과인지(1) 아닌지(0)를 판별해주는 함수

### 3.6 [[조인]] (Join)
- **[[INNER JOIN]]:** 기준 필드가 일치하는 레코드만 결합.
    - `SELECT * FROM A INNER JOIN B ON A.ID = B.ID;`
- **[[OUTER JOIN]]:** 일치하지 않는 레코드도 포함하여 결합.
    - **[[LEFT JOIN]]:** 왼쪽 테이블 모든 레코드 + 오른쪽 일치 레코드
    - **[[RIGHT JOIN]]:** 오른쪽 테이블 모든 레코드 + 왼쪽 일치 레코드

## 4. 데이터 제어어 ([[DCL]], Data Control Language)

### 4.1 권한 제어
- **[[GRANT]]:** 권한 부여. `GRANT 권한 ON 객체 TO 사용자;`
- **옵션:**
    - `WITH GRANT OPTION`: 부여받은 권한을 다른 사용자에게 재부여 가능
    - `WITH ADMIN OPTION`: 시스템 권한을 재부여 가능
- **[[REVOKE]]:** 권한 회수. `REVOKE 권한 ON 객체 FROM 사용자;` (부여한 권한은 연쇄 회수될 수 있음)
- **[[Role]]:** 권한들의 집합. **[[RBAC]]**(역할 기반 접근 제어)을 통해 효율적으로 관리.

### 4.2 트랜잭션 제어 ([[TCL]])
- **개념:** 논리적인 작업의 최소 단위. 하나 이상의 DML 모음.
> [!INFO] 💡 AI 보충 설명: 트랜잭션의 이해
> 트랜잭션은 은행 송금처럼 '출금'과 '입금'이 모두 성공해야 끝나는 하나의 묶음입니다. 중간에 실패하면 없었던 일로 되돌려야(Rollback) 합니다.

- **[[ACID]] 특징:**
    - **원자성(Atomicity):** 모두 반영되거나, 전혀 반영되지 않아야 함 (All or Nothing)
    - **일관성(Consistency):** 실행 완료 후 데이터베이스 상태가 항상 모순 없이 유지됨
    - **고립성(Isolation):** 트랜잭션 실행 중 타 트랜잭션의 간섭 불가
    - **지속성(Durability):** 성공한 결과는 영구적으로 반영됨
- **상태:** 활동(Active), 부분 완료, 완료(Committed), 실패, 철회(Aborted)
- **제어 명령:**
    - `COMMIT`: 변경사항 최종 반영 (복구 불가능)
    - `ROLLBACK`: 변경 사항 취소 및 복구
    - `[[SAVEPOINT]]`: 롤백할 특정 지점을 지정

## 5. 절차형 SQL
- **구조:** `DECLARE`(정의), `BEGIN~END`(실행), `OR REPLACE`(덮어쓰기)
- **[[프로시저]] (Procedure):** 호출을 통해 실행. **반환값 없음**.
    - 구성: CONTROL, EXCEPTION, SQL, TRANSACTION
- **[[사용자 정의 함수]]:** 호출을 통해 실행. **반환값 존재**.
    - 구성: 마지막에 `RETURN`을 통해 결과값 정의
- **[[트리거]] (Trigger):** 이벤트 발생 시 자동 호출. 입출력/반환값 없음.
    - 구성: `EVENT` (실행 조건) 정의
    - 시점: `AFTER`, `BEFORE` / 대상: `FOR EACH ROW`(레코드마다)
    - 데이터 참조: `NEW`(신규), `OLD`(기존)
- **유의사항:** 트리거 내에서 DCL 사용 금지.

## 6. 인덱스와 뷰

### 6.1 [[인덱스]] (Index)
- **특징:** 검색 속도 향상을 위한 자료구조. 수정이 잦으면 비효율적이며 오버헤드 발생.
- **문법 예시:**
    - 생성: `CREATE INDEX 인덱스명 ON 테이블(컬럼);`
    - 삭제: `DROP INDEX 인덱스명;`

### 6.2 [[뷰]] (View)
- **특징:** 가상 테이블. 보안성 향상 및 복잡한 쿼리 단순화.
- **문법:** `CREATE VIEW 뷰이름 AS SELECT...`
- **옵션:**
    - `FORCE`: 원본 테이블 존재 여부와 상관없이 생성
    - `WITH CHECK OPTION`: 뷰 조건에 위배되는 데이터 수정 불가
    - `WITH READ ONLY`: 데이터 수정(DML) 원천 차단
> [!INFO] 💡 AI 보충 설명: WITH CHECK OPTION은 언제 쓰나?
> 예를 들어 '성적 > 80'인 학생들만 보이는 뷰를 만들었을 때, 실수로 성적을 50으로 수정하면 해당 학생은 뷰에서 사라집니다. 이런 '뷰의 정의를 벗어나는 수정'을 막고 싶을 때 사용합니다.

## 7. 시스템 카탈로그 및 도구
- **[[시스템 카탈로그]]:** 메타데이터를 저장한 데이터 사전. DBMS가 자동 유지관리하며 사용자가 직접 수정 불가능.
- **[[PL/SQL]]:** SQL에 프로그래밍 언어 특성(변수, 제어문)을 추가한 확장 SQL.
- **도구:** `SQL Plus`(오라클), `APM`(모니터링), `TKPROF`(추적/분석), `EXPLAIN PLAN`(경로 분석).

## 8. 병행 제어와 회복

### 8.1 [[병행 제어]] (Concurrency Control)
- **문제점:** 분실된 갱신, 모순성, 연쇄 복귀, 비완료 의존성
- **기법:**
    - **[[로킹]] (Locking):** 타 트랜잭션의 접근 제어. 로킹 단위가 크면 단순하지만 병행성 낮아짐.
    - **타임스탬프:** 실행 순서대로 시간표 부여.
    - **낙관적 제어:** 종료 시점에 한 번에 검사 (충돌 적은 시스템용).

### 8.2 [[회복]] (Recovery)
- **연산:** `REDO`(로그로 재실행), `UNDO`(로그로 취소)
- **기법:**
    - **지연 갱신:** 완료 전까지 로그에만 기록, 완료 후 한 번에 반영
    - **즉시 갱신:** 즉시 반영, 장애 시 로그로 UNDO 수행
    - **[[체크포인트]] 회복:** 검사점 이전은 REDO, 이후는 UNDO
    - **그림자 페이징:** 복사본 페이지를 유지하여 회복

## 🎯 빈출 키워드
- **@([[CASCADE]] / [[RESTRICT]])**: 참조 무결성 유지를 위한 DDL 옵션.
- **@([[ACID]])**: 트랜잭션의 4가지 특성(원자성, 일관성, 고립성, 지속성).
- **@([[JOIN]])**: INNER, LEFT, RIGHT JOIN의 차이점 및 문법.
- **@([[병행 제어]])**: 로킹 단위에 따른 병행성과 오버헤드의 상관관계.
- **@([[윈도우 함수]])**: RANK, DENSE_RANK, ROW_NUMBER의 순위 산정 차이.
