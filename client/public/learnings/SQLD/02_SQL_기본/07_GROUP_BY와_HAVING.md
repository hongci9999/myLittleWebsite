# GROUP BY, HAVING 절

## 핵심 요약

**GROUP BY**(3번): 컬럼 기준 **그룹화** + **집계함수**. **HAVING**(4번): **그룹** 조건 — `AVG(급여) > 1000` 등 **집계함수는 HAVING만**. **WHERE**(2번)는 **행** 필터, **GROUP BY 전**에 먼저 걸러두는 것이 유리. 순서: **WHERE → GROUP BY → HAVING**.

## 왜 중요한가

- 부서별 평균 급여처럼 **요약·통계** 조회의 기본이다.
- **WHERE**(행) vs **HAVING**(그룹) 구분은 시험 **필수**.
- 실행 순서 **FWGHSO**에서 GROUP BY·HAVING 위치를 알아야 오류·함정을 피할 수 있다.

## 1. GROUP BY 절

| 항목 | 내용 |
|------|------|
| **역할** | 특정 **컬럼**을 기준으로 행들을 **그룹화**하여 **집계함수** 결과를 산출 |
| **위치** | `WHERE` 다음, `HAVING`·`SELECT` 이전 |
| **실행 순서** | **(3) GROUP BY** |

### 기본 문법

```sql
SELECT 부서, AVG(급여) AS 평균급여
  FROM 직원
 GROUP BY 부서;
```

| 절 | 역할 |
|----|------|
| **SELECT** | **그룹 기준 컬럼** + **집계함수** (별칭 가능) |
| **GROUP BY** | **같은 값**끼리 묶을 **컬럼**(들) |

---

## 2. 예 — \<직원\> 부서별 평균 급여

### 원본 테이블

| 사번 | 부서 | 급여 |
|------|------|------|
| 101 | 인사 | 800 |
| 102 | 개발 | 1200 |
| 103 | 기획 | 1200 |
| 104 | 개발 | 1300 |
| 105 | 개발 | 1100 |
| 106 | 인사 | 900 |
| 107 | 기획 | 1400 |
| 108 | 인사 | 1200 |
| 109 | 인사 | 1100 |
| 110 | 기획 | 700 |

```sql
SELECT 부서, AVG(급여) AS 평균급여
  FROM 직원
 GROUP BY 부서;
```

### 쿼리 수행 결과 (그룹당 1행)

| 부서 | 평균급여 | 계산 |
|------|----------|------|
| 인사 | **1000** | (800+900+1200+1100) ÷ **4** |
| 개발 | **1200** | (1200+1300+1100) ÷ **3** |
| 기획 | **1100** | (1200+1400+700) ÷ **3** |

```
[직원 10행]  --GROUP BY 부서-->  [부서별 3행 + 집계값]
```

→ **부서** 값이 같은 행끼리 묶이고, 각 그룹에서 **AVG(급여)** 가 **한 번** 계산된다.

---

## 3. GROUP BY 사용 규칙

| 규칙 | 설명 |
|------|------|
| **집계함수와 함께** | GROUP BY 없이 집계만 → **전체 1행**; GROUP BY 있으면 → **그룹별 1행** |
| **SELECT 목록** | **GROUP BY에 넣은 컬럼**(또는 동일 표현) + **집계함수**만 일반적으로 조회 (시험: **비집계 일반 컬럼 단독 SELECT X**) |
| **WHERE와 순서** | **WHERE**로 행을 먼저 걸러낸 뒤 **GROUP BY**로 묶음 |
| **집계함수 in WHERE** | **불가** — `WHERE AVG(급여) > 1000` → **HAVING** 사용 |

```sql
-- 행 필터 → 그룹화
SELECT 부서, AVG(급여) AS 평균급여
  FROM 직원
 WHERE 급여 IS NOT NULL
 GROUP BY 부서;
```

---

## 4. HAVING 절

| 항목 | 내용 |
|------|------|
| **역할** | **그룹화된 결과**에 대한 **조건** 지정 |
| **위치** | `GROUP BY` 다음, `SELECT` 이전 |
| **실행 순서** | **(4) HAVING** |
| **주 사용** | **`GROUP BY`와 함께** (집계 후 그룹 필터) |
| **단독 사용** | 집계 함수만으로 **전체 1그룹**에 조건을 걸 때도 가능(교재 범위) |

### WHERE vs HAVING

| 구분 | WHERE | HAVING |
|------|-------|--------|
| **대상** | **행(튜플)** | **그룹** |
| **실행 순서** | **(2)** | **(4)** |
| **집계함수** | **사용 불가** (`AVG`, `SUM` …) | **사용 가능** |
| **예** | `WHERE 부서 != '기획'` | `HAVING AVG(급여) > 1000` |

> §2 \<직원\> 테이블 공통 사용.

---

## 5. 예 — HAVING만 (평균급여 > 1000)

```sql
SELECT 부서, AVG(급여) AS 평균급여
  FROM 직원
 GROUP BY 부서
HAVING AVG(급여) > 1000;
```

| 단계 | 내용 |
|------|------|
| 1 | `GROUP BY 부서` → 인사 **1000**, 개발 **1200**, 기획 **1100** |
| 2 | `HAVING … > 1000` → **1000 초과**만 남음 |

| 부서 | 평균급여 |
|------|----------|
| 개발 | 1200 |
| 기획 | 1100 |

→ **인사(1000)** 는 `> 1000` **불만족**으로 **제외** (`=` 포함 안 됨).

---

## 6. 예 — WHERE + HAVING 혼합

```sql
SELECT 부서, AVG(급여) AS 평균급여
  FROM 직원
 WHERE 부서 != '기획'
 GROUP BY 부서
HAVING AVG(급여) > 1000;
```

| 단계 | 절 | 내용 |
|------|-----|------|
| 1 | **WHERE** | **기획** 부서 **행 전부 제거** (인사·개발만 남음) |
| 2 | **GROUP BY** | 인사 **1000**, 개발 **1200** |
| 3 | **HAVING** | 평균급여 **> 1000** → 개발만 |

| 부서 | 평균급여 |
|------|----------|
| 개발 | 1200 |

→ **WHERE**로 행을 줄인 뒤 **그룹화**·**HAVING** — 실행 순서와 동일.

### 성능·실무 팁

| 팁 | 설명 |
|----|------|
| **WHERE 먼저** | `GROUP BY`는 **부하가 큼** → 가능하면 **WHERE**에서 행을 먼저 걸러 **그룹 대상 축소** |
| **역할 분리** | 일반 컬럼 조건 → **WHERE**, **집계 결과** 조건 → **HAVING** |

---

## 7. 시험 포인트 / 함정

| 구분 | 내용 |
|------|------|
| GROUP BY | **그룹 기준 컬럼**으로 묶고 **집계함수** 결과 산출 |
| HAVING | **그룹** 조건, **집계함수 사용 가능** |
| 실행 순서 | **WHERE → GROUP BY → HAVING → SELECT** |
| WHERE vs HAVING | 행 vs 그룹, 집계함수 **WHERE X / HAVING O** |
| HAVING `> 1000` | **1000은 제외** (`>=` 와 구분) |
| 성능 | **WHERE**로 먼저 필터 후 **GROUP BY** |
| AVG | 그룹 내 **NULL 급여 제외** 후 평균 |
| 함정 | `SELECT 이름, AVG(급여) … GROUP BY 부서` — **이름** 단독 **X** |
| 함정 | `WHERE AVG(급여) > 1000` → **HAVING** |
| 함정 | `WHERE`에 **기획 제외** + `HAVING`에 **평균 조건** — **둘 다** 적용 순서 기억 |
