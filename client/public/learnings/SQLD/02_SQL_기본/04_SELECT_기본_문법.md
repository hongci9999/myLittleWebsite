# SELECT 기본 문법

## 핵심 요약

`SELECT *`는 **모든 열**을, `SELECT 이름, 나이`는 **지정 열**만 조회한다. **DISTINCT**는 **중복 행**을 제거한다. **WHERE**는 **행 조건** 필터(실행 순서 **2번**). **AS(별칭)** 는 컬럼·테이블 헤더를 바꾸며, **WHERE에서는 SELECT 별칭을 쓸 수 없다**(실행 순서). **문자열 연결**은 Oracle `||`, SQL Server `+` 등 DBMS별 문법이 다르다.

## 왜 중요한가

- 2과목에서 **가장 많이 쓰는** DML 패턴이다.
- DISTINCT·WHERE·별칭은 이후 **JOIN·집계·서브쿼리**의 기초이다.

## 0. 예제 테이블 \<학생\>

| 학번 | 이름 | 나이 |
|------|------|------|
| 101 | 홍길동 | 20 |
| 102 | 이순신 | 21 |
| 103 | 임꺽정 | 20 |
| 104 | 장보고 | 21 |
| 105 | 홍길동 | 20 |

*(교재 예 2·3은 5행 버전 — 101·105 중복 행 포함)*

---

## 1. 전체 열 조회 — `SELECT *`

```sql
SELECT *
  FROM 학생;
```

| 결과 |
|------|
| **모든 컬럼**(학번, 이름, 나이) × **모든 행** |

| 포인트 | 내용 |
|--------|------|
| `*` | 해당 테이블의 **전체 속성(열)** 조회 |
| 실행 | FROM(1) → … → SELECT(5)에서 `*` 확장 |

---

## 2. 특정 열 조회 — `SELECT 열1, 열2`

```sql
SELECT 이름, 나이
  FROM 학생;
```

| 이름 | 나이 |
|------|------|
| 홍길동 | 20 |
| 이순신 | 21 |
| … | … |
| 홍길동 | 20 |

→ **학번**은 빠지고, **이름·나이**만 5행

### DISTINCT (중복 행 제거)

```sql
SELECT DISTINCT 이름, 나이
  FROM 학생;
```

| 효과 |
|------|
| (홍길동, 20)이 **두 번** 있어도 **한 행**만 남김 |
| **행 전체**가 같을 때 중복 제거 (선택한 열 조합 기준) |

| 구분 | 설명 |
|------|------|
| `SELECT` | 중복 행 **그대로** |
| `SELECT DISTINCT` | 조회 결과에서 **중복 행 제거** |

---

## 3. 조건 필터 — `WHERE`

```sql
SELECT 이름, 나이
  FROM 학생
 WHERE 나이 > 20;
```

| 이름 | 나이 |
|------|------|
| 이순신 | 21 |
| 장보고 | 21 |

| 포인트 | 내용 |
|--------|------|
| 역할 | **조건에 맞는 행**만 남김 — 관계대수 **σ** |
| 실행 순서 | **FROM 다음(2번)**, GROUP BY **이전** |
| 비교 | `>`, `<`, `=`, `>=`, `<=`, `<>`, `BETWEEN`, `IN`, `LIKE` 등 |

> WHERE·비교 연산자 상세: [06_WHERE_절과_비교연산자](./06_WHERE_절과_비교연산자.md)

## 4. 컬럼·테이블 별칭(ALIAS)과 문자열 연결

### 4-1. 기본 예 — 컬럼 별칭

```sql
SELECT 학번 AS ID,
       이름
  FROM 학생
 WHERE 학번 = 101;
```

| ID | 이름 |
|----|------|
| 101 | 홍길동 |

→ **WHERE**에는 **원래 컬럼명** `학번` 사용 (`ID` 아님)

### 4-2. 별칭 사용 시 주의사항 (4가지)

| # | 규칙 | 설명 |
|---|------|------|
| 1 | **AS 생략 가능** | `SELECT 학번 ID` 처럼 **AS 없이** 별칭 지정 가능. **FROM** 절에서 **테이블 별칭**도 가능 (예: `FROM 학생 s`) |
| 2 | **이름 규칙** | 별칭은 **숫자로 시작 불가**, **특수문자** 사용 제한. **예약어** 사용 **금지** |
| 3 | **쌍따옴표** | `"전체 이름"` 처럼 **쌍따옴표**로 감싸면 **공백·특수문자** 포함 가능. 따옴표 없이 허용되는 특수문자: **`_`**, **`$`**, **`#`** |
| 4 | **WHERE에서 별칭 불가** | **WHERE**는 **SELECT보다 먼저** 실행되므로, **SELECT 절에서 정한 별칭**을 **WHERE**에서 **사용할 수 없음** |

```sql
-- 오류(또는 DBMS 거부) 예
SELECT 학번 AS ID FROM 학생 WHERE ID = 101;

-- 올바른 예
SELECT 학번 AS ID FROM 학생 WHERE 학번 = 101;
```

> 실행 순서: [03_SELECT_문_구조와_실행순서](./03_SELECT_문_구조와_실행순서.md) — SELECT는 **(5)번**

### 4-3. 문자열 연결(CONCAT) + 별칭

**\<멤버\>** 테이블

| 성 | 이름 |
|----|------|
| 홍 | 길동 |
| 이 | 순신 |
| 임 | 꺽정 |
| 장 | 보고 |

```sql
SELECT 성 || 이름 AS 전체이름
  FROM 멤버;
```

| 전체이름 |
|----------|
| 홍길동 |
| 이순신 |
| 임꺽정 |
| 장보고 |

| 항목 | 내용 |
|------|------|
| 연산 | **문자열 연결(CONCAT)** — 성 + 이름 |
| 별칭 | 결과 열 헤더를 **전체이름**으로 표시 |
| Oracle | **`||`** 로 연결 |
| SQL Server | **`+`** 로 연결 (문자열·숫자 혼용 시 주의) |

```sql
-- Oracle 스타일
SELECT 성 || 이름 AS 전체이름 FROM 멤버;

-- SQL Server 스타일 (참고)
SELECT 성 + 이름 AS 전체이름 FROM 멤버;
```

## 5. 예제 ↔ 실행 순서

| 예 | FROM | WHERE | SELECT(열) |
|----|------|-------|------------|
| `SELECT *` | 학생 | — | `*` |
| `이름, 나이` | 학생 | — | 이름, 나이 |
| `나이 > 20` | 학생 | 나이>20 | 이름, 나이 |
| `AS ID` | 학생 | 학번=101 | 학번→ID, 이름 |
| `성\|\|이름` | 멤버 | — | CONCAT + 별칭 전체이름 |

---

## 6. 시험 포인트 / 함정

| 구분 | 내용 |
|------|------|
| `*` | **모든 열** |
| DISTINCT | **행** 중복 제거 (선택 열 **조합** 기준) |
| WHERE | **행** 필터, 집계 전 |
| AS | **출력 헤더** 변경; **AS 생략**·**FROM 테이블 별칭** 가능 |
| 별칭 규칙 | 숫자 시작 X, 예약어 X; `" "` 로 공백·특수문자; 무인용 시 `_` `$` `#` |
| WHERE vs 별칭 | **SELECT 별칭을 WHERE에서 사용 불가** (FWGHSO 4번 규칙) |
| CONCAT | Oracle **`||`**, SQL Server **`+`** |
| 함정 | DISTINCT는 **열 하나**만 중복 제거하는 기능이 **아님** — **행** 단위 |
| 함정 | `WHERE ID = 101` (별칭) → 실행 순서상 **오류** |
| 함정 | `SELECT *` 남용 → 필요 열만 조회 권장(실무·성능) |
