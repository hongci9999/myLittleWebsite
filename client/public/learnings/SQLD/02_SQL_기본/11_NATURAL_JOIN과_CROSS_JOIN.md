# NATURAL · CROSS · OUTER JOIN

## 핵심 요약

**NATURAL**: **같은 이름 모든 컬럼** EQUI, **ON/USING 없음**. **CROSS**: **카티션 곱**(M×N). **OUTER**: 조건에 **매칭 없어도** 기준 테이블 행 **유지**, 반대쪽 컬럼 **NULL**. **LEFT**=왼쪽 기준, **RIGHT**=오른쪽, **FULL**=양쪽 **전부**.

## 왜 중요한가

- NATURAL·CROSS·OUTER는 **INNER와 결과 행 수**가 다르다.
- LEFT/RIGHT/FULL **NULL 채움 방향**이 시험 **필수**.
- CROSS는 관계대수 **×**, OUTER는 **미매칭 보존**이 핵심이다.

## 1. NATURAL JOIN

| 항목 | 내용 |
|------|------|
| **정의** | 두 테이블에서 **동일한 이름**을 가진 **모든 컬럼**에 대해 **EQUI JOIN** 수행 |
| **조건절** | **`ON`·`USING` 없이** 조인 가능 |
| **성격** | 매칭 없는 행 **제외** → **INNER JOIN**과 유사 |

```sql
SELECT *
  FROM 직원
 NATURAL JOIN 부서;
```

→ `부서ID`처럼 **양쪽에 같은 이름**인 컬럼이 **자동** 조인 키.

### \<직원\> · \<부서\>

**직원**

| 사번 | 이름 | 부서ID |
|------|------|--------|
| 101 | 홍길동 | 10 |
| 102 | 이순신 | 20 |
| 104 | 장보고 | 30 |

**부서**

| 부서ID | 부서명 |
|--------|--------|
| 10 | 인사 |
| 20 | 개발 |
| 40 | 영업 |

### 결과

| 사번 | 이름 | 부서ID | 부서명 |
|------|------|--------|--------|
| 101 | 홍길동 | 10 | 인사 |
| 102 | 이순신 | 20 | 개발 |

| 제외 | 이유 |
|------|------|
| **장보고**(부서ID 30) | 부서 테이블에 **30 없음** |
| **영업**(부서ID 40) | 직원 테이블에 **40 없음** |

→ **공통 컬럼명 `부서ID`** 로만 묶인 **EQUI INNER** 결과.

### NATURAL vs USING

| 구분 | NATURAL JOIN | USING (한 컬럼) |
|------|--------------|-----------------|
| **범위** | **이름 같은 모든 컬럼** 자동 | **지정한 컬럼**만 |
| **ON/USING** | **없음** | `USING (부서ID)` |

> ☞ **ON·USING 없이** 조인한다는 점이 시험 **함정**.

## 2. CROSS JOIN

| 항목 | 내용 |
|------|------|
| **정의** | 두 테이블 **모든 행**을 **조합**해 출력 |
| **연산** | **카티션 곱(Cartesian Product)** — 관계대수 **×** |
| **조건** | **조인 조건 없음** |

```sql
SELECT 학생.이름, 과목.과목명
  FROM 학생
 CROSS JOIN 과목;
```

### \<학생\> · \<과목\>

**학생**

| 학번 | 이름 |
|------|------|
| 101 | 홍길동 |
| 102 | 이순신 |

**과목**

| 과목ID | 과목명 |
|--------|--------|
| 1 | 수학 |
| 2 | 과학 |
| 3 | 컴퓨터 |

### 결과

| 이름 | 과목명 |
|------|--------|
| 홍길동 | 수학 |
| 홍길동 | 과학 |
| 홍길동 | 컴퓨터 |
| 이순신 | 수학 |
| 이순신 | 과학 |
| 이순신 | 컴퓨터 |

| 계산 | 내용 |
|------|------|
| **행 수** | 2 × 3 = **6행** |
| **규칙** | **학생 행 수 × 과목 행 수** |

```
[학생 2행] × [과목 3행] → [6행] (모든 조합)
```

### CROSS JOIN 동치 문법

```sql
FROM 학생, 과목          -- 콤마 조인 (조건 없으면 CROSS)
FROM 학생 CROSS JOIN 과목
```

> 조건을 `WHERE`에 넣으면 **곱집합 후 필터** — 의도와 다를 수 있어 **CROSS JOIN**은 **조건 없이** 쓰는 경우가 많음.

## 3. OUTER JOIN

| 항목 | 내용 |
|------|------|
| **정의** | JOIN 조건에 **동일한 값이 없어도** 행을 **반환할 수 있음** |
| **INNER와 차이** | INNER는 **매칭만**; OUTER는 **기준 테이블** 행을 **남기고** 상대 컬럼을 **NULL** |

```sql
SELECT 학생.이름, 과목.과목명
  FROM 학생
  LEFT JOIN 과목
    ON 학생.과목ID = 과목.과목ID;
```

### 유형

| 유형 | 기준 | 매칭 없을 때 |
|------|------|--------------|
| **LEFT OUTER JOIN** | **왼쪽(좌측)** 테이블 | **오른쪽** 컬럼 → **NULL** |
| **RIGHT OUTER JOIN** | **오른쪽(우측)** 테이블 | **왼쪽** 컬럼 → **NULL** |
| **FULL OUTER JOIN** | **양쪽 모두** | 한쪽만 있으면 **반대편 NULL** |

> `LEFT OUTER JOIN` = `LEFT JOIN` (OUTER 생략 가능). RIGHT·FULL도 동일.

### \<학생\> · \<과목\> (OUTER 예)

**학생**

| 학번 | 이름 | 과목ID |
|------|------|--------|
| 101 | 홍길동 | 1 |
| 102 | 이순신 | 2 |
| 103 | 임꺽정 | NULL |

**과목**

| 과목ID | 과목명 |
|--------|--------|
| 1 | 수학 |
| 3 | 컴퓨터 |

| 매칭 | 설명 |
|------|------|
| 101 ↔ 수학 | 과목ID **1** 일치 |
| 102 | 과목ID **2** — 과목 테이블에 **없음** |
| 103 | 과목ID **NULL** — 매칭 없음 |
| 컴퓨터 | 과목ID **3** — 학생에 **없음** |

### LEFT OUTER JOIN

**왼쪽(학생) 전부** + 오른쪽 없으면 **NULL**

| 이름 | 과목명 |
|------|--------|
| 홍길동 | 수학 |
| 이순신 | NULL |
| 임꺽정 | NULL |

### RIGHT OUTER JOIN

**오른쪽(과목) 전부** + 왼쪽 없으면 **NULL**

| 이름 | 과목명 |
|------|--------|
| 홍길동 | 수학 |
| NULL | 컴퓨터 |

### FULL OUTER JOIN

**양쪽 모든 데이터** — 한쪽만 있어도 포함

| 이름 | 과목명 |
|------|--------|
| 홍길동 | 수학 |
| 이순신 | NULL |
| 임꺽정 | NULL |
| NULL | 컴퓨터 |

```
INNER:  (홍길동, 수학) 만
LEFT:   학생 3행 모두 (미매칭 → 과목명 NULL)
RIGHT:  과목 2행 모두 (미매칭 → 이름 NULL)
FULL:   LEFT ∪ RIGHT (중복 매칭 1행)
```

---

## 4. 조인 유형 한눈에

| 조인 | 매칭 없는 행 | 대표 결과 |
|------|--------------|-----------|
| **INNER** | **제외** | 교집합 |
| **NATURAL** | **제외** (INNER 성격) | 동일 컬럼명 EQUI |
| **CROSS** | (조건 없음) | **M × N** |
| **LEFT** | **왼쪽 유지** | 왼쪽 NULL 채움 없음, **오른쪽 NULL** |
| **RIGHT** | **오른쪽 유지** | **왼쪽 NULL** |
| **FULL** | **양쪽 유지** | **양쪽 NULL** 가능 |

---

## 5. 시험 포인트 / 함정

| 구분 | 내용 |
|------|------|
| NATURAL | **같은 이름 모든 컬럼** EQUI, **ON/USING 없음** |
| CROSS | **카티션 곱**, 행 수 = **A × B** |
| OUTER | **미매칭도** 행 반환, 상대 컬럼 **NULL** |
| LEFT | **좌측 기준**, 우측 없으면 **NULL** |
| RIGHT | **우측 기준**, 좌측 없으면 **NULL** |
| FULL | **좌+우 전체** |
| 함정 | NATURAL에 **ON 필수** → **X** |
| 함정 | LEFT vs RIGHT — **기준 테이블** 방향 혼동 |
| 함정 | OUTER에서 조인 키 **NULL** 행 → 매칭 없음 → **상대 컬럼 NULL** (임꺽정) |
