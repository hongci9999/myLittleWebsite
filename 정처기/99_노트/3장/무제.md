## 관계형 데이터베이스(중요,예시 생성)

개체 타입 entity
현실 세계의 개념이나 대상을 표현하고자 하는 논리적인 표현 단위
다수의 속성으로 표현
개체 타입은 rdb에서 표에 해당아며 테이블 형태로 표현
개체 : 개체 타입의 규ㅜ칙을 통해 생성된 대상, 파일시스템에서 레코드

속성 attribute
개체를 구성하는 고유의 특성, 의미있는 데이터의 가장 작은 논리적 단위
솬련 속성을 모아서 의미있는 개체를 구성
한 개체의 속성값은 파일 시스템에서 필드를 의미, 데이블의 열 에 해당

도메인 domain
하나의 속서ㅗㅇ값이 가질 수 있는 모든 원자 값의 집합
도메인 정으ㅏㅣ가 어려운 경우에는 속성의 특성을 고려한 데이터 타입을 정의
등급 속성 : vip, gold, silver 등으로 정의
성별 속성:  남,냐
이름 속성: 20개의 문자로 데이터 타입 정의


튜플 tuple
하나의 개체(레코드)를 표현하는 완전하고 고유한 정보 단위
각 튜플은 고유해야 하며 순서가 없다
테이블의 행
모이면 릴레이션 인스턴스 라고도 하나?

릴레이션 relation
2차원 테이블 구조
릴레이션 스키마와 릴레이션 인스텀스의 결합
스키마 - 속성명의 집합
어커런스 - 인스턴스와 같은 의미

차수(degree) -속성의 개수
기수 - 튜플의 개수
null - 없음


### 키 key
분류, 정렬, 검색에서 기준이 되는 속성
투플을 유일하게 구별할 수 있는 식별자의 역할

후보기 
키가 될 수 있는 후보들
유일성과 최소성을모두 만족
유일성 - 유일하게 구분할 수 있는 성질
최소성 - 가장 적은 수의 속성으로 구성되는 성질

기본키
후보키들 중 튜플의 식별을 위해 지정된 속성
대체키 - 키본키 속성을 제외한 후보키들

슈퍼키
유일성을 만족하는 속성
둘 이사으이 속성을 합칠 떄
후보키와 달리 최소성 만족 못함

외래키
다른 릴레이션의 키본키를 참조
외래키 자체는 키의 속성을 만족하지 못할수도
