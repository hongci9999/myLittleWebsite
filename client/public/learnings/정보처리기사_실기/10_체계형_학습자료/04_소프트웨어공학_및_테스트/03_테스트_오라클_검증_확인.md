# 테스트 오라클, 검증, 확인

## 핵심 요약

이 단원은 비슷해 보이는 용어를 구분하는 것이 핵심이다. 테스트 오라클 종류, 정적/동적 테스트, Verification/Validation 차이를 제대로 정리하면 단답형과 비교형 문제를 안정적으로 맞힐 수 있다.

## 1. 테스트 오라클

| 종류 | 설명 |
|---|---|
| 참 오라클 | 모든 입력값에 대한 기대 결과 제공 |
| 샘플링 오라클 | 일부 입력값만 기대 결과 제공 |
| 휴리스틱 오라클 | 경험과 추정 활용 |
| 일관성 검사 오라클 | 수정 전후 결과 일관성 확인 |

## 2. 정적 테스트와 동적 테스트

| 구분 | 설명 |
|---|---|
| 정적 테스트 | 실행 없이 문서/코드 분석 |
| 동적 테스트 | 실제 실행 후 결과 확인 |

## 3. Verification vs Validation

| 용어 | 의미 |
|---|---|
| Verification | 제품을 올바르게 만들고 있는가 |
| Validation | 올바른 제품을 만들었는가 |

## 4. 디버깅과 테스트

- 테스트: 결함을 찾는 활동
- 디버깅: 결함 원인을 분석하고 수정하는 활동

## 5. 예제

- 코드 리뷰, 워크스루 -> 정적 테스트
- 프로그램 실행 후 결과 비교 -> 동적 테스트
- 요구사항에 맞는 기능인지 확인 -> Validation

## 6. 자주 틀리는 함정

- Verification과 Validation의 시각(개발자/사용자)을 뒤집는다
- 디버깅을 테스트 기법으로 착각한다
- 샘플링 오라클과 휴리스틱 오라클을 혼동한다

## 7. 시험장에서 푸는 순서

1. 실행 여부로 정적/동적 테스트를 구분한다.
2. 개발 과정 중심인지 결과 적합성 중심인지 보고 Verification/Validation을 판단한다.
3. 오라클은 범위와 정확도로 구분한다.

## 한 페이지 요약

- 참/샘플링/휴리스틱/일관성 검사 오라클을 구분해야 한다.
- 정적 테스트는 실행하지 않고, 동적 테스트는 실행한다.
- Verification은 과정, Validation은 결과 중심이다.
- 디버깅은 테스트 그 자체가 아니다.
