페이징 기법
주기억 장치의 영역을 동일한 크기의 페이지와 페이지 프레임으로 나눈 후 페이지를 페이지 프레임에 적재하는 기법
내부 단편화는 발생 가능
비용 증가, 처리속소 ㄷ감고

세그먼테이션
다양한 크기의 논리적인 세그먼트로 나눈 루 주기억 장치에 적재
외부 단ㅍ쳔화 발생 가능
세그먼트의 시작주소와 상대주ㅗ를 ㅇ니요ㅕㅇ해 실제 물리주소를 계산할 수 있음

페이지 교체 알고리즘
페이지 부재 - 적재하려는 페이지가 페이지의프레임에 존재하지 않는 상태
프레임에 빈 공간이 없을 때 주기억 장치에 적재하는 프레임을 교체하는 방식데 대한 알고리즘


opt
가장 오래 사용하지 않을 페이지를 교체하는 기법
페이지 적중률 hit ratio = 1- 페이지 부재 횟수/참조 횟수
낮은걸 교체

fifo

페이지마다 스택을 두어 현 시점에서 가장 오래 사용하지 않은 페이지 교체
사용하지 않으면 스택 증가
//예시

lfu

참조비트 - 호출되면 1

## 페이지 관리 방식

공산 구혁성 - 특정 위치의 페이지 집중적

워킹 셋

시간에 지나면 변함

## 순차 편성
데이터의 논리적 순서에 따라 물리적으로 연속된 위치에 기록
일괄 처리에 효율적
밀도 좋음

백업에 사용
탐색 효율 떨어짐

임의 편성
직접 편성이라고로
해싱 탐색
처리 속도가 일정
데이터에 직접 접근 ㅏ능

색인 순차 편성 (인덱스)
저장 ㅌ데이터에 대한 색인을 구성하여 랜덤 처리와 일반 순차 처리를 병향
정적 인덱스 - 구조는 변하지 않음 
동적 인덱스 - ㅖ비 구역 필요

색인 구역-  목차를 구성하는 구역

기본 데이터 구혁 - 실제 데이터 편선

오버플로우 구성 - 추가될 데이터가 기록되지 못할 때 별도로 기록하기 위한 구역

융통성 좋음
