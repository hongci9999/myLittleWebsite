# 보안 솔루션 및 보안 운영체제

## 📌 핵심 요약
외부의 불법적인 침입을 차단하는 **[[IPS]]**와 비정상적인 접근을 탐지하는 **[[IDS]]**의 차이점을 분석하고, 네트워크 구성 방식에 따른 보안 강도를 정리합니다. 또한 운영체제 자체에 보안 기능을 강화한 **[[Secure OS]]**의 메커니즘과 주요 기능을 다룹니다.

---

## 📝 상세 정리

## 1. 침입 방지 시스템 ([[IPS]], Intrusion Prevention System)
불법적인 외부 침입을 실시간으로 차단하여 내부 네트워크를 보호합니다. 알려진 패턴뿐만 아니라 알려지지 않은 공격도 자동으로 차단하는 기능을 갖추고 있습니다.

### 1.1 기본 구성 요소
- **스크리닝 라우터 (Screening Router)**: 라우터에 패킷 필터링 기능을 추가. IP 주소와 포트 번호 기반의 차단만 가능하여 세부 제어가 어려움.
- **배스천 호스트 (Bastion Host)**: 외부 네트워크와 내부 네트워크 사이의 게이트웨이 역할을 하는 호스트. 공격의 집중 표적이 되므로 방어 기능이 매우 견고해야 함.

### 1.2 네트워크 구축 방식 (방화벽 구조)

- **단일 홈 게이트웨이**: 배스천 호스트에 네트워크 카드(NIC) 1개 장착.
- **이중 홈 게이트웨이 (Dual-Homed)**: NIC 2개 장착하여 외부와 내부 네트워크를 물리적으로 분리.
- **스크린 호스트 게이트웨이**: 스크리닝 라우터와 배스천 호스트를 결합하여 2단계 방어.
- **스크린 서브넷 게이트웨이**: 외부/내부 스크리닝 라우터 사이에 배스천 호스트를 배치하여 **DMZ(완충 지역)**를 형성. 가장 안전하지만 비용이 많이 듦.

## 2. 침입 탐지 시스템 ([[IDS]], Intrusion Detection System)
시스템이나 네트워크에 대한 비정상적인 접근을 실시간으로 탐지합니다. 방어보다는 **탐지 및 통보**에 중점을 둡니다.

### 2.1 탐지 오류 유형
- **오탐 (False Positive)**: 정상적인 행위를 공격으로 잘못 판단. (정상을 비정상으로)
- **미탐 (False Negative)**: 실제 공격을 탐지하지 못하고 통과시킴. (비정상을 정상으로)

### 2.2 IDS 분류
- **호스트 기반 (HIDS)**: 개별 서버의 로그, 파일 등을 분석. 탐지가 정확하나 시스템 부하가 발생함.
- **네트워크 기반 (NIDS)**: 네트워크 트래픽 전체를 감시. 패킷 손실 가능성이 있으며, 내부(서버 내부)에서 벌어지는 행위는 감지 불가.
- **다중 호스트 기반**: 여러 호스트의 정보를 종합적으로 분석.

### 2.3 탐지 기법 (지식 기반 vs 이상 탐지)
- **지식 기반 탐지 (Knowledge-based / Signature)**:
    - 알려진 공격 패턴(Signature)을 DB에 저장하여 비교.
    - **오탐률이 낮으나**, 새로운 공격(Zero-Day) 탐지가 어려움.
- **이상 탐지 (Anomaly Detection)**:
    - 정상적인 행위 패턴을 설정하고 이를 벗어나는 행위를 탐지.
    - **미탐률이 낮으나**, 정상적인 변화에도 알람이 울려 오탐률이 높음.

## 3. 백도어 및 탐지 도구

### 3.1 [[백도어]] (Backdoor)
정상적인 인증 절차 없이 시스템에 접근하기 위해 설치된 악의적인 통로입니다.
- **탐지 기법**:
    - **무결성 검사**: 파일의 변조 여부 확인.
    - **로그 분석**: 비정상적인 접속 기록 확인.
    - **열린 포트 확인**: `netstat` 등을 통해 의심스러운 포트 확인.

### 3.2 탐지 도구 분류
- **스캐너**: 취약점이나 포트 활성화 여부 점검.
- **침입 발견 / 로그 감시**: 실시간 침입 징후 포착.
- **무결성 검사 도구**: 파일 시스템의 변경 사항 추적.

## 4. 보안 운영체제 ([[Secure OS]])
기존 운영체제(OS)의 커널에 보안 기능을 추가하여 보안 취약점을 근본적으로 해결한 신뢰성 있는 운영체제입니다.

### 4.1 보안 매커니즘
- **암호적 매커니즘**: 데이터 저장 및 전송 시 암호화.
- **논리적 매커니즘**: 접근 제어 논리 설계.
- **시간적 매커니즘**: 특정 시간에만 접근 허용 등.
- **물리적 매커니즘**: 장치 보호.

### 4.2 주요 기능
- **식별 및 인증**: 사용자의 신원을 명확히 확인.
- **접근 통제 (Access Control)**: 사용자별 자원 접근 권한 관리.
- **완전한 중재 (Complete Mediation)**: 모든 접근 시도는 반드시 보안 정책에 따라 검사됨.
- **메모리 재사용 방지**: 이전 프로세스가 사용한 메모리 데이터를 삭제하여 정보 유출 방지.
- **안전한 경로 (Trusted Path)**: 사용자와 보안 커널 간의 신뢰할 수 있는 통신 경로 제공.

> [!INFO] 💡 AI 보충 설명: IDS와 IPS의 결정적 차이
> IDS는 'CCTV'와 같습니다. 침입이 발생하면 관리자에게 알람을 울리지만 직접 막지는 못합니다. 반면 IPS는 '무장 경비원'과 같습니다. 침입이 감지되는 즉시 해당 패킷을 차단하거나 연결을 끊는 능동적인 조치를 취합니다.
