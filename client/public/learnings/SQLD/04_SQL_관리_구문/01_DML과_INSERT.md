# 관리 구문 — DML과 INSERT

## 핵심 요약

**DML**(데이터 조작언어)은 **INSERT·UPDATE·DELETE·MERGE**로 **행 데이터**를 바꾼다. **INSERT**는 테이블에 **새 행**을 넣는다. **전체 컬럼**이면 `VALUES(값…)`만 쓰고 **테이블 정의 순서**대로 값을 넣는다. **일부 컬럼**이면 `(컬럼 목록)`을 쓰고, **명시하지 않은 컬럼은 NULL**.

## 왜 중요한가

- 3과목 **관리 구문**의 출발점이다.
- **컬럼 순서**·**NULL 기본**은 시험 **선택형·서술형** 함정이다.
- 이후 **UPDATE·DELETE·MERGE**·**DDL**과 함께 묶어 출제된다.

> 조회(DQL): [02_SQL_기본](../02_SQL_기본/) · 활용: [03_SQL_활용](../03_SQL_활용/)

---

## 1. DML (Data Manipulation Language)

| 항목 | 내용 |
|------|------|
| **한글** | 데이터 **조작**언어 |
| **역할** | 테이블 **행 데이터** 추가·변경·삭제 |
| **구문** | **INSERT**, **UPDATE**, **DELETE**, **MERGE** |

| 구문 | 역할 |
|------|------|
| **INSERT** | **새 행** 추가 |
| **UPDATE** | 기존 행 **수정** |
| **DELETE** | 행 **삭제** |
| **MERGE** | 조건에 따라 **INSERT 또는 UPDATE** (DBMS) |

## 2. 예제 테이블 — \<학생\>

| 학번 | 이름 | 나이 | 학년 |
|------|------|------|------|
| 1 | … | … | … |
| 2 | … | … | … |
| 3 | … | … | … |
| 4 | … | … | … |

| 컬럼 | 설명 |
|------|------|
| **학번** | 학생 식별 |
| **이름** | 이름 |
| **나이** | 나이 |
| **학년** | 학년 |

---

## 3. INSERT — 행 추가

| 항목 | 내용 |
|------|------|
| **역할** | 테이블에 **새로운 데이터(행)** 추가 |
| **기본 형태** | `INSERT INTO 테이블 … VALUES (…)` |

---

## 4. (1) 전체 컬럼 값 추가

**모든 컬럼**에 값을 넣을 때는 **컬럼 이름을 생략**할 수 있다.

```sql
INSERT INTO 학생 VALUES (5, '길동', 23, 1);
```

| 항목 | 내용 |
|------|------|
| **VALUES(값1, 값2, …)** | 삽입할 값을 **테이블 컬럼 정의 순서**대로 입력 |
| **순서** | `학번` → `이름` → `나이` → `학년` 순과 **일치**해야 함 |

### Before → After

| 학번 | 이름 | 나이 | 학년 |
|------|------|------|------|
| 1 | … | … | … |
| … | … | … | … |
| 4 | … | … | … |

↓ **1행 추가**

| 학번 | 이름 | 나이 | 학년 |
|------|------|------|------|
| … | … | … | … |
| **5** | **길동** | **23** | **1** |

---

## 5. (2) 명시된 컬럼 값 추가

**일부 컬럼**만 지정해 넣을 수 있다.

```sql
INSERT INTO 학생 (학번, 이름, 학년)
VALUES (6, '선영', 1);
```

| 항목 | 내용 |
|------|------|
| **(학번, 이름, 학년)** | 값을 넣을 **컬럼만** 나열 |
| **VALUES** | 나열한 컬럼 **순서**에 맞게 값 입력 |
| **명시 안 한 컬럼** | **`나이`** → **NULL** (기본) |

### Before → After

| 학번 | 이름 | 나이 | 학년 |
|------|------|------|------|
| … | … | … | … |
| 5 | 길동 | 23 | 1 |

↓ **1행 추가**

| 학번 | 이름 | 나이 | 학년 |
|------|------|------|------|
| … | … | … | … |
| **6** | **선영** | **NULL** | **1** |

> `나이`를 넣지 않았으므로 **NULL**이 들어간다. (NOT NULL 제약이 있으면 **오류**.)

---

## 6. INSERT 방식 비교

| 구분 | 전체 컬럼 | 지정 컬럼 |
|------|-----------|-----------|
| **컬럼 목록** | 생략 가능 | **필수** `(col1, col2, …)` |
| **VALUES 순서** | **테이블 정의 순서** | **나열한 컬럼 순서** |
| **빠진 컬럼** | 없음(전부 값 필요) | **NULL** (또는 DEFAULT) |
| **예** | `INSERT INTO 학생 VALUES (5, '길동', 23, 1)` | `INSERT INTO 학생 (학번, 이름, 학년) VALUES (6, '선영', 1)` |

---

## 7. 시험 포인트 / 함정

| 구분 | 내용 |
|------|------|
| DML 4종 | **INSERT·UPDATE·DELETE·MERGE** |
| INSERT | **행 추가** |
| VALUES 순서 | **컬럼 정의·명시 순서**와 일치 |
| 지정 컬럼 | **미명시 컬럼 = NULL** |
| 함정 | 컬럼 생략 시 **개수·순서** 틀리면 오류 |
| 함정 | NOT NULL 컬럼에 NULL 삽입 시 **실패** |
| 함정 | SELECT로 넣기(`INSERT … SELECT`)는 별도 유형 |
