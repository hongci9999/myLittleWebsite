# 탐색(Search) 알고리즘 및 복잡도

## 핵심 요약
데이터 집합에서 원하는 항목을 찾는 **[[탐색]]** 기법(선형, 이분, 보간, 블록, 해싱)과 알고리즘의 효율성을 평가하는 **[[시간 복잡도]]**(Big-O)를 정리함. 특히 **이분 탐색**의 조건(정렬)과 **해싱**의 충돌 해결 방법은 시험의 핵심 출제 포인트임.

---

## 1. 탐색(Search) 개요

### 1) 탐색 장소에 따른 분류
* **내부 탐색**: 데이터가 **주기억 장치(Memory)**에 있을 때 수행. 속도가 빠름.
* **외부 탐색**: 데이터가 **보조기억 장치(HDD/SSD)**에 있을 때 수행. 대용량 데이터, 속도가 상대적으로 느림.

### 2) 주요 탐색 기법 요약
* **[[선형 탐색]] (Linear)**: 처음부터 끝까지 순차적으로 비교.
* **[[이분 탐색]] (Binary)**: 데이터를 절반씩 나누어가며 검색 (정렬 필수).
* **[[보간 탐색]] (Interpolation)**: 데이터의 위치를 예상하여 검색하는 사전식 탐색.
* **[[블록 탐색]] (Block)**: 데이터를 그룹별로 블록화하여 인덱싱 후 검색.
* **[[이진 트리 탐색]] (BST)**: 이진 탐색 트리로 변환하여 검색.
* **[[해싱 탐색]] (Hashing)**: 해시 함수를 이용하여 즉시 주소 계산.

---

## 2. 알고리즘 복잡도 (Complexity)

### 1) 공간 복잡도 (Space Complexity)
* 프로그램 실행 시 사용되는 **메모리 공간의 크기**.
* 최근 하드웨어 발달로 시간 복잡도에 비해 비중은 낮음.

### 2) [[시간 복잡도]] (Time Complexity) `중요`
* 연산이 수행되는 횟수. (실행 시간은 하드웨어마다 다를 수 있으므로 횟수 기준)
* 주로 **최악의 경우(Worst Case)**를 기준으로 판단함.
* **[[빅오 표기법]] (Big-O Notation)** 사용.

#### 속도 비교 (빠름 $\rightarrow$ 느림)
> $O(1) < O(\log n) < O(n) < O(n \log n) < O(n^2) < O(2^n) < O(n!)$

| 표기법 | 의미 | 대표 알고리즘 |
| :--- | :--- | :--- |
| **$O(1)$** | 상수형 | **[[해싱]]** (데이터 수와 무관하게 일정) |
| **$O(\log n)$** | 로그형 | **[[이분 탐색]]**, 이진 트리 (증가 폭이 점점 줄어듦) |
| **$O(n)$** | 선형 | **[[선형 탐색]]** (데이터 수에 정비례) |
| **$O(n \log n)$** | 선형 로그형 | **퀵 정렬**, **병합 정렬**, **힙 정렬** (효율적인 정렬) |
| **$O(n^2)$** | 제곱형 | **버블 정렬**, **선택 정렬**, **삽입 정렬** (기본 정렬) |
| **$O(2^n)$** | 지수형 | 피보나치 수열 (재귀 호출 시) |
| **$O(n!)$** | 팩토리얼 | 외판원 순회 문제 (가장 느림) |

## 3. 주요 탐색 알고리즘 상세

### 1) [[선형 탐색]] (Linear Search)
* **특징**: 첫 번째 데이터부터 마지막까지 **순차적**으로 비교.
* **조건**: 데이터의 정렬 여부나 개수를 몰라도 사용 가능. 데이터 수가 적을 때 유리.
* **복잡도**: $O(n)$.
* **평균 비교 횟수**: $\frac{n+1}{2}$.

### 2) [[이분 탐색]] (Binary Search) `빈출`
* **특징**: 중앙값(Middle)과 비교하여 검색 범위를 **절반**으로 줄여나감.
* **조건**: 반드시 데이터가 **정렬(Sort)** 되어 있어야 함. 데이터 개수를 알아야 함.
* **복잡도**: $O(\log_2 n)$.
* **종료 조건**: 검색 범위의 시작(Low)이 끝(High)보다 커지면 검색 실패. ($L > H$)

### 3) [[블록 탐색]] (Block Search)
* **특징**: 전체 데이터를 여러 블록으로 나누고, 각 블록의 대표값(최대값)을 **인덱스**에 저장하여 검색. (색인 순차 검색)
* **과정**: 인덱스에서 블록을 찾음 $\rightarrow$ 해당 블록 내부에서 **순차 탐색** 진행.
* **조건**: 인덱스는 정렬되어 있어야 함. 블록 내 데이터는 정렬될 필요는 없으나, 논리적 순서는 지켜야 함.
* **이상적인 블록 개수**: $\sqrt{n}$.

### 4) [[보간 탐색]] (Interpolation Search)
* **특징**: 찾으려는 키값이 있을만한 위치를 공식으로 **예상**하여 검색. (사전에서 단어 찾기 방식)
* **조건**: 데이터가 일정한 규칙(분포)을 가지고 나열되어 있어야 함.
* **복잡도**: 평균 $O(\log(\log n))$.
* **위치 계산 공식**:
    $$Position = Low + \frac{Find - Min}{Max - Min} \times (High - Low)$$
    *(찾을 값에서 최소값을 뺀 비율을 전체 범위에 적용)*

### 5) [[이진 탐색 트리]] (BST)
* **특징**: 데이터를 이진 트리로 구성하여 검색.
* **규칙**:
    * **Left < Root < Right** (왼쪽 자식은 부모보다 작고, 오른쪽은 부모보다 큼).
* **과정**: 첫 데이터를 근(Root) 노드로 지정하고 규칙에 따라 삽입/검색.
* **복잡도**: 평균 $O(\log n)$, 최악(경사 트리) $O(n)$.

## 4. [[해싱]] (Hashing) 탐색

### 1) 개념
* **[[해시 함수]] (Hash Function)**를 이용하여 데이터의 키(Key) 값을 메모리 주소(Index)로 변환, 즉시 접근하는 방식.
* **복잡도**: $O(1)$ (가장 빠름).

### 2) 주요 용어
* **해시 테이블 (Hash Table)**: 해싱된 데이터가 저장되는 기억 공간.
* **슬롯 (Slot)**: 한 개의 레코드를 저장하는 공간.
* **버킷 (Bucket)**: 여러 개의 슬롯이 모인 덩어리. (주소 하나에 할당된 공간).
* **동의어 (Synonym)**: 서로 다른 키(Key)가 해시 함수를 통과했는데 **같은 주소**가 나온 경우.
* **[[충돌]] (Collision)**: 동의어 발생으로 인해 데이터를 저장할 공간이 부족한 현상.
* **오버플로우 (Overflow)**: 버킷이 꽉 차서 더 이상 저장할 수 없는 상태.

### 3) 충돌 해결 방법 (Collision Resolution)
* **개방 주소법 (Open Addressing)**: 충돌 발생 시 비어있는 다른 공간을 찾아 저장 (**Probing**).
    * 선형 탐사(Linear Probing): +1 칸씩 이동.
* **체인법 (Chaining)**: 버킷에 연결 리스트(Linked List)를 할당하여 데이터를 계속 연결.

> [!INFO] 💡 AI 보충 설명: 해싱 예시 (Modulo 연산)
> **상황**: 해시 테이블의 크기(버킷 수)가 **5**이고, 해시 함수가 **Key % 5** (나머지 연산)일 때.
>
> 1.  **데이터 저장**:
>     * `Key=12`: $12 \% 5 = 2$ $\rightarrow$ **2번지** 저장.
>     * `Key=7`: $7 \% 5 = 2$ $\rightarrow$ **2번지** 저장 시도. (이미 12가 있음)
>
> 2.  **문제 발생**:
>     * 서로 다른 키(12, 7)가 같은 주소(2)를 가리킴. 이를 **동의어(Synonym)**라 하고, 이 현상을 **충돌(Collision)**이라 함.
>
> 3.  **해결 (체인법 사용 시)**:
>     * 2번지 버킷에 **연결 리스트**를 달아서 `12 -> 7` 순서로 줄줄이 매달아 저장함.
>
> 4.  **검색**:
>     * `7`을 찾을 때: $7 \% 5 = 2$번지로 가서, 연결된 리스트를 뒤져서 `7`을 발견함.

---

## 5. 핵심 단어 정리
- **시간 복잡도**: $O(1)$(해시) < $O(\log n)$(이분) < $O(n)$(선형).
- **이분 탐색 조건**: 데이터가 **정렬**되어 있어야 함.
- **해싱**: 충돌(Collision) 해결책으로 **체이닝**과 **개방 주소법**이 있음.
- **보간 탐색**: 위치를 예상(비례식)하여 탐색.
