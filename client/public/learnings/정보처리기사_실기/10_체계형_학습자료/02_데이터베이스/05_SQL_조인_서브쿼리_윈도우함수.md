# SQL 조인, 서브쿼리, 윈도우 함수

## 핵심 요약

실기 SQL의 고득점 포인트는 조인과 서브쿼리다. 여기에 `EXISTS`, `IN`, `ANY`, `ALL`, 순위 함수까지 정확히 구분하면 까다로운 문제도 안정적으로 풀 수 있다.

## 왜 중요한가

- 다중 테이블 결과를 해석하는 문제는 실기에서 매우 자주 출제된다.
- `JOIN` 과 `서브쿼리` 는 비슷한 결과를 만들 수 있어 헷갈리기 쉽다.
- `RANK`, `DENSE_RANK`, `ROW_NUMBER` 차이는 대표 암기 포인트다.

## 1. JOIN

### INNER JOIN

- 양쪽에서 일치하는 행만 반환

```sql
SELECT e.이름, d.부서명
FROM 사원 e
JOIN 부서 d ON e.부서코드 = d.부서코드;
```

### LEFT OUTER JOIN

- 왼쪽 테이블 행은 모두 유지
- 오른쪽에 일치값 없으면 NULL

```sql
SELECT e.이름, d.부서명
FROM 사원 e
LEFT JOIN 부서 d ON e.부서코드 = d.부서코드;
```

## 2. 서브쿼리

서브쿼리는 쿼리 안에 포함된 또 다른 쿼리다.

### 단일 행 서브쿼리

```sql
SELECT *
FROM 사원
WHERE 급여 > (SELECT AVG(급여) FROM 사원);
```

### 다중 행 서브쿼리

```sql
SELECT *
FROM 사원
WHERE 부서코드 IN (SELECT 부서코드 FROM 부서 WHERE 지역 = '서울');
```

## 3. EXISTS, ANY, ALL

| 연산 | 의미 |
|---|---|
| `EXISTS` | 결과가 한 행이라도 있으면 참 |
| `ANY` | 여러 값 중 하나 이상과 비교 만족 |
| `ALL` | 모든 값과 비교 만족 |
| `IN` | 목록 안에 포함되면 참 |

### 예시

```sql
SELECT *
FROM 고객 c
WHERE EXISTS (
  SELECT 1
  FROM 주문 o
  WHERE o.고객번호 = c.고객번호
);
```

## 4. 상관 서브쿼리

- 바깥 쿼리의 값을 안쪽 쿼리에서 참조한다.
- 보통 `EXISTS` 와 함께 자주 나온다.

## 5. 윈도우 함수

윈도우 함수는 집계와 달리 행을 유지한 채 순위나 누적값을 계산할 수 있다.

| 함수 | 특징 |
|---|---|
| `RANK()` | 공동 순위 뒤 순위 건너뜀 |
| `DENSE_RANK()` | 공동 순위 뒤 순위 건너뛰지 않음 |
| `ROW_NUMBER()` | 무조건 고유 번호 |

### 예시

```sql
SELECT 이름, 점수,
       RANK() OVER (ORDER BY 점수 DESC) AS 순위
FROM 학생;
```

## 6. 코드 추적 예제

### 예제 1: INNER vs LEFT JOIN

사원 테이블에 부서코드가 없는 사원이 있다고 하자.

- `INNER JOIN` -> 부서가 없는 사원은 결과에서 제외
- `LEFT JOIN` -> 부서가 없는 사원도 결과에 나오고 부서명은 NULL

### 예제 2: `IN` 과 `EXISTS`

```sql
SELECT *
FROM 고객 c
WHERE EXISTS (
  SELECT 1
  FROM 주문 o
  WHERE o.고객번호 = c.고객번호
);
```

- 주문 이력이 있는 고객만 출력
- 존재 여부만 보므로 `SELECT 1` 형태를 자주 사용

### 예제 3: 순위 함수 차이

점수가 `100, 90, 90, 80` 이라면:

- `RANK()` -> `1, 2, 2, 4`
- `DENSE_RANK()` -> `1, 2, 2, 3`
- `ROW_NUMBER()` -> `1, 2, 3, 4`

## 7. 자주 틀리는 함정

### 7.1 JOIN 조건 누락

- 카테시안 곱이 발생해 행 수가 급증한다.

### 7.2 `NOT IN` 과 NULL

- 서브쿼리 결과에 NULL이 섞이면 직관과 다르게 동작할 수 있다.
- 이런 경우 `NOT EXISTS` 가 더 안전한 선택인 문제 유형이 있다.

### 7.3 `ANY` 와 `ALL` 혼동

- `> ANY` 는 하나보다만 크면 됨
- `> ALL` 은 전부보다 커야 함

## 8. 시험장에서 푸는 순서

1. 단일 테이블인지 다중 테이블인지 먼저 본다.
2. 조인 문제면 기준 테이블과 조인 조건을 적는다.
3. 서브쿼리 문제면 결과가 단일 행인지 다중 행인지 본다.
4. `EXISTS`, `IN`, `ANY`, `ALL` 의 의미를 비교한다.
5. 순위 함수가 나오면 공동 순위 처리 방식을 확인한다.

## 한 페이지 요약

- INNER JOIN 은 일치 행만, LEFT JOIN 은 왼쪽 전체를 유지한다.
- 서브쿼리는 단일 행과 다중 행으로 나눠 본다.
- `EXISTS` 는 존재 여부, `IN` 은 포함 여부를 본다.
- `ANY` 는 하나 이상, `ALL` 은 전부를 만족해야 한다.
- `RANK`, `DENSE_RANK`, `ROW_NUMBER` 차이는 자주 출제된다.
