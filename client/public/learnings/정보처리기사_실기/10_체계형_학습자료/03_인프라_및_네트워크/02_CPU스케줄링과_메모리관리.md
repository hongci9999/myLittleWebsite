# CPU 스케줄링과 메모리 관리

## 핵심 요약

운영체제 계산형 문제의 중심은 CPU 스케줄링이다. 여기에 메모리 관리, 가상기억장치, 페이징 개념이 함께 묶여 출제되므로 알고리즘 특징과 용어를 같이 정리해야 한다.

## 1. CPU 스케줄링 목적

- CPU 이용률 향상
- 처리량 증가
- 반환시간/대기시간 감소

## 2. 주요 스케줄링 알고리즘

| 알고리즘 | 특징 |
|---|---|
| FCFS | 먼저 온 작업 먼저 처리 |
| SJF | 작업 시간이 짧은 것 우선 |
| HRN | 응답률 고려 |
| RR | 시간 할당량 방식 |
| SRT | 남은 시간이 짧은 작업 우선 |

## 3. 선점형과 비선점형

| 구분 | 설명 |
|---|---|
| 비선점형 | CPU를 스스로 반납할 때까지 사용 |
| 선점형 | 운영체제가 강제로 CPU 회수 가능 |

- FCFS, SJF, HRN: 비선점형
- RR, SRT: 선점형

## 4. RR 핵심

- 일정 시간 할당량(Time Quantum)을 준다
- 시간이 끝나면 다음 프로세스로 교체
- 응답시간 개선에 유리

## 5. 메모리 관리 핵심

### 연속 할당

- 메모리를 연속된 공간에 할당
- 외부 단편화 발생 가능

### 페이징

- 논리 메모리를 페이지 단위로 분할
- 물리 메모리를 프레임 단위로 분할
- 외부 단편화 문제를 줄인다

### 세그먼테이션

- 프로그램의 논리적 단위(코드, 데이터 등)로 분할

## 6. 가상기억장치

- 보조기억장치를 주기억장치처럼 사용하는 기법
- 실제 메모리보다 큰 프로그램 실행 가능

### 관련 용어

- 페이지 부재(Page Fault)
- 작업 집합(Working Set)
- 스래싱(Thrashing)

## 7. 예제

### 예제 1: FCFS 평균 대기시간

작업 실행시간이 `P1=3`, `P2=5`, `P3=2` 라고 하자.

- P1 대기시간 = 0
- P2 대기시간 = 3
- P3 대기시간 = 3 + 5 = 8
- 평균 대기시간 = `(0 + 3 + 8) / 3 = 11/3`

### 예제 2: RR 특징

- 응답은 빨라질 수 있음
- 문맥 교환이 많아질 수 있음

## 8. 자주 틀리는 함정

- 선점형/비선점형을 뒤집어 외운다
- 평균 대기시간에서 마지막 작업의 누적 시간을 놓친다
- 페이징과 세그먼테이션을 같은 개념으로 본다
- 페이지 부재와 스래싱을 구분하지 못한다

## 9. 시험장에서 푸는 순서

1. 스케줄링 문제면 선점형인지 비선점형인지 먼저 본다.
2. 작업 도착 순서와 실행 시간을 표로 쓴다.
3. 대기시간을 각 작업별로 계산한 뒤 평균을 낸다.
4. 메모리 문제는 페이징/세그먼테이션/가상기억장치를 먼저 분류한다.

## 한 페이지 요약

- FCFS, SJF, HRN은 비선점형, RR, SRT는 선점형이다.
- RR은 시간 할당량 기반이다.
- 페이징은 페이지와 프레임 단위로 메모리를 관리한다.
- 가상기억장치는 보조기억장치를 활용해 큰 프로그램을 실행하게 한다.
