## 📌 핵심 요약
빅데이터 플랫폼은 자원 관리부터 분석 서비스까지 계층적 구조(인프라, 플랫폼, 소프트웨어)를 가지며, 인공지능(AI)은 머신러닝과 딥러닝을 포괄하는 개념으로 발전하고 있습니다. 또한, 데이터 활용의 핵심인 '데이터 3법'을 통해 가명정보 개념이 도입되었으며, 개인정보 비식별 조치와 알고리즘 부작용에 따른 통제 방안이 강조됩니다.

---

## 📝 상세 정리

### 1. 빅데이터 플랫폼 (Big Data Platform)
데이터의 수집부터 저장, 처리, 분석 등의 파이프라인(Pipeline) 전 과정을 통합적으로 제공하는 환경을 의미합니다.

#### (1) 빅데이터 플랫폼의 계층 구조
| 계층 | 명칭 | 주요 역할 및 기능 |
| :--- | :--- | :--- |
| **상위 계층** | **소프트웨어 계층 (Software Layer)** | 데이터 수집 및 정제, 데이터 처리 및 분석, 사용자/서비스 관리 |
| **중간 계층** | **플랫폼 계층 (Platform Layer)** | 데이터 및 자원의 관리, 작업 스케줄링(Scheduling), 프로파일링(Profiling) |
| **하위 계층** | **인프라스트럭처 계층 (Infrastructure Layer)** | 자원의 배치 및 관리, 저장장치 및 네트워크 관리 |

> [!INFO] 💡 구조 요약
> 소프트웨어 계층이 최상단에서 사용자 서비스를 담당하며, 인프라스트럭처 계층이 최하단에서 물리적/가상적 자원을 지원합니다.

#### (2) 범위에 따른 구분
1. **광의(廣義, Broad Sense)의 분석 플랫폼**: 분석 서비스 제공 엔진, 분석 어플리케이션, 분석 서비스 API, 하드웨어를 모두 포함.
2. **협의(狹義, Narrow Sense)의 분석 플랫폼**: 데이터 처리 프레임워크, 분석 엔진, 분석 라이브러리 위주의 핵심 구성 요소.
> [!INFO] 💡 관계
> 광의의 플랫폼은 협의의 플랫폼 요소를 모두 포함하는 상위 개념입니다.

### 2. 빅데이터와 인공지능 (Artificial Intelligence, AI)

#### (1) 인공지능, 머신러닝, 딥러닝의 관계
세 개념은 포함 관계를 가집니다.
- **관계식**: $Deep Learning \subset Machine Learning \subset Artificial Intelligence$

#### (2) 머신러닝 (Machine Learning, ML)의 종류
학습 방식과 **정답(Label)**의 유무에 따라 구분됩니다.
- **지도 학습 (Supervised Learning)**: 정답(Label)을 알려주고 모델을 학습시키는 방법.
- **비지도 학습 (Unsupervised Learning)**: 정답 없이 데이터의 패턴이나 구조를 스스로 학습하는 방법.
- **준지도 학습 (Semi-supervised Learning)**: 정답이 있는 소량의 데이터와 정답이 없는 대량의 데이터를 모두 활용.
- **강화 학습 (Reinforcement Learning)**: 에이전트(Agent)가 환경과 상호작용하며 보상(Reward)을 최대화하기 위해 학습.

#### (3) 인공지능의 수준별 분류
- **약인공지능 (Weak AI)**: 특정 도메인이나 한정된 조건에서만 작업을 수행 (예: 알파고, 챗봇).
- **강인공지능 (Strong AI)**: 인간과 동일한 사고가 가능하며, **튜링 테스트(Turing Test)**를 통과할 수 있는 수준.
- **초인공지능 (Super AI)**: 기술적 특이점(Singularity)을 넘어 인간의 지능을 초월한 수준.

#### (4) 경량 딥러닝 학습 기법
- **전이 학습 (Transfer Learning)**: 사전에 훈련된 모델(Pre-trained Model)의 지식을 새로운 과업에 재사용.
- **미세 조정 (Fine-Tuning)**: 사전 학습된 모델을 특정 데이터셋(Target)에 맞게 가중치를 미세하게 재조정.
- **지식 증류 (Knowledge Distillation)**: 거대 모델(Teacher Network)의 지식을 작은 모델(Student Network)에 전달하여 효율화.

#### (5) 최신 인공지능 기술 동향
- **AutoML**: 머신러닝 모델 구축의 전 과정(전처리, 모델 선택, 하이퍼파라미터 튜닝)을 자동화.
- **MLOps**: Machine Learning + Operations의 합성어로, ML 모델의 개발과 운영을 통합하는 체계.
- **XAI (eXplainable AI)**: 인공지능의 판단 결과에 대해 인간이 이해할 수 있도록 이유를 설명하는 기술.
- **생성형 AI (Generative AI)**: 기존 데이터를 학습하여 새로운 콘텐츠를 생성.
    - **텍스트 생성**: ChatGPT, BERT 등
    - **이미지 생성**: Midjourney, Stable Diffusion 등
- **LLM (Large Language Model)**: 수십억 개 이상의 파라미터를 가진 거대 언어 모델.
- **확산 모델 (Diffusion Models)**: 노이즈를 점진적으로 제거하는 확률적 프로세스를 통해 고품질 데이터를 생성.

### 3. 개인정보 법제도 및 활용

#### (1) 데이터 3법 (개·정·신)
1. **개인정보 보호법**
2. **정보통신망 이용 촉진 및 정보보호 등에 관한 법률 (정보통신망법)**
3. **신용정보의 이용 및 보호에 관한 법률 (신용정보법)**

#### (2) 데이터 3법의 주요 특징
- **가명정보(Pseudonymized Data)** 개념 도입: 통계 작성, 과학적 연구, 공익적 기록보존 목적 시 동의 없이 활용 가능.
- 개인정보보호 거버넌스 체계 효율화 (개인정보보호위원회 중앙행정기관 격상).
- 개인정보처리자의 책임성 강화 및 판단 기준 명확화.

#### (3) 데이터 식별 가능성에 따른 비교
| 구분 | 정의 | 특징 및 예시 |
| :--- | :--- | :--- |
| **개인정보** | 특정 개인을 직접 알아볼 수 있는 정보 | 활용 시 정보 주체의 동의 필수 (예: 홍길동, 33세) |
| **가명정보** | 추가 정보 없이는 특정 개인을 식별할 수 없도록 처리한 정보 | 특정 목적 하에 동의 없이 활용 가능 (예: 홍OO, 30대 초반) |
| **익명정보** | 시간·비용·기술적으로 더 이상 개인을 알아볼 수 없는 정보 | 제한 없이 자유롭게 활용 가능 (예: OOO, 30대) |

> [!INFO] 💡 중요 포인트 (재식별 가능성)
> 익명정보는 재식별이 불가능한 상태이지만, 가명정보는 다른 추가 정보와 결합 시 다시 개인을 특정할 수 있는 잠재적 가능성이 존재합니다.

#### (4) 개인정보 비식별 조치 가이드라인 (4단계)
1. **사전 검토**: 대상 데이터에 개인정보가 포함되어 있는지 확인.
2. **비식별 조치**: 총계 처리, 데이터 삭제, 마스킹(Masking) 등을 적용.
3. **적정성 평가**: 비식별 수준이 충분한지 평가 ($K$-익명성, $L$-다양성, $T$-근접성 모델 활용).
4. **사후 관리**: 데이터 활용 중 재식별 가능성을 모니터링하고 안전하게 관리.

### 4. 빅데이터의 부작용 및 통제 방안

#### (1) 발전으로 인한 부작용
- **알고리즘 편향 (Algorithm Bias)**: 데이터의 편향성으로 인해 불공정한 결과 산출.
- **프라이버시 침해**: 개인 의사와 무관한 정보 수집 및 노출.
- **책임 소재 불명확**: 사고 발생 시 설계자와 사용자 간 책임 공방.

#### (2) 위기 요인과 통제 방안
| 위기 요인 | 상세 내용 | 통제 방안 |
| :--- | :--- | :--- |
| **사생활 침해** | SNS 등 데이터 노출로 인한 프라이버시 위협 | 제공자 중심에서 **사용자 책임**으로 전환 |
| **책임 원칙 훼손** | 발생하지 않은 잠재적 위협으로 불이익 부여 | 결과에 대해서만 책임을 묻는 **결과 중심 원칙** 강화 |
| **데이터의 오용** | 분석 결과의 맹신 및 오류 가능성 | 알고리즘을 감시/해석하는 **알고리즘미스트(Algorithmist)** 도입 |

> [!INFO] 💡 알고리즘미스트 (Algorithmist)
> 알고리즘의 편향이나 오류로 인해 부당하게 피해를 입은 사람들을 구제하고, 알고리즘의 타당성을 평가하는 전문 인력을 말합니다.
