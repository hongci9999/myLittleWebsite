# 서브쿼리 (Subquery)

## 핵심 요약

**서브쿼리**는 메인 쿼리 **안의 SELECT**. **단일행**(1값·`>`), **다중열**(`(A,B) IN`), **다중행**(`IN`·`ANY`·`ALL`·`EXISTS`), **스칼라**(SELECT 절), **인라인 뷰**(FROM 절), **상관**(행마다 서브쿼리·외부 컬럼 참조). **뷰**는 논리 객체 — **보안·독립·편의**.

## 왜 중요한가

- 위치(WHERE·SELECT·FROM)와 **반환 형태**에 따라 이름·연산자가 달라진다.
- **IN / ANY / ALL / EXISTS**·상관 서브쿼리는 **빈출**.
- 인라인 뷰·뷰 개념은 **조인 대체**·시험 **서술형**에 자주 나온다.

## 1. 서브쿼리란

| 항목 | 내용 |
|------|------|
| **정의** | **하나의 쿼리(메인 쿼리)** 안에 포함된 **또 다른 쿼리문** |
| **역할** | 메인 쿼리의 **WHERE·FROM·SELECT·HAVING** 등에 **중첩**해 사용 |
| **실행** | 보통 **서브쿼리 먼저** → 그 결과로 **메인 쿼리** 처리(문맥에 따라 다름) |

```sql
SELECT …
  FROM …
 WHERE … ( SELECT … FROM … );   -- WHERE 절 서브쿼리 예
```

## 2. 예제 테이블

### \<직원\>

| 사번 | 이름 | 연봉 | 부서ID | 직무ID |
|------|------|------|--------|--------|
| 101 | 김사원 | 5000 | 10 | J1 |
| 102 | 김과장 | 6000 | 20 | J2 |
| 103 | 서대리 | 4000 | 10 | J1 |
| 104 | 이부장 | 7000 | 30 | J3 |

### \<부서\>

| 부서ID | 부서명 | 지역 |
|--------|--------|------|
| 10 | 기획 | 서울 |
| 20 | 개발 | 부산 |
| 30 | 마케팅 | 서울 |

### \<인사팀업무\>

| 부서ID | 직무ID |
|--------|--------|
| 10 | J1 |
| 30 | J3 |

## 3. 단일행 서브쿼리

| 항목 | 내용 |
|------|------|
| **정의** | 서브쿼리가 **하나의 값**(1행 1열)만 반환 |
| **연산자** | `=`, `>`, `<`, `>=`, `<=`, `!=`, `<>` 등 **단일 값 비교** |

```sql
SELECT 이름, 연봉
  FROM 직원
 WHERE 연봉 > (SELECT AVG(연봉) FROM 직원);
```

| 단계 | 내용 |
|------|------|
| **서브쿼리** | `AVG(연봉)` = (5000+6000+4000+7000) ÷ 4 = **5500** |
| **메인** | `연봉 > 5500` |

### 결과

| 이름 | 연봉 |
|------|------|
| 김과장 | 6000 |
| 이부장 | 7000 |

→ **평균 연봉보다 큰** 직원만 조회. (5500 **초과** — `>` 이므로 5500과 **같으면 제외**)

> 집계함수 `AVG`·`MAX` 등은 **단일행** 결과 → 단일행 서브쿼리에 자주 사용.

---

## 4. 다중열 서브쿼리

| 항목 | 내용 |
|------|------|
| **정의** | 서브쿼리가 **두 개 이상의 열**을 반환 |
| **비교** | 메인 쿼리도 **같은 개수·순서**의 열로 비교 — 주로 **`(열1, 열2) IN (서브쿼리)`** |

```sql
SELECT 이름
  FROM 직원
 WHERE (부서ID, 직무ID) IN (SELECT 부서ID, 직무ID FROM 인사팀업무);
```

| 단계 | 내용 |
|------|------|
| **서브쿼리** | `(10, J1)`, `(30, J3)` |
| **메인** | `(부서ID, 직무ID)` 가 위 **쌍 중 하나**와 일치 |

### 결과

| 이름 |
|------|
| 김사원 |
| 서대리 |
| 이부장 |

| 이름 | 부서ID | 직무ID | 매칭 |
|------|--------|--------|------|
| 김사원 | 10 | J1 | (10, J1) O |
| 김과장 | 20 | J2 | X |
| 서대리 | 10 | J1 | (10, J1) O |
| 이부장 | 30 | J3 | (30, J3) O |

→ **(부서ID, 직무ID) 조합**이 인사팀 업무 목록과 **동시에** 일치해야 함.

---

## 5. 다중행 서브쿼리

| 항목 | 내용 |
|------|------|
| **정의** | 서브쿼리가 **여러 행**(1열 또는 그 이상) 반환 |
| **대표 연산자** | **IN**, **ANY**, **ALL**, **EXISTS** |

```sql
SELECT 이름
  FROM 직원
 WHERE 부서ID IN (SELECT 부서ID FROM 부서 WHERE 지역 = '서울');
```

| 단계 | 내용 |
|------|------|
| **서브쿼리** | 서울 지역 부서ID **10**, **30** |
| **메인** | `부서ID IN (10, 30)` |

### 결과

| 이름 |
|------|
| 김사원 |
| 서대리 |
| 이부장 |

→ **김과장**(부서 20·부산) **제외**.

### 다중행 연산자

| 연산자 | 의미 |
|--------|------|
| **IN** | 서브쿼리 결과 **목록에 포함**되면 참 |
| **ANY** | 목록 값 **하나라도** 조건 만족 |
| **ALL** | 목록 **모든 값**이 조건 만족 |
| **EXISTS** | 서브쿼리 결과가 **존재**하면 참(행 개수·값보다 **존재 여부**) |

---

## 6. 스칼라 서브쿼리

| 항목 | 내용 |
|------|------|
| **정의** | **SELECT 절**에 두며, **1행 1열(값 하나)** 를 반환 |
| **용도** | 행마다 **다른 계산값**을 열로 붙임(조인과 유사 효과) |

```sql
SELECT 이름,
       (SELECT 부서명
          FROM 부서 D
         WHERE D.부서ID = E.부서ID) AS 부서명
  FROM 직원 E;
```

### 결과

| 이름 | 부서명 |
|------|--------|
| 김사원 | 기획 |
| 김과장 | 개발 |
| 서대리 | 기획 |
| 이부장 | 마케팅 |

→ 직원 **각 행**마다 부서 테이블에서 **부서명 1개** 조회.

---

## 7. 인라인 뷰 (Inline View)

| 항목 | 내용 |
|------|------|
| **정의** | **FROM 절**의 서브쿼리 — **임시 테이블(뷰)** 처럼 사용 |
| **별칭** | 서브쿼리에 **반드시 별칭** 부여 후 조인·참조 |

```sql
SELECT E.이름, D.부서명
  FROM 직원 E,
       (SELECT 부서ID, 부서명
          FROM 부서
         WHERE 지역 = '서울') D
 WHERE E.부서ID = D.부서ID;
```

| 단계 | 내용 |
|------|------|
| **인라인 뷰 D** | (10, 기획), (30, 마케팅) |
| **조인** | `E.부서ID = D.부서ID` |

### 결과

| 이름 | 부서명 |
|------|--------|
| 김사원 | 기획 |
| 서대리 | 기획 |
| 이부장 | 마케팅 |

→ §5 **다중행 IN** 과 **동일 인원**, **FROM 서브쿼리 + 조인** 방식.

### 뷰(View) 개념·장점

| 항목 | 내용 |
|------|------|
| **특징** | 테이블과 달리 **디스크에 데이터를 저장하지 않는** **논리적 객체** |
| **보안성** | 필요한 **열·행만** 노출, 전체 테이블 **숨김** |
| **독립성** | 기본 테이블 **구조 변경** 시에도 뷰로 **일관된** 조회 |
| **편의성** | **복잡한 쿼리**를 뷰로 정의해 **단순하게** 재사용 |

> 인라인 뷰 = **한 번 쓰고 버리는** FROM 절 서브쿼리. **CREATE VIEW** 는 DB에 **저장**하는 뷰(별도 DDL).

## 8. 상관 서브쿼리 (Correlated Subquery)

| 항목 | 내용 |
|------|------|
| **정의** | 서브쿼리가 **메인 쿼리의 컬럼**을 참조 — **메인 행마다** 서브쿼리 **재실행** |
| **비상관** | 서브쿼리가 **먼저 한 번** 끝난 뒤 메인 처리 |

```sql
SELECT E1.이름, E1.연봉
  FROM 직원 E1
 WHERE E1.연봉 < (SELECT MAX(E2.연봉)
                   FROM 직원 E2
                  WHERE E2.부서ID = E1.부서ID);
```

| 직원 | 부서 | 부서 내 MAX | 조건 `연봉 < MAX` |
|------|------|-------------|-------------------|
| 김사원 | 10 | 5000 | X (5000 < 5000 아님) |
| 김과장 | 20 | 6000 | X |
| 서대리 | 10 | 5000 | **O** (4000 < 5000) |
| 이부장 | 30 | 7000 | X |

### 결과

| 이름 | 연봉 |
|------|------|
| 서대리 | 4000 |

→ **같은 부서 최고 연봉보다 적은** 직원만 — **부서별**로 서브쿼리 **MAX** 가 달라짐.

---

## 9. 종류 한눈에

| 종류 | 반환 | 위치·연산 | 예 |
|------|------|-----------|-----|
| **단일행** | 1값 | WHERE `> (SELECT AVG…)` | 평균 연봉 비교 |
| **다중열** | 2열+ | WHERE `(A,B) IN (SELECT …)` | 부서ID+직무ID |
| **다중행** | 여러 행 | WHERE `IN` / `ANY` / `ALL` / `EXISTS` | 서울 부서ID 목록 |
| **스칼라** | 1값·행마다 | **SELECT** 절 | 부서명 열 추가 |
| **인라인 뷰** | 테이블 형태 | **FROM** 절 | 서울 부서 임시뷰 D |
| **상관** | 행마다 1값 | WHERE + **외부 별칭** | 부서별 MAX보다 작은 연봉 |

## 10. 시험 포인트 / 함정

| 구분 | 내용 |
|------|------|
| 단일행 | **1값** — `>`, `=` |
| 다중열 | **(열, 열) IN** — 열 **개수·순서** |
| 다중행 | **IN·ANY·ALL·EXISTS** |
| 스칼라 | **SELECT** 절, **1행 1열** |
| 인라인 뷰 | **FROM** 서브쿼리 + **별칭** |
| 상관 | **E1** 참조 → **행마다** 서브쿼리 |
| 뷰 장점 | **보안·독립·편의** (논리 객체) |
| 함정 | 단일행에 **2행 반환** → 오류 |
| 함정 | 상관 vs 비상관 — **외부 컬럼 참조** 여부 |
| 함정 | 인라인 뷰 = **저장 뷰(CREATE VIEW)** 와 **다름** |
