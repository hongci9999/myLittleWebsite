# DB/SQL 핵심용어

- 스키마(Schema) = 데이터베이스 구조와 제약조건의 정의
- 릴레이션(Relation) = 테이블 전체
- 튜플(Tuple) = 행
- 속성(Attribute) = 열
- 도메인(Domain) = 속성이 가질 수 있는 값의 범위
- 후보키(Candidate Key) = 유일성과 최소성을 만족하는 키
- 기본키(Primary Key) = 후보키 중 대표로 선택된 키
- 외래키(Foreign Key) = 다른 릴레이션의 기본키를 참조하는 키
- 정규화(Normalization) = 중복과 이상 현상을 줄이기 위한 분해 과정
- 이상 현상(Anomaly) = 삽입/삭제/갱신 과정에서 발생하는 문제
- DDL = 구조 정의 언어
- DML = 데이터 조작 언어
- DCL = 데이터 제어 언어
- TCL = 트랜잭션 제어 언어
- 트리거(Trigger) = 특정 이벤트 발생 시 자동 실행되는 객체
- 뷰(View) = SELECT 결과를 기반으로 한 가상 테이블
- 인덱스(Index) = 검색 속도 향상을 위한 자료구조
- ACID = 원자성, 일관성, 고립성, 지속성

## 한 페이지 요약

- DB/SQL 용어는 구조, 키, 언어, 객체, 트랜잭션 축으로 묶어 외운다.
