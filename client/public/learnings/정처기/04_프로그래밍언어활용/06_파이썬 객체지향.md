# 파이썬(Python) 객체지향 프로그래밍 및 클래스 설계

## 📌 핵심 요약
파이썬의 **[[클래스]]** 구조와 **[[self]]** 예약어의 메커니즘을 이해하고, 생성자 메서드인 **[[__init__]]**을 통한 객체 초기화 과정을 학습합니다. 특히 파이썬 특유의 **[[Private 변수]]** 선언 방식(`__`)과 **[[상속]]** 및 **[[오버라이딩]]**의 문법적 차이를 파악하는 것이 중요합니다.

---

## 📝 상세 정리

### 1. 클래스 구현 및 인스턴스 생성
- **클래스 정의**: `class` 키워드를 사용하여 구현합니다.
- **인스턴스 생성**: 자바와 달리 `new` 연산자 없이 **클래스명()** 호출만으로 객체를 생성합니다.
- **//예시 및 설명**:
  ```python
  class Car:
      def display(self):
          print("차량입니다.")

  my_car = Car()  # new 없이 생성
  my_car.display() # 인스턴스를 통한 호출
  ```

### 2. self의 개념과 역할
- **개념**: 인스턴스 자기 자신을 가리키는 참조 변수로, 자바의 `this`와 동일한 역할을 합니다.
- **특징**: 
    - 참조 변수를 통해 호출되는 메서드는 **자기 자신이 첫 번째 인수로 자동 지정**됩니다.
    - 메서드 정의 시 반드시 첫 번째 매개변수로 `self`를 명시해야 합니다.
    - 외부에서 인스턴스 변수에 접근할 때 사용되는 참조 변수명이 내부에서는 `self`가 됩니다.
- **//예시 및 설명**:
  ```python
  class Person:
      def set_info(self, name):
          self.name = name  # self를 통해 인스턴스 변수 생성 및 접근
  ```

### 3. 생성자 메서드 (__init__)
- **정의**: 파이썬에서 이미 정의된 이름인 `__init__`을 사용하며, 객체 생성 시 자동으로 호출됩니다.
- **역할**: 객체의 멤버 변수를 초기화하는 용도로 사용됩니다.
- **//예시 및 설명**:
  ```python
  class Student:
      def __init__(self, grade):
          self.grade = grade  # 생성 시점에 학년 정보 저장
  
  s1 = Student(3) # 생성과 동시에 __init__ 호출
  ```

### 4. 캡슐화와 변수의 종류
- **클래스 변수 (Class Variable)**:
    - 클래스 구역에서 직접 선언하며, 해당 클래스의 모든 인스턴스가 공유합니다.
    - 메서드 내부에서 접근할 때는 `클래스명.변수명` 형식을 사용합니다.
- **Private 변수 (Information Hiding)**:
    - 변수명 앞에 **언더바 두 개(`__`)**를 붙여 선언합니다.
    - 클래스 내부에서는 자유롭게 접근 가능하나, 클래스 외부에서의 직접 접근은 차단됩니다.
- **//예시 및 설명**:
  ```python
  class Bank:
      count = 0  # 클래스 변수 (공유)
      def __init__(self, money):
          self.__balance = money # Private 인스턴스 변수 (외부 접근 불가)
          Bank.count += 1        # 클래스명으로 클래스 변수 접근
  ```

### 5. 상속 (Inheritance)
- **문법**: `class 하위클래스(상위클래스):` 형태로 작성합니다.
- **특징**: 상위 클래스의 모든 속성과 메서드를 물려받습니다.
- **//예시 및 설명**:
  ```python
  class Animal:
      def speak(self): print("소리를 냅니다.")

  class Dog(Animal): # Animal 상속
      pass # 상위 클래스 기능 그대로 사용
  ```

### 6. super()와 메서드 오버라이딩
- **super()**: 상위 클래스의 메서드나 생성자를 명시적으로 호출할 때 사용합니다. 특히 하위 클래스에서 생성자를 정의할 때 상위 클래스의 초기화 로직이 필요하면 반드시 호출해야 합니다.
- **메서드 오버라이딩 (Overriding)**: 상위 클래스에서 정의된 메서드를 하위 클래스에서 동일한 이름으로 재정의하여 기능을 변경하거나 추가하는 기술입니다.
- **//예시 및 설명**:
  ```python
  class Parent:
      def __init__(self, name):
          self.name = name

  class Child(Parent):
      def __init__(self, name, age):
          super().__init__(name) # 상위 생성자 호출 (이름 전달)
          self.age = age

      def welcome(self): # 오버라이딩 예시
          print(f"{self.name}님 환영합니다.")
  ```

> [!INFO] 💡 AI 보충 설명
> 1. **인스턴스 생성 시 `new` 부재**: 파이썬은 모든 것이 객체인 동적 타이핑 언어이므로, 함수 호출과 같은 문법으로 인스턴스를 생성합니다.
> 2. **`__` (Double Underscore)**: 이는 실제로 '이름 장식(Name Mangling)' 기술을 사용하여 `_클래스명__변수명` 형태로 이름을 바꾸어 외부 접근을 막는 방식입니다.
> 3. **다중 상속**: 자바와 달리 파이썬은 `class C(A, B):`와 같이 여러 부모 클래스로부터의 다중 상속을 허용합니다.

## 🎯 빈출 키워드
- **[[self]]**: 메서드의 첫 번째 매개변수로 지정되는 인스턴스 자기 자신.
- **[[__init__]]**: 파이썬의 표준 생성자 메서드명.
- **[[Private 변수]]**: `__`를 접두어로 사용하여 외부 접근을 제한하는 정보 은닉 기법.
- **[[클래스 변수]]**: 모든 인스턴스가 공유하며 `클래스명.변수명`으로 접근하는 변수.
- **[[super()]]**: 자식 클래스에서 부모 클래스의 메서드나 생성자를 호출하기 위한 함수.
