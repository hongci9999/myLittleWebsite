하향식 통합
깊이우선방식 - 종속외어있는 모듈 우선(트리 내려가면서)
너비우선방식 - 같은 레벨 우선(트리 가로로)

메인 제어 모듈은 작성된 프로그램 활용
아직 통합되지 않은건 stub(상위 모듈의 테스트를 위한 최소한의 기능만을 가지는 더미모듈)를 활용

시각화가 가능하지만 테스트 케이스의 사용이 어려움
전체 통합 하면 회귀테스트를 통해 오류 확인


상행식 통합
상위 모듈에 종속되있는 하위 모듈을 클러스터화 하여 테스트
데이터 입출력을 위한 드라이버(상위 역할을 하는 더미) 작성하여 진행

회귀 테스트 케이스 선정 방법
대표적인 샘플, 변경에 의한 영향도가 가장 높은 기능에 집중한 테스트케이스 도츌, 실제 수정이 발생한 모듈


시스템 테스트
목표 컴퓨터 시스템에서 수행되는지 확인하는 테스트

## 태스트 커버리지
기능 기반 커버리지
전체 기능을 모수로 설명 수해ㅑㅇ된 기능의 수 측정
100% 달성 목표

라인 커버리지
전체ㅐ 소스코드 라인 수 모수

코드 커버리지
소스코드의 구조 코드 자체가 얼마나 테스트되었는지
-구문 커버리지 : 모든 구문을 한 번 이상 수행하는 테스트 커버리지
-결정 커버리지: 결정문의 결과가 참과 거짓을 수행하는 
-조건 : 내부 개별 조건식의 결과가 참과 거짓의 결과를 수행하는
-조건/결정 : 결정문과 내부 조건식의 결과가 참과 거짓의 결과를 모두 수행하는지
-변형 조건/결정: 독립적으로 전체 조건식의 결과에 영향을 주는 (가장 안전)
