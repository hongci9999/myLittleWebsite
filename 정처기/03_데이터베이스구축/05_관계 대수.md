# 구조적 질의어 (SQL) 및 관계 대수·해석

## 📌 핵심 요약
본 강의는 데이터베이스 관리 시스템(DBMS)의 표준 언어인 **[[SQL]]**(Structured Query Language)의 분류(DDL, DML, DCL)와 그 이론적 기초가 되는 **[[관계 대수]]**(Relational Algebra) 및 **[[관계 해석]]**(Relational Calculus)의 개념과 연산 상세를 다룹니다. 특히 관계 대수의 순수 관계 연산자와 집합 연산자의 차이, 그리고 쿼리 최적화의 기본 원칙을 학습합니다.

---

## 📝 상세 정리

### 1. [[SQL]] (Structured Query Language)
* **정의:** [[DBMS]]를 제어하는 표준 언어로, [[관계 대수]]를 기초로 설계되었습니다.
* **특징:** 자연어와 유사한 구조를 가져 사용자 편의성이 높습니다.
* **[[내장 SQL]] (Embedded SQL):**
    * 호스트 프로그램(C, Java 등)이 실행될 때 함께 실행되는 SQL입니다.
    * 일반 질의문(DQL)은 다수의 [[튜플]]을 반환할 수 있으나, 내장 SQL은 단 하나의 튜플만 반환하는 것이 원칙입니다. (다수 처리 시 커서 활용)
    * 호스트 변수와 데이터베이스 필드 이름은 달라도 무방하나, **데이터 타입은 반드시 일치**해야 합니다.

### 2. SQL 명령어 분류

#### 2.1 [[DDL]] (Data Definition Language, 데이터 정의어)
* 데이터베이스 객체(Table, Index, View 등)를 생성, 수정, 변경하는 언어입니다.
* **[[DBA]]**(Database Administrator, 데이터베이스 관리자)가 주로 사용합니다.
* 수행 시 즉시 반영되며, 이전 상태로 되돌릴 수 없습니다. (Auto Commit)
    * `CREATE`: 객체 생성
    * `ALTER`: 객체 수정
    * `DROP`: 객체 삭제
    * `TRUNCATE`: 객체 내의 데이터 전체 삭제 (구조는 유지)

#### 2.2 [[DML]] (Data Manipulation Language, 데이터 조작어)
* 데이터를 검색, 삽입, 갱신, 삭제하는 데 사용합니다.
* **[[TCL]]**을 통해 작업 내용을 확정하거나 복구할 수 있습니다.
    * `SELECT`: 데이터 조회
    * `INSERT`: 데이터 삽입
    * `UPDATE`: 데이터 수정
    * `DELETE`: 데이터 삭제

#### 2.3 [[DCL]] (Data Control Language, 데이터 제어어)
* 접근 통제(보안) 및 병행 수행 제어를 위한 언어입니다.
    * `GRANT`: 권한 부여
    * `REVOKE`: 권한 회수
    * `COMMIT`: 트랜잭션 성공 시 결과 영구 반영
    * `ROLLBACK`: 트랜잭션 실패 시 작업 취소 및 복구
    * `CHECKPOINT`: 복귀 지점 설정

### 3. [[관계 해석]] (Relational Calculus)
* **정의:** 원하는 정보가 무엇인지만 정의하는 **비절차적 언어**입니다.
* **특징:** 실행 결과는 반드시 참($True$) 또는 거짓($False$)으로 반환되며 질의어를 사용합니다.
* **분류:**
    * **튜플 관계 해석:** 튜플을 기준으로 조회합니다.
    * **도메인 관계 해석:** 속성(도메인)을 기준으로 조회합니다.
* **논리 기호:**
    * $OR$ ($\lor$): 논리합
    * $AND$ ($\land$): 논리곱
    * $NOT$ ($\neg$): 부정
    * **전칭 정량자 ($\forall$):** 모든 튜플에 대하여 성립해야 함을 의미합니다.
    * **존재 정량자 ($\exists$):** 조건을 만족하는 튜플이 하나라도 존재함을 의미합니다.

> [!INFO] 💡 AI 보충 설명
> 관계 해석과 관계 대수는 처리하는 기능과 능력 면에서 동등합니다. 이를 **관계적으로 완전(Relationally Complete)**하다고 표현합니다.

---

### 4. [[관계 대수]] (Relational Algebra)
* **정의:** 원하는 결과를 얻기 위해 어떻게(How) 유도하는가를 정의하는 **절차적 언어**입니다.



#### 4.1 일반 집합 연산자
이 연산자들은 합병 가능 조건(차수와 도메인이 동일)을 만족해야 하는 경우가 많습니다.
* **[[합집합]] (UNION):** 중복 없이 합하는 연산 ($R \cup S$)
* **[[교집합]] (INTERSECT):** 공통된 튜플 추출 ($R \cap S$)
* **[[차집합]] (DIFFERENCE):** $R$에는 있고 $S$에는 없는 튜플 추출 ($R - S$)
* **[[교차곱]] (CARTESIAN PRODUCT):** 모든 가능한 순서쌍의 집합 ($R \times S$)
    * **기수(Cardinality):** 두 릴레이션 기수의 곱
    * **차수(Degree):** 두 릴레이션 차수의 합

#### 4.2 순수 관계 연산자
* **[[SELECT]] ($\sigma$):** 조건에 맞는 튜플을 추출하는 **수평적** 부분집합 연산입니다.
* **[[PROJECT]] ($\pi$):** 주어진 속성 리스트만 추출하는 **수직적** 부분집합 연산입니다.
* **[[JOIN]] ($\bowtie$):** 공통 속성을 기준으로 두 릴레이션을 하나로 합칩니다.
    * **동일 조인(Equi Join):** $=$ 연산자 사용.
    * **자연 조인(Natural Join):** 동일 조인 후 중복 속성 제거.
    * **외부 조인(Outer Join):** 조건에 맞지 않는 튜플도 포함(NULL 처리).
    * **세타 조인(Theta Join):** $=$ 이외의 비교 연산자 사용.
* **[[DIVISION]] ($\div$):** $S$의 모든 조건을 만족하는 $R$의 튜플에서 $S$와 겹치는 속성을 제외하고 추출합니다.

---

### 5. 쿼리 최적화 규칙
절대적인 규칙은 없으나 일반적인 성능 향상을 위해 다음을 적용합니다.
* **선택(Select) 연산을 가급적 일찍 수행**하여 처리할 데이터 양을 최소화합니다.
* 프로젝트(Project) 연산도 일찍 수행하여 속성 수를 줄입니다.
* **조인(Join)은 비용이 크므로 가급적 마지막에 수행**하거나 최소화합니다.

## 🎯 빈출 키워드
- **[[DDL]] (Data Definition Language)**: 데이터 구조를 정의(CREATE, ALTER, DROP, TRUNCATE)하며 DBA가 주로 사용함.
- **[[DML]] (Data Manipulation Language)**: 데이터를 조작(SELECT, INSERT, UPDATE, DELETE)하며 사용자가 주로 사용함.
- **[[DCL]] (Data Control Language)**: 보안 및 제어(GRANT, REVOKE, COMMIT, ROLLBACK)를 담당함.
- **[[관계 대수]] (Relational Algebra)**: 어떻게(How) 데이터를 유도할지 명시하는 절차적 언어. ($\sigma, \pi, \bowtie, \div$)
- **[[관계 해석]] (Relational Calculus)**: 무엇(What)을 원하는지 명시하는 비절차적 언어. (정량자 $\forall, \exists$ 사용)
