# 논리 개체 상세화
## 논리 개체 정의

개체
현실 세계의 식별 가능한 대상을 디지텅화 하기 위해 추상화하여 표현한 단위
하나의 개체 타입에 의해 여러 개체 생성, 각 개체는 독립적

개체 타입(개체 스키마)
개체명과 개체 속성을 정의라여 명세한 것
모든 개체는 개체 타입을 통해 생성됨
개체 인스턴스: 특정 개체 타입에 의해 생성된개체만을 호칭할 떄
약어 사용 지양, 단수명사 사용

속성 attribute
고유의 특성, 개체의 ㅓㅇ격을 구체적으로
개체를 구성하는 가장 작은 단위

기본속성-  업무 분석을 통해 정의
설계 - 새로 만들거나 기존 데이터를 변형하여 저으이
파생-  다른 속성의 영향을 받아 정의(점수, 등급)

기본키 속성- 개체를 식별할 수 있는 속성
외래키 속성 - 다른 개체와 관계를 가지느 속성

원시 속성은 제거할 경우 재현 불가능

관계 스키마
속성들의 논리적 구조
표
다수의 릴레이션으로 분할
분리 - 서로 독립적인 돤계정을 가진 속성
중복 감소
정보 무손실 원칙

식별자 identifier
개체를 유일하게 구분
 키와 같음변하지 않아야 하며 값이 존재해야함

주 식별자
기본키와 같음

보조 식별자
주 식별자 대신 개체 식별 가능
고유 인덱스 - 칼럼에 중복값을 인정하지 않는 인덱스

내부 식별자 - 자신의 개체 타입으로부터 생성되는 식별자
외부 - 다른 개체타입에서 주식별자를 상속 = 외레키

단일 식별자 - 속성이 하나인 식별자
복합 식별자 - 둘 이상 속성 = 슈퍼키

원조 식별자 - 가공되지 않은 원래 식별자
대리 식별자 - 복합 식별자의 속성들을 하나의 속성으로 묶어 하나의 식별자로 활용하는 인조 식별자
코드, 일련번호 등 데이터를 새ㄹ 생성하여 식별자 지정

## 데이터 모델링
기업의 정보 구조를 디지털의 논리적인 데이터 모델로 변환하여 문화
현실의 비즈니스 요구하상을 기반으로 실체 entity 와 관계 relation을 통해 정보구조 표현

데이터 모델의 표시 요소
논리적 구조 - 개체들 간의 관걔
연산 - 데이터 처리 방법
제약조건 

데이터 모델의 구성 요소

속성
관계

데이터 모델링 특징
효율적인 의사소통 수단 제공

업무 흐음 파악 용이, 데이터 무결성 보장

### 개념 데이터 모델링
상위 수준 모델, 데이터 모델의 골격 설계
엔티티타입, 기본 속성, 관계, 주요 업무 기능 등 포함

독립적인 표현이 가능한 모델을 구현
순서보다 필수요소로 이해
요구분석, 주요 개체 선별, 개체 정의, 식별자 정의, 개체 통합, 관계 도출

### 논리 데이터 모델링
업무 데이터 및 규칙을 구체적으로 표현한 모델
모든 업무 영역에 대한 업무용 개체,속성,관계,프로세스 등 포함
관계형, 계층적, 네트워크형 데이터 모델

모델링 기법
컴퓨터가 이해할 수 있는 논리적 구조로 매핑

특징
요구사항을 춘분히 수집하지 않으면 다음의 요구사항 변경에 많은 비용
특정 시스템의 성능 및 제역사향으로부터 독립적
e-r 모델을 황용하려 의사소통
모든 데이터를 정규화하여 표현

## 논리 객체의 관계 정의
관계relationship
의미 있는 연결 상태
존재에 의한 관계 - 정보의 흐름이 정적인 상태(학교와 학생)
행위레 의한 관계 - 동적인 상태(고객과 주문 개체)

개체 간 관계는 1,0,n 을 활용해 표현
적절한 관계는 개체간 참조 무결성을 보장


관계 표현의 의미
어떻게 관리되느냐에 따라 표현이 달라질 수 있음
사용 빈도나 보안을 위해 복수의개체로 분할하여 연결할 있음

다대다 관계의 해소
다대다 관계는 어떤 방법으로도 구현 불가능
논리적 불안정 유지
정규화 문서화 작업 불가능
일대다 관계로 변경해야함 - 정규화
