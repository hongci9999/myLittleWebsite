# 탐색과 구현형 문제

## 핵심 요약

탐색과 구현형 문제는 화려한 알고리즘보다 "코드를 끝까지 정확히 읽는 힘"이 중요하다. 실기에서는 배열 순회, 조건 분기, 누적 합, 최댓값/최솟값 탐색, 이진 탐색 흐름 추적 같은 기본기가 자주 출제된다.

## 왜 중요한가

- 구현형 문제는 복잡한 이론보다 실수 방지가 점수를 좌우한다.
- 탐색 문제는 자료가 정렬되어 있는지 여부를 먼저 판단하는 것이 핵심이다.
- 이진 탐색은 개념은 간단하지만 중간 인덱스 이동 흐름을 잘못 읽으면 오답이 난다.

## 1. 선형 탐색

- 처음부터 끝까지 하나씩 확인하는 방법
- 자료가 정렬되어 있지 않아도 사용할 수 있다.
- 시간복잡도는 `O(n)`

```c
for(i = 0; i < n; i++) {
    if(arr[i] == key) {
        return i;
    }
}
```

## 2. 이진 탐색

- 정렬된 배열에서만 사용할 수 있다.
- 중간값과 비교하며 탐색 범위를 절반씩 줄인다.
- 시간복잡도는 `O(log n)`

```text
left = 0, right = n-1
mid = (left + right) / 2
```

## 3. 구현형 문제의 대표 유형

- 합계/평균 구하기
- 최댓값/최솟값 찾기
- 조건에 맞는 개수 세기
- 배열 요소 교환
- 누적 계산
- 자리수 처리, 문자열 순회

## 4. 코드 추적 예제

### 예제 1: 선형 탐색

```c
int arr[5] = {7, 3, 9, 1, 5};
int key = 1;
int idx = -1;
for(int i = 0; i < 5; i++) {
    if(arr[i] == key) {
        idx = i;
        break;
    }
}
printf("%d\n", idx);
```

- `7` 검사 -> 다름
- `3` 검사 -> 다름
- `9` 검사 -> 다름
- `1` 검사 -> 같음, `idx=3`
- 출력: `3`

### 예제 2: 이진 탐색

정렬 배열: `[1, 3, 5, 7, 9, 11, 13]`, 찾을 값 `9`

1. `left=0`, `right=6`, `mid=3`, 값 `7`
2. `9 > 7` 이므로 오른쪽 탐색, `left=4`
3. `left=4`, `right=6`, `mid=5`, 값 `11`
4. `9 < 11` 이므로 왼쪽 탐색, `right=4`
5. `left=4`, `right=4`, `mid=4`, 값 `9`
6. 탐색 성공

### 예제 3: 구현형 누적 계산

```c
int arr[4] = {2, 4, 6, 8};
int sum = 0;
for(int i = 0; i < 4; i++) {
    if(arr[i] > 3) {
        sum += arr[i];
    }
}
printf("%d\n", sum);
```

- `2` -> 조건 불만족
- `4` -> `sum=4`
- `6` -> `sum=10`
- `8` -> `sum=18`
- 출력: `18`

## 5. 탐색 알고리즘 비교

| 항목 | 선형 탐색 | 이진 탐색 |
|---|---|---|
| 정렬 필요 | 없음 | 있음 |
| 방법 | 처음부터 하나씩 비교 | 중간값 기준 범위 절반 축소 |
| 시간복잡도 | `O(n)` | `O(log n)` |
| 장점 | 단순 | 빠름 |

## 6. 자주 틀리는 함정

### 6.1 이진 탐색에 정렬 조건 누락

- 정렬되지 않은 배열에서는 이진 탐색을 적용하면 안 된다.

### 6.2 인덱스와 값 혼동

- 문제는 값을 묻는지, 위치를 묻는지 구분해야 한다.

### 6.3 `break` 후 반복 종료를 놓침

- 탐색 성공 후 `break`가 있으면 반복문은 즉시 끝난다.

### 6.4 누적 변수 초기화 누락

- `sum`, `count`, `max`, `min` 의 초기값이 무엇인지 반드시 확인해야 한다.

## 7. 시험장에서 푸는 순서

1. 정렬 여부를 먼저 본다.
2. 탐색이면 선형/이진 중 어떤 방식인지 구분한다.
3. 구현형이면 누적 변수와 조건식을 먼저 표시한다.
4. 반복 회차별 변수 변화를 적는다.
5. 출력값이 인덱스인지 실제 값인지 마지막에 다시 확인한다.

## 한 페이지 요약

- 선형 탐색은 순서대로 찾고, 이진 탐색은 절반씩 줄인다.
- 이진 탐색은 정렬된 데이터에서만 사용한다.
- 구현형 문제는 누적 변수 초기값과 조건식이 핵심이다.
- `break` 유무에 따라 탐색 흐름이 바뀐다.
- 문제를 보면 먼저 "탐색형인지 구현형인지" 분류하는 습관이 중요하다.
