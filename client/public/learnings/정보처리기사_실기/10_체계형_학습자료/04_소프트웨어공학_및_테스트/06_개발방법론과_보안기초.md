# 개발방법론과 보안기초

## 핵심 요약

소프트웨어공학의 마지막 축은 "어떻게 개발하는가"와 "어떻게 안전하게 지키는가"다. 실기에서는 SDLC, 애자일, 객체지향, DevOps 같은 개발 방식과 보안 위협 용어가 함께 출제되는 경우가 많다.

## 1. SDLC

- 계획
- 요구분석
- 설계
- 구현
- 테스트
- 유지보수

## 2. 개발방법론

| 방법론 | 특징 |
|---|---|
| 폭포수 | 순차적 진행 |
| 프로토타입 | 시제품 중심 |
| 나선형 | 위험 분석 반복 |
| 애자일 | 짧은 반복과 협업 |

## 3. 애자일 관련 용어

- 스크럼
- 스프린트
- 백로그
- 회고

## 4. 객체지향 핵심

- 캡슐화
- 상속
- 다형성
- 추상화

## 5. DevOps

- 개발과 운영 협업
- 자동화, 배포, 모니터링 강조

## 6. 보안 기초 용어

| 용어 | 의미 |
|---|---|
| 기밀성 | 인가된 사용자만 접근 |
| 무결성 | 데이터가 임의로 변조되지 않음 |
| 가용성 | 필요할 때 사용 가능 |

## 7. 자주 나오는 보안 위협

- 악성코드
- 랜섬웨어
- 피싱
- 스니핑
- 스푸핑
- 제로 트러스트

## 8. 예제

- 짧은 반복 개발과 고객 피드백 강조 -> 애자일
- 위험 분석을 반복하는 방법론 -> 나선형

## 9. 자주 틀리는 함정

- SDLC 단계 순서를 바꿔 기억한다
- 애자일을 특정 도구 이름처럼 이해한다
- 기밀성/무결성/가용성 의미를 섞어 외운다

## 10. 시험장에서 푸는 순서

1. 방법론인지 보안 용어인지 구분한다.
2. 순차형/반복형/위험분석형/협업형으로 개발방법론을 분류한다.
3. 보안은 CIA(기밀성, 무결성, 가용성) 축으로 정리한다.

## 한 페이지 요약

- SDLC는 계획부터 유지보수까지의 생명주기다.
- 폭포수, 프로토타입, 나선형, 애자일 특징을 비교해야 한다.
- 보안의 기본은 기밀성, 무결성, 가용성이다.
- 애자일과 DevOps는 현대 개발 방식 문제에서 자주 나온다.
