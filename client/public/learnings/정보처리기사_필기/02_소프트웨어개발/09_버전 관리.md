# 제품 소프트웨어 버전 관리

## 핵심 요약
소프트웨어 개발 과정에서 발생하는 변경 사항을 체계적으로 관리하는 **[[형상 관리]]**의 개념과 주요 도구의 기능을 학습합니다. 공유 폴더, 클라이언트/서버, 분산 저장소 방식의 차이점을 이해하고 각 도구(CVS, SVN, Git 등)의 특징을 파악하는 것이 핵심입니다.

---

## 상세 정리

### 1. 형상 관리(SCM, Software Configuration Management) 개요
* **정의**: 소프트웨어의 개발 생태계 전반에서 발생하는 변경 사항을 체계적으로 관리하고 통제하는 활동입니다.
* **필요성 및 중요성**:
    * **개발 통제**: 무절제한 변경을 방지하고 개발 과정을 가시화합니다.
    * **추적성 확보**: 버그 수정 및 변경 이력에 대한 추적 결여 문제를 해결합니다.
    * **가시성 증대**: 프로젝트의 필요한 요소들을 한눈에 파악할 수 있도록 관리합니다.

### 2. 형상 관리 도구의 주요 기능
형상 관리의 효율적인 운영을 위해 다음과 같은 핵심 기능들을 제공합니다.

| 기능명 | 설명 |
| :--- | :--- |
| **Check-Out** | 저장소(Repository)에서 형상의 사본을 로컬 컴퓨터로 가져오는 기능 |
| **Check-In** | 수정한 소스 코드를 저장소에 업로드하여 갱신하는 기능 |
| **Commit** | 체크인 후 변경 사항을 최종적으로 승인하고 저장소에 반영하는 기능 |
| **Update** | 저장소의 최신 버전과 로컬 컴퓨터의 형상을 동기화하는 기능 |
| **Import** | 관리되지 않던 소스 코드를 처음으로 저장소에 업로드하는 기능 |
| **Export** | 버전 관리 정보를 제외한 순수 소스 코드 파일만 추출하는 기능 |
| **Diff** | 서로 다른 버전 간의 소스 코드 차이점을 비교하는 기능 |

> [!INFO] 💡 AI 보충 설명: Git의 기본 명령어 흐름
> 사용자가 언급한 **Git**의 경우, 작업 디렉토리의 변경 내용을 `add`를 통해 스테이징 영역에 올린 후, `commit`을 통해 로컬 저장소에 반영하며, `push`를 통해 원격 저장소와 동기화합니다. `check-in`과 `check-out` 개념은 Git에서 `commit`과 `checkout/switch` 등으로 세분화되어 사용됩니다.

### 3. 관련 관리 체계 및 프레임워크
소프트웨어 자산 관리와 연관된 주요 개념들입니다.

* **[[ALM]] (Application Lifecycle Management)**: 소프트웨어 개발의 전체 라이프사이클을 관리합니다.
* **[[ITIL]] (IT Infrastructure Library)**: IT 서비스를 효과적으로 제공하고 관리하기 위한 프레임워크입니다.
* **[[ITSM]] (IT Service Management)**: 서비스화 및 비즈니스 중심으로 재설계된 IT 관리 체계입니다.
* **[[SLA]] (Service Level Agreement)**: 고객과 서비스 제공자 간에 협의된 서비스 수준 계약입니다.
* **IT Governance**: 의사결정 및 책임에 대한 프레임워크입니다.
* **[[EAMS]] (Enterprise Architecture Management System)**: EA의 관리 및 활용을 위한 구현 시스템입니다.
* **[[EA]] (Enterprise Architecture)**: 기업의 IT 청사진을 의미합니다.
* **[[PPM]] (Project Portfolio Management)**: 중앙 집중식 프로젝트 관리 체계입니다.

### 4. 도구 사용의 목적 및 유의사항
* **목적**: 관리 편의성 상승, 개발 생산성 개선, 운영 비용 절감.
* **유의사항**: 도구 도입 시 문제 발생에 대비한 **매뉴얼 사전 준비**가 반드시 필요합니다.

### 5. 형상 관리 도구 유형별 특징

#### A. 관리 방식에 따른 분류
1.  **공유 폴더 방식**: 로컬 컴퓨터의 공유 폴더를 이용하는 초기 방식입니다.
    * 종류: **RCS**, **SCCS**
2.  **클라이언트/서버 방식**: 중앙 서버에 형상 관리 시스템을 두고 클라이언트가 접속하여 관리하며 모니터링 기능을 제공합니다.
    * 종류: **CVS**, **SVN**
3.  **분산 저장소 방식**: 원격 저장소의 내용을 개인 로컬 저장소에 완전히 복사하여 개발한 후, 다시 원격 저장소에 반영합니다.
    * 종류: **Git**, **Bitkeeper**

#### B. 기타 구분에 따른 분류
* **저장소 위치**: 
    * 로컬 형상: **RCS**
    * 중앙 집중형: **CVS**, **SVN**, **ClearCase**
    * 분산형: **Git**, **Mercurial**
* **소스 공개 여부**:
    * 오픈 소스: **CVS**, **SVN**, **Git**
    * 상용(유료): **PVCS**, **ClearCase**

## 3. 빈출 내용 정리

- **[[형상 관리]]**: 소프트웨어 변경 사항을 관리하며 가시성, 추적성, 무결성을 보장하는 활동입니다.
- **[[형상 관리 기능]]**: **Check-In, Check-Out, Commit, Update**의 정의를 명확히 구분해야 합니다.
- **도구별 특징**:
    - **[[CVS]]**: 클라이언트/서버 방식, 다수 인원 동시 작업 가능.
    - **[[SVN]]**: CVS의 단점을 보완하여 파일명 변경, 이동 등이 자유로움.
    - **[[Git]]**: 리눅스 토발즈가 개발한 **분산형** 시스템으로 속도가 빠르고 대규모 프로젝트에 유리하며 중앙 서버 의존도가 낮습니다.
    - **[[RCS]]**: 소스 파일 수정을 한 사람으로 제한(Locking)하는 로컬 방식입니다.
    - **[[Bitkeeper]]**: 중앙 통제 방식의 분산 형상 관리 도구입니다.
    - **[[ClearCase]]**: IBM에서 제작한 상용 형상 관리 도구입니다.
