# WHERE 절과 기본 비교 연산자

## 핵심 요약

**WHERE**는 조건에 맞는 **행**만 조회(실행 **2번**). **우선순위**: `()` → 산술 → 비교·LIKE·BETWEEN·IN·IS NULL → **NOT** → **AND** → **OR**. **LIKE**·**NULL**·**IN·BETWEEN**·날짜 비교 포함.

## 왜 중요한가

- SELECT **필터링**의 기본이며 σ(선택)에 해당한다.
- `<>` vs `!=` 등 **부등호 표기**와 연산자별 **포함 행** 지문이 빈출이다.

> 실행 순서: [03_SELECT_문_구조와_실행순서](./03_SELECT_문_구조와_실행순서.md) · SELECT 입문: [04_SELECT_기본_문법](./04_SELECT_기본_문법.md)

## 1. WHERE 절

| 항목 | 내용 |
|------|------|
| **역할** | **특정 조건**을 만족하는 **행(튜플)** 만 조회 |
| **위치** | `FROM` 다음, `GROUP BY` 이전 |
| **관계대수** | **σ (선택)** |

### 예제 테이블 \<학생\>

| 학번 | 이름 | 나이 |
|------|------|------|
| 101 | 홍길동 | 20 |
| 102 | 이순신 | 21 |
| 103 | 임꺽정 | 20 |
| 104 | 장보고 | 21 |
| 105 | 강감찬 | 22 |
| 106 | 이성계 | 23 |

```sql
SELECT 이름, 나이
  FROM 학생
 WHERE 나이 = 21;
```

| 이름 | 나이 |
|------|------|
| 이순신 | 21 |
| 장보고 | 21 |

→ **나이가 21인 행만** 남음 (2행)

## 2. 기본 비교 연산자

| 연산자 | 의미 | 예 | 결과에 포함되는 이름 |
|--------|------|-----|----------------------|
| **=** | 같다 | `WHERE 나이 = 21` | 이순신, 장보고 |
| **!=** | 다르다 | `WHERE 나이 != 21` | 홍길동, 임꺽정, 강감찬, 이성계 |
| **<>** | 다르다 (표준) | `WHERE 나이 <> 21` | 위와 동일 |
| **^=** | 다르다 (일부 DBMS) | `WHERE 나이 ^= 21` | 위와 동일 |
| **>** | 크다 | `WHERE 나이 > 22` | 이성계 |
| **<** | 작다 | `WHERE 나이 < 22` | 홍길동, 이순신, 임꺽정, 장보고 |
| **>=** | 크거나 같다 | `WHERE 나이 >= 22` | 강감찬, 이성계 |
| **<=** | 작거나 같다 | `WHERE 나이 <= 21` | 홍길동, 이순신, 임꺽정, 장보고 |

### 부등호 표기

| 연산자 | 비고 |
|--------|------|
| **<>** | SQL **표준** 부등호 |
| **!=** | 많은 DBMS에서 허용 |
| **^=** | 일부 제품(문항에 나오면 **다르다**로 해석) |

### 나이 기준 빠른 참고 (본 예제)

| 조건 | 나이 | 해당 이름 |
|------|------|-----------|
| = 21 | 21 | 이순신, 장보고 |
| != 21 | 20,22,23 | 홍길동, 임꺽정, 강감찬, 이성계 |
| > 22 | 23 | 이성계 |
| < 22 | 20,21 | 홍길동, 이순신, 임꺽정, 장보고 |
| >= 22 | 22,23 | 강감찬, 이성계 |
| <= 21 | 20,21 | 홍길동, 이순신, 임꺽정, 장보고 |

## 3. 조건 표현 연산자

\<학생\> 테이블(§1) 기준 — **나이**로 필터.

| 표현 | 설명 | 예 | 결과 이름 |
|------|------|-----|-----------|
| **AND** | **두 조건 모두** 만족 | `WHERE 나이 >= 21 AND 나이 <= 22` | 이순신, 장보고, 강감찬 |
| **OR** | **둘 중 하나** 만족 | `WHERE 나이 = 20 OR 나이 = 23` | 홍길동, 임꺽정, 이성계 |
| **IN** | **여러 값 중 하나** | `WHERE 나이 IN (20, 21, 22)` | 홍길동, 이순신, 임꺽정, 장보고, 강감찬 |
| **BETWEEN A AND B** | **범위**(A 이상 B 이하, **양끝 포함**) | `WHERE 나이 BETWEEN 21 AND 22` | 이순신, 장보고, 강감찬 |
| **NOT** | 조건 **부정** | 아래 참고 | |

### BETWEEN과 AND의 관계

```sql
WHERE 나이 BETWEEN 21 AND 22
-- 동치(의미 같음)
WHERE 나이 >= 21 AND 나이 <= 22
```

| 연산자 | 범위 |
|--------|------|
| **BETWEEN 21 AND 22** | **21 ≤ 나이 ≤ 22** (21, 22 **포함**) |

### NOT 예

| SQL | 결과 이름 |
|-----|-----------|
| `WHERE 나이 NOT IN (20, 21, 22)` | **이성계** |
| `WHERE 나이 NOT BETWEEN 21 AND 22` | 홍길동, 임꺽정, **이성계** |

```sql
-- IN vs OR (나이 20 또는 23)
WHERE 나이 IN (20, 23)
WHERE 나이 = 20 OR 나이 = 23   -- 동일 의미
```

> **IN**, **BETWEEN**은 시험 **빈출**.

## 4. 연산자 우선순위

**높은 것부터** 먼저 계산된다(왼쪽 → 오른쪽 순서로 나열).

```
괄호 ()
  → 곱셈(*), 나눗셈(/), 나머지(%)
  → 비교(=, !=, <>, <, >, <=, >=), LIKE, BETWEEN, IN, IS NULL, IS NOT NULL
  → NOT
  → AND
  → OR
```

| 순위 | 연산자·표현 | 비고 |
|------|-------------|------|
| 1 | **괄호 `()`** | **최우선** — 의도를 명확히 할 때 사용 |
| 2 | **`*` `/` `%`** | 산술(SELECT 식·WHERE 숫자 조건) |
| 3 | **비교**, **LIKE**, **BETWEEN**, **IN**, **IS NULL**, **IS NOT NULL** | 같은 층(비교·조건 연산) |
| 4 | **NOT** | **AND·OR보다 먼저** |
| 5 | **AND** | **OR보다 먼저** |
| 6 | **OR** | **가장 나중** |

### 해석 예

| SQL | 실제 해석 |
|-----|-----------|
| `WHERE 나이 = 20 OR 나이 = 21 AND 나이 <= 21` | `OR`보다 **AND가 먼저** → `(나이=20) OR (나이=21 AND 나이<=21)` |
| `WHERE NOT 나이 IN (20, 21)` | **NOT**이 **IN** 결과에 적용 |

```sql
-- 의도가 헷갈리면 괄호로 고정
WHERE (나이 = 20 OR 나이 = 21) AND 나이 <= 21;
WHERE 나이 = 20 OR (나이 = 21 AND 나이 <= 21);
```

> ☞ **AND > OR**, **NOT > AND** — 교재·시험 **빈출**. 복합 조건은 **`()`** 로 묶는 습관.

## 5. 날짜 조건 조회

날짜·문자형 컬럼도 **비교 연산자**·**BETWEEN**으로 필터한다. 리터럴은 보통 **작은따옴표** `'YYYY-MM-DD'`.

### \<주문\> 예

| 주문번호 | 주문일자 |
|----------|----------|
| … | 2024-12-01 |
| … | 2024-12-15 |
| … | 2025-01-05 |

#### `<` (미만)

```sql
SELECT *
  FROM 주문
 WHERE 주문일자 < '2024-12-15';
```

→ **2024-12-01** 만 (12-15 **당일 제외**)

#### BETWEEN (기간)

```sql
SELECT *
  FROM 주문
 WHERE 주문일자 BETWEEN '2024-12-01' AND '2024-12-31';
```

→ **2024-12-01**, **2024-12-15** (12월 전체 — **양끝 날짜 포함**)

| 포인트 | 내용 |
|--------|------|
| 형식 | DB·컬럼 타입에 맞는 **날짜 문자열** (`'2024-12-01'`) |
| BETWEEN | **시작일·종료일 모두 포함** |
| `<` vs `<=` | `'2024-12-15'` **당일 포함 여부** 문항 확인 |

## 6. LIKE 연산자

**문자열 패턴** 검색. 와일드카드는 **문자열 컬럼**에 사용.

| 와일드카드 | 의미 |
|------------|------|
| **%** | **0글자 이상** (임의 길이) |
| **_** | **정확히 1글자** |

### \<라면\> 예

| 이름 | 가격 |
|------|------|
| 틈새라면 | 1100 |
| 진라면 순한맛 | 1000 |
| 라면볶이 | 1200 |

| SQL | 패턴 의미 | 결과 |
|-----|-----------|------|
| `WHERE 이름 LIKE '%라면%'` | 어디에든 **라면** 포함 | 틈새라면, 진라면 순한맛, 라면볶이 (**3건**) |
| `WHERE 이름 LIKE '%라면'` | **라면으로 끝남** | 틈새라면 (**1건**) |
| `WHERE 이름 LIKE '_라면%'` | **앞에 1글자** + 라면 + 이후 | 진라면 순한맛 (**1건**) |

```sql
SELECT * FROM 라면 WHERE 이름 LIKE '%라면%';
SELECT * FROM 라면 WHERE 이름 LIKE '%라면';
SELECT * FROM 라면 WHERE 이름 LIKE '_라면%';
```

### ESCAPE — `%`, `_` 를 글자 그대로 찾기

`%`·`_`를 와일드카드가 아니라 **문자 그대로** 검색할 때 사용.

```sql
SELECT *
  FROM 라면
 WHERE 이름 LIKE '%/_라면' ESCAPE '/';
```

| 항목 | 설명 |
|------|------|
| `/_` | **`/` 다음 `_`** = 와일드카드 아닌 **밑줄 문자 `_`** |
| **ESCAPE '/'** | 이스케이프 문자로 **`/`** 지정 |
| 의미 | 이름이 **`_라면`으로 끝나는** 경우 검색 (문자 `_` 포함) |

> ☞ **`%`·`_` 자체를 검색**할 때 **ESCAPE** 절 사용.

## 7. NULL 비교 연산자

**NULL**은 “값 없음”. **`=`**, **`!=`**, **`<>`** 로는 비교 **불가**(UNKNOWN) → 전용 연산자 사용.

### \<회원\> 예

| 회원번호 | 직업 |
|----------|------|
| 1 | NULL |
| 2 | 공무원 |
| 3 | 회사원 |
| 4 | 학생 |
| 5 | NULL |
| 6 | 교수 |

| 연산자 | 용도 | SQL | 결과 |
|--------|------|-----|------|
| **IS NULL** | 값이 **NULL** | `WHERE 직업 IS NULL` | 해당 **2행** |
| **IS NOT NULL** | 값이 **있음** | `WHERE 직업 IS NOT NULL` | 해당 **4행** |

```sql
SELECT COUNT(*) FROM 회원 WHERE 직업 IS NULL;      -- 2
SELECT COUNT(*) FROM 회원 WHERE 직업 IS NOT NULL;  -- 4
```

| 금지 | 이유 |
|------|------|
| `WHERE 직업 = NULL` | **오류** 또는 **기대와 다른 결과** |
| `WHERE 직업 != NULL` | 동일 — **사용 금지** |

## 9. 시험 포인트 / 함정

| 구분 | 내용 |
|------|------|
| WHERE | **행** 필터, 실행 **2번** |
| AND | **모두** 만족 |
| OR | **하나라도** 만족 |
| IN | `(20,21,22)` 중 **하나** — OR 나열과 동치 가능 |
| BETWEEN | **A 이상 B 이하**, **양끝 포함** = `>= AND <=` |
| NOT IN / NOT BETWEEN | 조건 **부정** |
| 우선순위 | `()` → `* / %` → 비교·LIKE·BETWEEN·IN·IS NULL → **NOT** → **AND** → **OR** |
| 함정 | **AND가 OR보다 먼저** — 괄호 없으면 의도와 다를 수 있음 |
| 날짜 | `'2024-12-01'` 문자 리터럴, `<`·BETWEEN 동일 규칙 |
| LIKE `%` | **0자 이상** |
| LIKE `_` | **정확히 1자** |
| ESCAPE | `%`·`_` **문자 그대로** 검색 |
| NULL | **IS NULL** / **IS NOT NULL** only |
| 함정 | `= NULL`, `!= NULL` **금지** |
| 함정 | `%라면` vs `%라면%` vs `_라면%` **패턴 차이** |
| 함정 | `BETWEEN 21 AND 22`에 20·23 **포함 안 됨** |
| 함정 | `WHERE AVG(나이)>20` → **HAVING** |
