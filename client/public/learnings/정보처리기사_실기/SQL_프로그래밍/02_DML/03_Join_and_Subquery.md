# [DML-3] 조인(JOIN) 및 서브쿼리(Subquery)

다중 테이블 **결과 집합**을 손으로 그리는 연습이 실기 고득점에 직결됩니다.

---

## 1. 개념

### 조인

- **INNER JOIN**: 양쪽에 **매칭되는 행만** 결과에 포함.
- **LEFT OUTER JOIN**: 왼쪽 테이블 행은 **전부** 유지, 오른쪽에 매칭 없으면 NULL.
- **RIGHT OUTER JOIN**: 오른쪽 기준(문제에서 드물게 명시).
- **FULL OUTER JOIN**: 양쪽 모두 유지(DB/교재에 따라 생략 가능).
- **쉼표 조인(암시적)**: `FROM A, B WHERE A.키 = B.키` — 조건 누락 시 **카테시안 곱**. 기출·교재에 나오면 문법 그대로.
- **NATURAL JOIN / USING**: 컬럼명이 같다고 자동 매칭하는 방식은 **의도와 다르게 붙는** 경우가 있어 실기·실무 모두 지문을 꼼꼼히 읽을 것.

### 서브쿼리

- **스칼라 서브쿼리**: 단일 값(한 행 한 열).
- **IN (서브쿼리)**: 서브쿼리 결과 집합에 **값이 포함**되면 참.
- **비교 연산자 + ANY | ALL**: 스칼라와 집합의 비교.
- **EXISTS (서브쿼리)**: 서브쿼리가 **한 행이라도** 반환하면 참(존재 여부).

---

## 2. 문법

### INNER JOIN

```sql
SELECT A.컬럼, B.컬럼
FROM   테이블A A
JOIN   테이블B B ON A.키 = B.키;
```

### LEFT OUTER JOIN

```sql
SELECT A.컬럼, B.컬럼
FROM   테이블A A
LEFT JOIN 테이블B B ON A.키 = B.키;
```

### 자기 자신과 조인 (SELF JOIN)

같은 테이블을 **별칭 두 개**로 읽어 관계를 표현(예: 사원–상사, 부품–상위부품).

```sql
SELECT e.성명 AS 사원, m.성명 AS 상사
FROM   사원 e
JOIN   사원 m ON e.상사번호 = m.사원번호;
```

### 서브쿼리

```sql
WHERE 컬럼 IN (SELECT ...);

WHERE 컬럼 > ANY (SELECT ...);
WHERE 컬럼 > ALL (SELECT ...);

WHERE EXISTS (SELECT 1 FROM ... WHERE 상관조건);
```

---

## 3. IN vs ANY vs ALL (정리)

> **IMPORTANT**
>
> - **IN**: 서브쿼리 결과 **하나라도 일치**하면 참( `= 값1 OR = 값2 ...` 와 유사).
> - **ANY**: `> ANY`면 결과 중 **최소 하나보다 크면** 참(예: `5 > ANY(3,7,9)` → 참).
> - **ALL**: `> ALL`면 결과 **모든 값보다 커야** 참(예: `10 > ALL(3,7,9)` → 거짓).

---

## 4. 예시

**사원(사원번호, 성명, 부서코드)**, **부서(부서코드, 부서명)**

```sql
SELECT s.성명, d.부서명
FROM   사원 s
JOIN   부서 d ON s.부서코드 = d.부서코드;
```

```sql
-- 부서가 배정되지 않은 사원까지(부서코드 NULL 또는 매칭 실패 케이스는 데이터에 따라)
SELECT s.성명, d.부서명
FROM   사원 s
LEFT JOIN 부서 d ON s.부서코드 = d.부서코드;
```

```sql
-- 급여가 부서 10 평균보다 큰 사원
SELECT *
FROM   사원
WHERE  급여 > (SELECT AVG(급여) FROM 사원 WHERE 부서코드 = '10');
```

```sql
-- 주문을 한 적 있는 고객만
SELECT *
FROM   고객 c
WHERE  EXISTS (
  SELECT 1 FROM 주문 o WHERE o.고객번호 = c.고객번호
);
```

---

## 5. 실행 결과(예시)

**사원**

| 사원번호 | 성명 | 부서코드 |
|---------|------|----------|
| 1 | 김 | 10 |
| 2 | 이 | 20 |
| 3 | 박 | NULL |

**부서**

| 부서코드 | 부서명 |
|---------|--------|
| 10 | 개발 |
| 20 | 영업 |

- **INNER JOIN** `사원·부서` → **2행**(박 제외).
- **LEFT JOIN** → **3행**, 박의 부서명은 **NULL**.

---

## 6. 실기 팁

- 조인 조건 누락 → **카테시안 곱**(행 폭발). 문제에서 의도한지 확인.
- **상관 서브쿼리**는 바깥 행마다 안쪽이 다시 평가된다는 점으로 EXISTS 의미를 기억.
- **INNER vs LEFT**: “전부 나와야 한다”는 지문이면 **LEFT**(또는 기준 테이블이 어느 쪽인지)를 의심.
- 여러 테이블을 이을 때는 **한 번에 한 조인씩** 조건을 확인(중간에 조인 키가 바뀌는 함정).

---

## 7. IN · ANY · ALL · EXISTS 한눈에 비교

| 연산 | 의미(직관) | 서브쿼리가 여러 행일 때 | 자주 쓰는 패턴 |
|------|------------|-------------------------|----------------|
| **`IN (서브쿼리)`** | 값이 결과 집합에 **있으면** 참 | 각 행과 `=` 비교의 OR 묶음과 비슷 | “목록에 포함?” |
| **`NOT IN (서브쿼리)`** | 결과 집합에 **없으면** 참 | 서브쿼리 결과에 **NULL이 있으면** 전체가 UNKNOWN→ 보통 **거짓으로 처리**되어 함정 | NULL 있으면 조심 |
| **`> ANY` / `< ANY` …** | 비교 연산자가 **한 행이라도** 성립하면 참 | `> ANY(3,7,9)` → 3,7,9 중 **하나보다만 커도** 참 | “최솟값보다 큰가?” 류 |
| **`> ALL` / `< ALL` …** | 비교 연산이 **모든 행**에 성립해야 참 | `> ALL(3,7,9)` → **9보다 커야** 참 | “최댓값보다 큰가?” 류 |
| **`EXISTS (서브쿼리)`** | 서브쿼리가 **1행이라도** 나오면 참 | SELECT 목록 내용은 거의 무관(`SELECT 1` 관례) | “존재하나?” 상관 서브쿼리 |
| **`NOT EXISTS`** | 서브쿼리가 **한 행도** 없으면 참 | IN/NOT IN보다 **NULL에 안전**한 편 | “한 번도 없는” 고객 등 |

---

## 8. 자주 틀리는 함정

### 8.1 `NOT IN`과 NULL

서브쿼리 결과에 NULL이 섞이면 `NOT IN`은 직관과 어긋날 수 있습니다. 실기에서는 **NOT EXISTS**로 바꾼 문장이 답인 유형이 나옵니다.

```sql
-- 안전한 패턴 예: 주문 이력이 전혀 없는 고객
SELECT *
FROM   고객 c
WHERE  NOT EXISTS (
  SELECT 1 FROM 주문 o WHERE o.고객번호 = c.고객번호
);
```

### 8.2 `IN` vs `EXISTS` (행이 많을 때 의미 차이)

- `WHERE 컬럼 IN (SELECT …)` : 보통 **값의 일치**가 본질.
- `WHERE EXISTS (SELECT …)` : **행 존재 여부**만 본다. 중복이 있어도 EXISTS는 참/거짓만.

### 8.3 스칼라 서브쿼리가 2행 이상이면

`WHERE 급여 = (SELECT …)` 형태는 서브쿼리가 **정확히 1행 1열**이어야 합니다. 2행 이상이면 **오류**인 DB가 일반적입니다.
