# Java 객체지향 핵심

## 핵심 요약

Java는 실기에서 객체지향의 대표 언어로 다뤄진다. 문제는 단순 문법보다 클래스 관계, 생성자 호출, 상속 구조, 오버로딩과 오버라이딩의 차이를 이해하고 있는지를 확인하는 방향으로 출제된다.

## 왜 중요한가

- 객체지향 개념은 Java 문제뿐 아니라 소프트웨어 공학 파트의 개념 문제와도 연결된다.
- 실기에서는 클래스 상속 구조를 제시하고 실행 결과를 묻는 문제가 자주 나온다.
- 특히 "어떤 메서드가 호출되는가"를 묻는 문제에서 다형성과 오버라이딩을 정확히 알아야 한다.

## 1. 클래스, 객체, 인스턴스

- 클래스: 객체를 만들기 위한 설계도
- 객체: 실제로 생성된 실체
- 인스턴스: 특정 클래스로부터 생성된 객체

```java
class Car {
    int speed;

    void run() {
        System.out.println("run");
    }
}

Car c = new Car();
```

## 2. 객체지향의 핵심 개념

### 2.1 캡슐화

- 데이터와 메서드를 하나로 묶는다.
- 내부 구현을 감추고 필요한 기능만 외부에 공개한다.

### 2.2 상속

- 상위 클래스의 속성과 메서드를 하위 클래스가 물려받는다.

```java
class Animal {
    void sound() {
        System.out.println("sound");
    }
}

class Dog extends Animal {
}
```

### 2.3 다형성

- 같은 이름의 메서드 호출이 객체 종류에 따라 다르게 동작한다.

## 3. 오버로딩과 오버라이딩

| 구분 | 오버로딩 | 오버라이딩 |
|---|---|---|
| 의미 | 같은 이름의 메서드를 여러 형태로 정의 | 부모 메서드를 자식이 재정의 |
| 조건 | 매개변수 타입/개수 달라야 함 | 메서드 이름/매개변수 같아야 함 |
| 관계 | 한 클래스 안에서도 가능 | 상속 관계 필요 |

### 오버로딩 예시

```java
class Calc {
    int add(int a, int b) { return a + b; }
    int add(int a, int b, int c) { return a + b + c; }
}
```

### 오버라이딩 예시

```java
class Parent {
    void show() {
        System.out.println("Parent");
    }
}

class Child extends Parent {
    void show() {
        System.out.println("Child");
    }
}
```

## 4. 생성자와 `super`

- 생성자는 객체 생성 시 호출된다.
- 자식 클래스 객체를 만들면 부모 생성자가 먼저 호출된다.
- `super()`는 부모 생성자를 호출할 때 사용한다.

```java
class Parent {
    Parent() {
        System.out.println("P");
    }
}

class Child extends Parent {
    Child() {
        System.out.println("C");
    }
}
```

`new Child()` 를 하면 출력 순서는 `P` -> `C` 이다.

## 5. 추상 클래스와 인터페이스

- 추상 클래스: 공통 기능을 일부 구현하고 일부는 미완성으로 남길 수 있다.
- 인터페이스: 구현해야 할 규약을 정의한다.

실기에서는 "공통 규약", "다중 구현", "추상 메서드" 정도를 구분할 줄 알면 충분한 경우가 많다.

## 6. 코드 추적 예제

### 예제 1: 오버라이딩

```java
class A {
    void show() {
        System.out.print("A");
    }
}

class B extends A {
    void show() {
        System.out.print("B");
    }
}

public class Main {
    public static void main(String[] args) {
        A obj = new B();
        obj.show();
    }
}
```

- 참조 변수 타입은 `A`
- 실제 객체는 `B`
- 오버라이딩된 메서드는 실제 객체 기준으로 호출
- 출력: `B`

### 예제 2: 생성자 호출 순서

```java
class A {
    A() {
        System.out.print("1");
    }
}

class B extends A {
    B() {
        System.out.print("2");
    }
}
```

- `new B()` 수행 시 부모 생성자 먼저
- 출력: `12`

## 7. 자주 틀리는 함정

### 7.1 오버로딩과 오버라이딩 혼동

- 이름만 같다고 오버라이딩이 아니다.
- 상속 관계와 매개변수 동일 여부를 함께 봐야 한다.

### 7.2 참조 변수 타입만 보고 메서드 결정

- 오버라이딩 메서드는 실제 객체 타입 기준으로 동작한다.

### 7.3 생성자 호출 순서 착각

- 자식 생성자보다 부모 생성자가 먼저 실행된다.

## 8. 시험장에서 푸는 순서

1. 클래스 간 상속 관계를 그림처럼 적는다.
2. 객체 생성 시 실제 객체 타입을 확인한다.
3. 메서드가 오버로딩인지 오버라이딩인지 구분한다.
4. 생성자가 있으면 부모 -> 자식 순서를 먼저 본다.
5. 출력문이 있으면 호출 순서를 줄 단위로 써 본다.

## 한 페이지 요약

- Java 문제의 핵심은 클래스 관계와 다형성이다.
- 오버로딩은 매개변수 차이, 오버라이딩은 상속 후 재정의다.
- 부모 타입 참조 변수라도 실제 객체가 자식이면 자식 메서드가 실행된다.
- 생성자는 부모가 먼저, 자식이 나중이다.
- 객체지향 문제는 관계를 표로 그리면 훨씬 쉽게 풀린다.
