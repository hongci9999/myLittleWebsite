# 정보처리기사 실기 데이터베이스 단원 재설계

**날짜:** 2026-04-14  
**관련:** `client/public/learnings/정보처리기사_실기/10_체계형_학습자료`

**Goal:** 기존 요약형 데이터베이스 단원을 실기 시험용 강의 교안 수준의 세부 문서 세트로 재작성한다.

---

## 1) 배경

- 현재 `10_체계형_학습자료/02_데이터베이스`는 2개 파일로 구성되어 있다.
- 기본 분류는 맞지만, 실제 시험 대비용으로 보기에는 SQL 세부 유형, 정규화 판단 기준, 트랜잭션/회복/병행제어 설명이 부족하다.
- 프로젝트 안에는 필기용 데이터베이스 문서와 실기 SQL 문서가 이미 있으므로, 이번 단원은 이를 바탕으로 실기용 강의 교안으로 재구성한다.

---

## 2) 목표 품질

- 정의만 나열하는 요약집이 아니라, 개념과 문제 풀이를 연결하는 강의노트로 작성한다.
- SQL은 문법만 적지 않고 "언제 어떤 절을 쓰는가", "왜 틀리기 쉬운가"를 함께 설명한다.
- 정규화는 단계명 암기에 그치지 않고, 함수 종속과 이상 현상 기준으로 판단하는 흐름을 넣는다.
- 트랜잭션/병행제어는 ACID, 상태 변화, 장애/회복, 동시성 문제를 함께 다룬다.

## 4) 문서 템플릿

모든 문서는 아래 흐름을 기본으로 통일한다.

1. 메타 정보 (`날짜`, `태그`, `주제`, `중요도`)
2. 핵심 요약
3. 왜 시험에 중요한가
4. 상세 개념 설명
5. 비교표 / 규칙 정리
6. SQL 또는 개념 추적 예제
7. 자주 틀리는 함정
8. 시험장에서 푸는 순서
9. 한 페이지 요약

## 5) 내용 범위

### DB 기본 이론

- DB 정의와 특성
- DBMS 기능
- 스키마 3계층
- 릴레이션, 속성, 튜플, 도메인

### 키/무결성

- 후보키, 기본키, 대체키, 슈퍼키, 외래키
- 개체/참조 무결성
- NULL, 차수, 카디널리티

### SQL

- DDL: CREATE, ALTER, DROP, TRUNCATE, 제약조건
- DML: SELECT, INSERT, UPDATE, DELETE
- 그룹화, 집계, 정렬
- JOIN, 서브쿼리, EXISTS, ANY, ALL, 윈도우 함수

### 정규화

- 이상 현상
- 함수 종속
- 1NF~BCNF, 4NF, 5NF
- 반정규화

### 트랜잭션/병행제어/회복

- ACID, COMMIT, ROLLBACK, SAVEPOINT
- 교착상태, 로킹, 타임스탬프, 낙관적 제어
- UNDO, REDO, 체크포인트, 즉시/지연 갱신

### 기타 객체

- 인덱스
- 뷰
- 프로시저, 함수, 트리거
- 시스템 카탈로그

---

## 6) 품질 기준

- 모든 SQL 문서는 최소 2개 이상의 SQL 예제를 포함한다.
- 개념 문서도 표, 사례, 함정 설명을 반드시 포함한다.
- DBMS별 차이는 "문제에서 제시된 문법 우선"이라는 원칙으로 정리한다.
- 기존 실기 SQL 폴더와 중복되더라도, 데이터베이스 단원 안에서 전체 체계를 스스로 설명할 수 있어야 한다.
