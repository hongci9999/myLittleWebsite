# [기타] 내장 함수 및 뷰(View)

실기에서는 **함수 이름 맞히기**, **함수 적용 결과 값**, **뷰 정의/갱신 가능 여부**가 단답형으로 나올 수 있습니다.

---

## 1. 개념

- **내장 함수**: DBMS가 제공하는 단일행 함수(문자열, 숫자, 날짜 등). 시험 범위는 교재에 나온 목록이 핵심.
- **뷰(View)**: 저장된 **질의(SELECT)** 에 이름을 붙인 가상 테이블. 보안·단순화에 사용.
- **단순 뷰 / 복합 뷰**: 조인·집계·GROUP BY 등이 들어가면 **복합 뷰**가 되어 **갱신 제한**이 커지는 경우가 많음(문제에서 “삽입/수정 가능?” 유형).

---

## 2. 문법 (자주 나오는 함수 예시)

> 함수 이름·인자 순서는 **제품/교재 기준**이 다를 수 있음. 아래는 **개념 학습용**입니다.

### 문자열

```sql
LENGTH(문자열)           -- 길이
SUBSTRING(문자열 FROM 시작 FOR 길이)
TRIM(문자열)
UPPER(문자열), LOWER(문자열)
CONCAT(a, b)             -- 또는 || 연산자
```

### 숫자

```sql
ABS(n), MOD(n, m), ROUND(n [, 자릿수]), CEIL/FLOOR(n)
```

### 날짜(교재별)

```sql
CURRENT_DATE
EXTRACT(YEAR FROM 날짜컬럼)
```

### 뷰 생성

```sql
CREATE VIEW 뷰이름 [(컬럼목록)] AS
SELECT ...
[WITH CHECK OPTION];  -- 문제에 나오면 “조건 위반 삽입/갱신 방지”로 이해
```

```sql
DROP VIEW 뷰이름 [CASCADE | RESTRICT];
```

---

## 3. 예시

```sql
SELECT 성명, LENGTH(성명) AS 이름길이
FROM   사원;

SELECT SUBSTRING(전화번호 FROM 1 FOR 3) AS 국번
FROM   고객;

CREATE VIEW v_고급사원 AS
SELECT 사원번호, 성명, 급여
FROM   사원
WHERE  급여 >= 500;
```

---

## 4. 실행 결과(예시)

**성명 `'김철수'`**

- `LENGTH` 결과는 DB 문자 집합에 따라 바이트/문자 단위가 달라질 수 있음 → **문제에서 정의한 규칙** 따름.
- `UPPER('kim')` → `'KIM'`.

**뷰 `v_고급사원`**

- `SELECT * FROM v_고급사원`은 내부적으로 저장된 SELECT를 다시 실행한 결과와 동일.
- 복잡한 뷰에 대한 `INSERT` 가능 여부는 **문제/교재 설명**을 그대로 선택.

---

## 5. 실기 체크

- [ ] 집계·DISTINCT·GROUP BY가 들어간 뷰는 **갱신 불가**인 경우가 많다는 직관.
- [ ] `WITH CHECK OPTION`이 붙으면 **뷰 조건을 벗어나는 행** 삽입/갱신이 막힐 수 있음.
