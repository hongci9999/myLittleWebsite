# 분석모형 설계 (Analysis Model Design)

## 📌 핵심 요약
분석 절차 수립 시 **모형 선정·정의**, **구축 4단계**(요건정의→모델링→검증→적용), **분석 도구(R/Python)**, **데이터 분할**(훈련·검증·평가)을 이해합니다.

---

## 📝 상세 정리

### 1. 분석 절차 수립 — 분석모형 선정

#### ● 분석모형 선정 (분석 방법론별)

| 기반 | 대표 모형 |
| :--- | :--- |
| **(1) 통계 기반** | 회귀분석, 상관분석, 주성분분석(PCA), 분산분석(ANOVA), 판별분석 |
| **(2) 데이터마이닝 기반** | 분류모델, 예측모델, 군집모델, 연관규칙모델 |
| **(3) 머신러닝 기반** | 지도학습, 비지도학습, 준지도학습, 강화학습 |
| **(4) 비정형데이터 기반** | 텍스트 마이닝, 오피니언 마이닝, 소셜네트워크 분석 |

---

### 2. 분석모형 정의

#### ● 분석모형 정의 시 고려사항

| 구분 | 설명 |
| :--- | :--- |
| **(1) 과대적합 (Overfitting)** | 모델이 지나치게 데이터를 학습하여 **매우 복잡해진** 모델. 훈련 데이터에는 잘 맞지만 일반화 성능 저하. |
| **(2) 과소적합 (Underfitting)** | 데이터를 **충분히 설명하지 못하는** 단순한 모델. 훈련·테스트 모두 낮은 성능. |
| **(3) 모형 선택의 오류** | 목적·데이터 특성에 **적합하지 않은 모델** 선택. |
| **(4) 변수 선택의 오류** | **관련 있는 변수 누락** 또는 **관련 없는 변수 포함**. |
| **(5) 데이터 편향 (Data Bias)** | **대표성 없는** 훈련 데이터로 인한 편향. |

#### ● 적합도 시각적 이해 (Overfitting / Justfitting / Underfitting)

| 유형 | 특징 | 의사결정 경계 |
| :--- | :--- | :--- |
| **Overfitting** | 훈련 데이터를 거의 완벽히 분리. 경계가 과도하게 구불구불. **일반화 실패**. | 복잡·꼬임 |
| **Justfitting** | 적당한 곡선으로 대부분 분리. 일부 오분류 허용. **균형 잡힌 일반화**. | 부드럽고 적절 |
| **Underfitting** | 단순한 직선 수준. 빨강·파랑이 섞여 분리 실패. **패턴 포착 실패**. | 지나치게 단순 |

> [!TIP] 목표
> **Justfitting** 수준의 모델을 찾는 것이 이상적. 과대적합·과소적합을 피하고, 검증 데이터로 일반화 성능을 확인한다.

---

### 3. 분석모형 구축 절차

#### ● 4단계 절차 (순차 진행)

| 단계 | 내용 |
| :--- | :--- |
| **(1) 요건정의** | 분석요건 도출 → 수행방안 설계 → 요건 확정 |
| **(2) 모델링** | 설계 및 구축, 탐색적 분석 및 유의변수 도출, 모델링, **모델링 성능평가** |
| **(3) 검증 및 테스트** | 실제 테스트 수행, **비즈니스 영향도 평가** |
| **(4) 적용** | 운영시스템에 적용, **주기적 리모델링** |

---

### 4. 분석 환경 구축

#### ● 분석 도구 선정 (R vs Python)

| 도구 | 특징 |
| :--- | :--- |
| **(1) R** | 통계 분석에 특화, 처리속도 느림, **강력한 시각화**, SPSS/SAS 등 연동 가능 |
| **(2) Python** | 간결함과 높은 가독성, **R보다 속도 빠름**, C 구현 모듈과 연동, R보다 시각화 약함 |

---

### 5. 데이터 분할 (Data Splitting)

**목적:** 과대적합·과소적합 방지, 불균형 데이터 문제 해결.

| 구분 | 용도 |
| :--- | :--- |
| **(1) 훈련용 (Training Set)** | **모델 학습**에 활용 |
| **(2) 검증용 (Validation Set)** | **과대·과소적합 조정**에 활용 (하이퍼파라미터 튜닝 등) |
| **(3) 평가용 (Test Set)** | **최종 모델 평가**에 활용 (학습·검증에 사용하지 않은 데이터) |

> [!NOTE] 검증 vs 평가
> **검증용**: 모델 튜닝·조정 과정에서 사용. **평가용**: 최종 성능 측정용으로, 모델 선택·튜닝에 관여하지 않아야 함.
