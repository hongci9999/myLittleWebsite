# 데이터 정제 (Data Cleaning)

## 📌 핵심 요약
데이터 분석 전 반드시 수행해야 하는 단계로, 기록되지 않은 값인 **결측치(Missing Value)**와 데이터의 일반적인 패턴에서 현저히 벗어난 **이상치(Outlier)**를 탐지하고 처리합니다. 이는 분석 결과의 왜곡을 방지하고 모델의 성능을 향상시키는 데 목적이 있습니다.

---

## 📝 상세 정리

### 1. 결측치 (Missing Value)
데이터셋에서 특정 변수의 값이 기록되지 않은 상태를 의미합니다. 존재하지 않는 데이터는 `null` 또는 `NA`로 표시됩니다.

* **결측값 종류:**
    * **완전 무작위 결측 (MCAR, Missing Completely At Random):** 다른 변수들과 아무런 상관이 없이 결측이 발생하는 경우.
        * 예시: 입력 실수, 전산 오류
    * **무작위 결측 (MAR, Missing At Random):** 특정 변수와 관련되어 발생하였지만, 결과와는 관계가 없는 경우.
        * 예시: 특정 정치 성향 유권자들의 응답률이 낮으나 정당의 득표율이 낮은 것은 아님
    * **비 무작위 결측 (MNAR, Missing Not At Random):** 결측치가 변수의 결과에 상관이 있는 경우.
        * 예시: 소득이 낮은 응답자들의 응답률이 낮음

* **발생 원인:**
    * **데이터 수집 오류:** 센서 고장, 전송 누락 등 기계적 결함.
    * **의도적인 미응답:** 설문 조사 시 민감한 질문에 대해 답변을 거부한 경우.
    * **측정 불가능:** 특정 조건에서 값을 측정할 수 없는 경우.
* **결측값 처리 방법:**
    * **(1) 단순대치법 (Single Imputation Methods)**
        * **완전분석법 (Complete Case Analysis / Listwise Deletion):** 결측값을 가진 모든 데이터를 **삭제**.
        * **평균 대치법 (Mean Imputation):**
            * 비조건부 평균 대치법: **단순 평균**으로 대치.
            * 조건부 평균 대치법: **회귀분석**의 결과로 대치.
        * **단순 확률 대치법 (Single Random Imputation):** 확률적으로 선택하여 대치.
            * **Hot-Deck:** 현재 데이터셋에서 비슷한 성향으로 대체.
            * **Cold-Deck:** 유사한 외부 출처에서 비슷한 성향으로 대체.
            * **Nearest Neighbor (최근접 이웃):** 바로 가까운 응답으로 대체.
    * **(2) 다중 대치법 (Multiple Imputation Methods)**
        * 여러 번 대치하여 종합 (**대치 → 분석 → 결합**).
        * 단순대치법의 단점인 **분산 과소 추정** 문제 해결이 가능.

### 2. 이상치 (Outlier)
극단적으로 크거나 작은 값을 의미하며, 의미 있는 데이터일 수도 있습니다 (예: 키 220cm). **이상값을 항상 제거하는 것은 아님**에 유의합니다.

* **발생 원인:**
    * **입력 오류:** 수치 입력 시 오타(예: 10을 1000으로 입력).
    * **자연스러운 변동:** 실제 현상에서 아주 드물게 발생하는 극단적인 값.
    * **데이터 처리 오류:** 데이터 변환이나 결합 과정에서 발생하는 왜곡.
* **이상값 탐지 방법:**
    * **(1) ESD (Extreme Studentized Deviation):** 평균으로부터 표준편차의 3배 넘어가는 데이터는 이상값으로 판단. ($\bar{X} \pm 3\sigma$ 범위 밖)
    * **(2) 사분위수 (IQR) 방법:** $Q_1 - 1.5 \times IQR$보다 작거나, $Q_3 + 1.5 \times IQR$보다 크면 이상값으로 판단. 상한·하한·최솟값·1~3사분위값·최댓값 등을 표현하며, 평균값은 표현하지 않음.
    * **(3) Z-Score 방법:** 데이터를 정규화(평균 0, 표준편차 1)한 뒤, 일정 임계값을 초과할 경우 이상값으로 판단.
    * **(4) DBSCAN 방법:** 밀도를 이용하여 밀도가 적은 부분의 데이터를 이상값으로 판단.
* **처리 방법:**
    * **삭제:** 분석 목적에 맞지 않는 오류성 이상치는 제거.
    * **값 대체:** 상한값이나 하한값으로 보정하거나 평균 등으로 대체.
    * **변환 (Transformation):** 로그 변환($\log$), 제곱근 변환($\sqrt{x}$) 등을 통해 데이터의 치우침(Skewness)을 완화.

> [!INFO] 💡 AI 보충 설명
> 
> **1. 범주형(형태형) 데이터의 결측치 처리**
> 수치형과 달리 평균을 낼 수 없으므로 주로 다음 방법을 사용합니다.
> * **최빈값 대체:** 가장 자주 등장하는 항목으로 채움.
> * **새로운 범주 생성:** '알 수 없음(Unknown)'이라는 별도의 카테고리로 분류.
> * **예측 모델:** 분류 알고리즘(예: Logistic Regression, Random Forest)을 사용하여 적절한 카테고리를 예측.
> 
> **2. 자가회귀(Autoregression)를 이용한 결측치 대체**
> 시계열 데이터에서 유용한 방법입니다.
> * 이전 시점의 데이터들이 현재 시점의 데이터에 영향을 준다는 가정하에, 과거의 관측값을 독립변수로 하여 결측 시점의 값을 예측합니다.
> * 데이터의 추세(Trend)나 계절성(Seasonality)이 뚜렷할 때 효과적입니다.
> 
> **3. 일변량 분석(Univariate Analysis)이란?**
> * **정의:** 단 하나의 변수(Feature)에 대해서만 데이터의 특성을 파악하는 분석 기법입니다.
> * **목적:** 변수의 분포, 중심 경향성(평균, 중앙값), 분산도(표준편차, 범위) 등을 파악하여 데이터의 기본적인 성질을 이해하는 것입니다.
> * **방법:** 히스토그램, 박스플롯, 빈도 분석 등이 주로 쓰입니다.
