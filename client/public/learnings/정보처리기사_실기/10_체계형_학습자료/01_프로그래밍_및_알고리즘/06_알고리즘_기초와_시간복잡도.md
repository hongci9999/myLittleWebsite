# 알고리즘 기초와 시간복잡도

## 핵심 요약

알고리즘 문제를 잘 푼다는 것은 코드를 많이 아는 것보다 "이 문제의 계산량이 어느 정도인가"와 "반복이 몇 번 일어나는가"를 읽어내는 능력에 가깝다. 실기에서는 빅오 표기, 반복 구조, 자료 이동 횟수를 묻는 문제가 자주 나온다.

## 왜 중요한가

- 알고리즘 기본 개념은 정렬/탐색 파트의 기초가 된다.
- 시간복잡도를 이해하면 어떤 알고리즘이 더 효율적인지 비교할 수 있다.
- 실기에서는 수학적 증명보다 반복 횟수와 중첩 구조 해석 능력이 중요하다.

## 1. 알고리즘이란

- 어떤 문제를 해결하기 위한 절차 또는 방법
- 입력이 들어왔을 때 정해진 순서대로 처리해 원하는 출력을 만든다.

좋은 알고리즘은 보통 다음 기준으로 평가한다.

- 정확성
- 효율성
- 단순성
- 확장성

## 2. 시간복잡도와 공간복잡도

- 시간복잡도: 입력 크기에 따라 연산 횟수가 얼마나 늘어나는가
- 공간복잡도: 문제 해결에 추가로 필요한 메모리 양이 얼마나 되는가

실기에서는 대부분 시간복잡도를 중심으로 본다.

## 3. 빅오 표기

| 표기 | 의미 | 예시 |
|---|---|---|
| `O(1)` | 상수 시간 | 배열 인덱스 접근 |
| `O(log n)` | 로그 시간 | 이진 탐색 |
| `O(n)` | 선형 시간 | 순차 탐색 |
| `O(n log n)` | 선형로그 시간 | 퀵/병합 정렬 평균 |
| `O(n^2)` | 제곱 시간 | 버블/선택/삽입 정렬 |

## 4. 반복문과 시간복잡도 해석

### 4.1 단일 반복문

```c
for(i = 0; i < n; i++) {
    sum += i;
}
```

- `n`번 반복
- 시간복잡도: `O(n)`

### 4.2 중첩 반복문

```c
for(i = 0; i < n; i++) {
    for(j = 0; j < n; j++) {
        count++;
    }
}
```

- 바깥쪽 `n`번
- 안쪽 `n`번
- 총 `n * n`
- 시간복잡도: `O(n^2)`

## 5. 상수와 낮은 차수는 생략

`O(2n + 3)` 는 `O(n)` 으로 본다.

이유:

- 입력이 커질수록 가장 영향이 큰 항만 중요하기 때문이다.

## 6. 코드 추적 예제

### 예제 1: 반복 횟수 계산

```c
int count = 0;
for(i = 1; i <= 5; i++) {
    count++;
}
```

- 1부터 5까지 총 5회
- 시간복잡도는 입력이 고정이면 상수지만, 일반화하면 `n`회 반복 구조로 `O(n)` 형태로 이해한다.

### 예제 2: 절반씩 줄어드는 반복

```c
int i = n;
while(i > 1) {
    i = i / 2;
}
```

- `n -> n/2 -> n/4 -> ...`
- 반복 횟수는 로그 형태
- 시간복잡도: `O(log n)`

## 7. 자주 틀리는 함정

### 7.1 연산 횟수와 코드 줄 수를 혼동

- 코드가 짧아도 반복 구조가 깊으면 복잡도가 커질 수 있다.

### 7.2 `O(n + n)` 을 `O(n^2)`로 오해

- 순차적으로 두 번 도는 것은 `O(2n)` -> `O(n)` 이다.

### 7.3 로그 시간 직관 부족

- 입력을 절반씩 줄이는 구조는 보통 `O(log n)` 이다.

## 8. 시험장에서 푸는 순서

1. 반복문이 몇 겹인지 본다.
2. 반복 횟수가 `n`, `n*n`, `n/2` 중 어느 형태인지 분류한다.
3. 순차 반복인지 중첩 반복인지 구분한다.
4. 상수와 낮은 차수는 마지막에 제거한다.
5. 정렬/탐색과 연결해서 함께 암기한다.

## 한 페이지 요약

- 시간복잡도는 반복 횟수 증가율을 보는 개념이다.
- 단일 반복문은 보통 `O(n)`, 중첩 반복문은 `O(n^2)` 이다.
- 절반씩 줄어들면 `O(log n)` 을 의심한다.
- 상수와 낮은 차수는 생략한다.
- 복잡도는 정렬/탐색 문제의 비교 기준으로 자주 쓰인다.
