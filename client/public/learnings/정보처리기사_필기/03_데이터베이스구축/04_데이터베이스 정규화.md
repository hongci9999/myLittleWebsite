# 데이터베이스 정규화 (Normalization)

## 📌 핵심 요약
데이터의 중복성을 최소화하고 **[[이상 현상(Anomaly)]]**을 제거하여 데이터의 일관성과 무결성을 보장하는 과정입니다. 함수적 종속성을 분석하여 무손실 분해를 수행하는 것이 핵심입니다.

---

## 📝 상세 정리

### 1. 함수적 종속 (Functional Dependency)
데이터베이스 내의 속성들 간에 존재하는 의존 관계를 의미합니다.

#### 1.1 종속성의 종류
- **[[함수 종속(FD)]]:** 어떤 속성 $X$의 값을 알면 다른 속성 $Y$의 값이 유일하게 결정되는 관계 ($X \to Y$).
    - **결정자(Determiner):** 종속성을 규명하는 기준이 되는 속성 ($X$).
    - **종속자(Dependent):** 결정자에 의해 값이 정해지는 속성 ($Y$).
- **[[다가 종속(MVD, Multi-valued Dependency)]]:** 하나의 결정자에 대해 여러 개의 독립적인 속성 값이 종속되는 경우 ($X \twoheadrightarrow Y$).
- **[[조인 종속(JD, Join Dependency)]]:** 릴레이션을 여러 개로 분해했다가 다시 조인했을 때 원래대로 복원될 수 있는 성질입니다.

#### 1.2 폐포 (Closure)
특정 속성 집합으로부터 함수적 종속성에 의해 유도될 수 있는 모든 속성의 집합을 의미하며 $X^+$로 표기합니다.
> **예시:** {회원번호}$^+$ = {회원번호, 이름, 연락처, 주소}

#### 1.3 암스트롱의 추론 규칙 (Armstrong's Axioms)
1. **반사 규칙:** $Y \subseteq X$ 이면 $X \to Y$ 이다.
2. **증가 규칙:** $X \to Y$ 이면 $XZ \to YZ$ 이다.
3. **이행 규칙:** $X \to Y$ 이고 $Y \to Z$ 이면 $X \to Z$ 이다.
4. **결합 규칙:** $X \to Y$ 이고 $X \to Z$ 이면 $X \to YZ$ 이다.
5. **분해 규칙:** $X \to YZ$ 이면 $X \to Y$ 이고 $X \to Z$ 이다.

---

### 2. 정규화 수행 및 이상 현상
정규화되지 않은 데이터베이스에는 데이터 조작 시 부작용인 **이상 현상**이 발생합니다.

- **[[삽입 이상(Insertion Anomaly)]]:** 데이터를 삽입할 때 원치 않는 정보까지 함께 삽입해야 하거나 삽입이 불가능한 현상.
- **[[갱신 이상(Update Anomaly)]]:** 중복된 데이터 중 일부만 수정되어 데이터 불일치가 일어나는 현상.
- **[[삭제 이상(Deletion Anomaly)]]:** 정보를 삭제할 때 연쇄적으로 필요한 정보까지 삭제되는 현상.

### 3. 정규화 단계 (Normal Forms)
정규화는 하위 단계의 조건을 모두 만족하면서 추가 조건을 충족해야 합니다.

| 단계 | 조건 | 설명 |
| :--- | :--- | :--- |
| **[[1NF (제1정규형)]]** | **원자값(Atomic Value)** | 모든 속성값이 더 이상 분해되지 않는 원자값이어야 함. |
| **[[2NF (제2정규형)]]** | **부분 함수 종속 제거** | 기본키가 복합키일 때, 기본키의 일부에만 종속되는 속성을 분리함. (완전 함수 종속화) |
| **[[3NF (제3정규형)]]** | **이행적 함수 종속 제거** | 기본키 → 일반속성 → 일반속성 관계를 분리함. ($A \to B, B \to C$ 관계 해소) |
| **[[BCNF (보이스-코드)]]** | **결정자이면서 후보키가 아닌 것 제거** | 모든 결정자가 후보키여야 함. (강한 제3정규형) |
| **[[4NF (제4정규형)]]** | **다가 종속(MVD) 제거** | 다치 종속 관계를 분리함. |
| **[[5NF (제5정규형)]]** | **조인 종속(JD) 제거** | 후보키를 통하지 않은 조인 종속을 제거함. |

#### 1NF: 제1정규형 (원자값 확보)
* **대상:** 한 칸에 여러 값이 들어있는 경우 (예: '취미' 컬럼에 '축구, 독서'가 같이 있음)
* **해결:** 중복된 행을 만들더라도 모든 속성을 **원자값(Atomic Value)**으로 분리.

#### 2NF: 제2정규형 (부분 함수 종속 제거)
* **대상:** 기본키가 **복합키(속성A + 속성B)**일 때, 일반 속성이 키의 **일부(속성A)**에만 종속된 경우.
* **예시:** `[학번, 과목코드]`가 기본키인데, `이름` 속성이 `학번`에만 종속되어 있다면 분리.
* **공식:** $2NF = 1NF - \text{Partial Functional Dependency}$



#### 3NF: 제3정규형 (이행 함수 종속 제거)
* **대상:** 기본키가 아닌 속성 간에 종속성이 존재하는 경우.
* **예시:** `학번(A) → 학과코드(B) → 학과명(C)` 관계에서 `학과명`은 `학번`에 이행적으로 종속됨.
* **해결:** `[학번, 학과코드]` 테이블과 `[학과코드, 학과명]` 테이블로 분리.
* **공식:** $3NF = 2NF - \text{Transitive Functional Dependency}$

#### BCNF: 보이스-코드 정규형
* **대상:** 모든 결정자가 **후보키**가 아닌 경우.
* **예시:** 수강 테이블에서 `[학생, 과목] → 교수`인데, 특정 `교수`가 한 과목만 담당하여 `교수 → 과목` 종속성이 생길 때, `교수`는 결정자이지만 후보키가 아니므로 분리.

### 4. 논리 데이터 모델 품질 검증

#### 4.1 주요 평가 요소
- **완전성:** 업무 요구사항에 필요한 모든 데이터가 누락 없이 정의되었는가?
- **중복 배제:** 동일한 사실은 한 번만 기록되었는가? (필요에 따라 의도적 중복 허용 가능)
- **업무 규칙:** 비즈니스 로직에 따라 데이터를 정확히 조작할 수 있는가?
- **데이터 재사용:** 통합 설계를 통해 여러 서비스에서 활용 가능한가?
- **안정성/간결성:** 데이터 구조가 단순하고 변화에 유연한가?

#### 4.2 품질 기준
1. **정확성:** 실제 데이터와 모델이 일치하는가?
2. **준거성:** 명명 규칙, 표준화 가이드라인, 법적 요건을 준수했는가?
3. **최신성:** 현재의 업무 프로세스를 정확히 반영하고 있는가?
4. **일관성:** 데이터 간 모순이 없는가?

> [!INFO] 💡 AI 보충 설명
> **반정규화(Denormalization):** 정규화는 데이터 무결성을 높이지만 테이블 수가 늘어나 조인(Join) 연산이 많아지고 성능이 저하될 수 있습니다. 이를 해결하기 위해 성능 향상을 목적으로 의도적으로 정규화를 위배하는 과정을 반정규화라고 합니다.

## 🎯 빈출 키워드
- **[[함수적 종속성]]**: 정규화의 판단 기준이 되는 속성 간 의존 관계입니다.
- **[[이행적 함수 종속]]**: $A \to B$이고 $B \to C$일 때 $A \to C$가 성립하는 관계이며, 3NF의 제거 대상입니다.
- **[[이상 현상]]**: 정규화를 수행하는 근본적인 원인인 삽입, 삭제, 갱신 시의 부작용입니다.
