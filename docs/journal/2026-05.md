## 2026-05-16

- **S3 + CloudFront 연동 완료** — 프론트 버킷 `mylittlewebsite-dev-661596276927-ap-northeast-2-an`을 CloudFront 배포 `EAV5ODYEOW4VD` 오리진으로 연결(OAC `oac-mylittlewebsite-dev-…`), 버킷은 퍼블릭 차단·`GetObject`는 `cloudfront.amazonaws.com` + `AWS:SourceArn` 조건으로만 허용하는 버킷 정책 적용.
- **접속** — 브라우저는 **CloudFront 도메인**(`https://….cloudfront.net`) 사용. S3 버킷 REST/웹사이트 URL 직접 접속은 OAC 정책상 `AccessDenied`가 나는 것이 정상.
- **실수 수정** — 오리진 **Origin path**에 `/index.html`을 넣으면 `/assets/...` 등이 잘못된 S3 키로 매핑됨 → **비움**. 루트 문서는 배포 **Default root object** = `index.html`로만 지정.
- **콘솔 안내** — 버킷에 정적 웹 호스팅이 켜져 있으면 “website endpoint 추천” 메시지가 뜰 수 있으나, **버킷 엔드포인트 + OAC** 유지(학습 문서·walkthrough §3 권장과 동일).
- **다음** — SPA면 CloudFront 커스텀 에러(403/404 → `index.html`, 200) 확인, `VITE_API_BASE_URL` 재빌드·S3 재배포, EB `CORS_ALLOWED_ORIGINS`에 CloudFront HTTPS 출처 추가, GitHub Actions 변수 `CLOUDFRONT_DISTRIBUTION_ID` 연동 시 배포 후 invalidation.
