# 데이터 및 빅데이터의 기초 개념과 데이터베이스의 이해

## 📌 핵심 요약

본 강의는 데이터의 정의와 **DIKW 피라미드**, **지식 순환 프로세스(SECI 모델)**를 통해 데이터가 지혜로 변모하는 과정을 다룹니다. 또한 **데이터베이스(DB)**의 특징과 설계 절차, **빅데이터**의 3V/5V/7V 특징 및 기술적·사회적 출현 배경을 학습하며, 데이터 사이언티스트가 갖춰야 할 하드/소프트 스킬과 분석 조직 구조의 유형을 정리합니다.

---

## 📝 상세 정리

### 1. 데이터의 정의 및 특징

- **데이터(Data):** 있는 그대로의 객관적 사실이며, 가공되지 않은 상태를 의미함. (예: 주문 수량)
  - **존재적 특성:** 현실 세계를 그대로 반영함.
  - **당위적 특성:** 맥락 속에서 의미 부여가 가능함.
- **정보(Information):** 데이터로부터 가공된 자료를 의미함. (예: 베스트셀러 목록)

### 2. DIKW 피라미드

데이터가 가공을 거쳐 지혜로 발전하는 계층 구조입니다.

1. **데이터(Data):** 가공 전의 객관적인 사실. (예: A대리점 핸드폰 100만 원, B대리점 200만 원)
2. **정보(Information):** 데이터를 통해 패턴을 인식함. (예: A대리점이 더 저렴하다는 사실 파악)
3. **지식(Knowledge):** 패턴을 의사결정에 활용함. (예: A대리점에서 사는 것이 이득이라는 판단)
4. **지혜(Wisdom):** 창의적인 전략을 도출함. (예: A대리점의 다른 기기도 저렴할 것이라는 유추)

### 3. 암묵지와 형식지의 상호작용 (SECI 모델)

- **암묵지(Tacit Knowledge):** 개인에게 습득되어 겉으로 드러나지 않는 지식.
- **형식지(Explicit Knowledge):** 문서, 매뉴얼 등 형상화된 지식.
- **지식 전환 4단계 (공표연내):**
  - **공통화(Socialization):** 암묵지 지식을 타인에게 직접 전수함.
  - **표출화(Externalization):** 암묵지를 매뉴얼이나 문서로 전환함.
  - **연결화(Combination):** 기존 매뉴얼에 새로운 지식을 추가/결합함.
  - **내면화(Internalization):** 문서화된 지식을 보고 타인의 지식을 본인의 것으로 터득함.

### 4. 데이터베이스(Database)의 이해

#### (1) 데이터베이스의 4가지 핵심 특징 (공통저변)

- **공용 데이터(Shared Data):** 여러 사용자가 서로 다른 목적으로 공동 이용함.
- **통합된 데이터(Integrated Data):** 동일 데이터의 중복을 최소화하여 통합함.
- **저장된 데이터(Stored Data):** 컴퓨터가 접근 가능한 저장 매체에 저장됨.
- **변화되는 데이터(Continuous Evolution):** 삽입, 삭제, 수정 등을 통해 항상 최신의 정확한 데이터를 유지함(무결성).

#### (2) 데이터베이스 설계 절차 (개논물)

1. **개념적 설계(Conceptual Design):** 추상적인 개념 스키마(ERD) 설계.
2. **논리적 설계(Logical Design):** 특정 DBMS에 맞는 논리적 구조(테이블) 설계.
3. **물리적 설계(Physical Design):** 실제 저장 장치에 물리적인 구조 설계.

#### (3) 데이터베이스 시스템 구성 및 용어

- **DBMS(Database Management System):** 데이터베이스를 관리하고 사용자의 접근을 지원하는 소프트웨어.
- **데이터베이스 스키마(Schema):** DB의 구조와 제약조건에 관한 전반적인 명세.
- **메타데이터(Metadata):** 데이터를 설명하는 데이터. 구조 정의 및 검색에 활용.
- **데이터 사전(Data Dictionary):** 메타데이터를 포함하는 시스템 내 저장소.
- **인스턴스(Instance):** 특정 시점에 DB에 저장된 실제 값들의 집합.
- **인덱스(Index):** 데이터 탐색 속도를 높이기 위한 보조 도구.

#### (4) SQL (Structured Query Language) 분류

- **DDL(Data Definition Language):** `CREATE`, `ALTER`, `DROP`
- **DML(Data Manipulation Language):** `SELECT`, `INSERT`, `DELETE`, `UPDATE`
- **DCL(Data Control Language):** `GRANT`, `REVOKE`
- **TCL(Transaction Control Language):** `COMMIT`, `ROLLBACK`

#### (5) 기업 내부 활용 데이터베이스 용어

| **용어**        | **주요 특징 및 활용**                                                 |
| --------------- | --------------------------------------------------------------------- |
| **OLTP**        | 다량의 단기 거래를 실시간 처리 (On-Line Transaction Processing)       |
| **OLAP**        | 다차원 데이터를 대화식으로 분석 (On-Line Analytical Processing)       |
| **CRM**         | 고객 관련 자료 분석 및 마케팅 활용 (Customer Relationship Management) |
| **SCM**         | 공급망 연결 최적화 (Supply Chain Management)                          |
| **ERP**         | 기업 경영 자원 효율화 (Enterprise Resource Planning)                  |
| **RTE**         | 실시간 정보로 신속한 의사결정 지원 (Real-Time Enterprise)             |
| **BI**          | 데이터 정리/분석 리포트 중심 도구 (Business Intelligence)             |
| **BA**          | 통계 기반 비즈니스 통찰력 확보 (Business Analytics)                   |
| **Block Chain** | 정보를 네트워크 참여자에게 분산 저장                                  |
| **KMS**         | 기업 내 지식 관리 시스템 (Knowledge Management System)                |

### 5. 빅데이터(Big Data)의 특징 및 가치

#### (1) 빅데이터의 출현 배경

- **기술적 요인:** 클라우드 컴퓨팅(On-demand), 분산 처리 기술(Hadoop 등).
- **사회/경제적 요인:** 비정형 데이터 확산, IoT 발달, 모바일 기기 보급.

#### (2) 빅데이터의 특징 (3V / 5V / 7V)

- **3V (Gartner 정의):** **Volume**(규모), **Variety**(다양성), **Velocity**(속도).
- **추가 요소:** **Value**(가치), **Veracity**(신뢰성), **Validity**(정확성), **Volatility**(휘발성).

#### (3) 데이터 단위 (패지요!)

- 순서: $KB < MB < GB < TB < PB < EB < ZB < YB$
- 주요 단위: **Peta**(PB), **Exa**(EB), **Zetta**(ZB), **Yotta**(YB)
- 수식 관계: $1 PB = 2^{50} \text{ bytes}$, $1 EB = 2^{60} \text{ bytes}$ 등

#### (4) 빅데이터 활용 3대 요소 (인자기)

- **인력:** 데이터 사이언티스트, 분석 전문가.
- **자원(데이터):** 분석의 대상이 되는 원천 데이터.
- **기술:** 하둡(HADOOP), NoSQL 등 처리/분석 기술.

#### (5) 빅데이터가 가져온 변화 (전후양상)

| **구분**        | **과거 (전통적 방식)**    | **현재 (빅데이터 방식)**     |
| --------------- | ------------------------- | ---------------------------- |
| **조사 범위**   | 표본조사 (Sample)         | **전**수조사 (Whole)         |
| **처리 방식**   | 사전처리 (Pre-processing) | **후**처리 (Post-processing) |
| **데이터 중심** | 질 (Quality)              | **양** (Quantity)            |
| **관계 분석**   | 인과관계 (Causality)      | **상**관관계 (Correlation)   |

### 6. 데이터 사이언스 및 조직

#### (1) 데이터 사이언티스트 역량

- **Hard Skill:** 수학, 통계학, 가설 검정, 머신러닝 기술 (이과적 소양).
- **Soft Skill:** 통찰력, 설득력(스토리텔링), 협력/커뮤니케이션 (문과적 소양).

#### (2) 빅데이터 가치 패러다임 변화 (Digital CA메라)

1. **Digitalization(디지털화):** 아날로그의 디지털 전환.
2. **Connection(연결):** 정보 간의 네트워크 형성.
3. **Agency(에이전시):** 지능화된 주체가 스스로 행동.

#### (3) 분석 조직 구조 (집기분)

- **집중 구조:** 전담 분석 조직 구성. 전략적 중요도가 높으나 현업과 업무 중복 가능.
- **기능 구조:** 별도 조직 없이 현업 부서에서 직접 수행 (DSCoE 없음).
- **분산 구조:** 분석 인력을 현업 부서에 직접 배치하여 협업 강화.

> [!INFO] 💡 AI 보충 설명
>
> **하둡(HADOOP)이란?**
>
> 여러 개의 저렴한 컴퓨터를 하나로 묶어 대용량 데이터를 처리하는 자바 기반 오픈소스 프레임워크입니다. **HDFS(분산 파일 시스템)**와 **MapReduce(분산 처리 프레임워크)**가 핵심 구성 요소입니다.
>
> **DSCoE (Data Science Center of Excellence):**
>
> 데이터 분석 역량을 집중시키고 전사적으로 분석 문화를 확산시키기 위한 '분석 거점 센터'를 의미합니다. 집중 구조와 분산 구조에서 주로 나타납니다.
