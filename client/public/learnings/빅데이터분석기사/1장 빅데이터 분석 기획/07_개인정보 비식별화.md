# 데이터 비식별화 (Data De-identification)

## 📌 핵심 요약

데이터 비식별화는 개인정보 포함 데이터에서 특정 개인을 식별할 수 있는 요소를 삭제하거나 대체하여 사생활을 보호하는 기술입니다. 식별자(Identifier)와 속성자(Attribute)를 구분하여 가명처리, 총계처리, 데이터 삭제, 범주화, 마스킹 등의 기법을 통해 재식별 가능성을 최소화하면서 데이터의 유용성을 유지하는 것이 핵심입니다.

---

## 📝 상세 정리

### 1. 데이터 비식별화의 정의 및 원칙

- **비식별 정보 (De-identified Information):** 정보의 집합물에 대해 가이드라인에 따라 비식별 조치가 완료된 정보입니다.
- **비식별 조치 (De-identification):** 개인을 식별할 수 있는 요소를 삭제하거나 대체하는 일련의 과정을 의미합니다.
- **비식별 정보의 활용:** \* 정보 주체의 동의 없이 이용하거나 제3자에게 제공이 가능합니다.
  - 단, 불특정 다수에게 공개하는 것은 원칙적으로 금지됩니다.
- **비식별 정보의 보호:** \* 현재 개인정보가 아니더라도 기술 발전이나 결합 가능한 정보의 증가로 인해 **재식별(Re-identification)** 가능성이 항상 존재합니다.
  - 재식별이 발생할 경우 해당 데이터를 즉시 파기해야 합니다.

### 2. 식별 대상의 분류

#### 2.1 식별자 (Identifier)

- **정의:** 개인 또는 관련한 사물에 고유하게 부여된 값 또는 이름입니다.
- **조치 원칙:** 데이터셋에 포함된 식별자는 원칙적으로 **삭제** 조치합니다.
- **예외:** 데이터 이용 목적상 반드시 필요한 경우 비식별 조치 후 활용합니다.
- **항목:** 고유식별정보(주민등록번호 등), 성명, 전화번호, 사진, 지문 등.

#### 2.2 속성자 (Attribute)

- **정의:** 단독으로는 식별이 어려우나, 다른 정보와 결합하는 경우 특정 개인을 알아볼 수도 있는 정보입니다.
- **조치 원칙:** \* 데이터 이용 목적과 관련 없는 경우에는 원칙적으로 삭제합니다.
  - 분석과 관련이 있다면 가명처리, 총계처리 등의 기법을 활용하여 비식별화합니다.
- **항목:** 개인특성(주소, 생년월일 등), 신체특성(키, 몸무게 등).

### 3. 주요 비식별 조치 방법 (5대 기법)

#### 3.1 가명처리 (Pseudonymization)

- **개념:** 식별자를 다른 값으로 대체하는 방법입니다.
- **특징:** 데이터의 변형 수준이 적으나, 대체 값 부여 시에도 식별 가능한 고유 속성이 계속 유지될 위험이 있습니다.
- **세부 기법:**
  - **휴리스틱 가명화 (Heuristic Pseudonymization):** 식별자에 해당하는 값들을 정해진 규칙이나 사람의 판단에 따라 가공하여 숨기는 방법입니다. 모든 데이터를 동일한 방법으로 가공하지만, 대체 변수의 한계 및 규칙 노출 시 취약점이 존재합니다.
  - **암호화 (Encryption):** 알고리즘을 적용하여 데이터를 변환합니다. 복호화 키의 보안 관리가 필수적이며, **일방향 암호화 (One-way Encryption)**는 복호화가 원천적으로 불가능합니다.
  - **교환 방법 (Swapping):** 기존 데이터베이스 레코드를 정해진 변수나 다른 레코드의 값과 교체합니다.

#### 3.2 총계처리 (Aggregation)

- **개념:** 데이터의 총합(평균, 합계 등) 값을 보여주고 개별 값은 보여주지 않는 방법입니다.
- **특징:** 민감 정보 보호 및 통계 분석용 데이터셋 작성에 유리하나, 데이터 정밀 분석이 어렵고 수량이 적으면 추론이 가능합니다.
- **세부 기법:**
  - **부분총계 (Partial Aggregation):** 데이터셋 내 일정 부분의 레코드만 총계처리하거나 오차 범위가 큰 항목을 통계값으로 변환합니다.
  - **라운딩 (Rounding):** 올림, 내림, 반올림을 적용하여 수치를 단순화합니다.
  - **재배열 (Shuffling):** 기존 정보는 유지하되 타인의 정보와 뒤섞어 식별되지 않도록 합니다.

#### 3.3 데이터 삭제 (Data Suppression)

- **개념:** 불필요한 값 또는 개인 식별에 중요한 값을 완전히 삭제합니다.
- **특징:** 확실한 비식별화가 가능하나 분석의 다양성, 유효성 및 신뢰성이 저하됩니다.
- **세부 기법:**
  - **식별자 삭제 (Identifier Removal):** 원본 데이터에서 이름, 주민번호 등의 식별자를 삭제합니다.
  - **레코드 삭제 (Record Suppression):** 다른 정보와 뚜렷하게 구별되는 특이 레코드 전체를 삭제합니다. (Outlier 제거 등)
  - **식별요소 전부 삭제:** 잠재적인 식별자까지 모두 제거하여 유용성은 매우 낮아집니다.

#### 3.4 데이터 범주화 (Data Categorization/Generalization)

- **개념:** 데이터의 구체적인 값을 해당 범주의 대표값이나 구간값으로 변환합니다.
- **특징:** 통계형 데이터 형식으로 가공 가능하나 정확도가 낮고 추론 가능성이 있습니다.
- **세부 기법:**
  - **감추기 (Masking/Clustering):** 명확한 값을 숨기기 위해 평균값이나 범주값으로 표현합니다.
  - **랜덤 라운딩 (Random Rounding):** 임의의 수를 기준으로 올림/내림을 수행합니다.
  - **범위 방법 (Range/Binning):** 임의의 수 기준으로 구간(예: 20대, 30대)을 설정합니다.
  - **제어 라운딩 (Controlled Rounding):** 행과 열의 합이 일치하지 않는 문제를 해결하기 위해 합계를 제어하며 라운딩합니다. 구현이 어려워 대중화에는 한계가 있습니다.

#### 3.5 데이터 마스킹 (Data Masking)

- **개념:** 식별 기여도가 높은 부분의 값을 보이지 않게 처리(예: \*표시)합니다.
- **특징:** 원본 데이터 변형이 적지만, 과도하게 적용하면 활용도가 떨어지고 적게 적용하면 추론이 가능합니다.
- **세부 기법:**
  - **임의 잡음 추가 (Adding Random Noise):** 데이터에 임의의 값을 더하거나 곱하여 실제 값을 숨깁니다.
  - **공백과 대체 (Blanking & Substitution):** 식별자 항목을 공백으로 두거나 특수문자( 대체문자)로 바꿉니다.

### 4. 비식별화 적정성 평가 모델 (k-익명성, l-다양성, t-근접성)

비식별 조치 후 해당 데이터가 충분히 안전한지 평가하기 위한 **정량적 지표**입니다. 세 모델은 순차적으로 한계를 보완하며 발전했습니다.

#### 4.1 k-익명성 (k-Anonymity)

- **정의:** 준식별자(Quasi-Identifier)를 변형하여, **동일한 속성 조합을 가진 레코드가 최소 k개 이상** 존재하도록 하는 모델입니다.
- **목적:** 특정 개인이 누구인지 **신원 재식별(Identity Disclosure)** 되지 않도록 방지합니다.
- **방식:** 일반화(Generalization), 억제(Suppression) 등으로 준식별자를 넓은 범주로 변환합니다.
- **한계:** 같은 그룹 내 **민감정보가 동질적**이면(예: k명 모두 같은 질병) 민감정보가 노출될 수 있습니다. → **동질성 공격(Homogeneity Attack)**에 취약합니다.

> [!EXAMPLE] 예시
> 나이·지역·성별이 모두 같은 3명이 한 그룹이면 k=3. 이때 3명 모두 "당뇨"라면, 해당 그룹에 속한 개인의 질병을 추론 가능합니다.

#### 4.2 l-다양성 (l-Diversity)

- **정의:** k-익명성을 만족하는 각 동질 집합 내에서 **민감정보(SA)의 서로 다른 값이 최소 l개 이상** 존재하도록 하는 모델입니다.
- **목적:** k-익명성의 한계를 보완하여 **동질성 공격**, **배경지식 공격(Background Knowledge Attack)**을 방지합니다.
- **방식:** 민감정보의 다양성을 강제하여, 그룹 내에서 특정 민감값을 추론하기 어렵게 합니다.
- **한계:** 희귀한 민감값이 있거나, 민감정보 분포가 전체와 크게 다를 때 **분포 기반 추론**에 취약할 수 있습니다.

> [!EXAMPLE] 예시
> k=3인 그룹에서 민감정보(질병)가 "당뇨, 고혈압, 천식" 3가지로 다양하면 l=3. 특정 개인의 질병을 특정하기 어렵습니다.

#### 4.3 t-근접성 (t-Closeness)

- **정의:** 각 동질 집합 내 **민감정보의 분포**가 **전체 데이터의 분포와 t 이내로 유사**하도록 제한하는 모델입니다.
- **목적:** l-다양성의 한계를 보완하여 **분포 공격(Distribution Attack)**, **유사성 공격(Similarity Attack)**까지 방지합니다.
- **방식:** 그룹별 민감정보 분포가 전체 분포와 크게 벗어나지 않도록 제어합니다.
- **한계:** 구현이 복잡하고, 제약이 강해질수록 **정보 손실**이 증가하는 경향이 있습니다.

> [!EXAMPLE] 예시
> 전체 데이터에서 "당뇨" 비율이 10%일 때, 특정 그룹에서 "당뇨" 비율이 80%라면 분포가 왜곡되어 추론 가능합니다. t-근접성은 그룹 분포를 전체와 유사하게 유지합니다.

#### 4.4 세 모델의 관계

| 모델         | 방어 대상            | 한계                          |
| ------------ | -------------------- | ----------------------------- |
| **k-익명성** | 신원 재식별          | 민감정보 동질성 → 동질성 공격 |
| **l-다양성** | 동질성·배경지식 공격 | 분포 왜곡 → 분포 공격         |
| **t-근접성** | 분포·유사성 공격     | 구현 복잡, 정보 손실 증가     |

> [!INFO] 💡 평가 시 고려사항
> 비식별 적정성은 **정성적 평가**(식별자 삭제 여부, 분류 적절성 등)와 **정량적 평가**(k, l, t 값 측정·검증)를 병행합니다. 데이터 특성과 활용 목적에 따라 적절한 모델과 수준을 선택해야 합니다.

#### 4.5 차등정보보호 (Differential Privacy, DP)

- **정의:** 데이터셋 또는 질의 결과에 **계산된 노이즈**를 추가하여, **한 개인의 데이터 포함 여부를 구별하기 어렵게** 만드는 수학적 프라이버시 보호 기법입니다.
- **핵심 원리:** 데이터에 1명을 추가하거나 제거해도 통계·질의 결과가 거의 동일하게 보이도록 하여, 개인 추론을 방지합니다.
- **특징:** k-익명성 계열과 달리 **확률적(수학적) 보장**을 제공하며, 적대적 공격에 대해서도 프라이버시를 보장합니다. 구글, 애플, 마이크로소프트 등 빅테크에서 활용 중입니다.

**한계**

- **프라이버시-유용성 트레이드오프:** 보호 수준(ε 감소)이 높을수록 데이터 유용성이 감소합니다.
- **소규모 데이터:** 적용 시 노이즈 영향이 커질 수 있습니다.
- **구현 복잡도:** ε 설정, 노이즈 설계 등 실무 적용을 위한 경험과 비용이 필요합니다.
