- 메인에서 AI 개요 위젯을 제거하고, `FavoriteLinksWidget` 중심으로 레이아웃과 스타일을 재정렬했다.
- 즐겨찾기 링크 위젯 크기를 `5x4`로 조정하고, 우측에 `7x7` 빈 슬롯(`EmptyPlaceholderWidget`)을 추가했다.
- 링크 위젯 헤더 높이/타이포를 줄이고 회색 톤으로 조정해 정보 밀도를 낮췄다.
- 링크 hover 시 제목 색이 과도하게 바뀌던 동작을 제거했다.
- 빈 슬롯 문구 인코딩 깨짐 이슈를 확인하고 문구를 복구했다.

### 배운 점 / 확인한 점
- `col-span`과 `row-span`은 같은 "칸 수"처럼 보여도 기준이 다르다. `row-span`은 `grid-auto-rows` 단위에 따라 실제 픽셀 높이가 결정된다.
- CSS 계산식만으로는 브라우저/상황별 편차가 생길 수 있어, 실제 컨테이너 너비를 기준으로 셀 크기를 계산하는 방식이 안정적이었다.
- PowerShell 파일 생성 시 인코딩 영향으로 한글 문자열이 `??`로 저장될 수 있어, 생성/저장 방식에 주의가 필요하다.

### 막혔던 부분
- 행 높이가 의도보다 짧게 보이는 현상이 반복되어, 단순 Tailwind arbitrary value 계산식만으로는 원하는 비율 유지가 어려웠다.
- 한글 텍스트가 파일 저장 과정에서 깨져 보이는 문제가 발생했다.

### 해결 방법
- `WidgetGrid`에 `ResizeObserver`를 적용해 컨테이너 폭 기반 셀 단위를 계산하고 `grid-auto-rows`에 반영했다.
- 깨진 한글 문구는 파일을 직접 수정해 정상 텍스트로 복구했다.
