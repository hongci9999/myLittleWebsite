# 앙상블 분석 (Ensemble Analysis) ⚠️

> [!IMPORTANT] 시험 핵심
> **앙상블**은 여러 예측 모형을 조합하여 성능을 높이는 기법입니다. **보팅·배깅·부스팅·스태킹** 구분, **랜덤포레스트·XGBoost** 특징, **병렬처리 가능 여부**를 반드시 숙지하세요.

---

## 📌 핵심 요약
**앙상블**은 **여러 개의 예측 모형**을 조합하는 기법입니다. **보팅**은 다수결, **배깅**은 붓스트랩 샘플링 후 보팅, **랜덤포레스트**는 배깅+의사결정나무로 **과적합에 강함**입니다. **부스팅**은 오분류에 가중치를 주며 **병렬처리 불가**, **XGBoost**는 GBM보다 빠르고 **규제 포함**입니다. **스태킹**은 메타 학습기로 예측 결과를 재학습합니다.

---

## 📝 상세 정리

### 1. 앙상블 (Ensemble)

**정의:** **여러 개의 예측 모형**을 조합하는 기법.

---

### 2. 보팅 (Voting)

| 유형 | 설명 |
| :--- | :--- |
| **하드보팅 (Hard Voting)** | **예측값**을 다수결 투표 |
| **소프트보팅 (Soft Voting)** | 각 모델의 **확률값 평균**으로 최종 결과 결정 |

---

### 3. 배깅 (Bagging)

**복원추출**로 **붓스트랩(Bootstrap)** 샘플 생성 → 각 샘플로 모델 학습 → **보팅**으로 결합.

#### ● 붓스트랩(Bootstrap)이란?

**복원추출** = 뽑은 데이터를 **다시 넣고** 또 뽑는 방식. 같은 데이터가 **여러 번 뽑힐 수 있음**.

| 단계 | 설명 |
| :--- | :--- |
| **1** | 원본 데이터 N개에서 **N번** 복원추출 (한 번 뽑을 때마다 다시 넣음) |
| **2** | 이렇게 만든 샘플 = **붓스트랩 샘플** 1개 (크기 N, 일부 데이터는 중복·일부는 누락) |
| **3** | 붓스트랩 샘플로 모델 1개 학습 |
| **4** | 위 과정을 **여러 번** 반복 → 모델 여러 개 → **보팅**으로 최종 예측 |

**예시:** 데이터 [A,B,C,D,E] 5개 → 5번 복원추출 → [A,C,C,E,B] 같은 샘플 가능 (C 2번, D 없음)

#### ● 왜 붓스트랩을 쓰나?

| 목적 | 설명 |
| :--- | :--- |
| **모델 다양성** | 샘플마다 **다른 데이터 조합** → 각 모델이 **서로 다른 패턴** 학습 → 보팅 시 **과적합 완화** |
| **분산 감소** | 여러 모델의 예측을 평균하면 **분산이 줄어듦** (앙상블 효과) |
| **과적합에 강함** | 단일 모델은 훈련 데이터에 과적합되기 쉬운데, 샘플링 변동으로 **일반화 성능** 향상 |

> [!TIP] 한 줄 요약
> **같은 데이터로 여러 '다른 버전'의 모델**을 만들어 합치면, 하나만 쓸 때보다 **안정적이고 과적합에 덜 민감**해짐.

#### ● OOB (Out-of-Bag)

복원추출을 무한 반복할 때, 특정 데이터가 **한 번도 선택되지 않을 확률**:

$$\lim_{N \to \infty} \left(1 - \frac{1}{N}\right)^N \approx 36.8\%$$

#### ● 랜덤포레스트 (Random Forest)

**배깅** + **의사결정나무**. 여러 의사결정나무를 붓스트랩으로 학습 후 보팅.

| 특징 | 설명 |
| :--- | :--- |
| **편향·분산** | 편향은 높지만 **분산 감소** → **과적합에 강함** |
| **모델 특성** | **블랙박스** 모델 |

---

### 4. 부스팅 (Boosting)

**잘못 분류된 데이터**에 **큰 가중치** 부여. **이상치에 민감**.

| 모델 | 특징 |
| :--- | :--- |
| **AdaBoost** | 오분류 샘플 가중치 증가 |
| **GBM** | 경사 부스팅 |
| **XGBoost** | GBM보다 **빠름**, **규제 포함** |
| **Light GBM** | **학습 속도 개선** |

> [!NOTE] 병렬처리
> **보팅·배깅·랜덤포레스트** → **병렬처리 가능**  
> **부스팅** → **병렬처리 불가** (순차적 가중치 업데이트)

---

### 5. 스태킹 (Stacking)

각 모델의 **예측 결과**를 **메타 학습기**의 입력으로 사용하여 **재학습**.

---

> [!TIP] 앙상블 비교 요약
> | 기법 | 병렬 | 대표 모델 |
> | :--- | :--- | :--- |
> | 보팅 | ○ | - |
> | 배깅 | ○ | 랜덤포레스트 |
> | 부스팅 | × | XGBoost, Light GBM |
> | 스태킹 | × | 메타 학습기 |
