# 화이트박스와 블랙박스 테스트

## 핵심 요약

화이트박스는 내부 로직 중심, 블랙박스는 외부 명세 중심이다. 시험에서는 기법 이름을 보고 어떤 분류인지 맞히는 문제와 둘의 차이점을 비교하는 문제가 매우 자주 나온다.

## 1. 화이트박스 테스트

- 내부 구조를 보고 테스트
- 제어 흐름, 조건, 경로를 검사

### 대표 기법

- 기초 경로 검사
- 제어 구조 검사
- 조건 검사
- 루프 검사
- 데이터 흐름 검사

## 2. 블랙박스 테스트

- 내부 코드를 보지 않고 명세 기반으로 테스트

### 대표 기법

- 동등 분할
- 경계값 분석
- 원인-효과 그래프
- 오류 예측
- 비교 테스트

## 3. 비교표

| 항목 | 화이트박스 | 블랙박스 |
|---|---|---|
| 기준 | 내부 로직 | 외부 명세 |
| 목적 | 논리 검증 | 기능 검증 |
| 대표 기법 | 기초 경로 | 동등 분할, 경계값 |

## 4. 순환 복잡도

- 기초 경로 검사에서 자주 등장
- 공식: `V(G) = E - N + 2`

## 5. 예제

- 경계값 분석 -> 입력 0~100 범위면 0,1,99,100 등 경계 근처 검사
- 기초 경로 검사 -> 독립 경로 수 계산

## 6. 자주 틀리는 함정

- 경계값 분석을 화이트박스로 착각
- 기초 경로 검사를 블랙박스로 착각

## 7. 시험장에서 푸는 순서

1. 내부 구조를 보는지, 명세만 보는지 구분한다.
2. 기법 이름을 보고 분류한다.
3. 순환 복잡도 공식은 별도로 기억한다.

## 한 페이지 요약

- 화이트박스는 내부 로직, 블랙박스는 외부 명세다.
- 기초 경로 검사는 화이트박스 대표 기법이다.
- 동등 분할과 경계값 분석은 블랙박스 대표 기법이다.
