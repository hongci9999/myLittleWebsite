# 프로세스
메모리에 적재되어 실행되고 있는 프로그램
pcb에 기록
프로제스 대한 정보를 지록한 테이블
프로세스 생성될 때마다 고유의 pcb가 생성

타임 슬라이스에 의해 문맥 교환을 수행
저장된 후 다른 프로세스 정보를 적재하는 과정
작은 경우- 문맥 교환수, 인터럽트 횟수, 오버헤드 증가
큰 경우 - 반대

##  인터럽트 처리
일시 중지 후 복귀

내부 인터업트 trap - 잘못된 명령이나 데이터
소프트웨어 인터럽트 - 요청에 의해

인터럽트 시 실행 중인 연산 마무리 후 일시 중단
pcb,pc를 이용해 프로세스 정보 저장
문맥 교ㅕ환을 톨해 cpu 할당 전환 인터업트 처리 루틴 수행
저장된 프로세스 정보를 통해 프로세스 상태가 복구, 재수행

프로세스 통신 ipc
직접 통신 방법이 ㅇ벗는 프로세스들이 통신을 하는 방법
공유 메모리 기법 - 빠르지만 불안정
메시지 전달 기법 - os에 의해 데이터 교환, 특징은 반대

# 프로세스 관련 기술
## 상호배제
임계구역
프로세스가 서로 공유하는 자원
동시 사용 불가능

상호배제
다른 프로세스의 접근을 차단
단일 프로세스만 임계구역 준재
임계구혁 진입이 무한정 연기되지 않음
임계구역 내 프로세스가 다른 프로세스의 진입 차단 가능
속도나 개수에 영햐ㅐㅇ 받지 않음

잠금 -다른 프로세스 접근 금지
인터럽트 봉쇄 - 사용 중 인터럽트 막아 새 프로세스 접근 차단
엄격한 교대 - 다수의 프로세스가 하나의 임계구역을 교대로 접근

한 프로세스의 실행시가닝 길면 - 바븐 대시 현상
운영체제 부담됨 성능 저하
세마포어 알고리즘 사용

## 교착상태
조건
상호배제
점유와 대기
비선점
환형대기 - 각 프로세스가 순차적으로 다음 프로세스가 요구하는 자원을 가진 상태

예방
조건 중 하나 이상을 부정하여 교착상태 예방
상호배제 부정 - 동시 접근 가능 신뢰성 보장 어려움
비선점 부정 - 가장 현실적
점유와 대기 부정 - 자원 일부 점유한 프로세스가 추가 자원의 요구를 싱패하는 경우 기존 자원 반납 후 다시 요청
환형 대기 부정 - 자신이 가진 자원의 앞,뒤 순서 프로세스의 자원 요청 금지
