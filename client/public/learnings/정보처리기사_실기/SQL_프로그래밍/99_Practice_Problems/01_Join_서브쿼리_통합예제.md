# 조인 · 서브쿼리 통합 예제 (한 시나리오 끝까지)

**목표:** 같은 스키마·데이터로 **INNER / LEFT**, **스칼라 서브쿼리**, **IN**, **EXISTS**, **ANY/ALL** 감을 잡습니다.  
실기에서는 아래 표를 보고 **쿼리 결과 행을 직접** 적는 연습을 하세요.

---

## 1. 테이블 정의(개념)

**고객** `고객(고객번호, 고객명, 가입일)`

**주문** `주문(주문번호, 고객번호, 주문금액, 주문일시)`  
- `주문일시`: 최신 글처럼 **시간 순서**를 생각할 때 쓰는 컬럼(실기·실무 모두 흔함).

---

## 2. 샘플 데이터

**고객**

| 고객번호 | 고객명 | 가입일 |
|---------|--------|--------|
| C01 | 민지 | 2026-01-10 |
| C02 | 태호 | 2026-02-01 |
| C03 | 수연 | 2026-03-05 |

**주문**

| 주문번호 | 고객번호 | 주문금액 | 주문일시 |
|---------|---------|---------|----------|
| O1 | C01 | 12000 | 2026-04-01 10:00 |
| O2 | C01 | 8000 | 2026-04-02 14:00 |
| O3 | C02 | 5000 | 2026-04-03 09:00 |

- **C03**은 주문 **없음**.

---

## 3. 쿼리 A — INNER JOIN (고객 + 주문)

```sql
SELECT g.고객명, o.주문번호, o.주문금액, o.주문일시
FROM   고객 g
JOIN   주문 o ON g.고객번호 = o.고객번호
ORDER BY o.주문일시;
```

### 결과(직접 채점용)

| 고객명 | 주문번호 | 주문금액 | 주문일시 |
|--------|---------|---------|----------|
| 민지 | O1 | 12000 | 2026-04-01 10:00 |
| 민지 | O2 | 8000 | 2026-04-02 14:00 |
| 태호 | O3 | 5000 | 2026-04-03 09:00 |

→ **3행**. C03은 주문이 없어 **제외**.

---

## 4. 쿼리 B — LEFT JOIN

```sql
SELECT g.고객명, o.주문번호, o.주문금액
FROM   고객 g
LEFT JOIN 주문 o ON g.고객번호 = o.고객번호
ORDER BY g.고객번호, o.주문번호;
```

### 결과

| 고객명 | 주문번호 | 주문금액 |
|--------|---------|---------|
| 민지 | O1 | 12000 |
| 민지 | O2 | 8000 |
| 태호 | O3 | 5000 |
| 수연 | NULL | NULL |

→ **4행**. 수연은 주문 컬럼이 **NULL**.

---

## 5. 쿼리 C — 스칼라 서브쿼리 (평균보다 큰 주문)

```sql
SELECT 주문번호, 고객번호, 주문금액
FROM   주문
WHERE  주문금액 > (SELECT AVG(주문금액) FROM 주문);
```

전체 평균: `(12000 + 8000 + 5000) / 3 = 8333…`

### 결과

| 주문번호 | 고객번호 | 주문금액 |
|---------|---------|---------|
| O1 | C01 | 12000 |

→ **1행** (8000, 5000은 평균 이하).

---

## 6. 쿼리 D — IN (주문한 고객만)

```sql
SELECT 고객번호, 고객명
FROM   고객
WHERE  고객번호 IN (SELECT 고객번호 FROM 주문);
```

### 결과

| 고객번호 | 고객명 |
|---------|--------|
| C01 | 민지 |
| C02 | 태호 |

---

## 7. 쿼리 E — EXISTS (주문 **한 번도** 없는 고객)

```sql
SELECT g.고객번호, g.고객명
FROM   고객 g
WHERE  NOT EXISTS (
  SELECT 1 FROM 주문 o WHERE o.고객번호 = g.고객번호
);
```

### 결과

| 고객번호 | 고객명 |
|---------|--------|
| C03 | 수연 |

---

## 8. 쿼리 F — ANY (다른 고객 주문금액보다 **하나라도** 큰 금액?)

문제에서 이런 식으로 나오면 “ANY = OR 묶음”으로 생각합니다.

```sql
SELECT 주문번호, 주문금액
FROM   주문 o1
WHERE  o1.주문금액 > ANY (
  SELECT o2.주문금액 FROM 주문 o2 WHERE o2.주문번호 <> o1.주문번호
);
```

O1(12000): 다른 금액 8000,5000 중 **하나라도** 작은지 → 12000 > 8000 참 → **포함**.  
O2(8000): 12000,5000 중 5000보다 크면 참 → 포함.  
O3(5000): 12000,8000과 비교 → 둘 다보다 클 수 없음 → **제외**.

### 결과

| 주문번호 | 주문금액 |
|---------|---------|
| O1 | 12000 |
| O2 | 8000 |

(실기에서는 **상관 서브쿼리** 문장을 그대로 읽고 표를 그리는 연습이 핵심입니다.)

---

## 9. 쿼리 G — ALL

```sql
SELECT 주문번호, 주문금액
FROM   주문 o1
WHERE  o1.주문금액 > ALL (
  SELECT o2.주문금액 FROM 주문 o2 WHERE o2.주문번호 <> o1.주문번호
);
```

O1: 나머지 8000,5000 **모두**보다 커야 함 → 12000 > 8000 및 > 5000 → **포함**.  
O2,O3: 12000을 이길 수 없음.

### 결과

| 주문번호 | 주문금액 |
|---------|---------|
| O1 | 12000 |

---

## 10. 복습 체크

- [ ] INNER vs LEFT로 **행 수**가 어떻게 달라지는지 말할 수 있다.
- [ ] `NOT IN` 대신 `NOT EXISTS`를 쓰는 이유(NULL)를 한 문장으로 말한다.
- [ ] `> ANY` vs `> ALL`을 **최소/최대** 직관으로 바꿔 말한다.
