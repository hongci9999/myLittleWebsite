# UI 요구사항 및 웹 표준/접근성

## 핵심 요약
사용자와 시스템을 연결하는 **[[UI]]**(User Interface)의 유형(CLI, GUI, NUI)과 구성 요소, 그리고 웹 개발 시 필수적으로 준수해야 하는 **[[웹 표준]]**, **[[웹 접근성]]**, **[[웹 호환성]]**을 학습함. 또한 KWCAG의 4가지 원칙과 UI 프로토타입의 장단점을 비교 정리함.

---

## 1. [[UI]] (User Interface) 개요

### 1) 개념
사용자와 컴퓨터(시스템) 간의 상호작용을 돕는 연계 시스템. 정보 전달과 입력의 매개체.

> [!INFO] 💡 AI 보충 설명: UI의 3가지 분야
> * **물리적 제어 분야**: 사용자가 정보에 접근하기 위해 조작하는 하드웨어적 수단 (키보드, 마우스, 터치스크린 등).
> * **구성과 표현 분야**: 사용자가 정보를 시각적으로 인식하는 화면의 디자인, 레이아웃, 아이콘, 폰트 등의 구성.
> * **기능적 분야**: 시스템이 수행하는 실제 기능과 그 결과를 사용자에게 피드백하는 효율성 및 운영 방식.

### 2) UI 유형
발전 순서: CLI → GUI → NUI

* **[[CLI]] (Command Line Interface)**
    * 텍스트(명령어) 기반의 인터페이스.
    * 키보드를 주로 사용. (예: 도스, 리눅스 터미널)
    > [!INFO] 💡 AI 보충 설명: CLI의 특징
    > * 시스템 자원을 적게 소모하지만, 명령어를 암기해야 하므로 배우기 어렵습니다.

* **[[GUI]] (Graphical User Interface)**
    * 그래픽(아이콘, 메뉴) 기반의 인터페이스.
    * 마우스를 주로 사용. (예: 윈도우, 맥OS)
    > [!INFO] 💡 AI 보충 설명: GUI의 특징
    > * 직관적이고 사용하기 쉽지만, CLI보다 시스템 자원을 많이 소모합니다.

* **[[NUI]] (Natural User Interface)**
    * 사용자의 자연스러운 행동(신체) 기반 인터페이스.
    * 터치, 음성, 제스처 등 사용. (예: 스마트폰 터치, AI 스피커, 동작 인식)
    > [!INFO] 💡 AI 보충 설명: NUI의 확장
    > * **OUI (Organic UI)**: 모든 사물과 환경이 입력 장치가 되는 유기적 인터페이스.
    > * **VUI (Voice UI)**: 사람의 목소리로 제어하는 인터페이스.

## 2. UI 구성 요소 (Layout)

화면을 구성하는 기본적인 구조적 요소들.

1.  **인디케이터 (Indicator)**
    > [!INFO] 💡 AI 보충 설명
    > * 현재 시스템의 상태나 정보(시간, 배터리, 와이파이 신호 등)를 상단에 표시하는 영역입니다.
2.  **헤더 (Header)**
    > [!INFO] 💡 AI 보충 설명
    > * 화면의 최상단에 위치하며, 제목이나 검색 기능, 뒤로 가기 버튼 등을 포함하는 영역입니다.
3.  **내비게이션 (Navigation)**
    > [!INFO] 💡 AI 보충 설명
    > * 사용자가 원하는 메뉴나 정보로 이동할 수 있도록 안내하는 메뉴 바(GNB, LNB 등)입니다.
4.  **컨텐트 (Content)**
    > [!INFO] 💡 AI 보충 설명
    > * 사용자가 실제로 보거나 이용하는 핵심 정보(이미지, 텍스트, 영상)가 표시되는 영역입니다.
5.  **버튼 (Button)**
    > [!INFO] 💡 AI 보충 설명
    > * 사용자의 기능 실행 요청(확인, 취소, 전송 등)을 처리하는 조작 요소입니다.
6.  **푸터 (Footer)**
    > [!INFO] 💡 AI 보충 설명
    > * 화면의 최하단에 위치하며, 저작권 정보, 회사 주소, 패밀리 사이트 링크 등을 포함합니다.

## 3. [[UI 요소]] (UI Elements)

사용자와 서비스가 상호작용(Interaction) 할 수 있는 구체적인 조작 도구.

* **텍스트 박스 (Text Box)**
    > [!INFO] 💡 AI 보충 설명
    > * 사용자가 직접 텍스트를 입력할 수 있는 사각형 영역입니다.
* **라디오 버튼 (Radio Button)**
    > [!INFO] 💡 AI 보충 설명
    > * 여러 개의 옵션 중 **반드시 하나만 선택**해야 할 때 사용하는 둥근 버튼입니다. (상호 배타적)
* **체크 박스 (Check Box)**
    > [!INFO] 💡 AI 보충 설명
    > * 여러 개의 옵션 중 **복수 선택**이 가능하거나 선택하지 않을 수도 있을 때 사용하는 사각형 버튼입니다.
* **토글 버튼 (Toggle Button)**
    > [!INFO] 💡 AI 보충 설명
    > * On/Off 처럼 두 가지 상태 중 하나로 스위칭하는 버튼입니다.
* **콤보 상자 (Combo Box)**
    > [!INFO] 💡 AI 보충 설명
    > * 텍스트 박스와 드롭다운 목록이 결합된 형태입니다. 목록에서 선택하거나 직접 입력할 수 있습니다.
* **명령 버튼 (Command Button)**
    > [!INFO] 💡 AI 보충 설명
    > * 사용자가 이벤트를 발생시키기 위해 클릭하는 버튼(확인, 취소, 실행 등)입니다.

## 4. 웹 기술 3대 요소

### 1) [[웹 표준]] (Web Standards)
* **개념**: W3C 등 표준화 기구에서 제정한 기술 규약을 준수하는 것.
* **목적**: 브라우저 종류(크롬, 엣지, 사파리 등)에 상관없이 동일한 화면과 기능을 제공하기 위함.
> [!INFO] 💡 AI 보충 설명: 주요 기술
> * 구조(HTML), 표현(CSS), 동작(JavaScript)을 분리하여 개발하는 것이 핵심입니다.

### 2) [[웹 접근성]] (Web Accessibility)
* **개념**: 장애인, 노약자 등 정보 취약 계층을 포함한 **모든 사용자**가 웹 사이트를 불편 없이 이용할 수 있도록 하는 것.
* **디지털 접근성**: 다양한 환경(모바일, PC 등)과 사용자 능력에 상관없이 정보에 접근할 수 있어야 함.
> [!INFO] 💡 AI 보충 설명: 구체적 사례
> * 시각 장애인을 위한 이미지 대체 텍스트(Alt text) 제공.
> * 마우스를 못 쓰는 사용자를 위한 키보드 이용 보장.
> * 동영상에 자막 제공.

### 3) [[웹 호환성]] (Web Compatibility)
* **개념**: 특정 브라우저나 운영체제(OS)에 종속되지 않고, 어느 환경에서든 웹 사이트가 정상적으로 작동하는 것. (Cross Browsing)
> [!INFO] 💡 AI 보충 설명: 관련 이슈
> * 과거의 ActiveX나 Flash 등 특정 브라우저(IE)에서만 돌아가는 비표준 기술 사용을 지양하는 것입니다.

## 5. 한국형 웹 콘텐츠 접근성 지침 ([[KWCAG]])

장애인이 비장애인과 동등하게 웹 콘텐츠에 접근할 수 있도록 하는 4가지 원칙과 지침.

### 1) 인식의 용이성 (Perceivable)
> [!INFO] 💡 AI 보충 설명
> * 사용자가 콘텐츠를 감각(시각, 청각)으로 인식할 수 있어야 함.
> * **대체 텍스트**: 그림이나 이미지에 대한 텍스트 설명 제공.
> * **멀티미디어 대체 수단**: 동영상에 자막, 대본 제공.

### 2) 운용의 용이성 (Operable)
> [!INFO] 💡 AI 보충 설명
> * 사용자가 UI 구성요소를 조작하고 내비게이션 할 수 있어야 함.
> * **입력장치 접근성**: 마우스 없이 키보드만으로 모든 기능 사용 가능.
> * **충분한 시간 제공**: 시간 제한이 있는 경우 연장 기능 제공.

### 3) 이해의 용이성 (Understandable)
> [!INFO] 💡 AI 보충 설명
> * 사용자가 콘텐츠의 내용과 조작 방법을 이해할 수 있어야 함.
> * **가독성**: 텍스트 내용이 이해하기 쉬워야 함.
> * **예측 가능성**: 사용자 인터페이스가 예상대로 동작해야 함.

### 4) 견고성 (Robust)
> [!INFO] 💡 AI 보충 설명
> * 사용자가 이용하는 기술(브라우저, 보조 공학 기기 등)에 관계없이 콘텐츠가 견고하게 해석되어야 함.
> * **문법 준수**: 마크업 언어의 문법을 정확히 지켜야 함.

## 6. 전자정부 웹 표준 준수 지침

### 특징
* **보편성**: 모든 국민이 이용해야 하므로 화려한 디자인보다는 **정보 전달**과 **기능**에 충실해야 함.
* **엄격성**: 웹 표준과 웹 접근성 지침을 민간 사이트보다 훨씬 **엄격하게 준수**해야 함. (법적 의무)

> [!INFO] 💡 AI 보충 설명: 전자정부 UI 가이드라인
> * 시스템 간의 통일성을 위해 공통된 UI 컴포넌트와 디자인 패턴을 사용하여 사용자 혼란을 줄입니다.

## 7. [[UI 프로토타입]]

본격적인 개발 전, 시스템의 모양이나 동작을 모형으로 만들어 검증하는 단계.

### 1) 페이퍼 프로토타입 (Paper Prototype)
손으로 직접 종이에 그려서 만드는 아날로그 방식.
* **장점**: 제작 비용이 저렴하고 매우 빠름. 회의 중 즉석 수정 가능.
* **단점**: 실제 **테스트 불가**, 상호작용(인터랙션) 확인 불가, 버전 관리 어려움.
> [!INFO] 💡 AI 보충 설명
> * 기획 초기 단계에서 아이디어를 빠르게 공유할 때 적합합니다.

### 2) 디지털 프로토타입 (Digital Prototype)
파워포인트, 피그마(Figma), XD 등의 툴을 사용하여 만드는 방식.
* **장점**: 실제 작동하는 것처럼 **테스트 가능**, 수정 및 재사용 용이, 산출물 활용 가능.
* **단점**: 페이퍼에 비해 제작 시간과 비용이 소요됨.
> [!INFO] 💡 AI 보충 설명
> * 개발 직전 상세 설계 단계나 사용자 테스트(UT) 용도로 적합합니다.

## 8. 핵심 단어 정리
- **UI 유형**: CLI(텍스트), GUI(그래픽), NUI(자연/터치).
- **UI 요소**: 라디오 버튼(택1), 체크 박스(다중선택).
- **웹 3요소**: 표준(규약), 접근성(장애인/모두), 호환성(브라우저/OS).
- **KWCAG 원칙**: **인**식, **운**용, **이**해, **견**고 (**인운이견**).
