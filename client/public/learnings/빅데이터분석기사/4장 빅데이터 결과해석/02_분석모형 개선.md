# 분석모형 개선: 과대적합 방지 (Overfitting Prevention)

## 1. 과대적합 방지방안 (Overfitting Prevention Measures)

---

### (1) 데이터 관점 (Data Perspective)

| 방안 | 설명 |
| :--- | :--- |
| **데이터 양 증가** | 학습 데이터 규모 확대 |
| **데이터 증강** | 기존 데이터 변형·확장 (이미지 회전, 노이즈 추가 등) |
| **피처 선택 및 차원축소** | 불필요한 변수 제거, PCA 등으로 차원 축소 |

---

### (2) 규제 관점 (Regularization Perspective)

| 방안 | 설명 |
| :--- | :--- |
| **L1 규제 (Lasso)** | 계수에 L1 노름 부과 → 일부 계수를 0으로 축소 (특성 선택 효과) |
| **L2 규제 (Ridge)** | 계수에 L2 노름 부과 → 계수 크기를 전반적으로 줄임 |

---

### (3) 모델 알고리즘 관점 (Model Algorithm Perspective)

| 방안 | 설명 |
| :--- | :--- |
| **드롭아웃 (Dropout)** | 학습 시 뉴런을 확률적으로 비활성화 → 특정 뉴런에 대한 과의존 방지 |
| **가지치기 (Pruning)** | 의사결정나무 등에서 불필요한 노드·가지 제거 |

---

### (4) 학습 및 검증 프로세스 관점 (Training and Validation Process Perspective)

| 방안 | 설명 |
| :--- | :--- |
| **교차검증 (Cross-validation)** | 데이터를 여러 번 분할해 검증 → 과적합·과대적합 추정 |
| **조기종료 (Early Stopping)** | **검증 오차가 증가하기 시작하면** 학습 중단 → 과적합 구간에서 멈춤 |

---

## 2. 학습곡선 (Learning Curve)

> [!NOTE] 정의
> **학습곡선**: 학습 누적(Epoch)에 따른 **훈련 데이터**와 **검증 데이터**의 **오차 변화**를 시각화한 그래프.

### 그래프 구성

| 축 | 의미 |
| :--- | :--- |
| **X축** | Epoch (학습 반복 횟수) |
| **Y축** | Error (오차) |

### 곡선 특성

| 곡선 | 형태 | 해석 |
| :--- | :--- | :--- |
| **학습 데이터** | Epoch가 증가할수록 **지속 감소** | 모델이 훈련 데이터에 계속 맞춰짐 |
| **검증 데이터** | 초반 감소 후 **상승** | 과적합 구간 진입 시 검증 성능 악화 |

### 조기종료 시점

* **검증 데이터 곡선의 최저점** = **학습을 종료해야 하는 지점**
* 그 이후로 계속 학습하면 과대적합이 심해진다.

---

## 3. 매개변수 최적화 (Parameter Optimization)

### (1) 파라미터와 하이퍼파라미터 (Parameters vs. Hyperparameters)

| 구분 | 목적 | 설정 방식 |
| :--- | :--- | :--- |
| **파라미터 (Parameter)** | 최적화된 모델 구현 | **수동 설정 불가** — 모델이 학습을 통해 도출 |
| **하이퍼파라미터 (Hyperparameter)** | 최적의 파라미터 도출 | **사용자가 수동으로 설정** |

---

### (2) 경사하강법 (Gradient Descent)

> [!NOTE] 정의
> 모델 최적화를 위해 **손실함수의 최솟값**을 찾아가는 **반복 알고리즘**

$$W_{new} = W_{old} - \alpha \frac{d}{dW} cost(W)$$

* $W$: 가중치, $\alpha$: **학습률 (learning rate)**

| α 조건 | 결과 |
| :--- | :--- |
| **너무 클 때** | 이전보다 **높은 곳으로 발산** (overshooting) |
| **너무 작을 때** | 최솟값으로 **수렴하는 데 시간이 오래 걸림** |

---

### (3) Batch Size, Epoch, Iteration

| 용어 | 정의 |
| :--- | :--- |
| **Batch Size** | 하나의 소그룹(배치)에 속하는 **데이터 수** |
| **Epoch** | **전체 데이터셋**을 학습하는 **총 횟수** |
| **Iteration** | **1 Epoch를 마치는 데 필요한** 배치 수 |

$$\text{Iteration} = \frac{\text{전체 데이터 수}}{\text{Batch Size}}$$

* 예: 1000개 ÷ 100 = **10 Iteration** (1 Epoch당)

---

### (4) 하이퍼파라미터 튜닝 (Hyperparameter Tuning)

| 방법 | 설명 |
| :--- | :--- |
| **(1) Manual Search** | 경험 또는 감으로 값 설정 |
| **(2) Grid Search** | **모든 조합**을 탐색 |
| **(3) Random Search** | **임의**로 조합 탐색 |
| **(4) Bayesian Optimization** | **모델링**을 통한 최적의 조합 발견 |

---

### (5) 경사하강법 옵티마이저 (Gradient Descent Optimizers)

> [!NOTE] 목적
> 학습 속도 문제 및 **지역 극솟값(local minimum) 수렴** 문제 해결

#### 진화 흐름

| 구분 | 옵티마이저 | 핵심 아이디어 |
| :--- | :--- | :--- |
| 기반 | **GD** | 전체 데이터로 gradient 계산 |
| 기반 | **SGD** | 일부 데이터만 사용 → 빠른 이동 |
| **이동 방향** | Momentum | 이전 방향의 **관성**을 이용해 더 멀리 이동 |
| **이동 방향** | NAG | 관성 방향으로 먼저 이동 후, 그 지점에서 gradient 계산해 보정 |
| **스텝 사이즈** | Adagrad | 자주 갱신된 파라미터(이미 방문한 곳)는 스텝을 **줄임** |
| **스텝 사이즈** | RMSProp | **최근** gradient만 반영 → 학습률 급감 방지 |
| **스텝 사이즈** | AdaDelta | 스텝이 **너무 작아지는 것** 방지 |

**혼합 옵티마이저**
| 옵티마이저 | 조합 |
| :--- | :--- |
| **Adam** | Momentum + RMSProp |
| **Nadam** | NAG + RMSProp |

#### 대표 옵티마이저 정의

| 옵티마이저 | 특징 |
| :--- | :--- |
| **Momentum** | 이전 gradient를 일정 비율로 반영해 **진동을 줄이고** 최솟값 쪽으로 **가속 수렴** |
| **Adagrad** | 누적 gradient로 학습률을 **자동 조정**. 다만 학습률이 **시간이 지날수록 감소**하는 경향 |
| **RMSProp** | **지수이동평균**으로 학습률 조정 → Adagrad의 급격한 학습률 감소 완화 |
| **Adam** | Momentum과 RMSProp의 **장점을 결합** |
