# 계층형 질의 · 자기 조인

## 핵심 요약

**계층형 질의**는 **트리 구조** 데이터(조직도 등)를 **LEVEL 단위**로 탐색한다. **START WITH**(루트) + **CONNECT BY PRIOR**(부모·자식 연결). **PRIOR 자식=부모** → **하향**(사장→부하). **ORDER SIBLINGS BY**·**NOCYCLE**·**SYS_CONNECT_BY_PATH** 등. **자기 조인**은 **같은 테이블**을 붙여 **1단계** 상하 관계 조회(Oracle **CONNECT BY** 대안·단순 경우).

## 왜 중요한가

- 조직·카테고리·BOM 등 **실무 빈출** 구조이다.
- 1과목 **계층형 모델**([12_관계와_조인_계층_상호배타](../01_데이터_모델링의_이해/12_관계와_조인_계층_상호배타.md))의 SQL 구현이다.
- **PRIOR 방향**·**START WITH** 조건은 **시험 함정**.

> 조인 기본: [09_조인_개요](../02_SQL_기본/09_조인_개요.md) · INNER: [10_INNER_JOIN](../02_SQL_기본/10_INNER_JOIN.md)

## 1. 계층형 질의란

| 항목 | 내용 |
|------|------|
| **정의** | **계층(트리) 구조**를 가진 데이터를 **탐색**하는 SQL |
| **예** | 조직도(상사–부하), 분류 체계, 메뉴 트리 |
| **대표 문법** | Oracle **CONNECT BY** (교재·SQLD 범위) |

### 계층 구조 예 — \<사원\>

| 사원이름 | 직속상관 | 직급 |
|----------|----------|------|
| 영수 | NULL | 사장 |
| 수지 | 영수 | 부장 |
| 민수 | 수지 | 과장 |
| 민희 | 민수 | 대리 |
| 선영 | 민수 | 대리 |

```
LEVEL 1   영수 (사장)          ← 루트
LEVEL 2     수지 (부장)
LEVEL 3       민수 (과장)
LEVEL 4         민희, 선영 (대리)
```

---

## 2. 계층형 질의 키워드

| 키워드 | 설명 |
|--------|------|
| **LEVEL** | 현재 행의 **계층 깊이**(루트=1) |
| **START WITH** | **시작점(루트)** 조건 |
| **CONNECT BY** | **부모–자식** 연결 조건 |
| **PRIOR** | 탐색 **방향** 지정 |
| **NOCYCLE** | **순환** 참조 시 **무한 루프 방지** |
| **ORDER SIBLINGS BY** | **같은 LEVEL(형제)** 끼리 정렬 |
| **CONNECT_BY_ROOT** | **최상위 루트** 값 |
| **CONNECT_BY_ISLEAF** | **말단(리프)** 이면 1, 아니면 0 |
| **CONNECT_BY_ISCYCLE** | **순환**이면 1 |
| **SYS_CONNECT_BY_PATH** | **루트→현재** 경로 문자열 |

### PRIOR 방향

| 표현 | 방향 | 의미 |
|------|------|------|
| **PRIOR 자식 = 부모** | **하향** (Top → Down) | 부모 행에서 **자식**으로 |
| **PRIOR 부모 = 자식** | **상향** (Bottom → Up) | 자식에서 **상위**로 |

```sql
-- 하향: 사원이름이 상관의 '직속상관'과 연결
CONNECT BY PRIOR 사원이름 = 직속상관
```

---

## 3. 예 — 조직도 하향 조회

```sql
SELECT LEVEL, 사원이름, 직속상관, 직급
  FROM 사원
 START WITH 직속상관 IS NULL
CONNECT BY PRIOR 사원이름 = 직속상관;
```

| 단계 | 절 | 역할 |
|------|-----|------|
| 1 | **START WITH** `직속상관 IS NULL` | **루트** = 상사 없음 → **영수** |
| 2 | **CONNECT BY PRIOR** `사원이름 = 직속상관` | **이름**이 다음 행의 **직속상관**인 관계로 **하향** |

### 결과

| LEVEL | 사원이름 | 직속상관 | 직급 |
|-------|----------|----------|------|
| 1 | 영수 | NULL | 사장 |
| 2 | 수지 | 영수 | 부장 |
| 3 | 민수 | 수지 | 과장 |
| 4 | 민희 | 민수 | 대리 |
| 4 | 선영 | 민수 | 대리 |

→ **LEVEL 4**에 **민희·선영** 형제 — `ORDER SIBLINGS BY 사원이름` 등으로 순서 지정 가능.

### 순환 방지 예

```sql
SELECT …
  FROM 사원
 START WITH …
CONNECT BY NOCYCLE PRIOR 사원이름 = 직속상관;
```

→ 데이터에 **순환**이 있어도 **무한 루프** 방지.

## 4. 경로·루트 컬럼 (`SYS_CONNECT_BY_PATH` · `CONNECT_BY_ROOT`)

```sql
SELECT 사원이름, 직급, 직속상관,
       SYS_CONNECT_BY_PATH(사원이름, '/') AS 경로,
       CONNECT_BY_ROOT(사원이름) AS 최상위상급자
  FROM 사원
 START WITH 직속상관 IS NULL
CONNECT BY PRIOR 사원이름 = 직속상관;
```

### 결과

| 사원이름 | 직급 | 직속상관 | 경로 | 최상위상급자 |
|----------|------|----------|------|--------------|
| 영수 | 사장 | NULL | **/영수** | 영수 |
| 수지 | 부장 | 영수 | **/영수/수지** | 영수 |
| 민수 | 과장 | 수지 | **/영수/수지/민수** | 영수 |
| 민희 | 대리 | 민수 | **/영수/수지/민수/민희** | 영수 |
| 선영 | 대리 | 민수 | **/영수/수지/민수/선영** | 영수 |

| 함수 | 설명 |
|------|------|
| **SYS_CONNECT_BY_PATH(열, '구분자')** | **루트 → 현재**까지 경로를 **문자열**로 |
| **CONNECT_BY_ROOT(열)** | 해당 행이 속한 트리의 **최상위(루트)** 값 — 전원 **영수** |

## 5. 자기 조인 (Self Join)

**같은 테이블**에 **조인** — **1단계** 관계(직속 상사·멘토 등).

### 예 1 — 직속 상관 (이름)

```sql
SELECT E.사원이름, M.사원이름 AS 상사이름
  FROM 사원 E
  LEFT JOIN 사원 M
    ON E.직속상관 = M.사원이름;
```

### 예 2 — 멘토 (사번)

| 사번 | 이름 | 멘토사번 |
|------|------|----------|
| 1 | 영수 | NULL |
| 2 | 민희 | 4 |
| 3 | 수지 | 1 |
| 4 | 민수 | 3 |
| 5 | 선영 | 4 |

```sql
SELECT E.이름 AS 사원이름, M.이름 AS 멘토이름
  FROM 사원 E, 사원 M
 WHERE E.멘토사번 = M.사번;
```

| 사원이름 | 멘토이름 |
|----------|----------|
| 민희 | 민수 |
| 수지 | 영수 |
| 민수 | 수지 |
| 선영 | 민수 |

→ **영수**(멘토 없음)는 **WHERE 조인**이라 **제외** — `LEFT JOIN`이면 NULL 멘토 행 가능.

| 구분 | 자기 조인 | CONNECT BY |
|------|-----------|------------|
| **단계** | **1단계** 관계 | **전체 계층**·LEVEL |
| **용도** | 멘토·직속상사 **한 명** 붙이기 | **조직도 전체**·경로 |

---

## 6. 시험 포인트 / 함정

| 구분 | 내용 |
|------|------|
| START WITH | **루트** 조건 |
| CONNECT BY | **부모–자식** 연결 |
| PRIOR | **하향**: `PRIOR 자식열 = 부모열` (교재 표기) |
| LEVEL | **1** = 루트 |
| NOCYCLE | **순환** 데이터 |
| ORDER SIBLINGS BY | **형제** 정렬 |
| 자기 조인 | **동일 테이블** 2회 |
| 함정 | PRIOR **방향** 반대 → **상향/하향** 뒤바뀜 |
| 함정 | START WITH **누락** → 여러 루트·중복 |
| SYS_CONNECT_BY_PATH | **루트/…/현재** 경로 문자열 |
| CONNECT_BY_ROOT | **최상위** 루트 값 |
| 함정 | 자기 조인만으로 **전체 LEVEL** — **CONNECT BY** 필요 |
