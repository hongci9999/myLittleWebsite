# 접근 통제
## 시스템 인증
접근방법 4가지
지식기반 - 비번
소유기반 - otp
존재기반 - 지문
행위기반 - 서명
영어이름도 중요

새션
로그인 순간부터 로그아웃 까지
사용자 인증이 유지
추가적인 인증이 필요 없음
금융거래 같은건 지속적으로 

새선 가로채기 통제 - 연결되 있는 상태에서 가로채는 공격

비동기화 탐지 - 서버와 시퀀스 넘버 체크
ack strom 탐지 - 급격한 ark 증가
유실과 재선종 증가 탐지 - 
예상치 못한 접속의 리셋 탐지

## 인증 도구
디지털 서명
송신자가 자신의 비밀키로 암호화한 메시지를 수신자가 송신자의 공개키로 해독하는 과정
인증 - 정당한 비밀키로
무결성 - 위조 불가능
부인방지 - 서명 부인 못함
유일성 - 재사용 못함
진위확인용이성

디지털 인증서
신롸할 만한 인증기관의 서명료 개인키로 서명한 인증서

메시지 인증 코드
조금 변형되어도 크게 달라지는 코드 메시지 무결성 코드
해시함수 이용
임의 길이 메시지를 고정 실이의 해시값으로 변환시키는 단방향성 알고리즘
## 인증관련 프로토콜
aaa
인증, 권한부여, 계정관리
엑세스 저어 추적

pap
평문의 아이디 패스워드

chap
시스템측에서 인증 주관

radius

diameter

보안, 장애복구 강화
