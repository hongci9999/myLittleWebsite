# 통신 시스템 및 신호 변환/장비 상세 정리

## 📌 핵심 요약
컴퓨터와 회선이 결합된 **[[통신 시스템]]**의 구성 요소를 파악하고, 데이터 전송을 위한 **[[신호 변환]]**(Modem, DSU, Codec) 방식의 차이를 이해합니다. 또한, 각 계층별 **[[네트워크 장비]]**의 특징과 역할을 명확히 구분하여 학습합니다.

---

## 📝 상세 정리

### 1. 통신 시스템의 구성
- **구성 요소**: 송/수신자, 전송 매체, 메시지, 그리고 약속된 규칙인 **[[프로토콜]]**.
- **시스템 분류**:
    - **데이터 전송계**: 데이터 단말 장치(**[[DTE]]**), 데이터 회선 종단 장치(**[[DCE]]**), 통신 제어 장치(**[[CCU]]**).
        - **DTE**: 데이터 입출력 및 에러 제어 담당.
        - **DCE**: 신호 변환(변복조) 담당.
        - **CCU**: 컴퓨터와 회선 사이에서 속도 차이 조절 및 통신 기능 보강.
    - **데이터 처리계**: 컴퓨터(중앙처리장치 및 주변장치).

### 2. 신호 변환 방식 (Modem, DSU, Codec)
#### (1) 모뎀(Modem)의 디지털-아날로그 변환
반송파의 특성을 변화시켜 신호를 전달합니다.
- **[[ASK]] (진폭 편이 변조)**: 진폭의 크기를 변화.
- **[[FSK]] (주파수 편이 변조)**: 주파수를 변화.
- **[[PSK]] (위상 편이 변조)**: 위상(각도)을 변화.
- **[[QAM]] (직교 진폭 변조)**: 진폭과 위상을 동시에 변화 (고속 전송에 유리).

#### (2) DSU (Digital Service Unit)의 디지털-디지털 변환
변/복조 과정이 단순하며 전압의 극성을 이용합니다.
- **[[RZ]] (Return to Zero)**: 1이면 전압을 주고 비트 시간의 절반에서 0으로 복귀.
- **[[NRZ]] (Non-Return to Zero)**: 1이면 양극, 0이면 음극(또는 0V)으로 유지.
- **[[Manchester]]**: 비트 간격 중간에서 전압 변화를 일으켜 동기화 용이.
- **바이폴라 (Bipolar)**: 1이면 양극과 음극을 번갈아 사용(AMI), 0이면 전압을 주지 않음.

#### (3) Codec의 아날로그-디지털 변환 (**[[PCM]]** 방식)
1. **표본화 (Sampling)**: 연속적인 아날로그 신호를 일정 간격으로 추출.
2. **양자화 (Quantizing)**: 추출된 표본 값을 정수화(PAM 신호 변환).
3. **부호화 (Encoding)**: 정수화된 값을 2진수 디지털 신호로 변환.
4. **복호화 (Decoding)**: 수신측에서 디지털 신호를 다시 PAM 신호로 복원.
5. **여파화 (Filtering)**: 원래의 아날로그 신호와 유사하게 매끄럽게 복원.



### 3. 데이터 전송 시스템 및 제어 문자
- **전송 방향**:
    - **단방향 (Simplex)**: TV, 라디오처럼 한쪽으로만 전송.
    - **반이중 (Half-Duplex)**: 무전기처럼 양방향 가능하나 동시는 불가.
    - **전이중 (Full-Duplex)**: 전화기처럼 동시에 양방향 전송 가능.
- **동기식 전송 제어 문자**:
    - `SYN`: 문자 동기 유지.
    - `SOH`: 헤딩(Heading)의 시작.
    - `STX`: 본문의 시작 및 헤딩의 종료.
    - `ETX`: 본문의 종료.
    - `ETB`: 전송 블록의 종료.
    - `EOT`: 전송 종료 및 데이터 링크 해제.
    - `ENQ`: 응답 요구 및 데이터 링크 설정 요청.
    - `DLE`: 전송 제어 문자의 투명성 제공(구분자).
    - `ACK`: 긍정 응답 / `NAK`: 부정 응답.

### 4. 네트워크 장비
- **허브 (Hub)**: 여러 대의 컴퓨터를 연결.
    - **더미 허브**: 대역폭을 나누어 쓰므로 연결 대수가 늘면 속도 저하.
    - **스위칭 허브**: 목적지 포트에만 데이터를 전달하여 속도 유지.
- **리피터 (Repeater)**: 감쇠된 신호를 증폭하여 전송 거리 연장. (물리 계층)
- **브리지 (Bridge)**: LAN과 LAN을 연결하고 트래픽을 제어. (데이터링크 계층)
- **스위치 (Switch)**: 허브의 확장판. L2(MAC 주소), L3(IP/라우팅), L4(프로토콜/부하분산) 등으로 구분.
- **라우터 (Router)**: 최적의 경로(Routing)를 설정하여 데이터 전달. (네트워크 계층)
- **게이트웨이 (Gateway)**: 프로토콜이 전혀 다른 네트워크 간의 관문 역할.

---

> [!INFO] 💡 AI 보충 설명: L2, L3, L4 스위치의 차이
> 1. **L2 스위치**: OSI 2계층 장비로, MAC 주소를 기반으로 데이터를 전달합니다.
> 2. **L3 스위치**: 3계층 장비로, 라우터 기능을 내장하여 IP 주소를 기반으로 라우팅을 수행합니다.
> 3. **L4 스위치**: 4계층 장비로, TCP/UDP 포트 번호를 확인하여 서버 부하 분산(Load Balancing)을 수행합니다.

---

## 🎯 빈출 키워드
- **[[DCE]]**: 모뎀, DSU 등 신호 변환을 수행하는 종단 장치.
- **[[QAM]]**: 진폭과 위상을 조합하여 대용량 전송을 가능하게 하는 방식.
- **[[PCM]]**: 표본화-양자화-부호화 순서로 진행되는 신호 변환 기술.
- **[[라우터]]**: 네트워크 계층에서 최적의 경로를 결정하는 장비.
- **[[ACK/NAK]]**: 송신 데이터의 성공/실패 여부를 알리는 제어 문자.

# 데이터 오류 제어 및 ARQ 기법 상세 정리

## 📌 핵심 요약
데이터 전송 중 발생하는 **[[전송 오류]]**의 원인을 분석하고, 이를 검출 및 수정하기 위한 **[[순방향 오류 제어(FEC)]]**와 **[[역방향 오류 제어(BEC)]]** 방식을 학습합니다. 특히 오류 발생 시 재전송을 요구하는 **[[ARQ]]**의 네 가지 유형을 상세히 비교 분석합니다.

---

## 📝 상세 정리

### 1. 전송 오류의 원인 (신호 감쇠 및 왜곡)
데이터가 매체를 통해 전달될 때 물리적 요인으로 변형되는 현상입니다.
- **감쇠 (Attenuation)**: 전송 거리가 멀어짐에 따라 신호의 세기가 약해지는 현상.
- **지연 왜곡 (Delay Distortion)**: 하나의 매체로 여러 주파수를 보낼 때, 주파수별 전달 속도 차이로 인해 신호가 뒤섞이는 현상.
- **상호 변조 잡음 (Intermodulation Noise)**: 서로 다른 주파수들이 하나의 매체를 공유할 때 주파수 간의 합이나 차로 인해 발생하는 잡음.
- **충격 잡음 (Impulse Noise)**: 번개, 기계적 충격 등 외부 요인에 의해 발생하는 순간적이고 강한 잡음. (데이터 전송에 가장 큰 악영향)

### 2. 오류 제어 방식의 분류
- **순방향 오류 제어 ([[FEC]], Forward Error Correction)**: 송신측이 오류 수정 코드를 함께 보내 수신측이 직접 오류를 검출하고 **수정**까지 하는 방식. (역채널 불필요)
- **역방향 오류 제어 ([[BEC]], Backward Error Correction)**: 수신측이 오류를 검출하면 송신측에 **재전송(ARQ)**을 요청하는 방식. (역채널 필요)

### 3. 오류 검출 및 수정 기술
#### (1) 순방향 오류 제어 (FEC) 기법
- **[[해밍 코드]] (Hamming Code)**: 수신측에서 오류를 스스로 검출하고 **직접 수정**할 수 있는 대표적인 코드.
- **상승 코드 (Cyclic Code)**: 경계값 디코딩 등을 사용하여 여러 비트의 오류를 수정.

#### (2) 역방향 오류 제어 (BEC) 기법
- **패리티 검사 (Parity Check)**: 1의 개수가 짝수(Even) 또는 홀수(Odd)인지 확인하여 1비트 오류 검출.
- **블록합 검사 (BSC)**: 패리티 검사의 단점을 보완하기 위해 가로, 세로 두 방향으로 패리티를 체크.
- **[[CRC]] (순환 중복 검사)**: 다항식 코드를 이용하여 집단 오류를 검출하는 방식. HDLC 프레임 등에서 사용되며 신뢰도가 매우 높음.

### 4. ARQ (Automatic Repeat reQuest) 기법 상세
오류 검출 시 송신측에 재전송을 자동으로 요청하는 방식입니다.

- **Stop-and-Wait ARQ**:
    - 한 개의 프레임을 보내고 수신측의 `ACK` 또는 `NAK` 응답이 올 때까지 기다리는 방식.
    - 구조가 단순하지만 전송 효율이 매우 낮음. (BSC 프로토콜에서 사용)
- **Go-Back-N ARQ**:
    - 응답을 기다리지 않고 여러 프레임을 연속 전송.
    - 오류(`NAK`)가 발생한 프레임부터 그 이후에 보낸 **모든 프레임을 다시 전송**.
- **Selective Repeat ARQ**:
    - 오류가 발생한 **해당 프레임만 골라서 재전송**.
    - 전송 효율은 가장 좋으나 수신측에 복잡한 버퍼와 재조립 로직이 필요.
- **Adaptive ARQ**:
    - 채널의 상태(오류 발생률)에 따라 **프레임의 길이를 동적으로 변경**.
    - 오류가 적으면 긴 프레임, 오류가 많으면 짧은 프레임을 전송하여 효율 극대화.



### 5. 데이터 링크 프로토콜과 프레임
- **[[BSC]]**: 문자 위주의 프로토콜. 반이중 전송만 지원하며 Stop-and-Wait ARQ 사용.
- **[[HDLC]]**: 비트 위주의 프로토콜. 전이중 전송 및 모든 데이터 통신 방식 지원. Go-Back-N 또는 Selective Repeat ARQ 사용.

---

> [!INFO] 💡 AI 보충 설명: 해밍 거리에 따른 오류 제어 능력
> 송신된 코드와 수신된 코드 간의 차이가 나는 비트 수를 **해밍 거리(Hamming Distance)**라고 합니다.
> 1. 오류를 **검출**만 하려면 최소 해밍 거리가 $d+1$이어야 합니다.
> 2. 오류를 **수정**까지 하려면 최소 해밍 거리가 $2d+1$이어야 합니다. (여기서 $d$는 검출/수정하고자 하는 오류 비트 수)

## 🎯 빈출 키워드
- **[[해밍 코드]]**: 오류 검출과 수정을 동시에 수행하는 FEC의 대표 주자.
- **[[CRC]]**: 다항식을 이용한 강력한 오류 검출 기법.
- **[[Go-Back-N]]**: 오류 발생 시점부터의 모든 데이터를 다시 보내는 ARQ 방식.
- **[[Selective Repeat]]**: 오류가 난 특정 블록만 재전송하는 가장 효율적인 방식.
- **[[충격 잡음]]**: 디지털 데이터 전송 시 오류의 주된 원인이 되는 비연속적 잡음.
