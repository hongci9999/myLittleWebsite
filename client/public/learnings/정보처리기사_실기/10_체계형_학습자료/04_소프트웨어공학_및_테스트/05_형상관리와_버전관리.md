# 형상관리와 버전관리

## 핵심 요약

형상관리는 소프트웨어 변경을 체계적으로 통제하는 활동이다. 실기에서는 형상관리 기능 용어와 저장소 방식, CVS/SVN/Git 같은 도구 분류를 자주 묻는다.

## 1. 형상관리의 목적

- 변경 통제
- 추적성 확보
- 가시성 향상
- 협업 효율 향상

## 2. 형상 항목(CI)

- 관리 대상이 되는 산출물 단위
- 소스코드, 문서, 테스트 산출물 등이 될 수 있다

## 3. 주요 기능

| 기능 | 의미 |
|---|---|
| Check-Out | 저장소에서 사본 가져오기 |
| Check-In | 변경본 저장소 반영 |
| Commit | 변경 확정 |
| Update | 최신 상태 동기화 |
| Diff | 버전 차이 비교 |
| Import | 새 자원 등록 |
| Export | 버전 정보 제외 추출 |

## 4. 저장소 방식

| 방식 | 도구 예 |
|---|---|
| 공유 폴더 방식 | RCS, SCCS |
| 클라이언트/서버 | CVS, SVN |
| 분산 저장소 | Git, Mercurial |

## 5. 도구 특징

- SVN: 중앙집중형
- Git: 분산형
- RCS: 로컬 중심 초기 방식

## 6. 예제

- 로컬 작업본과 저장소 차이 비교 -> Diff
- 저장소 최신 반영 -> Update

## 7. 자주 틀리는 함정

- Check-In 과 Commit을 완전히 같은 의미로 본다
- Git을 중앙집중형으로 착각한다
- 형상관리와 프로젝트 관리 도구를 같은 범주로 본다

## 8. 시험장에서 푸는 순서

1. 기능 용어 문제인지 도구 분류 문제인지 구분한다.
2. 중앙집중형인지 분산형인지 먼저 본다.
3. Git, SVN, CVS, RCS를 방식별로 연결한다.

## 한 페이지 요약

- 형상관리는 변경을 체계적으로 관리하는 활동이다.
- Check-Out, Check-In, Commit, Update, Diff는 핵심 기능이다.
- Git은 분산형, SVN/CVS는 중앙집중형으로 기억한다.
