# 데이터 분석 기획 및 접근 방식의 이해

## 📌 핵심 요약

데이터 분석 기획은 분석 대상(What)과 방법(How)에 따라 4가지 유형으로 분류되며, 과제 중심의 단기적 접근과 마스터 플랜 중심의 중장기적 접근으로 나뉩니다. 하향식(Top-Down)과 상향식(Bottom-Up) 접근 방식을 반복적으로 수행하여 최적의 분석 과제를 도출하고 로드맵을 수립하는 것이 핵심입니다.

---

## 📝 상세 정리

### 1. 분석 기획의 유형 (Analysis Planning Types)

분석의 대상(What)과 방법(How)의 인지 여부에 따라 4가지 유형으로 구분됩니다.

- **Optimization (최적화):** 분석 대상과 방법을 알고 있는 경우.
- **Insight (통찰):** 분석 대상은 알지만 방법은 모르는 경우.
- **Solution (솔루션):** 분석 대상은 모르지만 방법은 알고 있는 경우.
- **Discovery (발견):** 분석 대상과 방법 모두 모르는 경우.

> [!INFO] 💡 AI 보충 설명: 4가지 유형 상세
>
> - **Optimization (최적화):** 기존 프로세스를 효율화하거나 정교화하는 단계입니다.
> - **Insight (통찰):** 기존 분석 방식으로는 한계가 있어 새로운 관점을 통해 숨겨진 의미를 찾는 단계입니다.
> - **Solution (솔루션):** 검증된 분석 도구나 기법을 새로운 비즈니스 영역에 이식하는 단계입니다.
> - **Discovery (발견):** 데이터 자체를 탐색하여 새로운 비즈니스 기회나 가치를 창출하는 가장 고도화된 단계입니다.

### 2. 기간에 따른 분석 기획 구분

- **단기적 접근 (과제 중심):** Quick-Win 방식으로 빠르게 분석 결과를 도출하여 성과를 가시화함.
- **중장기적 접근 (마스터 플랜):** 분석 문화를 내재화하고 전략적 가치를 창출하기 위한 종합 계획 수립.
- **혼합형:** 마스터 플랜으로 장기적 방향을 설정하되, 가치 증명을 위해 Quick-Win 과제를 병행 수행.

### 3. 분석 기획 절차 (Analysis Planning Process)

1.  **비즈니스 이해 및 범위 설정:** 도메인 이슈 도출, 현황 파악, 개선 과제 정의.
2.  **프로젝트 정의:** 분석 목표 수립, 분석 목표 정의서 작성.
3.  **프로젝트 수행 계획 수립:** 작업 분할 구조도(WBS; Work Breakdown Structure) 작성, 예산 및 기간 고려.
4.  **프로젝트 위험 계획 수립:** 보유 데이터 자산(품질, 규모) 확인 및 법률적 이슈 검토.

### 4. 분석 마스터 플랜 (Analysis Master Plan)

전체적인 분석 방향성을 제시하는 기본 계획입니다.

- **수립 절차:** 정보 전략 계획(ISP; Information Strategy Planning) 방법론 활용, 데이터 분석 기획의 특성을 고려하여 수행.
- **구성 요소:** 분석 과제 정의, 우선순위 결정, 분석 로드맵 수립.

### 5. 정보 전략 계획 (ISP; Information Strategy Planning)

- 조직의 경영 환경을 분석하여 기회 및 문제점을 도출하는 중장기 마스터 플랜 수립 절차.
- 사용자의 요구사항을 확인하여 정보 시스템 구축의 우선순위를 결정함.

### 6. 우선순위 평가 기준 (Priority Evaluation Criteria)

#### IT 프로젝트의 과제 우선순위

- **전략적 중요도:** \* **전략적 필요성:** 조직의 비전 및 목표와의 정합성.
  - **시급성:** 목표 달성을 위해 얼마나 빨리 수행되어야 하는가의 정도.
- **실행 용이성:**
  - **투자 용이성:** 비용 확보 및 예산 투입의 수월함.
  - **기술 용이성:** 현재 기술 수준으로 구현 가능한지 여부.

#### 데이터 분석 프로젝트의 ROI(Return on Investment) 요소

- **투자 비용 요소 (3V):**
  - **Volume (크기):** 데이터의 규모.
  - **Variety (형태):** 데이터의 종류 및 유형.
  - **Velocity (속도):** 데이터의 생성 및 처리 속도.
- **비즈니스 효과 (Value; 4V의 마지막 요소):**
  - 새로운 가치 창출 및 비즈니스 임팩트.

#### 분석 ROI를 고려한 우선순위 평가 기준

- **시급성 (Urgency):** 전략적 중요도와 KPI(Key Performance Indicator)를 기반으로 현재 및 중장기적 관점의 비즈니스 효과 판단. value 관점
- **난이도 (Difficulty):** 데이터 획득/가공/저장 비용, 분석 수준, 분석 적용 비용 등 투자 비용 요소 고려. 3v 관점

### 7. 분석 로드맵 수립 (Analysis Roadmap)

- 최종 우선순위를 결정하여 단계별 이행 로드맵 수립.
- 과제별 세부 일정 계획을 명확히 정의함.
- **특징:** 데이터 수집과 준비는 순차적으로 진행되나, 모델링 단계는 반복적인 정련 과정을 통해 완성도를 높이는 **혼합형(순차+반복)** 방식을 주로 사용함.

### 8. 분석 접근 방식 (Analysis Approach)

#### 하향식 접근 방식 (Top-Down Approach)

문제가 주어졌을 때 해법을 찾기 위해 체계적으로 단계화하여 수행하는 방식.

- **절차:** 문제 탐색 → 문제 정의 → 해결방안 탐색 → 타당성 평가.
- **문제 탐색:** 비즈니스 모델 기반 탐색, 외부 사례 기반 탐색.
  1. 비즈니스 모델 기반 탐색

  5대 영역: 업무, 제품, 고객, 규제와 감사, 지원인프라
  2. 발굴 범위 확장
     분석 대상을 더 넓은 시야에서 바라보는 관점들입니다.
     거시적 관점 (STEEP): 사회(Social), 기술(Technological), 경제(Economic), 환경(Environmental), 정치(Political)
     경쟁자 확대 관점: 대체자, 경쟁자, 신규 진입자
     시장의 니즈 탐색 관점: 고객, 채널, 영향자
     역량의 재해석: 내부 역량, 파트너 네트워크

  3. 내/외부 사례 참조

  외부 참조 모델 기반: 벤치마킹을 통해 유스케이스(과거 유사한 사례) 분석
  💡 문제 탐색 단계에서는 정성적 분석 기법(FGI, 인터뷰, 설문 등)을 활용할 수 있습니다.

  (2) 문제 정의
  핵심: 식별된 비즈니스 문제를 데이터 문제로 변환하여 정의하는 단계입니다.
  (예: "매출을 올리자" → "고객 이탈 요인을 분석하여 타겟 마케팅 데이터를 추출하자")

  (3) 해결방안 탐색
  방법: 기존 시스템 활용, 시스템 고도화, 인적 자원 확보, 아웃소싱 등 다양한 방안을 모색합니다.
  (4) 타당성 검토
  도출된 해결 방안이 실제로 실행 가능한지 평가합니다.

  경제적 타당성: 비용 대비 수익(ROI)이 충분한가?
  데이터 타당성: 분석에 필요한 데이터가 존재하는가?
  기술적 타당성: 현재의 분석 역량이나 기술로 구현 가능한가?

- **문제 정의:** 데이터 분석 문제로의 변환.
- **수행 옵션:** 기술적/경제적 타당성 검토 후 최적 옵션 도출.

#### 상향식 접근 방식 (Bottom-Up Approach)

데이터 자체에서 의미를 찾거나 왜 그러한 일이 발생하는지 역으로 추적하여 인사이트를 도출하는 방식.

- **특징:** 기존 하향식 방식의 한계(새로운 문제 발견의 어려움, 복잡한 환경 부적합)를 극복하기 위해 시행착오를 통한 발견 강조.
- **비고:** 비지도 학습(Unsupervised Learning)이나 디자인 씽킹(Design Thinking) 기법이 주로 활용됨.

### 9. 포트폴리오 사분면 분석 (Portfolio Quadrant Analysis) 기법

// 요청하신 사분면 분석 구조입니다.

- **1사분면:** 전략적 중요도가 높아 시급하지만, 기술적 난이도가 높아 실행이 어려운 영역
- **2사분면:** 전략적 중요도가 낮아 시급하지 않고, 기술적 난이도도 높은 영역
- **3사분면:** 전략적 중요도가 높아 시급하며, 기술적 난이도도 낮아 즉시 실행 가능한 영역 (**최우선순위**)
- **4사분면:** 전략적 중요도가 낮아 시급하지 않지만, 기술적 난이도가 낮아 쉽게 적용 가능한 영역

### 10. 우선순위 결정 기준에 따른 순서

우선순위는 기업의 전략적 방향(시급성 중심 vs 난이도 중심)에 따라 달라질 수 있습니다.

#### ① 시급성(Urgency)을 우선으로 고려할 경우

> 전략적 중요도가 높은 것부터 처리하는 방식
> **순서: 3사분면 → 4사분면 → 1사분면 → 2사분면**

#### ② 난이도(Difficulty)를 우선으로 고려할 경우

> 기술적 용이성을 먼저 고려하여 성공 사례(Quick-Win)를 확보하는 방식
> **순서: 3사분면 → 1사분면 → 4사분면 → 2사분면**

### 11. 적용 우선순위 조정 요인

단순 사분면 분석 외에도 다음과 같은 요소를 고려하여 최종 순위를 조정합니다.

- **기술적 요소에 따른 조정:** 데이터 수집 가능성, 분석 알고리즘의 복잡성, IT 인프라 환경 등
- **분석 범위에 따른 조정:** 분석 결과의 파급 효과, 유관 부서와의 협업 필요성 등

> [!INFO] 💡 AI 보충 설명
> **Quick-Win(퀵윈):** 단기간에 가시적인 성과를 낼 수 있는 과제를 의미합니다. 난이도가 낮은 3, 4사분면 과제를 먼저 수행하여 데이터 분석에 대한 조직 내 신뢰를 쌓는 전략으로 자주 활용됩니다.
> 일반적으로 **시급성 기준(3→4→1→2)**은 비즈니스 가치 중심이며, **난이도 기준(3→1→4→2)**은 기술적 실현 가능성 중심의 의사결정 방식입니다.
