# 데이터 탐색 (Data Exploration)

## 📌 핵심 요약
데이터의 의미를 찾기 위해 **통계**와 **시각화**를 통해 파악하는 과정입니다. EDA(탐색적 자료 분석)는 가설 검증보다는 데이터 자체의 특성을 발견하는 데 초점을 둡니다.

---

## 📝 상세 정리

### 1. 데이터 탐색 개요

#### ● EDA (Exploratory Data Analysis, 탐색적 자료 분석)

데이터의 의미를 찾기 위해 **통계**와 **시각화**를 통해 파악하는 분석 접근법입니다.

#### ● EDA의 4가지 주제

| 주제 | 설명 |
| :--- | :--- |
| **1) 저항성의 강조** | 자료 변동에 민감하지 않음 (극단값·이상치에 덜 영향받는 지표 활용) |
| **2) 잔차 계산** | 값들이 주경향(평균 등)으로부터 얼마나 벗어나 있는지 확인하는 척도 |
| **3) 자료변수의 재표현** | 원래 변수를 적당한 척도로 변환 (로그, 제곱근 등) |
| **4) 그래프를 통한 현시성** | 시각화를 통하여 효율적으로 파악 |

> [!TIP] 암기법: **저잔재현** (저항성, 잔차, 재표현, 현시성)

---

### 2. 고급 데이터 탐색 (Advanced Data Exploration)

#### ● 시공간 데이터 탐색 (Spatio-temporal Data Exploration)

* **시공간 데이터:** 공간적 정보에 **시간의 흐름**이 결합된 데이터.
* **활용:** 시공간 패턴을 통한 예측, 지도를 통한 위치정보, 지리공간의 격자 차트 결합.

#### ● 다변량 데이터 탐색 (Multivariate Data Exploration)

* **목표:** 변수 간 관계, 패턴 분석, 이상치 탐지, 데이터 요약.
* **방법:** 상관관계분석, PCA, 다차원척도법(MDS), 다중선형회귀, 군집 분석 등.

#### ● 비정형 데이터 탐색: 자연어 전처리 (Natural Language Preprocessing)

| 단계 | 설명 |
| :--- | :--- |
| **(1) 토큰화 (Tokenization)** | 의미 있는 말뭉치로 문자를 나누는 작업 |
| **(2) 불용어(Stop Words) 처리** | 조사, 접속사 등 의미 없는 정보 제거 |
| **(3) 정규화 (Normalization)** | 같은 의미이면서 표현이 다른 단어를 통합 |
| **(4) 어간추출 (Stemming)** | 접사를 제거하여 기본 형태를 찾아내는 작업 |
| **(5) 표제어 추출 (Lemmatization)** | 사전에 기반하여 단어를 원형으로 변환 |

> [!CAUTION] ⚠️ 한국어 토큰화
> 한국어는 **띄어쓰기가 일정하지 않고**, **형태소 단위**로 분리해야 하므로 토큰화가 어렵습니다.
