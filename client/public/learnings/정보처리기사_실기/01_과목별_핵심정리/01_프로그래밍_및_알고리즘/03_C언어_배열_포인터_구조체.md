# C언어 배열, 포인터, 구조체

## 핵심 요약

실기 C언어에서 가장 까다로운 영역은 포인터다. 배열 이름이 첫 요소의 주소처럼 동작한다는 점, `*`와 `&`의 의미, 포인터 연산이 자료형 크기를 기준으로 이동한다는 점을 이해하면 대부분의 문제를 풀 수 있다.

## 왜 중요한가

- 정보처리기사 실기 C언어에서 포인터와 배열은 대표 고난도 주제다.
- 단순히 선언법만 외우면 안 되고, 메모리 관점에서 값이 어떻게 연결되는지 이해해야 한다.
- 구조체, 함수 인자 전달, 2차원 배열 문제도 결국 포인터 해석으로 이어진다.

## 1. 배열

- 배열은 같은 자료형 데이터를 연속된 메모리에 저장한다.
- 인덱스는 0부터 시작한다.
- 배열 이름은 첫 번째 요소의 시작 주소처럼 사용된다.

```c
int arr[3] = {10, 20, 30};
printf("%d\n", arr[0]);   // 10
printf("%d\n", *(arr+1)); // 20
```

즉, `arr[i]` 와 `*(arr + i)` 는 같은 의미다.

## 2. 문자열

- C에서 문자열은 `char` 배열이다.
- 문자열 끝에는 널 문자 `\0`가 들어간다.

```c
char str[] = "ABC";
```

메모리에는 `A`, `B`, `C`, `\0` 가 저장된다.

## 3. 포인터

### 3.1 기본 개념

- 포인터는 주소를 저장하는 변수다.
- `&a` 는 변수 `a`의 주소
- `*p` 는 포인터 `p`가 가리키는 실제 값

```c
int a = 10;
int *p = &a;
printf("%d\n", *p); // 10
```

### 3.2 포인터 연산

- `p + 1` 은 무조건 1바이트 증가가 아니다.
- 포인터가 가리키는 자료형 크기만큼 이동한다.

예를 들어 `int`가 4바이트라면:

- `p`가 100번지를 가리킬 때
- `p + 1` 은 104번지를 가리킨다.

## 4. 함수와 참조 전달

C언어는 기본적으로 값에 의한 전달을 하지만, 포인터를 사용하면 원본 값을 바꿀 수 있다.

```c
void change(int *x) {
    *x = 50;
}

int a = 10;
change(&a);
printf("%d\n", a); // 50
```

## 5. 구조체

- 구조체는 서로 다른 자료형을 하나로 묶는 사용자 정의 자료형이다.
- 멤버 접근은 일반 변수일 때 `.` 연산자, 포인터일 때 `->` 연산자를 쓴다.

```c
struct Student {
    char name[10];
    int score;
};

struct Student s = {"Kim", 90};
printf("%d\n", s.score);
```

## 6. 2차원 배열과 포인터

2차원 배열도 결국 "배열의 배열"이다.

```c
int a[2][3] = {{1, 2, 3}, {4, 5, 6}};
printf("%d\n", a[1][2]);         // 6
printf("%d\n", *(*(a+1)+2));     // 6
```

`a[1][2]` 와 `*(*(a+1)+2)` 는 완전히 같은 의미다.

## 7. 코드 추적 예제

### 예제 1: 배열과 포인터

```c
int arr[3] = {5, 10, 15};
int *p = arr;
printf("%d %d\n", *p, *(p+2));
```

- `p = arr` -> 배열 첫 요소 주소 저장
- `*p` -> `arr[0]` -> `5`
- `*(p+2)` -> `arr[2]` -> `15`
- 출력: `5 15`

### 예제 2: 포인터를 통한 값 변경

```c
int a = 3;
int *p = &a;
*p = *p + 7;
printf("%d\n", a);
```

- `p`는 `a`의 주소
- `*p`는 곧 `a`
- `*p = *p + 7` -> `a = 10`
- 출력: `10`

### 예제 3: 구조체 포인터

```c
struct Data {
    int x;
};

struct Data d = {20};
struct Data *p = &d;
printf("%d %d\n", d.x, p->x);
```

- `d.x` 와 `p->x` 는 같은 멤버를 다른 방식으로 접근한 것이다.
- 출력: `20 20`

## 8. 자주 틀리는 함정

### 8.1 배열 이름은 변수처럼 바꿀 수 없다

- 배열 이름은 시작 주소 역할을 하지만 일반 포인터 변수처럼 다른 주소를 대입할 수는 없다.

### 8.2 `*`와 `&`의 의미 혼동

- `&a` 는 주소
- `*p` 는 그 주소가 가리키는 값
- 둘은 정반대 동작이다.

### 8.3 포인터 연산을 1바이트 이동으로 착각

- `p + 1` 은 자료형 크기만큼 이동한다.

### 8.4 문자열 끝의 `\0` 누락

- 문자열 길이 문제에서 널 문자를 포함하는지 여부를 조심해야 한다.

## 9. 시험장에서 푸는 순서

1. 배열인지 포인터 변수인지 먼저 구분한다.
2. 포인터가 무엇의 주소를 저장하는지 적는다.
3. `*`가 몇 번 붙었는지 확인한다.
4. 필요하면 `arr[i]` 를 `*(arr+i)` 형태로 바꿔 본다.
5. 구조체 포인터는 `->` 를 `(*p).멤버`로 바꿔 해석한다.

## 한 페이지 요약

- 배열 이름은 첫 요소의 주소처럼 동작한다.
- `arr[i]` 와 `*(arr+i)` 는 같다.
- 포인터는 주소를 저장하고, `*`는 실제 값을 꺼낸다.
- 포인터 연산은 자료형 크기 기준으로 이동한다.
- 구조체 포인터는 `->` 연산자로 멤버에 접근한다.
