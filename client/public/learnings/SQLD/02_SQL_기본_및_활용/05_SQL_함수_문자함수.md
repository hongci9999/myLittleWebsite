# SQL 함수 — 문자·숫자·날짜·변환

## 핵심 요약

**단일행 함수**는 행마다 1→1, **집계함수**는 N→1. **NULL 처리**: **NVL**(NULL→대체값), **NULLIF**(같으면 NULL), **COALESCE**(첫 번째 non-NULL). **집계**는 NULL 기본 제외, **`COUNT(*)`만 포함**. **WHERE**에 집계 불가 → **HAVING**.

## 왜 중요한가

- SELECT·WHERE·**GROUP BY·HAVING** 문제의 핵심이다.
- AVG·SUM의 **NULL 제외**, `COUNT(*)` vs `COUNT(열)` 구분이 **함정** 빈출.
- ROUND/TRUNC·TO_DATE 형식도 같은 문서에서 함께 복습한다.

> 실행 순서: [03_SELECT_문_구조와_실행순서](./03_SELECT_문_구조와_실행순서.md) — HAVING은 **(4)번**

## 1. 함수 개요

| 항목 | 내용 |
|------|------|
| **정의** | 특정 연산을 수행하고 **결과(Output)** 를 돌려주는 **미리 정의된** 코드 |
| **사용** | `SELECT`, `WHERE`, `ORDER BY` 등 **표현식** 위치 |

### 단일행 vs 다중행

| 구분 | 입력 → 출력 | 예 |
|------|-------------|-----|
| **단일행 함수** (Scalar) | **1 → 1** (행마다) | `UPPER(이름)`, `SUBSTR(이름,1,2)` |
| **다중행 함수** (집계) | **N → 1** | `COUNT(*)`, `SUM(학점)`, `AVG(나이)` |

> 집계함수 상세는 **§6**, GROUP BY는 다음 문서.

### Oracle 참고 — `DUAL`

| 항목 | 내용 |
|------|------|
| **DUAL** | Oracle에서 함수·식을 **테스트**할 때 쓰는 **더미 테이블** |
| 예 | `SELECT UPPER('test') FROM DUAL;` |

### 리터럴 표기

| 타입 | 표기 |
|------|------|
| **문자형** | **작은따옴표** `'David'` |
| **숫자형** | 따옴표 **없이** `101`, `1`, `2` |

## 2. 문자함수 (String Functions)

### 요약 표

| 함수 | 설명 | 예 |
|------|------|-----|
| **LOWER(string)** | **소문자** 변환 | `LOWER('David')` → `'david'` |
| **UPPER(string)** | **대문자** 변환 | `UPPER('David')` → `'DAVID'` |
| **SUBSTR(string, start, [length])** | 시작 위치부터 **길이만큼** 부분 문자열 | 아래 상세 |
| **TRIM(string, [char])** | **양쪽** 공백 또는 지정 문자 제거 | 아래 상세 |
| **LTRIM(string, [char])** | **왼쪽**만 제거 | 아래 상세 |
| **RTRIM(string, [char])** | **오른쪽**만 제거 | 아래 상세 |
| **LENGTH(string)** | 문자열 **길이** (공백 포함) | `LENGTH('Hello World')` → **11** |
| **REPLACE(string, target, [replacement])** | 문자열 **치환** | 아래 상세 |
| **ASCII(char)** | 문자 → **아스키 코드** | `ASCII('A')` → **65** |

> 교재에서 **SUBSTR**, **TRIM** 은 **빈출** 표시 — 위치·방향 함정 주의.

---

### SUBSTR (부분 문자열) — 빈출

```sql
SUBSTR('Hi David', 1, 2)   -- 'Hi'   (1번째부터 2글자)
SUBSTR('Hi David', -5)     -- 'David' (끝에서 5번째 위치부터 끝까지 — DBMS별 해석 확인)
```

| 포인트 | 내용 |
|--------|------|
| 인자 | `문자열`, **시작 위치**, `[길이]` (생략 시 끝까지) |
| 시작 위치 | **1부터** 시작하는 경우가 많음 (0 기준 DBMS 있음 — 문항 확인) |
| **음수** | **뒤에서** 세는 시작 위치 (예: -5 → 'David') |

---

### TRIM / LTRIM / RTRIM — 빈출

```sql
TRIM('  Hello World  ')           -- 'Hello World' (양쪽 공백)
TRIM('!!!Wow!!!', '!')            -- 'Wow' (양쪽 '!')

LTRIM('  Hello World  ')          -- 'Hello World ' (왼쪽만)
LTRIM('!!!Wow!!!', '!')           -- 'Wow!!!'

RTRIM('  Hello World  ')          -- ' Hello World' (오른쪽만)
RTRIM('!!!Wow!!!', '!')           -- '!!!Wow'
```

| 함수 | 제거 위치 |
|------|-----------|
| **TRIM** | **양쪽** |
| **LTRIM** | **왼쪽** |
| **RTRIM** | **오른쪽** |
| 2번째 인자 | 제거할 **문자** 지정 (없으면 공백) |

---

### REPLACE

```sql
REPLACE('Hello John', 'John', 'Tim')  -- 'Hello Tim'
REPLACE('Hello World', 'World')       -- 'Hello ' (replacement 생략 → 제거)
```

---

### LENGTH · ASCII

| 함수 | 예 | 결과 |
|------|-----|------|
| **LENGTH** | `'Hello World'` (공백 1개 포함) | **11** |
| **ASCII** | `'A'` | **65** |

## 3. 숫자함수 (Numeric Functions)

| 함수 | 설명 | 예 |
|------|------|-----|
| **ABS(n)** | **절댓값** | `ABS(-3)` → 3, `ABS(-10.4)` → 10.4 |
| **MOD(n1, n2)** | n1 ÷ n2의 **나머지** | `MOD(10,2)` → 0, `MOD(11,3)` → 2 |
| **ROUND(n, [자릿수])** | 지정 자릿수로 **반올림** | 아래 상세 |
| **TRUNC(n, [자릿수])** | 지정 자릿수에서 **버림(절삭)** | 아래 상세 |
| **FLOOR(n)** | **이하** 최대 정수 | `FLOOR(3.58)` → 3 |
| **CEIL(n)** | **이상** 최소 정수 | `CEIL(3.58)` → 4 |
| **SIGN(n)** | 양수 1, 음수 -1, 0 → 0 | `SIGN(3.58)` → 1, `SIGN(-3)` → -1 |
| **POWER(n, 지수)** | **거듭제곱** | `POWER(2,3)` → 8 |

### ROUND vs TRUNC — 빈출

| 식 | ROUND | TRUNC |
|----|-------|-------|
| `15.58`, **1** | **15.6** (소수 1자리 반올림) | **15.5** (소수 1자리 절삭) |
| `15.58`, 생략 | **16** (정수 반올림) | **15** (정수 절삭) |
| `15.58`, **-1** | **20** (10의 자리 반올림) | **10** (10의 자리 절삭) |

| 구분 | ROUND | TRUNC |
|------|-------|-------|
| 동작 | **반올림** | **잘라내기**(올림 없음) |
| 자릿수 | 양수=소수 자리, **음수=정수 자리** | 동일 |

| 함수 | 설명 | 예 |
|------|------|-----|

## 5. 변환함수 (Conversion Functions)

| 함수 | 설명 | 예 |
|------|------|-----|
| **TO_NUMBER(문자)** | 문자 → **숫자** | `TO_NUMBER('2025')` → 2025 |
| **TO_CHAR(값, 형식)** | 숫자·날짜 → **문자** (형식 지정) | 아래 |
| **TO_DATE(문자, 형식)** | 문자 → **날짜** (형식 지정) | 아래 |

### TO_CHAR 예

```sql
TO_CHAR(1234567.89, '9,999,999.99')  -- '1,234,567.89'
TO_CHAR(SYSDATE, 'YYYYMMDD')         -- '20250305' (예)
```

### TO_DATE 예

```sql
TO_DATE('2025-03-05', 'YYYY-MM-DD')  -- 2025-03-05
TO_DATE('2025/03/05', 'YYYY/MM/DD')  -- 구분자에 맞는 형식 문자열
```

| 주의 | 내용 |
|------|------|
| **타입** | 날짜·문자는 **겉으로 비슷해 보여도** 내부 **데이터 타입이 완전히 다름** |
| 형식 | `TO_CHAR` / `TO_DATE`의 **두 번째 인자(형식)** 가 **문자열 패턴**과 **일치**해야 함 |

## 6. 집계함수 (Aggregate / 다중행 함수)

### 개요

| 항목 | 내용 |
|------|------|
| **정의** | 여러 **행**을 묶어 **하나의 결과값**을 반환 |
| **다른 이름** | **다중행 함수**, **집계 함수** |
| **입출력** | 입력 **N행** → 출력 **1값** (그룹·전체 테이블 단위) |

| 함수 | 역할 |
|------|------|
| **COUNT** | **개수** |
| **SUM** | **합계** |
| **AVG** | **평균** |
| **MIN** | **최솟값** |
| **MAX** | **최댓값** |

### 예 — \<임직원\> 테이블

**데이터**

| 이름 | 급여 |
|------|------|
| 박이사 | 2000 |
| 이부장 | 1500 |
| 김과장 | 1300 |
| 서대리 | 1000 |
| 한수습 | **NULL** |

```sql
SELECT COUNT(*)   AS 인원수,
       SUM(급여)    AS 총급여,
       AVG(급여)    AS 평균급여,
       MAX(급여)    AS 최고급여,
       MIN(급여)    AS 최저급여
  FROM 임직원;
```

| 결과 열 | 값 | 계산 |
|---------|-----|------|
| **인원수** | **5** | `COUNT(*)` — **NULL 행 포함** 전체 행 수 |
| **총급여** | **5800** | 2000+1500+1300+1000 (**NULL 제외**) |
| **평균급여** | **1450** | 5800 ÷ **4** (NULL 제외 인원) |
| **최고급여** | **2000** | |
| **최저급여** | **1000** | |

### 집계함수 규칙 (주의사항)

| # | 규칙 |
|---|------|
| 1 | **다중행 함수** — 여러 행 → **결과 1개** |
| 2 | **NULL 제외** — SUM·AVG·MIN·MAX·`COUNT(열)` 은 **NULL인 행/값은 계산에서 제외** |
| 3 | **WHERE 절 사용 불가** — 행 필터는 **WHERE**, 집계 조건은 **HAVING**(GROUP BY 후) |
| 4 | **COUNT 예외** — `COUNT(*)`, `COUNT(1)`, `COUNT(0)` 은 **NULL을 포함**해 **전체 행 수** |

| COUNT 형태 | NULL 행 | 의미 |
|------------|---------|------|
| **COUNT(\*)** | **포함** | 테이블 **전체 행** 수 (= 5) |
| **COUNT(급여)** | **제외** | 급여가 **NOT NULL** 인 행 (= 4) |
| **COUNT(1)** / **COUNT(0)** | **포함** | `COUNT(*)` 와 같이 **행 개수** |

```sql
-- 오류(또는 불가) 예
SELECT 이름 FROM 임직원 WHERE AVG(급여) > 1000;

-- 집계 조건은 HAVING (GROUP BY와 함께)
SELECT 부서, AVG(급여) FROM 임직원 GROUP BY 부서 HAVING AVG(급여) > 1000;
```

## 7. NULL 관련 함수

> NULL 개념: [13_트랜잭션과_NULL](../01_데이터_모델링의_이해/13_트랜잭션과_NULL.md)

### 정의

| 함수 | 설명 |
|------|------|
| **NVL(열, 대체값)** | 열이 **NULL**이면 **대체값** 반환, 아니면 **원래 값** |
| **NULLIF(열1, 열2)** | 두 값이 **같으면 NULL**, 다르면 **열1** 반환 |
| **COALESCE(열1, 열2, …)** | 인자 중 **처음으로 NULL이 아닌 값** 반환 |

> **NVL**은 Oracle 계열에서 많이 씀. 표준 SQL에서는 **COALESCE**·**IFNULL**(MySQL) 등 DBMS별 대체 함수 확인.

### 공통 예제 테이블 \<임직원\>

| 사번 | 부서 | 급여 | 성과급 | 전화번호 | 이메일 | 지역 |
|------|------|------|--------|----------|--------|------|
| 101 | 경영 | 1300 | 100 | 123-456 | abc@ab | 서울 |
| 102 | 기획 | 1100 | 20 | NULL | tav@tw | 인천 |
| 103 | 개발 | 1200 | NULL | 789-012 | cd@pj | 대구 |
| 104 | 영업 | 1100 | 30 | NULL | NULL | 부산 |

### A. NVL — NULL을 다른 값으로

```sql
SELECT 사번, 급여, NVL(성과급, 0) AS 보너스
  FROM 임직원;
```

| 사번 | 급여 | 보너스 |
|------|------|--------|
| 101 | 1300 | 100 |
| 102 | 1100 | 20 |
| 103 | 1200 | **0** ← 성과급 NULL |
| 104 | 1100 | 30 |

### B. NULLIF — 같으면 NULL로

```sql
SELECT 사번, 부서, NULLIF(성과급, 100) AS 개별성과급
  FROM 임직원;
```

| 사번 | 개별성과급 | 설명 |
|------|------------|------|
| 101 | **NULL** | 성과급 **100** → 열2와 **같음** |
| 102 | 20 | |
| 103 | NULL | 원래 NULL |
| 104 | 30 | |

### C. COALESCE — 첫 non-NULL

```sql
SELECT 사번,
       COALESCE(전화번호, 이메일, 지역) AS 연락수단
  FROM 임직원;
```

| 사번 | 연락수단 | 선택된 값 |
|------|----------|-----------|
| 101 | 123-456 | 전화번호 |
| 102 | tav@tw | 전화 NULL → **이메일** |
| 103 | 789-012 | 전화번호 |
| 104 | 부산 | 전화·이메일 NULL → **지역** |

### 세 함수 비교

| 함수 | NULL이 있을 때 | 같을 때 / 순서 |
|------|----------------|----------------|
| **NVL** | **지정 값**으로 치환 | 2개 인자 |
| **NULLIF** | (원래 NULL 유지) | **같으면** NULL |
| **COALESCE** | **앞에서부터** 첫 non-NULL | 인자 **여러 개** |

## 8. SELECT에서 사용 예

```sql
-- 단일행 함수
SELECT UPPER(이름) AS 이름_대문자,
       ROUND(점수, 1) AS 점수_반올림
  FROM 학생;

-- 집계함수 (전체 1행 결과)
SELECT COUNT(*) AS cnt, AVG(나이) AS 평균나이 FROM 학생;
```

## 9. 시험 포인트 / 함정

| 구분 | 내용 |
|------|------|
| 단일행 | **행마다** 1개 결과 |
| 집계 | COUNT·SUM·AVG·MIN·MAX — **N→1** |
| NVL | NULL → **대체값** (`NVL(성과급,0)` → 0) |
| NULLIF | **같으면 NULL** (`NULLIF(성과급,100)` → 101번 NULL) |
| COALESCE | **왼쪽부터** 첫 non-NULL (전화→이메일→지역) |
| 집계 NULL | SUM·AVG·COUNT(열) → **NULL 제외** |
| COUNT(*) | **NULL 포함** |
| WHERE | 집계 함수 **불가** → **HAVING** |
| 함정 | NVL과 NULLIF **역할 혼동** |
| 함정 | COALESCE **순서** — 앞 열이 우선 |
| 함정 | `COUNT(급여)` = 5 → **4** |
| 함정 | WHERE에 `AVG(급여)>1000` |
