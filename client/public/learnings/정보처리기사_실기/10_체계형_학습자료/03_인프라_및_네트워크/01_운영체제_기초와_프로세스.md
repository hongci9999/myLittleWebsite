# 운영체제 기초와 프로세스

## 핵심 요약

운영체제는 CPU, 메모리, 파일, 입출력 장치를 효율적으로 관리하는 시스템 소프트웨어다. 실기에서는 운영체제의 목적, 프로세스/스레드 개념, 상태 전이, 성능 지표를 묶어서 자주 묻는다.

## 왜 중요한가

- 운영체제는 계산형 문제와 용어형 문제가 함께 나온다.
- 프로세스와 스레드는 스케줄링, 교착상태, 메모리 관리와 모두 연결된다.
- 반환시간, 대기시간, 응답시간 같은 지표는 다른 개념과 섞어 출제되기 쉽다.

## 1. 운영체제의 목적

- 처리 능력 향상
- 반환 시간 단축
- 사용 가능도 향상
- 신뢰도 향상

## 2. 운영체제의 주요 기능

| 기능 | 설명 |
|---|---|
| 프로세스 관리 | 생성, 종료, 스케줄링 |
| 기억장치 관리 | 할당, 회수, 보호 |
| 파일 관리 | 저장, 검색, 권한 |
| 장치 관리 | 입출력 제어 |

## 3. 프로세스와 스레드

### 프로세스

- 실행 중인 프로그램
- 독립된 자원(메모리 공간)을 가진다

### 스레드

- 프로세스 내부의 실행 흐름 단위
- 같은 프로세스 안에서는 코드/데이터/힙을 공유한다

| 항목 | 프로세스 | 스레드 |
|---|---|---|
| 자원 소유 | 독립적 | 프로세스 자원 공유 |
| 생성 비용 | 큼 | 작음 |
| 문맥 교환 비용 | 큼 | 상대적으로 작음 |

## 4. 프로세스 상태 전이

| 상태 | 의미 |
|---|---|
| 생성 | 작업 생성 직후 |
| 준비 | CPU 할당 대기 |
| 실행 | CPU 사용 중 |
| 대기 | 입출력 등 이벤트 대기 |
| 종료 | 작업 완료 |

### 대표 전이

- 생성 -> 준비
- 준비 -> 실행
- 실행 -> 대기
- 대기 -> 준비
- 실행 -> 종료

## 5. 성능 지표

| 지표 | 의미 |
|---|---|
| 처리 능력 | 단위 시간당 처리 작업 수 |
| 반환 시간 | 제출부터 완료까지 걸린 총 시간 |
| 대기 시간 | 준비 큐에서 기다린 시간 |
| 응답 시간 | 첫 반응이 나올 때까지 시간 |

## 6. 예제

### 예제 1: 프로세스/스레드 구분

- 브라우저 프로그램 실행 -> 프로세스
- 브라우저 안에서 탭 렌더링, 네트워크 처리 흐름 -> 스레드

### 예제 2: 시간 지표 해석

- 작업 실행시간 5초, 준비 큐 대기시간 3초라면
- 반환시간 = 8초

## 7. 자주 틀리는 함정

- 반환시간과 대기시간을 같은 개념으로 본다
- 프로세스와 프로그램을 동일시한다
- 스레드가 완전히 독립 자원을 가진다고 오해한다

## 8. 시험장에서 푸는 순서

1. 문제에서 묻는 것이 개념인지 시간 지표인지 구분한다.
2. 프로세스/스레드는 자원 공유 여부로 구분한다.
3. 상태 전이는 순서대로 화살표로 적는다.
4. 시간 지표는 실행/대기/응답을 분리해서 계산한다.

## 한 페이지 요약

- 운영체제는 시스템 자원을 관리한다.
- 프로세스는 실행 중인 프로그램, 스레드는 그 내부 실행 단위다.
- 프로세스 상태는 생성, 준비, 실행, 대기, 종료로 본다.
- 반환시간 = 실행시간 + 대기시간으로 자주 해석한다.
