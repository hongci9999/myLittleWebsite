# ORDER BY 절

## 핵심 요약

**ORDER BY**는 **(6)번·마지막** 정렬. **SELECT 이후** → **별칭** 가능, **GROUP BY 이후** → **집계함수**(`AVG` …)로 정렬 가능. **ASC** 기본, **DESC** 내림차순. **NULL**: Oracle=**최대**(ASC 맨 뒤), SQL Server=**최소**(ASC 맨 앞).

## 왜 중요한가

- 정렬 방향·다중 키·별칭/순번은 **빈출**이다.
- **WHERE**는 별칭 불가, **ORDER BY**는 가능 — **실행 순서** 차이를 묻는다.
- FWGHSO에서 **ORDER BY가 항상 마지막**임을 기억해야 한다.

## 1. ORDER BY 절

| 항목 | 내용 |
|------|------|
| **역할** | 쿼리 **결과**를 **정렬** |
| **위치** | 문장 **맨 끝**(작성상), 실행 **(6) ORDER BY** |
| **기본값** | **ASC** (오름차순) — **생략 가능** |

### 키워드

| 키워드 | 의미 |
|--------|------|
| **ASC** | **오름차순** (작은 값 → 큰 값) — **기본값** |
| **DESC** | **내림차순** (큰 값 → 작은 값) |

```sql
ORDER BY 급여          -- ASC와 동일
ORDER BY 급여 ASC
ORDER BY 급여 DESC
```

---

## 2. 예 1 — 급여 오름차순 (ASC 기본)

### \<직원\> (일부)

| 사번 | 급여 |
|------|------|
| 101 | 1300 |
| 102 | 1100 |
| 103 | 1200 |
| 104 | 1500 |
| 105 | 1100 |

```sql
SELECT 사번, 급여
  FROM 직원
 ORDER BY 급여;
```

| 사번 | 급여 |
|------|------|
| 102 | 1100 |
| 105 | 1100 |
| 103 | 1200 |
| 101 | 1300 |
| 104 | 1500 |

→ **ASC 생략** = **오름차순**. 급여 같으면 **사번** 등 나머지 순서는 DB 기본(문항에서 명시할 때만 따름).

---

## 3. 예 2 — 별칭 + DESC

```sql
SELECT 사번, 급여 AS 연봉
  FROM 직원
 ORDER BY 연봉 DESC;
```

| 사번 | 연봉 |
|------|------|
| 104 | 1500 |
| 101 | 1300 |
| 103 | 1200 |
| 102 | 1100 |
| 105 | 1100 |

| 포인트 | 내용 |
|--------|------|
| **DESC** | **내림차순** |
| **별칭 `연봉`** | `ORDER BY`는 **SELECT 이후** → **별칭 사용 가능** |
| **WHERE와 대비** | `WHERE 연봉 > 1000` 같은 **별칭 조건은 불가**(실행 순서) |

---

## 4. 예 3 — 다중 컬럼 정렬

### \<직원\> (보너스 포함)

| 사번 | 급여 | 보너스 |
|------|------|--------|
| 101 | 1300 | 100 |
| 102 | 1100 | 20 |
| 103 | 1200 | 0 |
| 104 | 1500 | 30 |
| 105 | 1100 | 50 |

```sql
SELECT 사번, 급여 AS 연봉, 보너스
  FROM 직원
 ORDER BY 급여 DESC, 보너스 ASC;
```

| 사번 | 연봉 | 보너스 |
|------|------|--------|
| 104 | 1500 | 30 |
| 101 | 1300 | 100 |
| 103 | 1200 | 0 |
| 102 | 1100 | 20 |
| 105 | 1100 | 50 |

| 순서 | 규칙 |
|------|------|
| 1 | **급여 DESC** — 큰 급여부터 |
| 2 | 급여 **같으면** **보너스 ASC** — 1100인 102(20) → 105(50) |

### 컬럼 번호로 정렬

`SELECT` 목록에서의 **위치(순번)** 로 지정 가능.

```sql
-- 위 쿼리와 동일 의미 (사번=1, 연봉=2, 보너스=3)
ORDER BY 2 DESC, 3 ASC;
```

| SELECT 순번 | 컬럼 |
|-------------|------|
| **1** | 사번 |
| **2** | 급여 (별칭 연봉) |
| **3** | 보너스 |

> `ORDER BY 급여 DESC, 보너스 ASC` ↔ **`2 DESC, 3 ASC`**

---

## 5. 예 4 — GROUP BY + ORDER BY (집계값 내림차순)

\<직원\> 전체(10행)는 [07_GROUP_BY와_HAVING](./07_GROUP_BY와_HAVING.md) §2 참고.

```sql
SELECT 부서, AVG(급여) AS 평균급여
  FROM 직원
 GROUP BY 부서
 ORDER BY AVG(급여) DESC;
```

| 단계 | 내용 |
|------|------|
| **GROUP BY** | 부서별 평균 — 인사 **1000**, 개발 **1200**, 기획 **1100** |
| **ORDER BY** | **평균급여 내림차순** |

| 부서 | 평균급여 |
|------|----------|
| 개발 | 1200 |
| 기획 | 1100 |
| 인사 | 1000 |

→ 부서별 **평균급여**를 구한 뒤, **평균급여 기준 내림차순** 정렬. `ORDER BY 평균급여 DESC`·`ORDER BY 2 DESC`도 동일 의미 가능.

## 6. ORDER BY 절 특징

| # | 특징 |
|---|------|
| 1 | **SELECT 절 이후** 실행 → **SELECT에서 정의한 별칭(ALIAS)** 으로 정렬 **가능** |
| 2 | **GROUP BY 절 이후** 실행 → **집계함수**(AVG, SUM, COUNT …)로 정렬 **가능** |

| 절 | 실행 순서 | ORDER BY에서 |
|----|-----------|--------------|
| SELECT | (5) | **별칭** 사용 O |
| GROUP BY | (3) | **집계 결과** 정렬 O |
| WHERE | (2) | 별칭·집계 **X** |

---

## 7. 데이터 형에 따른 정렬 순서 (오름차순 ASC 기준)

| 형 | ASC 정렬 |
|----|----------|
| **숫자형** | **작은 수 → 큰 수** |
| **문자형** | **사전(가나다·알파벳) 순** |
| **날짜형** | **과거 → 미래** (이른 날짜 → 늦은 날짜) |

### NULL 정렬 (DBMS별 — 시험 빈출)

| DBMS | NULL 취급 | ASC일 때 NULL 위치 | DESC일 때 NULL 위치 |
|------|-----------|-------------------|---------------------|
| **Oracle** | **가장 큰 값** | **맨 뒤** | **맨 앞** |
| **SQL Server** | **가장 작은 값** | **맨 앞** | **맨 뒤** |

> 문항에 **Oracle / SQL Server** 가 명시되면 위 표대로 선택.

---

## 8. WHERE·SELECT·ORDER BY 비교

| 절 | 실행 순서 | SELECT 별칭 | 집계함수 |
|----|-----------|-------------|----------|
| **WHERE** | (2) | **불가** | **불가** |
| **SELECT** | (5) | 정의 | 표현 |
| **ORDER BY** | **(6) 마지막** | **가능** | GROUP BY 쿼리에서 **가능**(별칭·집계) |

---

## 9. 시험 포인트 / 함정

| 구분 | 내용 |
|------|------|
| 실행 순서 | **ORDER BY = 6번·마지막** (FWGHSO **O**) |
| ASC | **기본값**, 생략 시 오름차순 |
| DESC | 내림차순 |
| 별칭 | **SELECT 이후** → ORDER BY에 **별칭** O |
| 집계 | **GROUP BY 이후** → `ORDER BY AVG(급여)` O |
| GROUP BY+ORDER BY | 그룹 결과를 **집계값**으로 정렬 |
| NULL Oracle | ASC **맨 뒤**, DESC **맨 앞** (NULL=최대) |
| NULL SQL Server | ASC **맨 앞**, DESC **맨 뒤** (NULL=최소) |
| 다중 정렬 | 앞 컬럼 우선, 같으면 다음 컬럼 |
| 컬럼 번호 | `ORDER BY 2 DESC, 3 ASC` |
| 함정 | `WHERE`에 **별칭·집계** → **불가** |
| 함정 | 작성 순서 = 실행 순서 → **오답** |
