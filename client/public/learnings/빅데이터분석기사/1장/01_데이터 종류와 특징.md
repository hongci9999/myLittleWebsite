# 데이터의 분류 및 유형별 특성 이해

## 📌 핵심 요약
데이터는 환원 여부에 따라 **가역(Reversible)** 및 **불가역(Irreversible)** 데이터로, 형태에 따라 **정형·반정형·비정형** 데이터로 분류됩니다. 또한 조직 내부 생성 여부에 따라 **내부 데이터**와 **외부 데이터**로 나뉘며, 각 유형은 수집 난이도와 처리 방식에서 뚜렷한 차이를 보입니다.

---

## 📝 상세 정리

### 1. 가역 데이터 vs 불가역 데이터 (Reversibility)
데이터의 원본 환원 가능 여부와 추적성에 따른 분류입니다.

| 구분 | 가역 데이터 (Reversible Data) | 불가역 데이터 (Irreversible Data) |
| :--- | :--- | :--- |
| **환원성** | 원본으로 일정 수준 환원 가능 | 원본 환원 불가능 |
| **추적성** | 추적 가능 | 추적 불가능 |
| **의존성** | 원본 데이터에 의존 | 독립된 새로운 객체 |
| **갱신 여부** | 갱신 가능 | 갱신 불가 |
| **관계** | $1:1$ 관계 | $1:n$ 관계 |
| **처리 과정** | 탐색(Search) 중심 | 결합(Combination) 중심 |
| **주요 분야** | 데이터 웨어하우스 (Data Warehouse) | 데이터 전처리 (Data Preprocessing) |

### 2. 정량적 데이터 vs 정성적 데이터 (Quantitative vs Qualitative)
데이터의 표현 방식과 분석 용이성에 따른 분류입니다.

| 구분 | 정량적 데이터 (Quantitative Data) | 정성적 데이터 (Qualitative Data) |
| :--- | :--- | :--- |
| **형태** | 주로 숫자, 수치 | 텍스트, 함축적 의미 |
| **의미 부여** | 여러 요소의 결합으로 의미 부여 | 객체 하나가 함축된 의미 내포 |
| **특성** | 객관적 (Objective) | 주관적 (Subjective) |
| **예시** | DBMS 내 수치 데이터 등 | 웹사이트, 플랫폼 등 외부 텍스트 |
| **분석** | 통계 분석 용이 | 통계 분석 어려움 |

### 3. 구조에 따른 데이터 분류 (Structure)
데이터가 가진 형식의 정형화 정도에 따른 분류입니다.

* **정형 데이터 (Structured Data)**
    * **정의:** 정해진 형식과 구조를 가진 데이터입니다.
    * **특징:** 정형화된 스키마(Schema)가 존재하며, 연산이 가능합니다.
    * **예시:** 관계형 데이터베이스(RDB)의 테이블, 스프레드시트 등.
* **반정형 데이터 (Semi-Structured Data)**
    * **정의:** 형식과 구조가 유연한 데이터입니다.
    * **특징:** 메타구조(Meta-structure)를 포함하고 있으며, 직접적인 연산은 불가능합니다.
    * **예시:** HTML, XML, JSON 등.
* **비정형 데이터 (Unstructured Data)**
    * **정의:** 구조가 정해지지 않은 데이터입니다.
    * **특징:** 일정한 구조가 없으며 연산이 불가능합니다.
    * **예시:** 동영상, 이미지, 음성 데이터, 소셜 미디어 텍스트 등.

### 4. 데이터 소재지에 따른 분류 (Source Location)

#### 4.1 내부 데이터 (Internal Data)
* **정의:** 조직 내부의 서비스 시스템, 마케팅 관련 시스템 등에서 생성되는 데이터입니다.
* **분석 가치:** 보통 수준입니다.
* **수집 방식:** 내부 조직 간의 협의를 통해 수집됩니다.
* **데이터 형태:** 대부분 **정형 데이터(Structured Data)** 형태입니다.
* **관리 특성:**
    * 담당자와의 협의가 원활하며 수집 비용이 낮습니다.
    * 서비스 수명 주기(Service Life Cycle) 관리가 용이합니다.
    * 필요 데이터 목록 작성, 보안 요소 확인, 실시간 수집 지원이 주요 고려 사항입니다.

#### 4.2 외부 데이터 (External Data)
* **정의:** 소셜 데이터, 기관 데이터, 공공 데이터, M2M(Machine to Machine, 센서/장비), LOD(Linked Open Data) 등을 포함합니다.
* **분석 가치:** 매우 높음(High)으로 평가됩니다.
* **수집 방식:** 외부 조직 협약 또는 데이터 구매(Data Purchase) 등을 통해 수집됩니다.
* **데이터 형태:** 대부분 **반정형(Semi-structured)** 또는 **비정형(Unstructured)** 데이터입니다.
* **관리 특성:**
    * 외부 기관과의 의사소통이 어려울 수 있으며 데이터 가공 작업이 필수적입니다.
    * 데이터 수집 및 정제 난이도가 높습니다.
    * 제공 기업의 명칭, 제공 방법, 법률적 검토(저작권 등)가 필요합니다.
    * 일괄 전송(Batch) 또는 일정 주기 수립 등 수집 정책 수립이 중요합니다.

> [!INFO] 💡 AI 보충 설명
> **1. LOD (Linked Open Data):** 웹상의 데이터를 식별자(URI)로 연결하여 기계가 읽을 수 있는 형태로 공개한 데이터입니다. 외부 데이터 수집 시 상호운용성을 높이는 중요한 자원입니다.
> **2. M2M (Machine to Machine):** 사물지능통신으로도 불리며, 사람이 개입하지 않고 기기 간에 발생하는 데이터를 의미합니다. 최근 IoT(사물인터넷) 확산으로 비정형/반정형 외부 데이터의 핵심 소스가 되고 있습니다.
> **3. 데이터 스키마(Schema):** 데이터베이스의 구조와 제약 조건에 관해 기술한 메타데이터의 집합입니다. 정형 데이터는 이 스키마가 엄격히 고정되어 있으나, 반정형 데이터는 데이터 내부에 스키마 정보가 포함된 구조를 가집니다.
