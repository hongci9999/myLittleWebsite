# 2026-04 개발 로그

## 2026-04-06

### 오늘 한 일
- GeekNews RSS 학습 문서(`learnings/0024`)를 작성해 RSS 구조·수집 시 주의점을 정리했다.
- 메인 빈 슬롯 위젯을 `오늘의 GeekNews` 위젯으로 교체했다.
- 서버에 공개 API `GET /api/geeknews/latest`를 추가해 RSS를 읽고 최신 항목을 내려주도록 구성했다.
- 외부 RSS 지연/장애 영향을 줄이기 위해 서버 메모리 캐시(TTL) 로직을 적용했다.
- 클라이언트에서 최신 5개 항목을 불러와 로딩/에러/빈 상태를 포함해 렌더링하도록 연결했다.

### 배운 점 / 확인한 점
- GeekNews RSS는 사이트 메인 랭킹과 다를 수 있으므로, 위젯 제목을 "최신" 기준으로 명확히 표현하는 것이 혼동을 줄인다.
- 피드 파싱은 `item`과 `entry`를 함께 처리하면 RSS/Atom 변형에 더 안정적이다.

### 다음 할 일
- 위젯 항목 클릭 로그(analytics) 필요 여부를 검토한다.
- RSS 실패 시 마지막 성공 데이터의 디스크 캐시까지 확장할지 판단한다.
- 필요하면 `GEEKNEWS_RSS_URL`, `GEEKNEWS_RSS_CACHE_TTL_MS`를 `.env.example`에 문서화한다.

## 2026-04-01

### 오늘 한 일
- 메인 위젯 섹션을 세로 스택에서 벤토 그리드 구조로 전환할 수 있도록 `WidgetGrid`와 `main-widget-layout` 설정을 도입했다.
- 메인에서 AI 개요 위젯을 제거하고, `FavoriteLinksWidget` 중심으로 레이아웃과 스타일을 재정렬했다.
- 즐겨찾기 링크 위젯 크기를 `5x4`로 조정하고, 우측에 `7x7` 빈 슬롯(`EmptyPlaceholderWidget`)을 추가했다.
- 링크 위젯 헤더 높이/타이포를 줄이고 회색 톤으로 조정해 정보 밀도를 낮췄다.
- 링크 hover 시 제목 색이 과도하게 바뀌던 동작을 제거했다.
- 빈 슬롯 문구 인코딩 깨짐 이슈를 확인하고 문구를 복구했다.

### 배운 점 / 확인한 점
- `col-span`과 `row-span`은 같은 "칸 수"처럼 보여도 기준이 다르다. `row-span`은 `grid-auto-rows` 단위에 따라 실제 픽셀 높이가 결정된다.
- CSS 계산식만으로는 브라우저/상황별 편차가 생길 수 있어, 실제 컨테이너 너비를 기준으로 셀 크기를 계산하는 방식이 안정적이었다.
- PowerShell 파일 생성 시 인코딩 영향으로 한글 문자열이 `??`로 저장될 수 있어, 생성/저장 방식에 주의가 필요하다.

### 막혔던 부분
- 행 높이가 의도보다 짧게 보이는 현상이 반복되어, 단순 Tailwind arbitrary value 계산식만으로는 원하는 비율 유지가 어려웠다.
- 한글 텍스트가 파일 저장 과정에서 깨져 보이는 문제가 발생했다.

### 해결 방법
- `WidgetGrid`에 `ResizeObserver`를 적용해 컨테이너 폭 기반 셀 단위를 계산하고 `grid-auto-rows`에 반영했다.
- 깨진 한글 문구는 파일을 직접 수정해 정상 텍스트로 복구했다.

### 다음 할 일
- 빈 슬롯을 실제 위젯(날씨/운세 등)으로 대체할지 결정한다.
- `main-widget-layout`에서 span 조합(`5+7`, `6+6`)을 시나리오별로 정리한다.
- 메인 위젯 섹션 시각 완성도(여백, 대비, 포커스)를 한 번 더 폴리시한다.
