# 웹사이트 구조·UX 용어 정리

날짜: 2026-02-21

## 요약

웹사이트 개발 시 알아야 할 페이지 구조, UX 패턴, 설계 단계 용어를 정리했다. 히어로, 랜딩, CTA 같은 기본 용어부터 시선 패턴, 정보 구조, 자주 쓰는 섹션·UI 패턴까지 포함한다.

---

## 1. 기본 페이지 구조 용어

| 용어 | 의미 | 활용 |
|------|------|------|
| **Hero Section (히어로 섹션)** | 페이지 최상단, 첫 화면 전체를 차지하는 임팩트 있는 메인 영역 | 헤드라인, 부제목, CTA 버튼, 배경 이미지/비디오 |
| **Landing Page (랜딩 페이지)** | 특정 목적(가입·구매·신청 등)에 최적화된, 외부에서 바로 들어오는 페이지 | 광고·검색 등에서 클릭 시 진입, 전환(conversion) 최적화 |
| **Above the fold** | 스크롤 없이 보이는 첫 화면 영역 | 가장 중요한 메시지·CTA 배치 |
| **Fold** | 화면을 잘랐을 때 보이지 않는 경계선 | 디바이스마다 다름(신문 접힌 위쪽에서 유래) |
| **Section** | 페이지 내 역할이 구분된 블록 단위 | Hero, Features, Testimonials 등 |
| **Navigation (Nav)** | 상단·사이드 등에 있는 메뉴·링크 모음 | 헤더(Header), 푸터(Footer) |
| **Footer** | 페이지 하단의 고정 영역 | 회사 정보, 링크, 저작권, SNS |

### Hero Landing Page

히어로 섹션을 중심으로 한 랜딩 페이지. 첫 화면에 임팩트 있는 메시지와 CTA, 스크롤 시 Features·Pricing 등이 이어지는 형태.

---

## 2. 전환·마케팅 관련

| 용어 | 의미 |
|------|------|
| **CTA (Call-to-Action)** | 사용자 행동을 유도하는 버튼·링크 ("무료 체험", "구매하기" 등) |
| **Conversion (전환)** | 목표 행동의 완료 (가입, 결제, 문의 등) |
| **Funnel (퍼널)** | 방문 → 관심 → 구매/가입까지의 단계 흐름 |
| **Bounce Rate (이탈률)** | 한 페이지만 보고 나가는 비율 |

---

## 3. 시선 흐름 패턴 (스캐닝 패턴)

### F-패턴 (F-Pattern)

- **특징**: 왼쪽 상단에서 수평으로 → 내려가며 다시 수평으로 → 전체가 F자 형태
- **적합한 페이지**: 블로그, 뉴스, 검색 결과, 긴 스크롤 페이지
- **디자인 원칙**: 중요 정보를 **상단과 왼쪽**에 배치

### Z-패턴 (Z-Pattern)

- **특징**: 왼쪽 상단 → 오른쪽 → 대각선 아래 → 오른쪽 (Z자)
- **적합한 페이지**: 랜딩 페이지, 포트폴리오, 광고 배너
- **디자인 원칙**: 로고는 왼쪽 상단, **CTA는 오른쪽 하단** 근처에 배치

> 모든 사용자가 동일하게 따르지 않으므로, 콘텐츠 유형에 맞게 패턴을 선택하고 직관적인 디자인 유지가 중요하다.

## 4. 설계·구조 단계 용어

### Information Architecture (IA, 정보 구조)

웹사이트/앱 내 정보를 논리적으로 조직·구조화하는 것. 사용자가 원하는 정보를 빠르게 찾게 하고 UX를 높인다.

**주요 요소:**
- **네비게이션 시스템**: 상단 메뉴, 사이드바, 푸터
- **카테고리 분류**: 콘텐츠를 논리적으로 그룹화 (예: 의류 > 남성복 > 셔츠)
- **사이트맵**: 전체 페이지 구조를 한눈에 보여주는 구조도
- **콘텐츠 흐름**: 사용자가 정보를 소비하는 순서

### Sitemap (사이트맵)

- **개념**: 웹사이트 페이지의 구조화된 목록/구조도
- **역할**: SEO 지원(검색 엔진이 링크를 따라가도록), 사용자에게 사이트 개요 제공

### Wireframe (와이어프레임)

- **개념**: 레이아웃·구조를 단순한 선과 박스로 표현한 스케치
- **목적**: 콘텐츠 배치·흐름 확인, 개발 전 논의용

### Mockup / Prototype

- **Mockup**: 색상·타이포·이미지가 반영된 정적인 디자인
- **Prototype**: 클릭·스크롤 등 상호작용이 들어간 시뮬레이션

## 5. 자주 쓰는 페이지 섹션 유형

| 섹션 | 역할 | 예시 |
|------|------|------|
| **Features** | 주요 기능을 카드 등으로 소개 | 빠른 속도, 반응형, 보안 등 |
| **Services** | 제공 서비스 목록 | 상담, 개발, 유지보수 |
| **Testimonials** | 고객 후기·리뷰 | 인용문, 별점, 이름·직함 |
| **Pricing** | 요금제 정보 | 플랜 비교, 월/연 구독 |
| **FAQ** | 자주 묻는 질문 | 아코디언 형태 |
| **About / Team** | 회사·팀 소개 | 비전, 팀원 소개 |
| **Contact** | 연락처·문의 폼 | 주소, 이메일, 문의 양식 |
| **CTA Section** | 행동 유도 (가입, 체험 등) | 버튼·간단한 설명 |

## 6. UI 패턴 용어

| 용어 | 의미 | 활용 |
|------|------|------|
| **Card** | 콘텐츠를 박스 단위로 정렬 | Features, 블로그 포스트 목록 |
| **Carousel / Slider** | 이미지·콘텐츠 슬라이드 | 갤러리, 후기 슬라이더 |
| **Tabs** | 여러 콘텐츠를 탭으로 구분 | 상세 설명, 설정 메뉴 |
| **Accordion** | 클릭 시 펼쳐지는 아코디언 | FAQ |
| **Skeleton Screen** | 로딩 중 보여주는 플레이스홀더 | 스켈레톤 UI |
| **Pagination** | 여러 페이지로 나눈 목록 | 게시판, 검색 결과 |
| **Progressive Disclosure** | 처음에는 축소, 필요 시 확장 | 복잡한 폼, 숨겨진 네비게이션 |
| **Badge** | 신규·할인 등 라벨 표시 | 알림, 카테고리 태그 |

---

## 7. 성능·로딩

| 용어 | 의미 |
|------|------|
| **Lazy Loading** | 필요할 때까지 리소스 로딩을 지연 |
| **Progressive Loading** | 점진적으로 콘텐츠를 로드 |
| **Throttle / Debounce** | 함수 실행 빈도 제한 (스크롤·입력 이벤트 등) |
| **CLS (Cumulative Layout Shift)** | 레이아웃 변화를 측정하는 Core Web Vital 지표 |

---

## 8. 접근성(Accessibility)

| 용어 | 의미 |
|------|------|
| **ARIA** | 스크린 리더 등 보조기기에 정보 전달용 HTML 속성 |
| **Keyboard Navigation** | 키보드만으로 조작 가능한 UI |
| **Screen Reader** | 화면 내용을 음성으로 읽어주는 보조 기술 |
| **Live Regions** | 동적 콘텐츠 변경을 화면에 공지하는 ARIA 기능 |

---

## 9. 기술·구조 관련

| 용어 | 의미 |
|------|------|
| **Semantic HTML** | 내용의 의미를 반영한 HTML 태그 (`header`, `main`, `article`, `section` 등) |
| **DOM (Document Object Model)** | 문서를 트리 구조로 표현하는 프로그래밍 인터페이스 |
| **Design Tokens** | 색상·간격·폰트 등을 변수로 관리해 디자인 시스템 일관성 유지 |
| **Responsive Design** | 다양한 화면 크기에 맞게 레이아웃이 변하는 디자인 |
