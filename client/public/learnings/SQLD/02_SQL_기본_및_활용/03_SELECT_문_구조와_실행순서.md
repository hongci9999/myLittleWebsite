# SELECT 문 구조와 실행 순서

## 핵심 요약

**SELECT**는 DB에서 데이터를 **조회**할 때 쓰는 **가장 기본적인** DML 명령이다. **작성 순서**와 **논리적 실행 순서**가 다르다 — DB는 **FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY** 순으로 처리한다(암기: **FWGHSO**, 「**공격수(FW)가 해 써 오!**」). `SELECT`는 맨 위에 쓰지만 **5번째**에 실행된다.

## 왜 중요한가

- WHERE vs HAVING, GROUP BY 유무 문제의 **전제**이다.
- 실행 순서를 모르면 **집계·필터·별칭** 오류를 이해하기 어렵다.
- [02\_관계대수](./02_관계대수_연산자.md)의 σ(선택)·π(프로젝션)가 **어느 단계**에 해당하는지 연결된다.

## 1. SELECT 문이란

| 항목 | 내용                                              |
| ---- | ------------------------------------------------- |
| 분류 | **DML** (또는 **DQL**로 분리하기도 함)            |
| 역할 | 릴레이션에서 **원하는 데이터(행·열)** 를 **조회** |
| 특징 | 여러 **절(CLAUSE)** 을 조합해 사용                |

---

## 2. SELECT 문의 절(구조)

| 절           | 역할 (한 줄)                                    |
| ------------ | ----------------------------------------------- |
| **SELECT**   | **조회할 컬럼**(표현식, 집계, DISTINCT 등) 지정 |
| **FROM**     | 데이터를 가져올 **테이블·뷰**(조인 포함)        |
| **WHERE**    | **행(튜플)** 단위 **조건** 필터                 |
| **GROUP BY** | 지정 컬럼 기준으로 **그룹** 묶기                |
| **HAVING**   | **그룹**에 대한 **조건** 필터                   |
| **ORDER BY** | 결과 **정렬** 기준                              |

> 교재·시험 범위에 따라 **JOIN**, **DISTINCT** 등이 별도 절·키워드로 추가됨 — 다음 문서에서 확장.

## 3. 논리적 실행 순서 (중요)

**DB 엔진이 처리하는 순서** — **작성 순서와 다름**

| 순서    | 절           | 설명                                       |
| ------- | ------------ | ------------------------------------------ |
| **(1)** | **FROM**     | **어떤 테이블**에서 데이터를 가져올지 결정 |
| **(2)** | **WHERE**    | 조건에 맞는 **행**만 필터링                |
| **(3)** | **GROUP BY** | 데이터를 **그룹**으로 묶음                 |
| **(4)** | **HAVING**   | **그룹**에 대한 조건 필터링                |
| **(5)** | **SELECT**   | 조회할 **컬럼**·집계 결과 확정             |
| **(6)** | **ORDER BY** | **정렬** 기준 적용                         |

### 암기: FWGHSO

| 글자  | 절           |
| ----- | ------------ |
| **F** | **FROM**     |
| **W** | **WHERE**    |
| **G** | **GROUP BY** |
| **H** | **HAVING**   |
| **S** | **SELECT**   |
| **O** | **ORDER BY** |

```mermaid
flowchart LR
  F[1 FROM]
  W[2 WHERE]
  G[3 GROUP BY]
  H[4 HAVING]
  S[5 SELECT]
  O[6 ORDER BY]
  F --> W --> G --> H --> S --> O
```

### 작성 순서 vs 실행 순서

| 구분                | 순서                                                     |
| ------------------- | -------------------------------------------------------- |
| **SQL에 적는 순서** | SELECT → FROM → WHERE → … → ORDER BY                     |
| **실행(논리) 순서** | **FROM → WHERE → GROUP BY → HAVING → SELECT → ORDER BY** |

→ **SELECT가 맨 위**에 있어도 **실행은 5번째**

---

## 4. WHERE vs HAVING (실행 순서로 이해)

| 절         | 단계                      | 필터 대상 | 예                     |
| ---------- | ------------------------- | --------- | ---------------------- |
| **WHERE**  | **(2)** — GROUP BY **전** | **행**    | `WHERE 학점 >= 3`      |
| **HAVING** | **(4)** — GROUP BY **후** | **그룹**  | `HAVING COUNT(*) >= 2` |

→ 집계 함수 조건은 원칙적으로 **HAVING** (WHERE에 집계 넣기 **불가** — 표준)

---

## 5. 관계대수와의 연결

| 실행 단계        | 관계대수 관점                                                 |
| ---------------- | ------------------------------------------------------------- |
| FROM (+ JOIN)    | 릴레이션·**⋈**                                                |
| WHERE            | **σ** (선택)                                                  |
| SELECT (열 목록) | **π** (프로젝션) — 실행 순서상 GROUP BY·HAVING 이후에 열 확정 |

---

## 6. 시험 포인트 / 함정

| 구분      | 내용                                                                                 |
| --------- | ------------------------------------------------------------------------------------ |
| 실행 순서 | **FWGHSO** — FROM이 **1번**, SELECT **5번**                                          |
| 암기 문장 | 공격수(FW)가 해 써 오                                                                |
| WHERE     | **행** 필터, GROUP BY **이전**                                                       |
| HAVING    | **그룹** 필터, GROUP BY **이후**                                                     |
| ORDER BY  | **마지막(6번)**                                                                      |
| 함정      | 적는 순서 = 실행 순서 → **오답**                                                     |
| 함정      | HAVING을 WHERE보다 **먼저** 실행 → **오답**                                          |
| 함정      | SELECT 절 별칭을 같은 쿼리 **WHERE**에서 사용 — DBMS에 따라 **불가**(실행 순서 때문) |
