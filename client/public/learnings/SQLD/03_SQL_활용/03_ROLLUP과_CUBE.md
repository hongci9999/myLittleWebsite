# ROLLUP · CUBE · GROUPING SETS

## 핵심 요약

**ROLLUP**: **계층** 소계(순서 중요). **CUBE**: **모든 조합**. **GROUPING SETS**: 그룹화 **조합을 직접 지정** — `((지역,상품), (상품), ())` 각 괄호가 **하나의 GROUP BY**. **`()`** = **전체 총계**. 결과 **NULL** = 그 열로 **묶지 않음**.

## 왜 중요한가

- 단순 `GROUP BY`보다 **소계·총계**를 한 번에 뽑는 **활용** 문항이다.
- ROLLUP **순서**·CUBE **조합 수**·NULL 의미가 **함정**이다.
- 일반 집계(`COUNT`·`SUM` …)는 [07_GROUP_BY와_HAVING](../02_SQL_기본/07_GROUP_BY와_HAVING.md) 선행.

> 집계함수: [05_SQL_함수_문자함수](../02_SQL_기본/05_SQL_함수_문자함수.md) §6

## 1. 그룹 함수 개요

| 구분 | 함수·연산 | 용도 |
|------|-----------|------|
| **일반 집계** | `COUNT`, `SUM`, `AVG`, `MIN`, `MAX` | `GROUP BY` 로 **한 단계** 요약 |
| **소계·다차원** | **ROLLUP**, **CUBE**, **GROUPING SETS** | 계층·전조합·**사용자 지정** 집계 |

→ `GROUP BY` 절에 **ROLLUP(…)** / **CUBE(…)** 를 사용한다.

---

## 2. 예제 테이블 \<판매\>

| 지역 | 상품 | 가격 |
|------|------|------|
| 서울 | A | 100 |
| 서울 | B | 200 |
| 서울 | A | 150 |
| 서울 | B | 100 |
| 부산 | A | 300 |
| 부산 | B | 200 |
| 부산 | A | 250 |
| 부산 | B | 150 |

| 지역·상품 | SUM(가격) |
|-----------|-----------|
| 서울·A | **250** |
| 서울·B | **300** |
| 부산·A | **550** |
| 부산·B | **350** |

---

## 3. ROLLUP

| 항목 | 내용 |
|------|------|
| **정의** | **계층 구조**로 그룹화 — **소계**·**총계** 자동 생성 |
| **순서** | `ROLLUP(컬럼1, 컬럼2)` — **컬럼1 → 컬럼2** 계층. **순서 바꾸면 결과 달라짐** |

### 그룹화 단계 (`ROLLUP(지역, 상품)`)

| 단계 | GROUP BY 수준 | 의미 |
|------|---------------|------|
| 1 | **지역, 상품** | 상세 합계 |
| 2 | **지역** | **지역 소계** (상품 NULL) |
| 3 | **(없음)** | **전체 총계** (지역·상품 NULL) |

```sql
SELECT 지역, 상품, SUM(가격) AS 총합
  FROM 판매
 GROUP BY ROLLUP(지역, 상품);
```

### 결과

| 지역 | 상품 | 총합 | 비고 |
|------|------|------|------|
| 서울 | A | 250 | 상세 |
| 서울 | B | 300 | 상세 |
| 서울 | NULL | **550** | **서울 소계** |
| 부산 | A | 550 | 상세 |
| 부산 | B | 350 | 상세 |
| 부산 | NULL | **900** | **부산 소계** |
| NULL | NULL | **1450** | **총계** |

→ **NULL** = 그 열로는 **그룹하지 않음** (해당 축 **전체 합**).

---

## 4. CUBE

| 항목 | 내용 |
|------|------|
| **정의** | **조합 가능한 모든 경우**에 대해 그룹화 — **다차원** 소계 |
| **부하** | ROLLUP보다 **큼** (그룹 **많음**) |

### 그룹화 단계 (`CUBE(지역, 상품)`)

| 단계 | GROUP BY 수준 |
|------|---------------|
| 1 | **지역, 상품** |
| 2 | **지역** 만 |
| 3 | **상품** 만 |
| 4 | **전체** |

```sql
SELECT 지역, 상품, SUM(가격) AS 총합
  FROM 판매
 GROUP BY CUBE(지역, 상품);
```

### 결과 (ROLLUP + 상품 축 소계)

| 지역 | 상품 | 총합 | 비고 |
|------|------|------|------|
| 서울 | A | 250 | |
| 서울 | B | 300 | |
| 서울 | NULL | 550 | 지역 소계 |
| 부산 | A | 550 | |
| 부산 | B | 350 | |
| 부산 | NULL | 900 | 지역 소계 |
| NULL | A | **800** | **상품 A 전체** (250+550) |
| NULL | B | **650** | **상품 B 전체** (300+350) |
| NULL | NULL | **1450** | **총계** |

→ ROLLUP 결과에 **(NULL, A)**, **(NULL, B)** **상품별 총계**가 **추가**됨.

---

## 5. ROLLUP vs CUBE

| 구분 | ROLLUP | CUBE |
|------|--------|------|
| **구조** | **계층적**(순서 **중요**) | **모든 조합** |
| **2열 시 그룹** | (지역·상품), (지역), (전체) **3종+상세** | (지역·상품), (지역), (상품), (전체) |
| **상품만 소계** | **없음** | **있음** (NULL, A) 등 |
| **부하** | 상대적으로 **적음** | **큼** |
| **행 수(개념)** | 계층 **n+1** 단계 | **2^n** 조합(열 n개) |

```
ROLLUP(지역, 상품):  지역 → 상품 계층으로만 小계
CUBE(지역, 상품):   지역·상품·교차 小계 모두
GROUPING SETS:     필요한 小계만 골라 지정
```

---

## 6. GROUPING SETS

| 항목 | 내용 |
|------|------|
| **정의** | **그룹화할 대상**을 사용자가 **직접 지정** |
| **문법** | `GROUP BY GROUPING SETS ( (열…), (열…), () )` |
| **`()`** | **모든 행**에 대한 **전체 그룹화(총계)** |
| **NULL** | 해당 그룹에 **참여하지 않은** 열 → **NULL** 표시 |

```sql
SELECT 지역, 상품, SUM(가격) AS 총합
  FROM 판매
 GROUP BY GROUPING SETS ( (지역, 상품), (상품), () );
```

### 지정한 3가지 GROUP BY

| # | 괄호 | 의미 |
|---|------|------|
| **(1)** | `(지역, 상품)` | **지역·상품**별 합계 |
| **(2)** | `(상품)` | **상품**별 전체 합계 — **지역 NULL** |
| **(3)** | `()` | **총계** — **지역·상품 NULL** |

### 결과

| 지역 | 상품 | 총합 | 구분 |
|------|------|------|------|
| 서울 | A | 250 | (1) |
| 서울 | B | 300 | (1) |
| 부산 | A | 550 | (1) |
| 부산 | B | 350 | (1) |
| NULL | A | **800** | (2) 상품 전체 |
| NULL | B | **650** | (2) 상품 전체 |
| NULL | NULL | **1450** | (3) **총계** |

→ **CUBE**와 달리 **(지역)만** 소계는 **없음** — **원하는 집계만** 나열.

### ROLLUP·CUBE·GROUPING SETS

| 구분 | 지정 방식 | 본 예에 (지역) 소계 |
|------|-----------|---------------------|
| **ROLLUP** | **계층** 자동 | **있음** (서울·부산 NULL 행) |
| **CUBE** | **전 조합** 자동 | **있음** + 상품·전체 |
| **GROUPING SETS** | **괄호마다** 수동 | **없음** (넣지 않으면 안 나옴) |

> `GROUPING SETS ((지역,상품), (지역), (상품), ())` 처럼 **ROLLUP/CUBE와 비슷한 결과**도 **직접** 만들 수 있음.

---

## 7. NULL 해석 (시험)

| 결과 NULL | 의미 |
|-----------|------|
| **상품 = NULL** (지역 있음) | 해당 **지역** 소계 |
| **지역 = NULL** (상품 있음) | 해당 **상품** 전체 소계 (CUBE) |
| **둘 다 NULL** | **전체 총계** |

> 실제 NULL 데이터와 구분이 필요하면 **`GROUPING()`** 함수 사용(DBMS 지원).

---

## 8. 시험 포인트 / 함정

| 구분 | 내용 |
|------|------|
| ROLLUP | **계층** 소계, **인수 순서** 중요 |
| CUBE | **모든 조합**, ROLLUP보다 **부하 큼** |
| GROUPING SETS | **괄호 하나** = **하나의 GROUP BY** |
| `()` | **전체 총계** |
| NULL | **소계·총계** 행 — “값 없음”이 아님 |
| 함정 | GROUPING SETS에 **(지역)만** 없으면 **지역 소계 행 없음** |
| 함정 | `ROLLUP(상품, 지역)` vs `ROLLUP(지역, 상품)` **결과 다름** |
| 함정 | CUBE vs GROUPING SETS — **자동 전조합** vs **수동 지정** |
