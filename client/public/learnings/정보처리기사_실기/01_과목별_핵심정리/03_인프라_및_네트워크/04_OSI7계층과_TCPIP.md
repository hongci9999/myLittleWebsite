# OSI 7계층과 TCP/IP

## 핵심 요약

네트워크 문제의 뼈대는 계층 구조다. 실기에서는 각 계층의 역할, 데이터 단위, 대표 장비, 대표 프로토콜을 짝지어 묻는 문제가 많다.

## 1. 프로토콜의 3요소

- 구문(Syntax)
- 의미(Semantics)
- 시간(Timing)

## 2. OSI 7계층

| 계층 | 역할 | 데이터 단위 |
|---|---|---|
| 응용 | 사용자 서비스 제공 | 데이터 |
| 표현 | 압축, 암호화, 코드 변환 | 데이터 |
| 세션 | 대화 제어, 동기화 | 데이터 |
| 전송 | 종단 간 전송, 오류/흐름 제어 | 세그먼트 |
| 네트워크 | 경로 설정, IP 주소 | 패킷 |
| 데이터링크 | 인접 노드 전송, MAC 주소 | 프레임 |
| 물리 | 실제 신호 전송 | 비트 |

## 3. 계층별 대표 장비

- 물리: 허브, 리피터
- 데이터링크: 브리지, 스위치
- 네트워크: 라우터

## 4. TCP/IP 4계층

| TCP/IP | OSI 대응 |
|---|---|
| 응용 | 응용/표현/세션 |
| 전송 | 전송 |
| 인터넷 | 네트워크 |
| 네트워크 액세스 | 데이터링크/물리 |

## 5. TCP와 UDP

| 항목 | TCP | UDP |
|---|---|---|
| 연결 방식 | 연결형 | 비연결형 |
| 신뢰성 | 높음 | 낮음 |
| 속도 | 상대적으로 느림 | 빠름 |
| 대표 특징 | 흐름/오류 제어 | 오버헤드 적음 |

## 6. 예제

### 예제 1: 계층 매핑

- HTTP -> 응용 계층
- TCP -> 전송 계층
- IP -> 네트워크 계층

### 예제 2: 장비 매핑

- 스위치 -> 데이터링크 계층
- 라우터 -> 네트워크 계층

## 7. 자주 틀리는 함정

- 세그먼트/패킷/프레임 단위를 뒤섞는다
- 스위치를 네트워크 계층 장비로 착각한다
- OSI 7계층과 TCP/IP 4계층을 1:1로 단순 대응한다

## 8. 시험장에서 푸는 순서

1. 문제에서 묻는 것이 계층 역할인지 프로토콜인지 장비인지 구분한다.
2. 데이터 단위를 함께 떠올린다.
3. TCP와 UDP는 연결성/신뢰성/속도로 비교한다.
4. TCP/IP는 OSI를 압축한 모델로 정리한다.

## 한 페이지 요약

- OSI 7계층은 물-데-네-전-세-표-응 순서로 외운다.
- 전송계층은 TCP/UDP, 네트워크계층은 IP가 핵심이다.
- 스위치는 데이터링크, 라우터는 네트워크 계층 장비다.
- TCP/IP 4계층은 OSI를 실무적으로 단순화한 구조다.
