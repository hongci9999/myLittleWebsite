# 회선 공유 기술 및 라우팅 프로토콜

## 📌 핵심 요약
본 강의는 효율적인 데이터 전송을 위한 **다중화(Multiplexing)** 및 **집중화(Concentration)** 기술, 공유 회선 점유를 위한 **매체 접근 제어(MAC)** 방식, 그리고 최적의 경로 설정을 위한 **라우팅 프로토콜(Routing Protocol)**의 종류와 특징을 다룹니다. 특히 네트워크 토폴로지의 구조적 특성과 패킷 교환 방식의 흐름 제어 기법을 체계적으로 정리합니다.

## 📝 상세 정리

## 1. 회선 공유 기술

### 1.1 다중화기 ([[MUX]], Multiplexer)
하나의 고속 전송 회선에 다수의 데이터 신호를 중복시켜 전송하는 방식으로 전송 효율을 높입니다.
- **특징**: 구조가 단순하고 저렴하며, 전송/수신 채널의 개수와 속도가 동일해야 합니다.
- **주요 방식**:
    - **[[FDM]] (주파수 분할 다중화)**: 대역폭을 다수의 작은 채널로 분할. 인접 채널 간 간섭 방지를 위해 **보호 대역(Guard Band)**이 필수적입니다.
    - **[[TDM]] (시간 분할 다중화)**: 타임 슬롯(Time Slot)을 나누어 할당합니다.
        - **STDM (동기식 TDM)**: 타임 슬롯을 정적으로 배분하여 단순하지만 낭비가 심함.
        - **ATDM (비동기식/통계적 TDM)**: 동적으로 슬롯을 할당하여 효율적이나 제어 기능이 필요함.
    - **[[CDM]] (코드 분할 다중화)**: 주파수와 시간을 공유하며 고유 코드를 부여. 보안성이 뛰어납니다.
    - **[[WDM]] (파장 분할 다중화)**: 광섬유에서 서로 다른 파장을 할당. 빛의 간섭이 없어 확장성이 좋습니다.

### 1.2 집중화기 ([[Concentrator]])
하나의 전송 회선을 다수의 단말기가 공유하되, 특정 시점에는 하나의 단말기가 회선을 독점하는 방식입니다.
- **특징**: 단말기의 합계 속도가 전송 회선 속도보다 큰 경우 사용하며 구조가 복잡하고 고가입니다.
- **매체 접근 제어(MAC) 방식**:
    - **예약 방식 (Reservation)**: 타임 슬롯을 사전에 예약.
    - **경쟁 방식 (Contention)**: 소량 전송에 적합하며 정해진 순서가 없음.
        - **[[ALOHA]]**: 최초의 무선 패킷망. ACK(긍정 응답)가 없으면 전송 포기.
        - **Slotted ALOHA**: ALOHA에 타임 슬롯 개념을 도입하여 효율 개선.
    - **[[CSMA]] (Carrier Sense Multiple Access)**: 회선 상태를 확인 후 접근.
        - **CSMA/CD**: 충돌 검출 및 재송신 기능 (유선 LAN, IEEE 802.3).
        - **CSMA/CA**: 충돌 회피를 위해 예비 신호 사용 (무선 LAN, IEEE 802.11).

### 1.3 [[IEEE 802]] 표준 규약
- **802.3**: CSMA/CD (이더넷)
- **802.4**: Token Bus
- **802.5**: Token Ring
- **802.8**: Fiber Optic LANs
- **802.9**: 음성/데이터 통합 LAN
- **802.11**: 무선 LAN (WLAN)
    - **11a/11g**: OFDM 방식 사용.
    - **11b**: HR-DSSS 방식 사용.
    - **11e**: QoS 및 보안 강화.

### 1.4 할당 및 선택 방식
- **순서적 할당 (Round Robin)**:
    - **Polling**: 중앙 컴퓨터가 단말기에 전송 데이터 유무를 확인.
    - **Selecting**: 중앙 컴퓨터가 수신 측 단말기를 선택하여 전송.
        - **Select-Hold**: 수신 준비 확인 후 전송 ([[BSC]] 프로토콜).
        - **Fast-Select**: 확인 없이 즉시 전송 ([[SDLC]] 프로토콜).
- **토큰 제어**: [[Token Ring]], [[Token Bus]], [[FDDI]] (광섬유 기반) 등.

## 2. 교환 회선 및 네트워크 토폴로지

### 2.1 네트워크 토폴로지 (Network Topology)
- **버스형 (Bus)**: 중앙 회선에 단말기 연결. 추가/삭제 용이하나 기밀 유지 어려움.
- **성형 (Star)**: 중앙 노드 중심 연결. 제어 용이하나 중앙 노드 고장 시 전체 마비.
- **환형 (Ring)**: 인접 노드 간 P2P 연결. 양방향 전송 가능하나 노드 추가 시 지연 발생.
- **계층형 (Tree)**: 트리 구조의 분산 처리 시스템. 특정 지점 병목 현상 발생 가능.
- **망형 (Mesh)**: 모든 노드를 상호 연결. 신뢰성 최상, 비용 최상. 가용 회선 수 계산식: $\frac{n(n-1)}{2}$

### 2.2 교환 방식
- **회선 교환 (Circuit Switching)**: 물리적 경로 독점. 지연이 없으나 접속 시간이 김.
- **축적 교환 (Store and Forward)**:
    - **메시지 교환**: 메시지 전체를 저장 후 전송. 지연 시간 매우 김.
    - **패킷 교환**: 데이터를 [[패킷(Packet)]] 단위로 분할 전송.
        - **가상 회선 (Virtual Circuit)**: 경로 설정 후 전송. 순서 보장.
        - **데이터그램 (Datagram)**: 독립적 경로 전송. 순서 보장 안 됨.

### 2.3 트래픽 제어 기법
- **흐름 제어 (Flow Control)**: 송수신 측 속도 차이 조절.
    - **Stop-and-Wait ARQ**: 하나 보내고 응답 대기.
    - **[[Sliding Window]]**: 응답 없이 여러 프레임을 연속 전송.
- **혼잡 제어 (Congestion Control)**: 네트워크 내 패킷 과부하 방지.

## 3. 라우팅 (Routing)

### 3.1 라우팅 개요
최적의 경로를 설정하는 기법으로 고정, 범람, 적응형 라우팅으로 나뉩니다.
- **[[IGP]] (내부 라우팅)**: AS(Autonomous System) 내부 통신.
    - **Distance Vector**: 거리와 방향 정보만 유지. (Bellman-Ford 알고리즘)
    - **Link State**: 전체 네트워크 지도를 유지. (Dijkstra/SPF 알고리즘)
- **[[EGP]] (외부 라우팅)**: AS 간 통신.

### 3.2 라우팅 프로토콜 종류
- **[[RIP]]**: 거리 벡터 방식. 최대 홉(Hop) 수 15 제한. 소규모 적합.
- **[[OSPF]]**: 링크 상태 방식. 대규모 네트워크, 빠른 수렴(Convergence), [[VLSM]] 지원.
- **[[IGRP]]/[[EIGRP]]**: Cisco 전용. EIGRP는 복합 메트릭과 빠른 순차 전달 지원.
- **[[BGP]]**: 경로 벡터 방식. AS 간 라우팅. 도달 가능성 중심. [[CIDR]] 지원.

> [!INFO] 💡 AI 보충 설명
> **1. CSMA/CD vs CSMA/CA**: 
> 유선 환경(CD)은 전압 변화로 충돌을 즉시 감지할 수 있지만, 무선 환경(CA)은 송신 중 충돌 감지가 어려워 사전에 'LBT(Listen Before Talk)'와 같은 회피 메커니즘을 사용합니다.
> **2. 컨버전스 타임(Convergence Time)**: 
> 네트워크에 변화가 생겼을 때 모든 라우터가 변경된 정보를 공유하고 라우팅 테이블을 업데이트하기까지 걸리는 시간을 의미합니다. OSPF가 RIP보다 압도적으로 빠릅니다.

## 🎯 빈출 키워드
- **[[다중화(Multiplexing)]]**: 하나의 회선을 여러 채널로 나누어 사용하는 기술.
- **[[CSMA/CD]]**: IEEE 802.3 표준의 유선 LAN 매체 접근 제어 방식.
- **[[패킷 교환(Packet Switching)]]**: 데이터를 패킷 단위로 나누어 전송하는 효율적인 교환 방식.
- **[[OSPF]]**: 대표적인 링크 상태(Link State) 기반 내부 라우팅 프로토콜.
- **[[흐름 제어(Flow Control)]]**: 슬라이딩 윈도우 등을 이용해 송수신 속도를 맞추는 기법.
