# 윈도우 함수 (Window Function)

## 핵심 요약

**윈도우 함수**는 **행 유지** + `OVER`. **비율**: **NTILE(n)** n등분, **CUME_DIST** 누적분포, **PERCENT_RANK** `(순위-1)/(전체-1)`, **RATIO_TO_REPORT** 부분/전체. **ROWS/RANGE**·LAG/LEAD·LAST_VALUE 포함.

## 왜 중요한가

- 집계·순위를 **행을 유지한 채** 표현한다.
- 세 순위 함수의 **동점 처리** 차이는 **필기 빈출**.
- `OVER (ORDER BY …)` 문법과 **GROUP BY 차이**를 묻는다.

> GROUP BY: [07_GROUP_BY와_HAVING](../02_SQL_기본/07_GROUP_BY와_HAVING.md)

---

## 1. 윈도우 함수란

| 항목 | 내용 |
|------|------|
| **정의** | **GROUP BY**와 달리 행을 **합치지 않고**, **행마다** 결과를 계산하는 함수 |
| **문법** | `함수() OVER ( … )` — **`OVER`** 키워드 **필수** |
| **용도** | 순위, 누적합, 이동 평균, 파티션별 집계 등 |

```sql
SELECT 이름, 점수,
       RANK() OVER (ORDER BY 점수 DESC) AS 등수
  FROM 성적;
```

| GROUP BY | 윈도우 함수 |
|----------|-------------|
| 그룹당 **1행**으로 **축소** | **모든 행 유지** + 계산 열 **추가** |
| `GROUP BY` | **`OVER`** |

---

## 2. 순위 함수 비교

| 함수 | 설명 | 동점 예 (점수 1위 2명) |
|------|------|------------------------|
| **ROW_NUMBER()** | 행마다 **고유 번호** (동점 무시) | **1, 2**, 3, 4 … |
| **RANK()** | 동점 **같은 순위**, 다음 순위 **건너뜀** | **1, 1**, **3**, 4 … |
| **DENSE_RANK()** | 동점 **같은 순위**, **연속** 순위 | **1, 1**, **2**, 3 … |

```
점수:  90  90  80  70
RANK:       1  1  3  4
DENSE:      1  1  2  3
ROW_NUM:    1  2  3  4  (동점 내 순서는 DB·정렬에 따름)
```

## 3. 예제 테이블 \<성적\>

| 이름 | 점수 |
|------|------|
| 철수 | 90 |
| 영희 | 80 |
| 민수 | 90 |
| 수지 | 70 |

`ORDER BY 점수 DESC` 기준.

---

## 4. RANK()

```sql
SELECT 이름, 점수,
       RANK() OVER (ORDER BY 점수 DESC) AS 등수
  FROM 성적;
```

| 이름 | 점수 | 등수 |
|------|------|------|
| 철수 | 90 | **1** |
| 민수 | 90 | **1** |
| 영희 | 80 | **3** |
| 수지 | 70 | **4** |

→ 1위 **2명** 후 **2위 없음** — **3위**부터 이어짐.

---

## 5. ROW_NUMBER()

```sql
SELECT 이름, 점수,
       ROW_NUMBER() OVER (ORDER BY 점수 DESC) AS 등수
  FROM 성적;
```

| 이름 | 점수 | 등수 |
|------|------|------|
| 철수 | 90 | **1** |
| 민수 | 90 | **2** |
| 영희 | 80 | **3** |
| 수지 | 70 | **4** |

→ 동점 90이어도 **1, 2** — **중복 순위 없음**. (동점 시 **어느 행이 1·2**인지는 **정렬 안정성**·DB에 따름)

---

## 6. DENSE_RANK()

```sql
SELECT 이름, 점수,
       DENSE_RANK() OVER (ORDER BY 점수 DESC) AS 등수
  FROM 성적;
```

| 이름 | 점수 | 등수 |
|------|------|------|
| 철수 | 90 | **1** |
| 민수 | 90 | **1** |
| 영희 | 80 | **2** |
| 수지 | 70 | **3** |

→ 1위 2명 후 **2위** — **순위 건너뛰지 않음**.

## 7. 집계 함수 + OVER (`SUM`)

`COUNT`·`SUM`·`AVG` 등 **집계 함수**도 **`OVER`** 와 함께 쓸 수 있다.

### \<판매\> 예제 테이블

| 지역 | 직원 | 판매액 |
|------|------|--------|
| 서울 | A | 100 |
| 경기 | B | 150 |
| 경기 | C | 200 |
| 서울 | D | 300 |
| 서울 | E | 400 |

| 합계 | 값 |
|------|-----|
| **전체** | **1150** |
| **서울** | **800** (100+300+400) |
| **경기** | **350** (150+200) |

---

### (1) `OVER()` — 전체 합계

```sql
SELECT 지역, 직원, 판매액,
       SUM(판매액) OVER() AS 총판매액
  FROM 판매;
```

| 지역 | 직원 | 판매액 | 총판매액 |
|------|------|--------|----------|
| (각 행) | … | … | **1150** (모든 행 동일) |

→ `OVER()` **비어 있음** = **전체**를 하나의 윈도우로 **합계**.

---

### (2) `PARTITION BY` — 그룹별 합계

```sql
SELECT 지역, 직원, 판매액,
       SUM(판매액) OVER(PARTITION BY 지역) AS 지역판매액
  FROM 판매
 ORDER BY 지역 DESC, 판매액;
```

| 지역 | 직원 | 판매액 | 지역판매액 |
|------|------|--------|------------|
| 서울 | … | … | **800** |
| 서울 | … | … | **800** |
| 경기 | … | … | **350** |
| 경기 | … | … | **350** |

→ **지역**마다 파티션 → 해당 지역 **판매액 합**이 **같은 값**으로 붙음. (GROUP BY와 달리 **행 수 유지**)

---

### (3) `ORDER BY` — 누적 합계

```sql
SELECT 지역, 직원, 판매액,
       SUM(판매액) OVER(ORDER BY 판매액) AS 누적판매액
  FROM 판매
 ORDER BY 판매액;
```

| 판매액 | 누적판매액 | 계산 |
|--------|------------|------|
| 100 | **100** | 100 |
| 150 | **250** | 100+150 |
| 200 | **450** | +200 |
| 300 | **750** | +300 |
| 400 | **1150** | +400 |

→ `OVER(ORDER BY 판매액)` — **정렬 순서대로** 행을 **누적**하며 합산 (**Running Total**).

---

## 8. OVER 절 옵션 정리

| `OVER` 구문 | 윈도우 범위 | 예 결과 |
|-------------|-------------|---------|
| `OVER()` | **전체** | 총판매액 **1150** |
| `OVER(PARTITION BY 지역)` | **지역별** | 서울 **800**, 경기 **350** |
| `OVER(ORDER BY 판매액)` | 정렬 기준 **누적** | 100→250→…→1150 |
| `OVER(PARTITION BY 지역 ORDER BY …)` | 지역 내 누적·순위 등 | (확장) |

### 출력 순서 주의

| 항목 | 내용 |
|------|------|
| **OVER** | **계산 방식**만 정함 — **결과 행 순서 보장 안 함** |
| **정렬 필요** | 화면에 순서대로 보려면 쿼리 **맨 끝 `ORDER BY`** 별도 지정 |
| 예 | `OVER(ORDER BY 판매액)` + `ORDER BY 판매액` (맨 아래) |

> ☞ **GROUP BY**: 그룹당 **1행**. **윈도우 집계**: **행 유지** + 집계 열 **추가**.

## 9. 행 간 값 참조 함수

**다른 행**의 값을 가져와 비교 — **셀프 조인** 없이 사용.

| 함수 | 설명 |
|------|------|
| **LAG()** | **이전 행**의 값 |
| **LEAD()** | **다음 행**의 값 |
| **FIRST_VALUE()** | 정렬 기준 **첫 번째** 값 |
| **LAST_VALUE()** | 정렬 기준 **마지막** 값 |

→ **`OVER` 안 `ORDER BY`** 로 “이전·다음” **순서** 결정. **`PARTITION BY`** 로 **그룹(파티션)별**로 끊어서 참조.

### 예 — `LAG()` · \<판매\>

**원본**

| 날짜 | 직원 | 매출 |
|------|------|------|
| 2020-04-01 | 홍길동 | 100 |
| 2020-04-02 | 홍길동 | 200 |
| 2020-04-03 | 홍길동 | 150 |
| 2020-04-01 | 임꺽정 | 300 |
| 2020-04-02 | 임꺽정 | 350 |

```sql
SELECT 날짜, 직원, 매출,
       LAG(매출) OVER (PARTITION BY 직원 ORDER BY 날짜) AS 이전매출
  FROM 판매;
```

### 결과

| 날짜 | 직원 | 매출 | 이전매출 |
|------|------|------|----------|
| 2020-04-01 | 홍길동 | 100 | **NULL** |
| 2020-04-02 | 홍길동 | 200 | **100** |
| 2020-04-03 | 홍길동 | 150 | **200** |
| 2020-04-01 | 임꺽정 | 300 | **NULL** |
| 2020-04-02 | 임꺽정 | 350 | **300** |

| 포인트 | 설명 |
|--------|------|
| **PARTITION BY 직원** | **직원별**로 LAG — 홍길동·임꺽정 **각각** 첫 날짜는 **이전 행 없음** → **NULL** |
| **ORDER BY 날짜** | **날짜 순** “이전” — 4/2의 이전매출 = 4/1 매출 |
### 예 — `LEAD()` · 같은 \<판매\>

```sql
SELECT 날짜, 직원, 매출,
       LEAD(매출) OVER (PARTITION BY 직원 ORDER BY 날짜) AS 다음매출
  FROM 판매;
```

| 날짜 | 직원 | 매출 | 다음매출 |
|------|------|------|----------|
| 2020-04-01 | 홍길동 | 100 | **200** |
| 2020-04-02 | 홍길동 | 200 | **150** |
| 2020-04-03 | 홍길동 | 150 | **NULL** |
| 2020-04-01 | 임꺽정 | 300 | **350** |
| 2020-04-02 | 임꺽정 | 350 | **NULL** |

→ **LAG**는 **이전**, **LEAD**는 **다음**. 파티션 **마지막 행** → **NULL**.

| LAG | LEAD |
|-----|------|
| 이전 행 값 | **다음 행** 값 |
| 첫 행 **NULL** | **마지막 행 NULL** |

---

### 예 — `FIRST_VALUE()` · 처음 매출

```sql
SELECT 날짜, 직원, 매출,
       FIRST_VALUE(매출) OVER (PARTITION BY 직원 ORDER BY 날짜) AS 처음매출
  FROM 판매;
```

| 날짜 | 직원 | 매출 | 처음매출 |
|------|------|------|----------|
| 2020-04-01 | 홍길동 | 100 | **100** |
| 2020-04-02 | 홍길동 | 200 | **100** |
| 2020-04-03 | 홍길동 | 150 | **100** |
| 2020-04-01 | 임꺽정 | 300 | **300** |
| 2020-04-02 | 임꺽정 | 350 | **300** |

→ **직원 파티션** 안에서 **날짜 순 첫 매출**이 **모든 행**에 동일하게 붙음.

| 함수 | 기본 윈도우에서 |
|------|----------------|
| **FIRST_VALUE** | 파티션 **시작~현재행** 중 **첫 값** → 보통 **첫 행 값**이 반복 |
| **LAST_VALUE** | 기본은 **현재행까지만** → **진짜 마지막 값**을 쓰려면 **범위 지정** 필요 |

> ☞ **LAST_VALUE**는 `ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING` 등 **프레임**을 명시해야 **파티션 끝 값**을 얻는 경우가 많음(교재·DBMS).

## 10. 윈도우 프레임 (범위 지정)

**어느 행들**을 윈도우에 넣을지 **경계**를 정한다.

```sql
ROWS | RANGE BETWEEN [시작 경계] AND [끝 경계]
```

| 경계 키워드 | 의미 |
|-------------|------|
| **UNBOUNDED PRECEDING** | 파티션 **맨 앞**까지 |
| **N PRECEDING** | **현재 행 기준 N행 전** |
| **CURRENT ROW** | **현재 행** |
| **M FOLLOWING** | **현재 행 기준 M행 후** |
| **UNBOUNDED FOLLOWING** | 파티션 **맨 끝**까지 |

```
     UNBOUNDED PRECEDING
            ↓
    …  N PRECEDING  [현재]  M FOLLOWING  …
            ↑                    ↓
                              UNBOUNDED FOLLOWING
```

### 기본 동작 (범위 생략 시)

| 항목 | 내용 |
|------|------|
| **생략 시** | `RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW` 로 동작 |
| **의미** | **파티션 시작 ~ 현재 행**까지가 윈도우 |
| **영향** | **누적합**·**FIRST_VALUE** 등과 연결 — **LAST_VALUE**는 기본만으로는 **현재행 값**에 가깝게 나올 수 있음 |

## 11. ROWS vs RANGE — 누적합 비교

**같은 날짜**가 **여러 행**일 때 차이가 난다.

### \<판매\> (날짜 중복)

| 날짜 | 매출 |
|------|------|
| 2020-04-01 | 100 |
| 2020-04-02 | 200 |
| 2020-04-03 | 150 |
| 2020-04-03 | 300 |
| 2020-04-04 | 350 |

### ROWS — **행 단위** 누적

```sql
SELECT 날짜, 매출,
       SUM(매출) OVER (
         ORDER BY 날짜
         ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
       ) AS 누적합
  FROM 판매;
```

| 날짜 | 매출 | 누적합 |
|------|------|--------|
| 2020-04-01 | 100 | 100 |
| 2020-04-02 | 200 | 300 |
| 2020-04-03 | 150 | **450** |
| 2020-04-03 | 300 | **750** |
| 2020-04-04 | 350 | 1100 |

→ **날짜가 같아도** **행 순서대로** 하나씩 더함 (450 → 750).

### RANGE — **값(날짜) 단위** 누적

```sql
SELECT 날짜, 매출,
       SUM(매출) OVER (
         ORDER BY 날짜
         RANGE BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
       ) AS 누적합
  FROM 판매;
```

| 날짜 | 매출 | 누적합 |
|------|------|--------|
| 2020-04-01 | 100 | 100 |
| 2020-04-02 | 200 | 300 |
| 2020-04-03 | 150 | **750** |
| 2020-04-03 | 300 | **750** |
| 2020-04-04 | 350 | 1100 |

→ **같은 날짜(4/3)** 는 **한 묶음** — 4/3 두 행 모두 **750** (100+200+150+300).

| 구분 | ROWS | RANGE |
|------|------|-------|
| **기준** | **물리적 행** 순서 | **ORDER BY 값**이 같은 행 **동시** 반영 |
| **동일 날짜 2행** | 450, **750** (달라짐) | **750**, **750** (같음) |
| **교재 요약** | 값이 같아도 **행 단위** | 값이 같으면 **묶어서** |

## 12. `LAST_VALUE` + 전체 프레임

```sql
SELECT 날짜, 직원, 매출,
       LAST_VALUE(매출) OVER (
         PARTITION BY 직원
         ORDER BY 날짜
         ROWS BETWEEN UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING
       ) AS 마지막매출
  FROM 판매;
```

| 날짜 | 직원 | 매출 | 마지막매출 |
|------|------|------|------------|
| 2020-04-01 | 홍길동 | 100 | **150** |
| 2020-04-02 | 홍길동 | 200 | **150** |
| 2020-04-03 | 홍길동 | 150 | **150** |
| 2020-04-01 | 임꺽정 | 300 | **350** |
| 2020-04-02 | 임꺽정 | 350 | **350** |

| 항목 | 설명 |
|------|------|
| **프레임** | **파티션 전체** (`PRECEDING` ~ `FOLLOWING`) |
| **의미** | 직원별 **날짜 순 마지막 매출**을 **모든 행**에 표시 |
| **기본만 쓸 때** | `… AND CURRENT ROW` → **현재 행**이 “마지막”으로 잡혀 **매출과 동일**해질 수 있음 |

→ **LAST_VALUE**로 **파티션 끝 값**을 쓰려면 **프레임을 끝까지** 지정.

---

## 13. 예 — 이동 평균 (`ROWS BETWEEN`)

**현재 행 기준 전날·다음날** 포함 **3일 이동 평균**.

### \<판매\> (날짜별)

| 날짜 | 매출 |
|------|------|
| 2020-04-01 | 100 |
| 2020-04-02 | 200 |
| 2020-04-03 | 150 |
| 2020-04-04 | 300 |
| 2020-04-05 | 350 |

```sql
SELECT 날짜, 매출,
       AVG(매출) OVER (
         ORDER BY 날짜
         ROWS BETWEEN 1 PRECEDING AND 1 FOLLOWING
       ) AS 이동평균
  FROM 판매;
```

| 날짜 | 매출 | 이동평균 | 계산 (해당 윈도우) |
|------|------|----------|-------------------|
| 2020-04-01 | 100 | **150.0** | (100+200) ÷ **2** |
| 2020-04-02 | 200 | **150.0** | (100+200+150) ÷ **3** |
| 2020-04-03 | 150 | **216.7** | (200+150+300) ÷ **3** |
| 2020-04-04 | 300 | **266.7** | (150+300+350) ÷ **3** |
| 2020-04-05 | 350 | **325.0** | (300+350) ÷ **2** |

→ **앞 1행·현재·뒤 1행** — 경계 행은 **있는 행만** 평균(2일·3일).

## 14. 비율 함수 (Ratio Functions)

**비율·분위·분포**를 **행마다** 계산하는 윈도우 함수.

| 함수 | 설명 |
|------|------|
| **NTILE(n)** | 데이터를 **n등분**해 그룹 번호(1~n) 부여 |
| **CUME_DIST()** | **누적 분포** — 현재 값 **이하**인 행의 **비율** |
| **PERCENT_RANK()** | **백분위 순위** (0~1) |
| **RATIO_TO_REPORT()** | **전체 대비 비율** (부분 ÷ 전체) |

```sql
SELECT 직원, 매출,
       NTILE(3) OVER (ORDER BY 매출) AS 분위,
       CUME_DIST() OVER (ORDER BY 매출) AS 누적분포,
       PERCENT_RANK() OVER (ORDER BY 매출) AS 백분위순위,
       RATIO_TO_REPORT(매출) OVER () AS 판매비율
  FROM 판매;
```

### \<판매\> · 결과 (매출 오름차순)

| 직원 | 매출 |
|------|------|
| 철수 | 100 |
| 선영 | 200 |
| 민수 | 300 |
| 영희 | 400 |
| 수지 | 500 |

**전체 합계** = **1500**

| 직원 | 매출 | 분위 | 누적분포 | 백분위순위 | 판매비율 |
|------|------|------|----------|------------|----------|
| 철수 | 100 | **1** | **0.2** | **0.000** | **0.066** |
| 선영 | 200 | **1** | **0.4** | **0.250** | **0.133** |
| 민수 | 300 | **2** | **0.6** | **0.500** | **0.200** |
| 영희 | 400 | **2** | **0.8** | **0.750** | **0.266** |
| 수지 | 500 | **3** | **1.0** | **1.000** | **0.333** |

### 함수별 해석

| 함수 | 계산·의미 (본 예) |
|------|-------------------|
| **NTILE(3)** | 5행을 **3그룹** — **2·2·1**개씩 → 분위 **1,1,2,2,3** |
| **CUME_DIST** | **이하 행 수 ÷ 전체** — 1/5=**0.2**, 2/5=**0.4** … 5/5=**1.0** |
| **PERCENT_RANK** | **(순위 − 1) ÷ (전체 행 − 1)** — 0, 0.25, 0.5, 0.75, 1.0 |
| **RATIO_TO_REPORT** | **매출 ÷ 1500** — 판매비율 합 ≈ **1** |

```
PERCENT_RANK = (RANK(ORDER BY 매출) - 1) / (COUNT(*) - 1)   -- 개념식
RATIO_TO_REPORT(매출) = 매출 / SUM(매출) OVER ()
```

> **ORDER BY 매출** 기준(오름차순). 내림차순이면 **분위·순위 의미**가 반대로 해석.

## 15. 시험 포인트 / 함정

| 구분 | 내용 |
|------|------|
| 윈도우 | **행 유지** + `OVER` |
| ROW_NUMBER | **유일** — 1,2,3,4 |
| RANK | 동점 후 **건너뜀** — 1,1,3,4 |
| DENSE_RANK | 동점 후 **연속** — 1,1,2,3 |
| `SUM() OVER()` | **전체** 합 — 모든 행 **동일 값** |
| `PARTITION BY` | **그룹별** 합 — GROUP BY와 달리 **행 안 줄임** |
| `OVER(ORDER BY)` | **누적 합** |
| 출력 순서 | **OVER ≠ ORDER BY** — 맨 끝 `ORDER BY` |
| GROUP BY | **축소** vs 윈도우 **유지** |
| 함정 | RANK vs DENSE (80점 → **3위** vs **2위**) |
| LAG | **이전 행**, 파티션 **첫 행** → **NULL** |
| LEAD | **다음 행** |
| LAG vs LEAD | **PARTITION BY**·**ORDER BY** 필수 |
| 함정 | 직원 바뀔 때 **이전 직원 매출** 안 가져옴 — **파티션** 경계 |
| LEAD | 파티션 **마지막 행** → **NULL** |
| FIRST_VALUE | 파티션 **첫 값** 반복 |
| LAST_VALUE | 기본 프레임 — **범위 지정** 필요(교재) |
| 프레임 생략 | `UNBOUNDED PRECEDING` ~ **CURRENT ROW** |
| ROWS BETWEEN | **1 PRECEDING AND 1 FOLLOWING** = 전·현·후 **3행** 평균 |
| ROWS 누적 | **행마다** 합 — 동일 날짜 **450·750** |
| RANGE 누적 | 동일 **ORDER BY 값** **같은 누적합** — 4/3 둘 다 **750** |
| LAST_VALUE | `UNBOUNDED PRECEDING AND UNBOUNDED FOLLOWING` → **파티션 마지막 값** |
| 함정 | LAST_VALUE **프레임 생략** → **현재 행**과 같게 나올 수 있음 |
| NTILE(3) | **3등분** 그룹 번호 — 5행이면 **2·2·1** 배분 |
| CUME_DIST | **누적 분포** 0.2~1.0 |
| PERCENT_RANK | **(순위−1)/(n−1)** — 최소 **0**, 최대 **1** |
| RATIO_TO_REPORT | **부분/전체**, `OVER()` |
| 함정 | PERCENT_RANK vs CUME_DIST **공식** 혼동 |
