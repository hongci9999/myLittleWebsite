# INNER JOIN

## 핵심 요약

**INNER JOIN**은 조인 조건에 **같은 값이 있는 행만** 반환한다. **`JOIN` 단독** = **INNER JOIN**. **USING**: **같은 컬럼명**일 때 `USING (부서ID)`. **ON**: 컬럼명이 **달라도** `ON 직원.부서ID = 부서.ID`. **NULL**인 조인 키 행은 **매칭 없음** → **결과에서 제외**.

## 왜 중요한가

- 표준조인에서 **가장 많이 쓰는** 형태이다.
- USING vs ON 문법·NULL 처리는 **빈출**이다.
- EQUI JOIN의 **표준 SQL** 구현이다.

> 조인 개요: [09_조인_개요](./09_조인_개요.md) · EQUI JOIN 예: §2

## 1. INNER JOIN

| 항목 | 내용 |
|------|------|
| **정의** | JOIN 조건에서 **동일한 값**이 있는 **행만** 반환 |
| **의미** | 두 테이블의 **교집합**에 해당하는 매칭 행 |
| **키워드** | `INNER JOIN` 또는 **`JOIN`만** 써도 INNER |

```sql
INNER JOIN … ON …
JOIN … ON …          -- INNER JOIN과 동일
```

> ☞ **`JOIN`만 단독** 사용 시 **INNER JOIN** 의미. **NULL** 값은 조인 **대상이 아님**(매칭 실패 → 제외).

---

## 2. USING 조건절

**양쪽 테이블에 같은 이름**의 컬럼이 있을 때, 그 컬럼으로 조인.

```sql
SELECT 이름, 부서명
  FROM 직원
 INNER JOIN 부서
 USING (부서ID);
```

| 항목 | 설명 |
|------|------|
| **조건** | 조인할 컬럼 **이름이 동일** (`부서ID` = `부서ID`) |
| **SELECT** | `USING` 컬럼은 **테이블 접두어 없이** `부서ID` 하나로 취급 |
| **동치** | `ON 직원.부서ID = 부서.부서ID` 와 같은 의미(이름 같을 때) |

### \<직원\> · \<부서\>

**직원**

| ID | 이름 | 부서ID |
|----|------|--------|
| 101 | 홍길동 | 10 |
| 102 | 이순신 | 20 |
| 103 | 임꺽정 | NULL |

**부서**

| 부서ID | 부서명 |
|--------|--------|
| 10 | 인사 |
| 20 | 개발 |

### 결과

| 이름 | 부서명 |
|------|--------|
| 홍길동 | 인사 |
| 이순신 | 개발 |

→ **임꺽정**(부서ID **NULL**)은 부서 테이블과 **매칭 불가** → **제외**.

---

## 3. ON 조건절

**컬럼명이 달라도** 조인 조건을 **직접** 지정.

```sql
SELECT 직원.이름, 부서.부서명
  FROM 직원
  JOIN 부서
    ON 직원.부서ID = 부서.ID;
```

| 항목 | 설명 |
|------|------|
| **용도** | `부서ID` vs `ID` 처럼 **이름이 다른** FK·PK 매핑 |
| **SELECT** | **테이블.컬럼** 으로 구분 (`직원.이름`, `부서.부서명`) |
| **조건** | `=`, `BETWEEN` 등 — **EQUI·NON EQUI** 모두 **ON**에 기술 |

### \<직원\> · \<부서\> (컬럼명 상이)

**직원**

| 사번 | 이름 | 부서ID |
|------|------|--------|
| 101 | 홍길동 | 10 |
| 102 | 이순신 | 20 |
| 103 | 임꺽정 | NULL |

**부서**

| ID | 부서명 |
|----|--------|
| 10 | 인사 |
| 20 | 개발 |

### 결과

| 이름 | 부서명 |
|------|--------|
| 홍길동 | 인사 |
| 이순신 | 개발 |

→ `ON 직원.부서ID = 부서.ID` — **이름이 달라도** 조인 가능.

---

## 4. USING vs ON

| 구분 | USING | ON |
|------|-------|-----|
| **컬럼명** | **양쪽 동일**해야 함 | **달라도** 됨 |
| **문법** | `USING (부서ID)` | `ON A.열 = B.열` |
| **SELECT** | 공통 컬럼 **하나**로 표기 | **테이블.컬럼** 구분 |
| **대표** | `USING (부서ID)` | `ON 직원.부서ID = 부서.ID` |

---

## 5. 시험 포인트 / 함정

| 구분 | 내용 |
|------|------|
| INNER | 조건 **일치 행만** |
| JOIN | 단독 = **INNER JOIN** |
| NULL | 조인 키 **NULL** → **미포함** |
| USING | **같은 컬럼명** 전용 |
| ON | **다른 컬럼명**·임의 조건 |
| 함정 | USING은 `ON 직원.부서ID = …` 처럼 **한쪽만** 다른 이름이면 **불가** → **ON** 사용 |
| 함정 | OUTER와 혼동 — INNER는 **매칭 없는 행 제외** |
