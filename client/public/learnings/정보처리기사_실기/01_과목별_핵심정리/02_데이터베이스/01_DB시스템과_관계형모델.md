# DB 시스템과 관계형 모델

## 핵심 요약

데이터베이스 단원의 출발점은 "데이터를 왜 파일이 아니라 DB로 관리하는가"를 이해하는 것이다. 실기에서는 DB 시스템의 특성, DBMS의 기능, 관계형 모델의 구성 요소를 정확히 구분하는 문제가 자주 나온다.

## 왜 중요한가

- 스키마, 키, SQL, 정규화, 트랜잭션은 모두 DB 시스템의 기본 구조 위에서 이해해야 한다.
- 용어형 문제에서 `릴레이션`, `튜플`, `도메인`, `차수`, `카디널리티`를 많이 묻는다.
- DBMS의 기능과 데이터 특성은 단답형/개념형 출제 빈도가 높다.

## 1. 데이터베이스의 정의와 특성

데이터베이스는 여러 사용자가 공유할 수 있도록 통합 저장된 운영 데이터의 집합이다.

### 데이터의 주요 특성

- 공유 데이터: 여러 사용자가 함께 사용
- 통합 데이터: 중복을 최소화해 일관성 유지
- 저장 데이터: 컴퓨터 저장 매체에 저장
- 운영 데이터: 조직 운영에 필요한 실제 데이터

### DB 시스템의 특징

| 특징 | 의미 |
|---|---|
| 실시간 접근성 | 사용자의 질의에 즉시 응답 |
| 계속적인 변화 | 삽입, 삭제, 갱신이 지속적으로 발생 |
| 동시 공유 | 여러 사용자가 동시에 사용 |
| 내용에 의한 참조 | 저장 위치가 아니라 값의 내용으로 접근 |

## 2. DBMS의 역할과 기능

DBMS는 사용자와 데이터베이스 사이에서 데이터를 정의, 조작, 제어하는 소프트웨어다.

### DBMS의 필수 기능

| 기능 | 설명 |
|---|---|
| 정의 기능 | 데이터 구조와 제약조건 정의 |
| 조작 기능 | 검색, 삽입, 수정, 삭제 수행 |
| 제어 기능 | 무결성, 보안, 병행제어, 회복 처리 |

### 기억 포인트

- 정의 = DDL
- 조작 = DML
- 제어 = DCL/TCL 및 내부 관리 기능

## 3. 관계형 데이터베이스의 기본 구성

관계형 데이터베이스는 테이블 형태의 릴레이션으로 데이터를 표현한다.

| 용어 | 의미 |
|---|---|
| 릴레이션 | 테이블 전체 |
| 튜플 | 행(Row), 레코드 |
| 속성 | 열(Column), 필드 |
| 도메인 | 속성이 가질 수 있는 값의 범위 |
| NULL | 알 수 없음 또는 해당 없음 |

## 4. 차수와 카디널리티

- 차수(Degree): 속성의 개수
- 카디널리티(Cardinality): 튜플의 개수

예를 들어 학생 테이블이 다음과 같다고 하자.

| 학번 | 이름 | 학과 |
|---|---|---|
| 1001 | 김민수 | 컴공 |
| 1002 | 이서연 | 전자 |

- 차수 = 3
- 카디널리티 = 2

## 5. 파일 시스템과 DBMS 비교

| 항목 | 파일 시스템 | DBMS |
|---|---|---|
| 중복 | 많음 | 적음 |
| 무결성 | 관리 어려움 | 제약조건으로 관리 가능 |
| 보안 | 약함 | 권한 관리 가능 |
| 공유 | 제한적 | 동시 공유 가능 |

## 6. 코드/문제 추적 예시

### 예제 1: 릴레이션 용어 판별

문제: `학생(학번, 이름, 학과)` 테이블에서 `학과`는 무엇인가?

- 정답: 속성(Attribute)
- 이유: 테이블의 열이기 때문

### 예제 2: 차수와 카디널리티

테이블에 열이 5개, 행이 20개라면:

- 차수 = 5
- 카디널리티 = 20

## 7. 자주 틀리는 함정

### 7.1 릴레이션과 튜플 혼동

- 릴레이션은 전체 테이블
- 튜플은 한 행

### 7.2 차수와 카디널리티 혼동

- 차수는 열
- 카디널리티는 행

### 7.3 NULL을 0이나 공백으로 생각

- NULL은 0이나 빈 문자열과 다르다.
- "값이 존재하지 않음"을 의미한다.

## 8. 시험장에서 푸는 순서

1. 문제에서 묻는 것이 테이블 전체인지, 행인지, 열인지 먼저 본다.
2. 차수와 카디널리티는 각각 열과 행으로 바로 연결한다.
3. DBMS 기능은 정의/조작/제어 세 축으로 외운다.
4. 파일 시스템과 DBMS 비교 문제는 중복/보안/공유/무결성 기준으로 판단한다.

## 한 페이지 요약

- DB는 통합 저장된 운영 데이터의 집합이다.
- DBMS의 기본 기능은 정의, 조작, 제어다.
- 릴레이션은 테이블, 튜플은 행, 속성은 열, 도메인은 값의 범위다.
- 차수는 열 수, 카디널리티는 행 수다.
- 관계형 모델 용어는 단답형으로 자주 출제된다.
