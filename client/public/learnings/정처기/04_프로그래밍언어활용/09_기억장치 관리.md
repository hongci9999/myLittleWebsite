# 기억장치 관리 및 저장장치 기술 상세 정리

## 📌 핵심 요약
주기억장치의 효율적 활용을 위한 **[[배치 전략]]**과 단편화 해결 기법을 학습하고, **[[가상기억장치]]**의 페이징 및 세그먼테이션 원리를 파악합니다. 또한 데이터 저장의 효율을 결정하는 **[[파일 편성]]** 방식과 하드디스크의 물리적 구조에 따른 **[[디스크 스케줄링]]** 기법을 마스터합니다.

---

## 📝 상세 정리

### 1. 주기억장치 관리 및 할당 기법
- **관리 전략**:
    - **반입(Fetch)**: 데이터를 **언제** 적재할 것인가? (요구 반입, 예상 반입)
    - **배치(Placement)**: 데이터를 **어느 위치**에 적재할 것인가?
        - **최초 적합(First Fit)**: 가용한 첫 번째 공간.
        - **최적 적합(Best Fit)**: 단편화가 가장 작은 공간.
        - **최악 적합(Worst Fit)**: 단편화가 가장 큰 공간.
    - **교체(Replacement)**: 빈 공간이 없을 때 **어떤 데이터**를 내보낼 것인가?
- **할당 기법 분류**:
    - **연속 할당**: 데이터를 주기억장치에 연속적으로 배치. (단일/다중 분할 할당)
    - **분산 할당**: 프로그램을 조각내어 분산 배치. (페이징, 세그먼테이션)

### 2. 단일 분할 및 다중 분할 할당
- **단일 분할 할당**: 한 명의 사용자만 사용. 영역 낭비나 실행 불가 문제가 발생할 수 있음.
    - **오버레이(Overlay)**: 주기억장치보다 큰 프로그램 실행을 위해 불필요한 조각을 중첩 적재.
    - **스와핑(Swapping)**: 프로그램 전체를 보조기억장치로 내보내거나(Swap-out) 들여옴(Swap-in).
- **다중 분할 할당과 단편화**:
    - **내부 단편화**: 할당된 영역 내에서 남는 공간.
    - **외부 단편화**: 대기 중인 프로그램보다 빈 공간이 작아 할당되지 못하는 공간.
    - **해결책**: **통합**(인접 공간 합침), **압축**(조각난 공간을 한곳으로 모음).

### 3. 가상기억장치 (Virtual Memory)
- **개념**: 스와핑 기법을 발전시켜 용량이 큰 프로그램을 실행하며 단편화 문제를 해결함.
- **[[페이징(Paging)]]**: 프로그램을 동일 크기(페이지)로 나눔. **내부 단편화** 발생 가능. 주소 변환(Mapping) 필요로 속도 저하 가능성.
- **[[세그먼테이션(Segmentation)]]**: 논리적 단위(세그먼트)로 나눔. **외부 단편화** 발생 가능. 실제 주소는 시작주소 + 상대주소로 계산.



### 4. 페이지 교체 알고리즘 및 관리
- **알고리즘**:
    - **OPT**: 미래에 가장 오래 사용 안 할 페이지 교체. (최고 적중률)
    - **FIFO**: 가장 먼저 들어온 페이지 교체.
    - **LRU**: 최근에 가장 오랫동안 사용하지 않은 페이지 교체. (스택 이용)
    - **LFU**: 참조 빈도가 가장 적은 페이지 교체.
    - **NUR**: 최근 참조/변형 비트(0, 1)를 사용하여 교체.
    - **SCR**: FIFO의 단점을 보완하여 자주 사용되는 페이지 교체 방지.
- **주요 현상**:
    - **[[Locality(국부성)]]**: 시간/공간 구역성.
    - **[[워킹 셋]]**: 자주 참조하는 페이지 집합.
    - **[[스래싱(Thrashing)]]**: 페이지 교체 시간이 실행 시간보다 길어지는 성능 저하 현상. (부재 빈도 조절 및 자원 증설로 해결)

### 5. 파일 편성 (File Organization)
- **순차 편성(SAM)**: 물리적으로 연속 기록. 일괄 처리에 효율적이나 추가/삭제 불편. 백업용.
- **임의(직접) 편성(DAM)**: 해싱을 통해 직접 접근. 처리 속도가 일정하고 빠름.
- **색인 순차 편성(ISAM)**: 인덱스를 구성하여 랜덤과 순차 처리 병행.
    - **색인 구역(Index Area)** / **기본 데이터 구역(Prime Data Area)** / **오버플로우 구역(Overflow Area)**으로 구성.

### 6. 디스크 드라이브 및 스케줄링
- **물리 구조**: 트랙, 섹터, 실린더, 액세스 암, 디스크 헤드.
- **접근 시간**: $Seek\ Time + Rotational\ Delay + Transfer\ Time$.
- **[[FCB(파일 디스크립터)]]**: 파일의 관리 정보 블록. 실행 시 주기억장치로 적재됨.
- **[[RAID]] 종류**:
    - **RAID-0**: 스트라이핑(분산). 속도 빠름.
    - **RAID-1**: 미러링(복제). 안정성 높음.
    - **RAID-5**: 패리티 분산 저장. 효율과 안전성 균형.
- **스케줄링 기법**:
    - **FCFS**: 들어온 순서대로.
    - **SSTF**: 가까운 트랙 우선 (기아 현상 주의).
    - **SCAN**: 양방향 끝까지 이동 (엘리베이터).
    - **C-SCAN**: 한쪽 방향으로만 처리, 반대쪽 복귀 시 급송.
    - **LOOK / C-LOOK**: 요청이 없으면 끝까지 가지 않고 방향 전환.

> [!INFO] 💡 AI 보충 설명: 페이지 크기에 따른 영향
> 페이지 크기가 **작으면** 페이지 맵 테이블이 커져 매핑 속도가 느려지지만, 꼭 필요한 정보만 적재하므로 내부 단편화는 줄어듭니다. 반대로 페이지 크기가 **크면** 테이블은 작아져 관리는 쉬우나 불필요한 정보까지 적재되어 내부 단편화가 증가할 수 있습니다.
