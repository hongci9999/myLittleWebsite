# 빅데이터 분석 방법론의 계층 구조 및 프로세스

## 📌 핵심 요약

빅데이터 분석 방법론은 단계(Phase), 태스크(Task), 스텝(Step)의 3계층 구조로 이루어져 있으며, 분석 기획부터 평가 및 전개에 이르는 5단계 절차를 거칩니다. 또한, 반복 가능한 4계층 프로세스 모델인 CRISP-DM(Cross-Industry Standard Process for Data Mining)을 통해 업무 이해에서 전개까지의 데이터 마이닝 절차를 체계화합니다.

---

## 📝 상세 정리

### 1. 분석 방법론의 3계층 구조 (3-Layer Structure)

- **단계 (Phase):** 데이터 분석을 수행하기 위한 최상위 절차입니다. 기준선(Baseline)을 설정하고 버전 관리를 통해 통제합니다.
- **태스크 (Task):** 단계를 구성하는 세부 업무 단위입니다.
- **스텝 (Step):** 단기에 수행 가능한 워크 패키지(Work Package)입니다. 입력 자료 처리 및 출력 자료로 구성된 단위 프로세스(Unit Process)입니다.

### 2. 데이터 분석 프로세스 5단계

#### ① 분석 기획 (Planning)

- **비즈니스 이해 및 범위 설정:** 비즈니스 이해, 프로젝트 범위 설정 스텝으로 구성됩니다.
- **프로젝트 정의 및 계획 수립:** 데이터 분석 프로젝트 정의, 프로젝트 수행 계획 수립 스텝을 포함합니다.
- **프로젝트 위험 계획 수립:** 데이터 분석 위험 식별, 위험 대응 계획 수립 스텝을 포함합니다.

#### ② 데이터 준비 (Preparing)

- **필요 데이터 정의:** 데이터 정의, 데이터 획득 방안 수립 스텝을 수행합니다.
- **데이터 스토어 설계:** 정형 및 비정형 데이터 스토어(Data Store)를 설계합니다.
- **데이터 수집 및 정합성 점검:** 데이터 수집 및 저장, 데이터 정합성(Data Integrity) 점검 스텝을 거칩니다.

#### ③ 데이터 분석 (Analyzing)

- **분석용 데이터 준비:** 비즈니스 룰(Business Rule) 확인, 데이터셋(Data Set) 준비 스텝입니다.
- **텍스트 분석:** 텍스트 데이터 확인 및 추출, 텍스트 데이터 분석 스텝을 포함합니다.
- **탐색적 분석:** 탐색적 데이터 분석(EDA, Exploratory Data Analysis), 데이터 시각화 스텝을 수행합니다.
- **모델링 (Modeling):** 데이터 분할, 데이터 모델링, 모델 적용 및 운영 방안 수립 스텝입니다.
- **모델 평가 및 검증:** 모델 평가, 모델 검증 스텝을 수행합니다.

#### ④ 시스템 구현 (Developing)

- **설계 및 구현:** 시스템 분석 및 설계, 시스템 구현 스텝을 거칩니다.
- **시스템 테스트 및 운영:** 시스템 테스트, 운영 계획 수립 스텝을 포함합니다.

#### ⑤ 평가 및 전개 (Deploying)

- **모델 발전 계획 수립:** 모델 발전 계획 수립 스텝입니다.
- **프로젝트 평가 및 보고:** 프로젝트 성과 평가, 프로젝트 종료 보고 스텝을 포함합니다.

### 3. CRISP-DM 분석 방법론 (Cross-Industry Standard Process for Data Mining)

- **개요:** 4계층 프로세스(수준)로 구성되며, 각 단계 간 피드백과 반복이 가능한 유연한 모델입니다.
- **주요 단계:**
  1.  **업무 이해 (Business Understanding):** 비즈니스 관점에서 목적과 요구사항을 이해하기 위한 초기 단계입니다.
  2.  **데이터 이해 (Data Understanding):** 분석을 위한 초기 데이터를 수집하고 데이터의 속성을 이해하며 인사이트를 발견합니다.
  3.  **데이터 준비 (Data Preparation):** 수집된 데이터를 분석 기법에 적합한 데이터셋으로 정제 및 변환합니다.
  4.  **모델링 (Modeling):** 다양한 모델링 기법과 알고리즘을 적용하고 하이퍼파라미터를 최적화합니다.
  5.  **평가 (Evaluation):** 구축된 분석 모델이 비즈니스 목적에 부합하는지 평가합니다.
  6.  **전개 (Deployment):** 분석 결과를 실제 업무 프로세스에 적용하기 위한 계획을 수립하고 실행합니다.

● 분석 모델의 유형
계층적 프로세스 모델: 상위 단계부터 세부 실행 단위까지 체계적으로 관리합니다.

구조: 단계(Baseline) → 태스크 → 스텝(WorkPackage)

특징: 빅데이터 분석 방법론의 기본 기반이 됩니다.

폭포수 모델 (Waterfall): 이전 단계가 완전히 끝나야만 다음으로 넘어가는 하향식(Top-Down) 모델입니다.

프로토타입 모델 (Prototype): 주요 기능을 미리 견본(시제품)으로 만들어 보고 사용자의 요구를 반영하며 보완합니다.

나선형 모델 (Spiral): 여러 번의 개발 과정을 거쳐 점진적으로 완성하며, 특히 **위험 요소 제거(위험 분석)**에 초점을 둡니다.

애자일 (Agile): 짧은 개발 주기(Sprint)를 반복하며 고객의 피드백을 지속적으로 반영하는 유연한 방식입니다.

### 4. KDD 분석 방법론

KDD(Knowledge Discovery in Databases)는 데이터 속에서 통계적 패턴이나 지식을 찾기 위한 체계적인 절차입니다.

[KDD 5단계 프로세스]

데이터 선택 (Selection): 원시 데이터나 DB에서 분석에 필요한 데이터셋을 선택합니다.

전처리 (Preprocessing): 데이터의 노이즈, 이상값(Outlier), 잡음을 식별하고 제거하는 등 데이터를 가공합니다.

변환 (Transformation): 분석 목적에 맞게 변수를 선택하거나 차원을 축소하여 효율적인 형태로 바꿉니다.

마이닝 (Data Mining): 실제 알고리즘을 선택하여 데이터의 패턴을 찾고 분석을 수행합니다.

평가 (Evaluation): 결과를 해석하고, 목표에 충족되지 않으면 다시 이전 절차를 반복합니다.

> [!INFO] 💡 AI 보충 설명
> **데이터 정합성(Data Integrity):** 데이터의 정확성, 일관성, 유효성이 유지되는 상태를 의미합니다. 분석 단계로 넘어가기 전 수집된 데이터가 신뢰할 수 있는지 검증하는 필수 과정입니다.
> **KDD(Knowledge Discovery in Databases):** CRISP-DM과 함께 자주 언급되는 분석 방법론으로, 선택(Selection) - 전처리(Preprocessing) - 변환(Transformation) - 데이터 마이닝(Data Mining) - 결과 평가(Interpretation/Evaluation) 순으로 진행되는 특징이 있습니다.
