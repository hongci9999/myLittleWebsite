# 트랜잭션, 병행제어, 회복

## 핵심 요약

트랜잭션은 데이터베이스 작업의 논리적 단위다. 실기에서는 ACID, 트랜잭션 상태, COMMIT/ROLLBACK, 동시성 문제, 회복 기법을 함께 묶어 출제하는 경우가 많다.

## 왜 중요한가

- 트랜잭션과 병행제어는 이론형과 SQL 상태 추적형 모두 자주 나온다.
- ACID는 단답형으로 매우 빈출이다.
- 교착상태, 로킹, UNDO/REDO는 개념 비교 문제에 자주 등장한다.

## 1. 트랜잭션의 개념

트랜잭션은 하나의 논리적인 작업 단위다.

예:

- 은행 송금
- 재고 차감과 주문 등록

중간에 실패하면 전부 취소돼야 한다.

## 2. ACID

| 요소 | 의미 |
|---|---|
| 원자성 | 전부 수행되거나 전부 수행되지 않음 |
| 일관성 | 실행 전후 데이터 무결성 유지 |
| 고립성 | 동시에 실행돼도 서로 간섭하지 않음 |
| 지속성 | COMMIT 된 결과는 영구 반영 |

## 3. 트랜잭션 제어 명령

### COMMIT

- 변경 내용을 최종 반영

### ROLLBACK

- 변경 내용을 취소하고 이전 상태로 복구

### SAVEPOINT

- 중간 복구 지점 지정

```sql
SAVEPOINT sp1;
ROLLBACK TO SAVEPOINT sp1;
```

## 4. 트랜잭션 상태

| 상태 | 의미 |
|---|---|
| 활동 | 실행 중 |
| 부분 완료 | 마지막 연산 수행 후 |
| 완료 | COMMIT 성공 |
| 실패 | 실행 중 오류 발생 |
| 철회 | ROLLBACK 된 상태 |

## 5. 병행제어

여러 트랜잭션이 동시에 실행될 때 데이터 일관성을 유지하는 기법이다.

### 병행 수행 시 문제

- 갱신 손실
- 비완료 의존성
- 모순성
- 연쇄 복귀

## 6. 병행제어 기법

| 기법 | 특징 |
|---|---|
| 로킹 | 가장 대표적인 제어 기법 |
| 타임스탬프 | 시간 순서 기반 제어 |
| 낙관적 제어 | 충돌이 적을 때 유리 |

### 로킹

- 공유 잠금과 배타 잠금 개념이 자주 나온다.
- 로킹 단위가 크면 관리가 단순하지만 병행성은 낮아진다.

## 7. 교착상태

둘 이상의 트랜잭션이 서로 자원을 기다리며 무한 대기하는 상태다.

### 교착상태 발생 조건

- 상호 배제
- 점유와 대기
- 비선점
- 환형 대기

이 네 조건이 모두 만족하면 교착상태가 발생할 수 있다.

## 8. 회복(Recovery)

장애 발생 시 DB를 일관된 상태로 되돌리는 과정이다.

### 핵심 연산

- `UNDO`: 취소
- `REDO`: 재실행

### 회복 기법

| 기법 | 설명 |
|---|---|
| 지연 갱신 | 완료 전까지 DB 반영 미룸 |
| 즉시 갱신 | DB에 즉시 반영, 필요 시 UNDO 수행 |
| 체크포인트 | 특정 시점을 기준으로 회복 범위 축소 |
| 그림자 페이징 | 복사본 페이지 활용 |

## 9. 코드/상태 추적 예제

### 예제 1: COMMIT과 ROLLBACK

```sql
UPDATE 사원 SET 급여 = 급여 + 100 WHERE 사원번호 = 1;
ROLLBACK;
```

- 증가한 급여는 취소
- 원래 값 유지

```sql
UPDATE 사원 SET 급여 = 급여 + 100 WHERE 사원번호 = 1;
COMMIT;
```

- 변경 내용 최종 확정

### 예제 2: SAVEPOINT

```sql
UPDATE 사원 SET 급여 = 100 WHERE 사원번호 = 1;
SAVEPOINT sp1;
UPDATE 사원 SET 급여 = 200 WHERE 사원번호 = 2;
ROLLBACK TO SAVEPOINT sp1;
```

- 사원번호 1의 변경은 유지
- 사원번호 2의 변경은 취소

## 10. 자주 틀리는 함정

### 10.1 COMMIT 이후 ROLLBACK 가능하다고 착각

- 일반적으로 COMMIT 후에는 이전 상태로 되돌릴 수 없다.

### 10.2 SAVEPOINT 이후 전체가 다 취소된다고 오해

- `ROLLBACK TO SAVEPOINT` 는 그 지점 이후 작업만 취소한다.

### 10.3 교착상태와 단순 대기 혼동

- 교착상태는 서로가 서로를 기다리는 순환 대기다.

## 11. 시험장에서 푸는 순서

1. 트랜잭션 명령의 실행 순서를 타임라인으로 적는다.
2. COMMIT/ROLLBACK/SAVEPOINT 중 무엇이 호출됐는지 확인한다.
3. ACID 중 무엇을 묻는지 키워드로 연결한다.
4. 병행 문제면 충돌 유형과 제어 기법을 구분한다.
5. 회복 문제면 UNDO/REDO와 갱신 방식부터 판단한다.

## 한 페이지 요약

- 트랜잭션은 논리적 작업 단위다.
- ACID는 원자성, 일관성, 고립성, 지속성이다.
- COMMIT 은 확정, ROLLBACK 은 취소, SAVEPOINT 는 중간 복구 지점이다.
- 병행제어는 동시성 문제를 막기 위한 기법이다.
- 회복은 UNDO/REDO, 체크포인트, 지연/즉시 갱신으로 정리한다.
