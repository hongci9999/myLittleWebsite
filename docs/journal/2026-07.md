# 개발·학습 로그 2026-07

## 2026-07-23

- **AWS → Vercel 이전 준비(코드)** — 분리 호스팅(S3+CloudFront / EB)을 Vercel 단일 호스팅으로 통합하기로 결정([decisions/0025](decisions/0025-vercel-single-hosting.md)).
  - **Express 분리** — `server/src/app.ts`가 `app`을 export(리스닝 제외), `server/src/index.ts`는 로컬 dev용 `app.listen`만. Vercel 함수 진입점 `api/index.ts`는 `export default app`.
  - **vercel.json** — `buildCommand`(`npm run build -w client`)·`outputDirectory`(`client/dist`)·rewrite(`/api/(.*) → /api`, SPA fallback)·`functions.maxDuration=60`·`includeFiles: docs/learnings/**`.
  - **same-origin 이득** — 프론트·API가 한 도메인이 되어 CORS·커스텀 API 도메인·`VITE_API_BASE_URL`이 불필요.
  - **project-learning FS 의존** — `docs/learnings`를 런타임 `fs`로 읽는 유일한 섹션. `includeFiles` + `learning-sections.ts`의 프로젝트 루트 해석을 `__dirname` 실패 시 `process.cwd()` 폴백으로 보정(로컬·서버리스 양쪽 동작).
  - **배포 워크플로** — `deploy-aws.yml` 자동 push 트리거 제거(수동 전용). Vercel Git 연동이 배포 담당.
  - **검증** — `npm run build -w server`·`npm run build -w client` 모두 성공.
- **막힌 점/유의** — 서버리스 함수 실행 시간 제한(AI·youtube-transcript 느린 호출), 인메모리 캐시 콜드스타트 초기화, ESM `.js` import의 esbuild 해석은 첫 배포에서 확인 필요.
- **다음** — Vercel 프로젝트 Import·환경변수 등록·`*.vercel.app` 검증 → `mylittlewebsite.p-e.kr` CNAME을 `cname.vercel-dns.com`으로 전환 → 검증 후 AWS 자원(EB·CloudFront·S3·ACM·IAM) 정리. 체크리스트: [plans/2026-07-23-vercel-migration](plans/2026-07-23-vercel-migration.md).
