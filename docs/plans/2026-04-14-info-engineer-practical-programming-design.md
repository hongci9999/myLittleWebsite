# 정보처리기사 실기 프로그래밍·알고리즘 단원 재설계

**날짜:** 2026-04-14  
**관련:** `client/public/learnings/정보처리기사_실기/10_체계형_학습자료`

**Goal:** 기존 요약형 프로그래밍·알고리즘 단원을 실기 시험용 강의 교안 수준의 세부 문서 세트로 재작성한다.

---

## 1) 배경

- 현재 `10_체계형_학습자료/01_프로그래밍_및_알고리즘`은 2개 파일로 구성되어 있다.
- 구조 소개용으로는 충분하지만, 실제 학습용 기준 문서로 쓰기에는 설명 깊이와 예제 양이 부족하다.
- 프로젝트 안에는 필기용 프로그래밍 언어 문서가 있으나, 실기 관점의 코드 추적·함정·출제 포인트 중심 재구성이 필요하다.

---

## 2) 목표 품질

- "요약 노트"가 아니라 "강사가 수업할 때 배포하는 교안" 수준으로 작성한다.
- 각 문서는 정의만 나열하지 않고:
  - 왜 중요한지
  - 시험에서 어떻게 나오는지
  - 어디서 헷갈리는지
  - 코드를 어떻게 추적해야 하는지
  를 함께 설명한다.
- 기존 실기 SQL 문서처럼 비교표, 함정, 예제, 실행 흐름 해설을 포함한다.

---

## 3) 폴더 구조 결정

기존 2개 문서를 유지하지 않고, 다음 8개 세부 문서로 재구성한다.

2. `02_C언어_제어문과_연산자.md`

4. `04_Java_객체지향_핵심.md`

7. `07_정렬_알고리즘.md`
8. `08_탐색과_구현형_문제.md`

기존 파일:

- `01_프로그래밍_언어_핵심정리.md`
- `02_알고리즘_빈출유형.md`

은 삭제한다.

---

## 4) 문서 템플릿

모든 문서는 아래 흐름을 기본으로 통일한다.

1. 메타 정보 (`날짜`, `태그`, `주제`, `중요도`)
2. 핵심 요약
3. 왜 시험에 중요한가
4. 상세 개념 설명
5. 비교표 / 규칙 정리
6. 코드 예제와 실행 흐름 추적
7. 자주 틀리는 함정
8. 시험장에서 푸는 순서
9. 한 페이지 요약

---

## 5) 내용 범위

### C언어

- 자료형, 상수, 변수, 서식 지정자, 입출력
- 연산자 우선순위, 전치/후치 증감, 형 변환
- 제어문, 반복문, 중첩 반복, break/continue
- 배열, 문자열, 포인터, 포인터 연산, 구조체

### Java

- 클래스/객체/인스턴스
- 캡슐화, 상속, 다형성
- 오버로딩/오버라이딩 차이
- 생성자, `super`, 추상 클래스/인터페이스 기초

### Python

- 자료형, 슬라이싱, 리스트/튜플/딕셔너리
- 함수, 기본 문법, 컴프리헨션
- 클래스, `self`, `__init__`, 상속, 오버라이딩

### 알고리즘

- 알고리즘 기본 개념, 시간복잡도/공간복잡도
- 정렬: 버블/선택/삽입/퀵
- 탐색: 선형/이진
- 구현형: 반복 추적, 배열 조작, 누적 계산

---

## 6) 품질 기준

- 모든 문서에 최소 2개 이상의 코드 예제를 넣는다.
- 계산형/추적형 문제는 "단계별 상태 변화"를 반드시 설명한다.
- 단답 암기용 키워드와 서술형 설명을 동시에 제공한다.
- 기존 필기 문서 내용을 가져오더라도 실기용 관점으로 재정리한다.

---

## 7) 결정 기록 점검

이번 작업은 "실기 학습자료를 세부 강의노트 구조로 재편"하는 구조 변경을 포함하지만, 현재는 `10_체계형_학습자료` 내부 국소 설계 변경으로 본다. 구현 후 다른 단원까지 동일 패턴으로 확장할 때 ADR 추가 여부를 다시 점검한다.
