# 소프트웨어공학/보안 핵심용어

- Verification = 제품을 올바르게 만들고 있는가
- Validation = 올바른 제품을 만들었는가
- 화이트박스 테스트 = 내부 로직 기반 테스트
- 블랙박스 테스트 = 명세 기반 테스트
- 동등 분할 = 입력 영역을 대표 집합으로 나누는 기법
- 경계값 분석 = 경계 근처 입력을 검사하는 기법
- 테스트 오라클 = 결과의 참/거짓을 판정하는 기준
- 형상관리(SCM) = 변경을 체계적으로 통제하는 활동
- Check-Out = 저장소 사본 가져오기
- Commit = 변경 사항 확정
- SDLC = 소프트웨어 개발 생명주기
- 애자일(Agile) = 짧은 반복과 협업 중심 방법론
- DevOps = 개발과 운영의 협업 문화/방식
- 기밀성(Confidentiality) = 인가된 사용자만 접근 가능
- 무결성(Integrity) = 데이터의 정확성과 변조 방지
- 가용성(Availability) = 필요 시 언제든 사용 가능
- 랜섬웨어(Ransomware) = 데이터를 암호화하고 금전을 요구하는 악성코드
- 제로 트러스트(Zero Trust) = 내부/외부 구분 없이 모든 접근을 검증하는 보안 모델

## 한 페이지 요약

- 소프트웨어공학은 테스트/형상/방법론, 보안은 CIA와 대표 위협 중심으로 묶어 암기한다.
