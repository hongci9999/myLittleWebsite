# 인덱스, 뷰, 절차형 SQL

## 핵심 요약

데이터베이스 단원 후반부는 객체별 역할을 구분하는 문제가 자주 나온다. 인덱스는 성능, 뷰는 가상 테이블, 프로시저/함수/트리거는 절차형 SQL 객체라는 큰 틀로 정리하면 헷갈림이 줄어든다.

## 왜 중요한가

- 용어는 많지만 비교 기준을 잡으면 빠르게 정리할 수 있다.
- 인덱스와 뷰, 프로시저/함수/트리거는 개념 비교형 문제로 자주 출제된다.
- 시스템 카탈로그는 메타데이터 개념과 연결된다.

## 1. 인덱스

인덱스는 검색 속도를 높이기 위한 자료구조다.

### 장점

- 조회 속도 향상
- 정렬/탐색 효율 개선 가능

### 단점

- 저장 공간 추가 사용
- 삽입/수정/삭제 시 관리 비용 발생

```sql
CREATE INDEX IDX_사원_부서코드
ON 사원(부서코드);
```

## 2. 뷰

뷰는 실제 데이터를 저장하지 않는 가상 테이블이다.

### 뷰의 목적

- 복잡한 조회문 단순화
- 보안성 향상
- 필요한 열만 노출

```sql
CREATE VIEW V_고급학생 AS
SELECT 학번, 이름, 점수
FROM 학생
WHERE 점수 >= 90;
```

### 자주 나오는 옵션

- `WITH CHECK OPTION`
- `WITH READ ONLY`

## 3. 프로시저, 함수, 트리거

| 객체 | 특징 |
|---|---|
| 프로시저 | 호출해서 실행, 보통 반환값 없음 |
| 함수 | 호출해서 실행, 반환값 있음 |
| 트리거 | 특정 이벤트 발생 시 자동 실행 |

## 4. 프로시저

- 반복 작업이나 업무 로직을 DB 안에 저장해 실행할 수 있다.

## 5. 사용자 정의 함수

- 값을 계산해 반환하는 데 적합하다.
- SQL 문 안에서 사용 가능한 경우가 많다.

## 6. 트리거

- `INSERT`, `UPDATE`, `DELETE` 같은 이벤트 발생 시 자동 실행
- `BEFORE`, `AFTER` 와 같은 실행 시점이 중요하다.

예:

- 급여 변경 전 로그 남기기
- 데이터 입력 시 자동 검증

## 7. 시스템 카탈로그

- 데이터베이스의 메타데이터를 저장하는 영역
- 테이블, 인덱스, 사용자, 권한, 제약조건 등의 정보를 보관
- 일반적으로 DBMS가 자동 관리한다.

## 8. 비교 포인트

### 인덱스 vs 뷰

| 항목 | 인덱스 | 뷰 |
|---|---|---|
| 목적 | 성능 향상 | 조회 단순화/보안 |
| 실체 | 자료구조 | 가상 테이블 |

### 프로시저 vs 함수 vs 트리거

| 항목 | 프로시저 | 함수 | 트리거 |
|---|---|---|---|
| 실행 방식 | 직접 호출 | 직접 호출 | 자동 실행 |
| 반환값 | 보통 없음 | 있음 | 없음 |
| 사용 목적 | 작업 처리 | 값 계산 | 이벤트 반응 |

## 9. 예제

### 예제 1: 뷰 생성

```sql
CREATE VIEW V_서울부서 AS
SELECT 부서코드, 부서명
FROM 부서
WHERE 지역 = '서울';
```

### 예제 2: 트리거 개념

```sql
-- 개념 예시
CREATE TRIGGER TRG_로그
AFTER UPDATE ON 사원
FOR EACH ROW
BEGIN
  -- 로그 기록
END;
```

## 10. 자주 틀리는 함정

### 10.1 인덱스는 무조건 좋은 것이라고 생각

- 조회는 빨라질 수 있지만 수정 비용이 증가한다.

### 10.2 뷰가 실제 데이터를 저장한다고 오해

- 일반 뷰는 가상 테이블이다.

### 10.3 프로시저와 함수 혼동

- 함수는 반환값이 있다는 점이 가장 중요하다.

### 10.4 트리거를 수동 호출한다고 착각

- 트리거는 이벤트가 발생하면 자동 실행된다.

## 11. 시험장에서 푸는 순서

1. 객체의 목적이 성능인지, 조회 편의인지, 자동 실행인지 먼저 본다.
2. 반환값 유무로 프로시저와 함수를 구분한다.
3. 자동 실행이면 트리거를 먼저 의심한다.
4. 인덱스는 조회 성능과 수정 오버헤드를 함께 기억한다.
5. 시스템 카탈로그는 메타데이터 저장소로 연결한다.

## 한 페이지 요약

- 인덱스는 조회 성능 향상용 자료구조다.
- 뷰는 가상 테이블로 보안성과 조회 편의성을 높인다.
- 프로시저는 작업 수행, 함수는 값 반환, 트리거는 자동 실행이다.
- 시스템 카탈로그는 메타데이터를 저장한다.
- 객체별 목적과 실행 방식을 비교해 외우면 정리가 쉽다.
