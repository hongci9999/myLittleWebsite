# [DML-2] 집계 함수와 GROUP BY · HAVING

**실행 결과 작성** 문제에서 “부서별 평균 급여”, “건수 세기”, “조건 만족 그룹만” 같은 유형이 반복됩니다.

---

## 1. 개념

- **집계 함수**: 여러 행을 입력으로 하나의 값을 냄 — `COUNT`, `SUM`, `AVG`, `MIN`, `MAX`.
- **GROUP BY**: 지정한 컬럼(들) 값이 같은 행끼리 **한 그룹**으로 묶음.
- **HAVING**: 그룹 단위 필터. **WHERE는 행 단위**, **HAVING은 그룹 단위**.

> `SELECT`에 집계 함수와 일반 컬럼을 같이 쓸 때, 일반 컬럼은 **GROUP BY에 포함**되거나 집계 함수 안에 있어야 합니다(표준 SQL 관점).

**집계와 NULL**

- `SUM` / `AVG` / `MIN` / `MAX`: 보통 **NULL 행은 집계에서 제외**.
- `COUNT(*)`: NULL 포함 **행 수**. `COUNT(컬럼)`: 그 컬럼이 **NULL이 아닌 행**만 셈.
- `COUNT(DISTINCT 컬럼)`: **서로 다른 값**(NULL은 보통 제외)의 개수 — 중복 제거 건수 문제에 자주 등장.

---

## 2. 문법

```sql
SELECT   그룹기준컬럼 [, 집계함수(컬럼) ...]
FROM     테이블
[WHERE   행조건]
GROUP BY 그룹기준컬럼 [, ...]
[HAVING  그룹조건]
[ORDER BY ...];
```

### COUNT 주의

- `COUNT(*)`: NULL 포함 **행 수**.
- `COUNT(컬럼)`: 해당 컬럼이 **NULL이 아닌 행**만 셈.
- `COUNT(DISTINCT 컬럼)`: 중복 제거 후 개수.

### 복합 GROUP BY

```sql
SELECT   부서코드, 직급, COUNT(*) AS 인원, AVG(급여) AS 평균급여
FROM     사원
GROUP BY 부서코드, 직급;
```

---

## 3. 예시

**주문(주문번호, 고객번호, 금액)**

```sql
-- 고객별 주문 건수와 총액
SELECT   고객번호,
         COUNT(*)      AS 주문건수,
         SUM(금액)     AS 총주문액
FROM     주문
GROUP BY 고객번호;

-- 총액이 100만 원 이상인 고객만
SELECT   고객번호,
         SUM(금액) AS 총주문액
FROM     주문
GROUP BY 고객번호
HAVING   SUM(금액) >= 1000000;

-- 특정 날짜 이후 주문만 대상으로 부서별 평균(사원-주문 조인 가정 시에는 JOIN 후 GROUP BY)
```

---

## 4. 실행 결과(예시)

**주문**

| 주문번호 | 고객번호 | 금액 |
|---------|---------|------|
| 1 | A | 300 |
| 2 | A | 500 |
| 3 | B | 200 |

```sql
SELECT 고객번호, COUNT(*), SUM(금액) FROM 주문 GROUP BY 고객번호;
```

| 고객번호 | COUNT(*) | SUM(금액) |
|---------|----------|-----------|
| A | 2 | 800 |
| B | 1 | 200 |

```sql
... HAVING SUM(금액) >= 500
```

→ **A** 행만.

**NULL이 섞인 SUM/AVG (손풀이)**

| 급여 |
|------|
| 100 |
| NULL |
| 300 |

- `SUM(급여)` → **400**(NULL 무시).
- `AVG(급여)` → 보통 **200**((100+300)/2)처럼 NULL 제외한 평균(지문·DBMS 정의 확인).

---

## 5. WHERE vs HAVING (암기 문장)

- **WHERE**: 그룹 만들기 **전**에 행을 걸러냄.
- **HAVING**: 그룹 만든 **후**에 그룹을 걸러냄.
- 집계 결과로 걸러야 하면 **`HAVING SUM(...) ...`**. 행 단위 조건은 **`WHERE`**.

---

## 6. 실기 체크

- [ ] `SELECT`에 나온 **비집계 컬럼**이 전부 `GROUP BY`에 있는지 확인.
- [ ] “몇 가지 **종류**의 …” 유형 → `COUNT(DISTINCT …)` 후보.
- [ ] `ORDER BY 1` 같은 **열 번호 정렬**은 기출·교재에 나오면 그대로, 아니면 컬럼명·별칭 사용.
