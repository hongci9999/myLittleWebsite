# 통합 구현 (Integration Implementation)

## 핵심 요약
- 모듈 간의 연계와 통합을 위해 **송신 시스템, 중계 시스템, 수신 시스템** 간의 데이터 흐름을 설계하고 구현하는 과정입니다.
- 연계 데이터의 표준화와 식별자 정의가 중요하며, 데이터 표현 방식으로는 **XML, JSON** 등이 주로 사용됩니다.

## 상세 정리

### 1. 연계 시스템의 구성 요소
연계 시스템은 네트워크를 통해 송신 모듈과 수신 모듈이 통신하는 구조로 이루어집니다.

- **송신 시스템:** 전송하고자 하는 데이터를 생성하고 송신하는 시스템.
- **중계 시스템 (Intermediate System):**
    - 시스템 간 연계 시 **보안성을 강화**하고 네트워크(망)를 연결하는 역할.
    - 데이터 전송 과정의 **오류 처리 및 데이터 변환(Mapping)** 수행.
- **수신 시스템:** 전달받은 데이터를 수신하여 최종적으로 처리하는 모듈.
- **연계 데이터:** 송수신되는 데이터 그 자체로, 속성/길이/타입 등을 포함하며 DB 테이블이나 파일 형태로 분류됩니다.

### 2. 요구사항 분석 및 도구
연계 요구사항을 명확히 하기 위해 다양한 기초 자료와 분석 도구를 활용합니다.

* **주요 입력물:**
    - **시스템 구성도:** 송수신 시스템의 물리적 배치 파악.
    - **응용 애플리케이션 구성:** 연계 데이터가 발생하는 메뉴 및 비즈니스 로직 확인.
    - **[[ERD]] (Entity Relationship Diagram):** 데이터베이스 구조 파악.
* **분석 도구:** 인터뷰, 체크리스트, 설문조사, **[[델파이 기법]]**, 연계 솔루션 비교 분석.
* **분석 출력물:** 요구사항 분석서, 인터페이스 정의서, 회의록 등.

### 3. 연계 데이터 식별 및 표준화
식별된 연계 데이터를 수신 시스템 규격에 맞춰 표준화하는 과정입니다.

- **정보 상세화:** 데이터 타입, 길이, 코드화 여부를 정의하며, 가급적 **수신 시스템 기준**으로 변환하여 적용합니다.
- **코드 전환 및 매핑 (Mapping):** 송신 시스템의 코드를 수신 시스템에서 사용하는 코드로 일대일 매핑합니다.
- **변경 데이터 구분 방식:** 연계 데이터에 관리 정보를 추가하여 처리합니다.
    - **식별자:** 데이터를 유일하게 구분하는 키.
    - **변경 구분:** 데이터의 추가(Insert), 수정(Update), 삭제(Delete) 여부 식별.
    - **관리 정보:** 송수신 일시, 처리 여부, 오류 코드 등 모니터링용 정보.

### 4. 연계 데이터 표현 방식 (중요)
데이터를 구조화하여 표현하는 방식입니다.

- **[[XML]] (eXtensible Markup Language):** 사용자가 임의로 태그를 생성할 수 있는 확장성 있는 마크업 언어.
- **[[JSON]] (JavaScript Object Notation):** 자바스크립트 기반으로, **속성-값 쌍(Attribute-Value Pairs)**으로 이루어진 경량 데이터 포맷.
- **[[AJAX]] (Asynchronous JavaScript and XML):** 브라우저가 페이지 전체를 새로고침하지 않고도 데이터를 주고받는 **비동기 통신** 기술.
- **Text:** 항목 분리자(Delimiter)를 사용하여 데이터를 나열하는 방식.

---

> [!INFO] 💡 AI 보충 설명: XML vs JSON
> 시험에서 두 방식의 차이점을 묻는 경우가 많습니다.
> - **XML:** 태그를 사용하여 가독성은 좋으나 데이터 크기가 커지는 단점이 있음. (중량)
> - **JSON:** 기호(`{ }`, `[ ]`)를 사용하여 구조를 표현하며, XML보다 가볍고 속도가 빨라 현대 웹 API에서 선호됨. (경량)

## 3. 빈출 내용 정리
- **@[[XML]], [[JSON]], [[AJAX]]**: 각각의 정의와 특징(비동기성, 확장성 등)을 매칭하는 문제.
- **중계 시스템의 역할**: 보안 및 데이터 변환 매핑 수행 여부.
- **연계 데이터 관리 정보**: 오류 코드 및 송수신 일시가 관리 정보에 해당함을 인지할 것.
