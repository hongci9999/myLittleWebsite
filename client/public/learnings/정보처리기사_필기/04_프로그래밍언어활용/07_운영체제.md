# 명령어 형식 및 CPU 구조 상세 정리

## 📌 핵심 요약
컴퓨터의 핵심인 CPU가 명령어를 처리하는 단계인 **[[메이저 스테이트]]**를 이해하고, 효율적인 데이터 접근을 위한 다양한 **[[주소 지정 방식]]**을 학습합니다. 또한, 연산과 제어를 담당하는 각 **[[레지스터]]**의 역할을 명확히 구분하여 암기하는 것이 중요합니다.

---

## 📝 상세 정리

### 1. 명령어의 흐름과 메이저 스테이트 (Major State)
CPU가 무엇을 하고 있는가를 나타내는 상태로, 4가지 상태가 반복 수행됩니다.
- **[[인출]] (Fetch)**: 메모리에서 명령어를 읽어 CPU로 가져오는 단계.
- **[[간접]] (Indirect)**: 유효 주소를 찾기 위해 메모리에 다시 접근하는 단계. (간접 주소 지정 시 발생)
- **[[실행]] (Execute)**: 명령어를 해석하고 실제 연산을 수행하는 단계.
- **[[인터럽트]] (Interrupt)**: 예기치 않은 사건 발생 시 이를 처리하기 위한 단계.



### 2. 명령어 형식 (Instruction Format)
명령어는 크게 무엇을 할지 결정하는 **[[OP-Code]]**와 대상이 되는 **[[Operand]]**로 구성됩니다.
- **OP-Code (연산부)**: 비트 수가 $n$이면 $2^n$개의 명령어 설계 가능. 기능별로 함수(Function), 전달(Transfer), 제어(Control), 입출력(I/O) 등이 있음.
- **주소 수에 따른 분류**:
| 형식 | 특징 | 관련 키워드 |
| :--- | :--- | :--- |
| **0-주소** | 오퍼랜드 없이 스택(Stack)을 이용 | PUSH, POP |
| **1-주소** | 하나의 오퍼랜드만 사용 | 누산기(AC) |
| **2-주소** | 두 개의 오퍼랜드 사용 (결과는 목적지에 덮어씀) | 가장 일반적 |
| **3-주소** | 세 개의 오퍼랜드 사용 (결과를 별도 장소에 저장) | 프로그램 길이 단축 |

### 3. 주소 지정 방식 (Addressing Mode)
메모리 참조 횟수가 적을수록 속도가 빠릅니다.
- **[[즉시 주소 지정]] (Immediate)**: 명령어 내에 실제 데이터가 포함됨. 메모리 참조 0회. 속도 최상.
- **[[직접 주소 지정]] (Direct)**: 주소부에 실제 데이터가 저장된 메모리 주소가 있음. 참조 1회.
- **[[간접 주소 지정]] (Indirect)**: 주소부에 주소의 주소가 저장됨. 참조 2회 이상. 주소 길이에 제약받지 않음.
- **[[계산에 의한 주소 지정]]**: 레지스터 값과 주소부를 연산(더하기 등)하여 유효 주소를 결정. (인덱스 레지스터, 베이스 레지스터, PC 등 활용)

### 4. 레지스터 (Register)
CPU 내부의 고속 임시 저장 장치입니다.
- **물리적 구성**: **[[플립플롭]]**(Flip-Flop, 동기식) 또는 **[[래치]]**(Latch, 비동기식)의 조합.
- **제어장치(CU) 관련 레지스터**:
    - **[[PC]] (Program Counter)**: 다음에 실행할 명령어의 주소 보관.
    - **[[IR]] (Instruction Register)**: 현재 실행 중인 명령어 보관.
    - **[[MAR]] (Memory Address Register)**: 기억장치 주소를 기억.
    - **[[MBR]] (Memory Buffer Register)**: 기억장치에 저장될/읽어온 데이터 임시 보관.
    - **[[Decoder]] (해독기)**: 명령어를 0과 1의 신호로 변환.
- **연산장치(ALU) 관련 레지스터**:
    - **[[AC]] (Accumulator, 누산기)**: 연산 결과를 일시적으로 저장.
    - **[[가산기]] (Adder)**: 덧셈 수행.
    - **[[보수기]] (Complementer)**: 뺄셈을 위해 보수로 변환.
    - **[[상태 레지스터]] (Status Register)**: 연산 결과의 상태(부호, Zero, Overflow 등) 저장.

---

> [!INFO] 💡 AI 보충 설명: 시스템 버스 (System Bus)
> 레지스터와 외부 장치 간의 데이터 이동 통로입니다.
> 1. **주소 버스 (Address Bus)**: CPU가 메모리나 입출력 기기의 주소를 보낼 때 사용 (단방향).
> 2. **데이터 버스 (Data Bus)**: 실제 데이터를 주고받을 때 사용 (양방향).
> 3. **제어 버스 (Control Bus)**: 읽기/쓰기 신호 등 제어 신호를 전달 (양방향).

---

## 🎯 빈출 키워드
- **[[메이저 스테이트]]**: Fetch -> Indirect -> Execute -> Interrupt의 흐름.
- **[[PC]] (Program Counter)**: 다음 실행 주소를 가리키는 가장 중요한 레지스터.
- **[[누산기]] (AC)**: 1-주소 명령어 및 산술 연산의 핵심 저장소.
- **[[간접 주소 지정]]**: 메모리 참조가 2회 이상 발생하며 주소 확장성이 좋음.
- **[[플립플롭]]**: 1비트를 저장할 수 있는 레지스터의 최소 단위 회로.

# 운영체제 및 UNIX/Linux 시스템 심화 정리

## 📌 핵심 요약
운영체제의 성능 평가 지표와 자원 관리 요소를 파악하고, **[[UNIX]]**와 **[[Linux]]**의 핵심 구조인 **[[커널]]**과 **[[쉘]]**의 역할을 구분합니다. 또한 유닉스 파일 시스템의 권한 체계인 **[[chmod]]**의 8진수 계산법을 마스터하여 실제 권한 할당 방식을 이해합니다.

---

## 📝 상세 정리

### 1. 운영체제(OS)의 성능 평가 및 시간 지표
- **성능 평가 요소**: 처리 능력(Throughput)↑, 반환 시간(Turnaround Time)↓, 사용 가능도(Availability)↑, 신뢰도(Reliability)↑.
- **시간 지표 정의**:
    - **[[응답 시간]] (Response Time)**: 명령 입력 후 첫 응답이 나올 때까지의 시간.
    - **[[반환 시간]] (Turnaround Time)**: 작업 제출부터 완료 후 결과 반환까지 걸린 전체 시간 ($실행 시간 + 대기 시간$).
    - **[[대기 시간]] (Waiting Time)**: 프로세스가 준비 큐에서 CPU 할당을 기다린 시간.

### 2. UNIX / Linux 시스템 구조
- **[[커널]] (Kernel)**: 하드웨어를 보호하고 프로세스 관리, 기억장치 관리 등 핵심 기능을 수행하는 운영체제의 심장.
- **[[쉘]] (Shell)**: 사용자의 명령을 해석하여 커널에 전달하는 명령 해석기(Interface).
- **특징**: C언어로 작성되어 이식성이 높고, 계층적 파일 시스템(트리 구조)을 가집니다.



### 3. UNIX 파일 시스템과 아이노드 (inode)
- **부트 블록 (Boot Block)**: 부팅 시 필요한 로더 코드가 저장된 영역.
- **슈퍼 블록 (Super Block)**: 전체 파일 시스템의 크기, 사용 가능한 블록 정보 등 관리 데이터 저장.
- **[[아이노드]] (inode)**: 파일의 소유주, 권한, 크기, 생성 시간, 데이터 블록의 주소 등 **파일의 상세 정보(메타데이터)**를 저장하는 블록.
- **데이터 블록 (Data Block)**: 실제 파일의 내용이나 디렉토리 엔트리가 저장되는 영역.

### 4. 권한 설정 (chmod) 및 8진수 계산
유닉스 권한은 **소유자(User) / 그룹(Group) / 기타 사용자(Other)** 순으로 지정하며, 각 권한은 숫자의 합으로 표기합니다.

- **권한별 수치**:
    - **읽기 (r)**: 4 (2진수 `100`)
    - **쓰기 (w)**: 2 (2진수 `010`)
    - **실행 (x)**: 1 (2진수 `001`)
- **//상세 예시**:
    - `chmod 754 file.txt`
        - **7 (4+2+1)**: 소유자는 읽기, 쓰기, 실행 모두 가능 (`rwx`)
        - **5 (4+0+1)**: 그룹은 읽기, 실행 가능 (`r-x`)
        - **4 (4+0+0)**: 기타 사용자는 읽기만 가능 (`r--`)
    - 결과 표기: `-rwxr-xr--` (맨 앞 `-`는 일반 파일, `d`는 디렉토리 의미)



---

> [!INFO] 💡 AI 보충 설명: OLE (Object Linking and Embedding)
> 윈도우 운영체제의 특징 중 하나인 **[[OLE]]**는 한 애플리케이션에서 작성한 개체(표, 그림 등)를 다른 애플리케이션에 연결하거나 포함하는 기술입니다. 예를 들어, 엑셀 차트를 워드 문서에 넣고 엑셀에서 수정하면 워드에서도 자동으로 업데이트되는 기능이 이에 해당합니다.

---

## 🎯 빈출 키워드
- **[[커널]] (Kernel)**: 유닉스의 핵심 자원 관리 및 하드웨어 제어 담당.
- **[[아이노드]] (inode)**: 파일의 메타데이터(권한, 소유자 등)를 보관하는 장소.
- **[[chmod]]**: 파일이나 디렉토리의 접근 권한을 변경하는 명령어.
- **[[반환 시간]]**: 실행 시간과 대기 시간을 합친 값으로, 짧을수록 성능이 좋음.
- **[[선점형 멀티태스킹]]**: OS가 CPU 제어권을 강제로 뺏어올 수 있는 윈도우/유닉스의 특징.

# 리눅스 쉘 스크립트 및 제어문 상세 정리

## 📌 핵심 요약
사용자의 명령을 해석하여 커널에 전달하는 **[[쉘(Shell)]]**에서 실행되는 스크립트 작성법을 학습합니다. 시스템 운영에 필요한 **[[환경 변수]]** 설정법과 `if`, `case`, `for` 등 리눅스 특유의 제어문 문법을 실무 예제와 함께 정리하여 스크립트 해석 문제에 대비합니다.

---

## 📝 상세 정리

### 1. 쉘 스크립트(Shell Script) 개요
- **정의**: 리눅스/유닉스 명령어를 텍스트 파일에 나열하고 실행 권한을 주어 프로그램처럼 실행하는 파일입니다.
- **특징**: 보통 `.sh` 확장자를 가지며, 첫 줄에 `#!/bin/bash`와 같이 사용할 쉘의 경로를 명시합니다.
- **입출력 명령어**:
    - **read**: 사용자로부터 값을 입력받아 변수에 저장.
    - **echo**: 문자열이나 변수 값(`$변수명`)을 화면에 출력.

### 2. 환경 설정 변수 및 명령어
시스템 전체 또는 특정 사용자 세션의 동작을 제어하기 위해 사용합니다.
- **주요 환경 변수**:
    - `HOME`: 사용자의 홈 디렉토리 경로.
    - `PATH`: 실행 파일을 찾는 디렉토리 경로 리스트.
    - `PWD`: 현재 작업 디렉토리 위치.
    - `SHELL`: 현재 사용 중인 쉘의 종류.
- **설정 명령어**:
    - **env**: 모든 환경 변수를 출력.
    - **export**: 새로운 환경 변수를 등록하거나 하위 쉘로 전달.
    - **set**: 로컬 변수 및 환경 변수를 포함한 모든 변수 출력.

### 3. 쉘 스크립트 조건문 (if / case)
- **if 문**: 조건식은 반드시 대괄호(`[ ]`)와 공백을 지켜야 하며, `fi`로 닫습니다.
    - **//상세 예시**:
      ```bash
      if [ $num -gt 10 ]  # -gt: Greater Than (>)
      then
          echo "10보다 큼"
      else
          echo "10보다 작거나 같음"
      fi
      ```
- **case 문**: 여러 조건 중 하나를 선택하며, `in`과 `esac`으로 감쌉니다.
    - **//상세 예시**:
      ```bash
      case $answer in
          "yes") echo "승인" ;;
          "no")  echo "거절" ;;
      esac
      ```

### 4. 쉘 스크립트 반복문 (for / while / until)
반복문의 끝은 항상 **done** 키워드를 사용합니다.
- **for 문**: 지정된 횟수나 리스트만큼 반복.
    - `for i in 1 2 3` 또는 `for ((i=0; i<3; i++))` 형식 사용.
- **while 문**: 조건이 **참(True)**인 동안 무한 반복.
- **until 문**: 조건이 **참(True)이 될 때까지** (즉, 거짓인 동안) 반복하며, 참이 되면 종료.
- **//상세 예시**:
  ```bash
  # 1부터 5까지 출력하는 while문
  i=1
  while [ $i -le 5 ]  # -le: Less than or Equal (<=)
  do
      echo $i
      i=`expr $i + 1` # 변수 연산 시 expr과 역따옴표(`) 사용
  done
  ```



### 5. 주요 조건식 비교 연산자
| 연산자 | 의미 | 영어 약자 |
| :--- | :--- | :--- |
| **-eq** | 같다 (=) | Equal |
| **-ne** | 같지 않다 (!=) | Not Equal |
| **-gt** | 크다 (>) | Greater Than |
| **-ge** | 크거나 같다 (>=) | Greater or Equal |
| **-lt** | 작다 (<) | Less Than |
| **-le** | 작거나 같다 (<=) | Less or Equal |

---

> [!INFO] 💡 AI 보충 설명: 쉘의 종류
> 리눅스에는 다양한 쉘이 존재합니다.
> 1. **Bourne Shell (sh)**: 가장 기본이 되는 초기 유닉스 쉘.
> 2. **Bash (Bourne Again Shell)**: 리눅스 표준 쉘로 가장 널리 사용됨.
> 3. **C Shell (csh)**: C언어와 유사한 문법을 가진 쉘.
> 4. **Korn Shell (ksh)**: sh와 csh의 장점을 결합한 쉘.

---

## 🎯 빈출 키워드
- **[[환경 변수]]**: 시스템 구동 환경을 저장하며 `echo $변수명`으로 확인 가능.
- **[[fi]] / [[esac]] / [[done]]**: 각 제어문의 종료를 알리는 리눅스 특유의 키워드.
- **[[expr]]**: 스크립트 내에서 산술 연산을 수행하기 위해 사용하는 명령어.
- **[[PATH]]**: 실행 파일의 경로를 지정하는 가장 대표적인 환경 변수.
- **[[until]]**: 조건이 참이 되면 종료하는 독특한 반복문 (while과 반대).
