# 정규 표현식 (Regular Expression)

## 핵심 요약

**정규표현식**은 **문자+기호**로 **패턴**을 만들어 **검색·일치 여부·치환**에 쓴다. **`^` 시작**, **`$` 끝**, **`.` 임의 1글자**, **`[]` 문자집합**, **`*` 0회 이상**, **`+` 1회 이상**. SQL: **REGEXP_LIKE**(일치 여부), **REGEXP_REPLACE**(패턴 치환). **한글**은 **한 음절 = 한 문자**.

## 왜 중요한가

- `LIKE`·`%`보다 **복잡한 형식**(전화번호·이메일 등) 검증에 쓴다.
- **기호 의미**·**이스케이프 `\`** 는 시험 **함정**.
- Oracle 등 **REGEXP_** 함수는 SQLD **활용** 범위.

## 1. 정규표현식이란

| 항목 | 내용 |
|------|------|
| **정의** | **특정 패턴**을 찾거나 **일치 여부**를 확인하기 위해 **문자·기호**를 조합한 표현 |
| **용도** | 형식 검증, 부분 문자열 검색, 치환 |
| **한글** | **한 음절 = 하나의 문자** 단위 |

---

## 2. 정규표현식 기호

| 기호 | 설명 | 예 |
|------|------|-----|
| **`.`** | **임의의 문자 1개** | `a.c` → `abc`, `aic` O / `ac` X |
| **`^`** | **문자열의 시작** | `^010` → **010으로 시작** |
| **`$`** | **문자열의 끝** | `com$` → **com으로 끝** |
| **`*`** | 앞 문자 **0회 이상** 반복 | `ho*` → `h`, `ho`, `hoo` … |
| **`+`** | 앞 문자 **1회 이상** 반복 | `ho+` → `ho`, `hoo` … (`h` 단독 X) |
| **`?`** | 앞 문자 **0 또는 1회** | `ho?` → `h`, `ho` |
| **`[]`** | **문자 집합** | `[abc]` → a,b,c / `[a-z]` → a~z |
| **`[^…]`** | **부정 문자 집합** | `[^abc]` → a,b,c **제외** |
| **`{n}`** | **정확히 n회** | `a{3}` → `aaa` |
| **`{n,m}`** | **n~m회** | `a{2,4}` → `aa`, `aaa`, `aaaa` |
| **`()`** | **그룹** | `(ab)+` → `ab`, `abab` … |
| **`|`** | **OR**(둘 중 하나) | `^ab\|cd$` → **ab로 시작** 또는 **cd로 끝** |
| **`\`** | **이스케이프**(기능 무효) | `a \| b` → 문자 그대로 `a\|b` |

### 자주 쓰는 조합

| 패턴 | 의미 |
|------|------|
| `^[a-z]+[0-9]+$` | **영문 소문자** 뒤 **숫자**만 |
| `^010` | 010 **시작** (휴대폰 등) |
| `com$` | **com 끝** (도메인 등) |

---

## 3. 정규표현식 함수 (SQL)

| 함수 | 역할 |
|------|------|
| **REGEXP_LIKE(문자열, 패턴)** | 패턴 **일치 여부** (TRUE/FALSE) |
| **REGEXP_REPLACE(문자열, 패턴, 대체)** | 일치 부분 **치환** |

### REGEXP_LIKE

```sql
REGEXP_LIKE('hello123', '^[a-z]+[0-9]+$');
-- → TRUE (소문자 + 숫자 형식)
```

| 항목 | 내용 |
|------|------|
| **문자열** | `'hello123'` |
| **패턴** | `^` 시작 + `[a-z]+` 소문자 1개 이상 + `[0-9]+` 숫자 1개 이상 + `$` 끝 |
| **결과** | 형식 맞음 → **TRUE** |

→ `WHERE REGEXP_LIKE(열, '패턴')` 으로 **조건 필터** 가능.

### REGEXP_REPLACE

```sql
REGEXP_REPLACE('010/1234/5678', '/', '-');
-- → '010-1234-5678'
```

| 항목 | 내용 |
|------|------|
| **대상** | `'010/1234/5678'` |
| **패턴** | `'/'` (슬래시) |
| **대체** | `'-'` |
| **결과** | **`010-1234-5678`** |

---

## 4. LIKE vs 정규표현식

| 구분 | LIKE | 정규표현식 |
|------|------|------------|
| **와일드카드** | `%`, `_` | `.`, `*`, `+`, `[]` … |
| **복잡도** | 단순 | **복잡 패턴** |
| **SQL** | 표준 WHERE | **REGEXP_** (DBMS) |

---

## 5. 시험 포인트 / 함정

| 구분 | 내용 |
|------|------|
| `.` | **1글자** — `*`와 혼동 금지 |
| `^` / `$` | **시작** / **끝** |
| `*` vs `+` | 0회 이상 vs **1회 이상** |
| `[^abc]` | **제외** 집합 |
| `\` | **특수문자 문자 그대로** |
| REGEXP_LIKE | **일치 여부** |
| REGEXP_REPLACE | **치환** |
| 한글 | **음절 1개 = 문자 1개** |
| 함정 | `ho*`는 **`h`만**도 매칭 |
| 함정 | `^`·`$` **중간**에 쓰면 위치 고정 |
