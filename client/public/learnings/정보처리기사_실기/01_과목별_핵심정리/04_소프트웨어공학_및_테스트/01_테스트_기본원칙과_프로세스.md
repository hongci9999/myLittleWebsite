# 테스트 기본원칙과 프로세스

## 핵심 요약

테스트는 결함이 없음을 증명하는 활동이 아니라, 결함이 있음을 찾아내는 활동이다. 실기에서는 테스트의 7대 원칙과 테스트 프로세스, 산출물을 정확히 구분하는 문제가 자주 나온다.

## 1. 테스트의 목적

- 결함 발견
- 품질 향상
- 신뢰도 확보

## 2. 테스트 7대 원칙

- 완전한 테스트는 불가능
- 테스트는 결함이 있음을 보여준다
- 결함 집중
- 살충제 패러독스
- 테스트는 정황 의존
- 오류 부재의 궤변
- 초기 테스트

## 3. 테스트 프로세스

| 단계 | 산출물 예 |
|---|---|
| 계획 | 테스트 계획서 |
| 분석/설계 | 테스트 조건, 시나리오 |
| 케이스 작성 | 테스트 케이스 |
| 수행 | 테스트 결과서 |
| 평가/종료 | 결함 보고, 요약 보고 |

## 4. 예제

- 계획 단계 -> 범위와 일정 결정
- 케이스 작성 단계 -> 입력값, 예상 결과 정의

## 5. 자주 틀리는 함정

- 테스트와 디버깅을 같은 활동으로 본다
- 오류 부재의 궤변 의미를 반대로 이해한다

## 6. 시험장에서 푸는 순서

1. 원칙 문제인지 프로세스 문제인지 먼저 본다.
2. 원칙은 문장 의미로, 프로세스는 순서로 기억한다.
3. 산출물은 단계와 짝지어 외운다.

## 한 페이지 요약

- 테스트는 결함 발견 활동이다.
- 7대 원칙은 단답형으로 자주 출제된다.
- 계획 -> 설계 -> 케이스 -> 수행 -> 평가 흐름을 기억한다.
