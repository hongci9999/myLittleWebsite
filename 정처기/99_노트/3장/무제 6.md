# 분산 설계
## 분산 데이터베이스
물리적으로 분산되어있는 데이터베이스르 단일 데이터베이스로 인식할 수 있도록 논리적으로 통합

점신적으로 확장, 대용향 처리
다른 db 사용가능 신회도 가용성 보장
설계 복잡 비용 증가

## 분산 데이터베리스 관리 시스템
투명성
사용자들이 별도로 알 필요가 없어야 함

분할 투명성
테이블의 분할 구조를 알아야 할 필요가 없어야

위치 투명성
물리적인 위치를 알 필요 없

중복 투명성
중복된 데이터와 저장 위치에 대한 걸 

장애 투명성
장애가 발생해도 데이터 무결성과 트랜잭션의 원자성이 보장되어야

병행 투먕성
다수 트랜잭션이 동시에 수행되도 결과에 일관성

## 분산 데이터베이스 설계 기법
1 테이블 위치 분산
중복되지 않게 분산
2 데이블 분할
완전성  -손실 벗이
재두성 - 원래 테이블로 재두성
상호 중첩 배체 - 칼럼이 중복되지 않아야
3 테이블 할당
비중복 할당 - 단일 노드에만 존재하도록
중복할당 - 노드에 분할 태이블을 중복 항당

## 접근 통제 보안 모델
접근 통제 행영

// 예시

기밀성 모델
무결성보다 기밀성 에 중점
접근 객체에 보안 등급을 분류하여 제약조건 적용

단순보안 규칙 - 주체는 자신보다 높은 등습의객체를 읽을 수 없음
스타 보안 규칙 - 주체ㅐ는 자시보다 낮은 등ㄱ습의 객체에 정보를 쓸 수 없음
강한 스타 보안 구ㅠ치 - 주체는 자신과 등급이 다른 객체에 대하여 읽거나 쓸 수 ㅓㅄ음

무결성 모델
정보의 부당 변경 등을 방지하기 위한 보안 모델
(스타 보안 규칙에서 발생하는 문제)

단순 보안 구칙 - 자신보다 낮은 등급의 객체를 읽을ㅇ 수 없음
스타 무결성 규칙 - 주체는 자신보다 높은 등습의 객체에 정보를 쓸 수 없음

## 접근 통제 정책
신분 기반 정책
개인 또는 그룹의 신분에 근거한 제한 정책
ibp - 단일 사용자가 하나의 객체에 대해 허가를 부여받는 정우
gbp - 복수 사용자가 하나의 객체에 같은 허가를 함께

규칙기반정책
mlp - 사용자 및 객체가 각각부여된  기밀 분류에 따른
cbp - 특정 집단별

역할 기반 정책
개인의 직무에 따라
