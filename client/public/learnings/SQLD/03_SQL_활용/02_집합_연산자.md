# 집합 연산자

## 핵심 요약

**집합 연산자**는 **2개 이상 SELECT** 결과에 **합집합·교집합·차집합**을 수행한다. **스키마 일치** 필수(열 **개수·순서·데이터 타입**). **UNION ALL**: 합치기·**중복 허용**. **UNION**: 합치기·**중복 제거**·**정렬**. **INTERSECT**: **공통 행**만. **MINUS**: **앞 집합 − 뒤 집합**(SQL Server는 **EXCEPT**).

## 왜 중요한가

- 관계대수 **∪·∩·−** 의 SQL 구현이다.
- UNION vs UNION ALL **중복·정렬** 차이는 **빈출**.
- 열 개수·타입 불일치 시 **오류** — 함정 문항.

## 1. 집합 연산자란

| 항목 | 내용 |
|------|------|
| **정의** | **두 개 이상**의 `SELECT` 결과에 대해 **집합 연산**(합·교·차)을 하는 연산자 |
| **대상** | 각 `SELECT`가 반환한 **행 집합** |
| **전제** | 두 집합의 **스키마가 일치**해야 동작 |

### 스키마 일치 조건

| 항목 | 요구 |
|------|------|
| **열 개수** | **같음** |
| **열 순서** | **대응** (1번째↔1번째 …) |
| **데이터 타입** | **호환** (같거나 변환 가능) |

```sql
SELECT 이름, 학번, 전공 FROM 학생1
UNION
SELECT 이름, 학번, 전공 FROM 학생2;
```

→ **SELECT 목록** 구조가 **양쪽 동일**해야 함.

### 연산자와 관계대수·벤 다이어그램

| SQL | 관계대수 | 의미 (시각) |
|-----|----------|-------------|
| **UNION ALL** | ∪ (중복 포함) | **양쪽 원 전체** 채움 |
| **UNION** | ∪ (중복 제거) | **양쪽 합** — **중복 1번** |
| **INTERSECT** | ∩ | **겹치는 부분만** |
| **MINUS** | − | **왼쪽만** (겹침 제외) |

> SQL Server: **MINUS** 대신 **EXCEPT** 사용.

## 2. 예제 테이블

### \<학생1\>

| 이름 | 학번 | 전공 |
|------|------|------|
| 철수 | 21 | 컴퓨터 |
| 영희 | 22 | 경영 |
| 민수 | 24 | 물리 |

### \<학생2\>

| 이름 | 학번 | 전공 |
|------|------|------|
| 영희 | 22 | 경영 |
| 수진 | 19 | 교육 |
| 민수 | 24 | 물리 |

| 공통 행 | 학생1만 | 학생2만 |
|---------|---------|---------|
| 영희, 민수 | **철수** | **수진** |

## 3. UNION ALL (합집합 — 중복 허용)

| 항목 | 내용 |
|------|------|
| **역할** | 두 결과를 **그대로** 이어 붙임 |
| **중복** | **허용** |
| **정렬** | **하지 않음** (일반적으로 **빠름**) |

```sql
SELECT 이름, 학번, 전공 FROM 학생1
UNION ALL
SELECT 이름, 학번, 전공 FROM 학생2;
```

### 결과 (6행)

| 이름 | 학번 | 전공 |
|------|------|------|
| 철수 | 21 | 컴퓨터 |
| 영희 | 22 | 경영 |
| 민수 | 24 | 물리 |
| 영희 | 22 | 경영 |
| 수진 | 19 | 교육 |
| 민수 | 24 | 물리 |

→ **3 + 3 = 6행**. 영희·민수 **중복 포함**.

---

## 4. UNION (합집합 — 중복 제거)

| 항목 | 내용 |
|------|------|
| **역할** | 두 결과를 합치고 **중복 행 제거** |
| **정렬** | 중복 제거 과정에서 **정렬(Sort) 수행** (DBMS·버전에 따라 최적화 가능) |
| **성능** | UNION ALL보다 **부하 클 수 있음** |

```sql
SELECT 이름, 학번, 전공 FROM 학생1
UNION
SELECT 이름, 학번, 전공 FROM 학생2;
```

### 결과 (4행)

| 이름 | 학번 | 전공 |
|------|------|------|
| 민수 | 24 | 물리 |
| 수진 | 19 | 교육 |
| 영희 | 22 | 경영 |
| 철수 | 21 | 컴퓨터 |

→ **유일한 4행**. 교재 예시는 **정렬된** 결과(이름 등 기준).

| UNION ALL | UNION |
|-----------|-------|
| **6행** | **4행** |
| 중복 **유지** | 중복 **제거** |
| 정렬 **없음** | **정렬** 수행(중복 제거용) |

---

## 5. INTERSECT (교집합)

| 항목 | 내용 |
|------|------|
| **역할** | **양쪽 모두**에 있는 **행만** 반환 |
| **비교** | **전체 열 값**이 **완전히 같아야** 동일 행 |

```sql
SELECT 이름, 학번, 전공 FROM 학생1
INTERSECT
SELECT 이름, 학번, 전공 FROM 학생2;
```

### 결과 (2행)

| 이름 | 학번 | 전공 |
|------|------|------|
| 영희 | 22 | 경영 |
| 민수 | 24 | 물리 |

---

## 6. MINUS (차집합)

| 항목 | 내용 |
|------|------|
| **역할** | **앞 SELECT**에는 있고 **뒤 SELECT**에는 **없는** 행 |
| **순서** | **첫 번째 MINUS 두 번째** — **방향** 중요 |
| **DBMS** | Oracle 등 **MINUS** / SQL Server **EXCEPT** |

```sql
SELECT 이름, 학번, 전공 FROM 학생1
MINUS
SELECT 이름, 학번, 전공 FROM 학생2;
```

### 결과 (1행)

| 이름 | 학번 | 전공 |
|------|------|------|
| 철수 | 21 | 컴퓨터 |

→ **학생1에만** 있는 행. (영희·민수는 학생2에도 있어 **제외**)

```
학생1 MINUS 학생2 = { 철수 }
학생2 MINUS 학생1 = { 수진 }
```

## 7. 집합 연산자 비교

| 연산자 | 중복 | 정렬 | 결과 (본 예) |
|--------|------|------|--------------|
| **UNION ALL** | 허용 | X | **6행** |
| **UNION** | 제거 | O (중복 제거용) | **4행** |
| **INTERSECT** | — | — | **2행** (공통) |
| **MINUS** | — | — | **1행** (철수) |

---

## 8. 시험 포인트 / 함정

| 구분 | 내용 |
|------|------|
| 전제 | 열 **개수·순서·타입** 일치 |
| UNION ALL | **중복 허용**, **정렬 안 함** |
| UNION | **중복 제거**, **정렬** 수행 |
| INTERSECT | **공통 행**만 |
| MINUS | **앞 − 뒤**, SQL Server **EXCEPT** |
| 관계대수 | ∪ ∩ − ↔ UNION INTERSECT MINUS |
| 함정 | UNION ALL 행 수 = **n + m** (중복 포함) |
| 함정 | MINUS **순서** 바꾸면 결과 **다름** |
| 함정 | `SELECT *` 열 구조가 다르면 **오류** |
