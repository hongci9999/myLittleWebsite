# 메인 위젯 섹션 UI 디자인 계획

**날짜:** 2026-03-31  
**관련:** [메인 벤토 구조·구현 계획](./2026-03-31-main-bento-widgets-design.md), [0009-design-rules](../decisions/0009-design-rules.md)

**Goal:** `/main` 히어로 아래 **위젯 그리드 영역**의 시각적 언어·위계·리듬을 정해, 구현 시 토큰·컴포넌트 선택이 일관되게 따르도록 한다.

---

## 1. 디자인 컨텍스트 (고정)

| 항목 | 결정 |
| --- | --- |
| **대상** | 본인·방문자 — 포트폴리오·학습·자료 정리 톤 |
| **톤** | **미니멀·프로페셔널**, 콘텐츠 우선 (0009와 동일). 벤토는 “블록 나열”이지 화려한 대시보드 코스프레가 아님 |
| **차별점** | 격자에 **의도적으로 다른 너비**를 준 블록 배치 + 카드 안에서 **제목·본문·보조 정보** 위계가 한눈에 들어오는 것 |
| **제약** | CSS 변수·`BentoCard` 패턴, Noto Sans KR / Consolas. 새 색은 `index.css` 테마에만 |

---

## 2. 레이아웃 리듬 (그리드 컨테이너)

| 토큰 | 권장 값 | 비고 |
| --- | --- | --- |
| 최대 너비 | `max-w-6xl mx-auto` | 0009; 기존 `max-w-4xl`에서 상향해 벤토 여유 확보 |
| 가로 패딩 | `px-4 sm:px-6` | Hero·본문과 동일 계열 |
| 세로 패딩 | `py-6 sm:py-8 md:py-10` | 히어로 직후라 과한 `py-24`는 피하고, 스크롤 첫 화면에 위젯이 보이게 |
| 그리드 gap | `gap-4 md:gap-6` | 0009의 `gap-4`~`gap-8` 안; 너무 벌리면 “한 화면에 한 덩어리” 느낌 약해짐 |
| 열 | `grid-cols-1 md:grid-cols-12` | 모바일 1열 전체 폭; `md+` 에서만 span 의미 |

**읽기 순서:** DOM 순서 = 시각적 중요도 순(위→아래, 좌→우). 스크린 리더·키보드와 맞추기 위해 **장식적 reorder 금지**(CSS `order` 남용 X).

---

## 3. 위젯 카드 패밀리 (공통 UI 문법)

모든 위젯은 **동일한 “카드 문법”**을 따른다. 예외는 `AiDevToolsOverviewWidget`처럼 **이미 확립된 긴 문서형** 블록만(아래 §5).

### 3.1 래퍼

- 기본: **`BentoCard`** (`rounded-2xl`, `border-border/60`, `shadow-sm`, 호버 `shadow-md`).
- 패딩: 콘텐츠 밀도에 따라 `p-4 sm:p-5`(컴팩트) 또는 `p-5 sm:p-6`(여유). **한 페이지 안에서 위젯끼리 패딩 티어를 섞지 않기**(둘 중 하나로 통일).

### 3.2 헤더 블록

- **위젯 제목:** `text-lg font-semibold tracking-tight text-foreground` (기존 FavoriteLinks와 동일 계열).
- **선택 키커:** 한 줄 설명이 필요하면 `text-xs font-semibold uppercase tracking-wider text-muted-foreground` (섹션 키커 패턴 재사용).
- **아이콘:** 선택. 날씨·운세 등 **기능 아이콘 1개**는 `text-muted-foreground` + `size-5` 정도; 제목 옆 또는 우측 정렬. 아이콘만 덩어리로 쓰지 않기 (0009·미니멀).

### 3.3 본문

- 본문: `text-sm leading-relaxed text-muted-foreground`; 강조 한 줄은 `text-foreground`.
- 링크·CTA: `text-primary`, 호버 `underline` 또는 기존 링크 패턴.

### 3.4 밀도 등급

| 등급 | 용도 | 예 |
| --- | --- | --- |
| **컴팩트** | 날씨, 운세 한 줄, 숫자+라벨 | 패딩 작게, `gap-2`~`3` |
| **리스트** | 추천 링크 | 행 높이 일정, `truncate` + 아이콘 |
| **문서** | AI 도구 개요 | 기존 위젯 유지, 그리드에서는 **항상 넓은 행** |

## 4. 위젯별 UI 방향 (초안)

### 4.1 추천 링크 (`FavoriteLinksWidget`)

- **변경 최소.** 제목·리스트·하단 “더 보기” 구조 유지.
- 그리드에서 **중간 폭 열**(예: `md:col-span-5`) 또는 사이드 열로 두면 “테트리스” 대비가 난다.

### 4.2 AI 개발 도구 개요 (`AiDevToolsOverviewWidget`)

- **내부 목차·섹션 구조 유지.**
- 그리드: **`md:col-span-12`(전체 폭)** 권장 — 높이·가독성 때문에 작은 칸에 넣지 않음.
- 카드 스타일이 `BentoCard`가 아니라면, Phase 1에서 **시각만** `BentoCard`와 맞출지(테두리·radius) 결정 — 맞추면 “한 패밀리”, 유지하면 “특별 블록”으로 의도 표현 가능.

### 4.3 날씨 (신규)

- **한 눈에:** 위치명(또는 “서울”) + 큰 온도 숫자 + 단위 + 짧은 상태 텍스트(맑음 등).
- **보조:** 작은 아이콘 또는 이모지 대신 **SVG/아이콘 라이브러리** 한 종만 사용해 톤 통일.
- **로딩:** 카드 높이 유지하는 **스켈레톤**(제목 줄 + 큰 사각 2개).
- **에러:** `text-muted-foreground` 한 줄 + “다시 시도”는 `Button` variant `ghost` 또는 텍스트 링크.

### 4.4 오늘의 운세 (신규)

- **짧게:** 제목 “오늘의 운세” + 본문 2~4문장 또는 한 줄 요약 + 선택적 럭키 키워드 배지(`rounded-full border border-border/60 px-2 py-0.5 text-xs`).
- 운세 특유의 **과한 그라데이션·네온**은 피하고 primary/muted만으로 위계.

## 5. 상태·모션·접근성

| 주제 | 계획 |
| --- | --- |
| **로딩** | 위젯마다 다른 스피너 대신 **동일 스켈레톤 패턴**(제목 2줄 + 본문 블록). `animate-pulse` + `bg-muted/40` 수준 |
| **빈 데이터** | 한 줄 메시지 + 필요 시 `outline` 버튼으로 설정 이동 |
| **모션** | 카드 호버는 `BentoCard` 기본(`shadow-md`)에 맡김. 진입 시 **전체 페이지 스태거**는 선택; `prefers-reduced-motion` 존중 |
| **포커스** | 그리드 내 링크·버튼은 **보이는 `focus-visible:ring`** (기존 shadcn/토큰) |
| **대비** | 새 UI 요소는 라이트/다크 모드에서 `foreground` / `muted-foreground` 조합으로 검증 |

## 6. 구현 체크리스트 (UI만)

- [ ] `MainPage` 컨테이너: `max-w-6xl`, gap, 패딩이 §2와 일치
- [ ] 위젯 공통: 제목·본문 클래스가 §3.2~3.3과 일치
- [ ] 날씨·운세: §4.3~4.4 밀도·에러·스켈레톤 반영
- [ ] AI 개요: 전체 폭 배치로 줄바꿈·가독성 문제 없음
- [ ] 0009 위반 요소(하드코드 색, 카드 안 카드 남용) 없음
