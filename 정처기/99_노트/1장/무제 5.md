모듈 품질 개선

모듈 내부 요소들이 얼마나 밀접한지
독립성을 높이도록

기능적 응집도
모든 요소들이 하나의 문제를 해결하기 위해

순차적 응집도
출력 결과를 다른 모듈의 입력값으로 사용하는 경우

통신적 응집도
출력 결과를 이용하여 서로 다른 기능을 수행하는 경우

절차적 응집도
하나의 문제를 해결하기 위해 여러 모듈이 순차적으로 수행

시간적
특정 시기에 함께 수행되어야 

논리적
유사한 성격이나 형태를 가진 기능을 하나의 모듈에서 수행되도록

우연적 응집고
아무 관연 없는 경우
다시 진행해야


모듈 결합도
모듈 간 관련성이 얼마나 깊은지 의조ㅓㄴ적인지
약할수록 모듈의 독립성이 높아짐(중요)

아래 요소들 ai 예시 추가

자료의 결합도
인수와 매개변수를 통해서만 상호작용
인수- 입력값
매개변수  -  전달받은 값

스탬프 결합도
동일한 자료 구조 공유
특정 모듈에서 자료 구조를 변화시켰다면 관련 있는 다른 모듈에 그 영향

제어 결합도
값 뿐 아니라 제어 요소를 함꼐 전달하는 경우
전달되는 제어 요소에 따라 대상 모듈의 처리절차가 달라짐

외부 결합도
인수 정달 없이 다른 모듈의 내부 데이터를 참조

공유
모듈 외부에 선언된 변수를 참조하여 

내용
다른 모듈의내부 기능과 데이터를 직접적으로 사용하는 경우
가장 좋지 않음


복잡도
공유도 fan-in : 호출하는 모듈의 수
공통 모듈화 츨면에서는 좋음, 실패할 경우를 대비해 중접관리, 더 많은 테스트 필요
제어도  fan-out : 자신이 호출하는 호듈의 수
더 단순화 시킬 필요 있음
