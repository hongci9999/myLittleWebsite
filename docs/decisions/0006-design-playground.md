# 디자인 플레이그라운드 도입

- **날짜**: 2026-02-26
- **상태**: 채택

## 배경 (왜 이 결정이 필요한가)

디자인 시스템(shadcn/ui) 결정 후, 실제 사이트에 적용할 폰트·색상·컴포넌트 스타일을 구체적으로 정해야 했다. 여러 옵션을 한 화면에서 비교하고 결정할 수 있는 도구가 필요했다.

## 결정 (무엇으로 했는지)

**디자인 플레이그라운드** 페이지(`/design-playground`)를 추가한다.

- 폰트, 색상 테마, 모서리·그림자·트랜지션, 각 컴포넌트 스타일을 실시간으로 선택·미리보기
- 선택 결과를 마크다운으로 복사해 AI에게 전달하면 index.css·컴포넌트에 반영 가능
- About 페이지에서 진입점 제공

## 이유 (다른 선택지를 배제한 이유)

- **임시 페이지**: 결정 후 삭제 가능. foundation의 "확장성"에 맞게, 결정 과정을 지원하는 도구로 설계
- **복사 → AI 반영**: 직접 코드 수정보다, 선택 결과를 문서화해 AI가 일괄 반영하는 흐름이 효율적
- **유형별 선택지**: 단일선택/다중선택/토글을 유형별로 독립 섹션으로 분리해, 각각 다른 스타일을 동시에 결정 가능

## 결과/참고

- `client/src/pages/DesignPlaygroundPage.tsx` - 메인 구현
- `client/src/pages/design-playground.css` - 플레이그라운드 전용 스타일 (전역과 분리)
- 사이트 소개 페이지(/about)에 "디자인 플레이그라운드" 링크
- learnings/0007: 사용법·구성 상세

### 스타일 독립성

플레이그라운드는 `design-playground-root` 클래스로 전역 스타일(body 폰트·크기, code 폰트) 상속을 차단한다. 폰트·색상·타이포는 플레이그라운드 내부 선택값만 적용된다.
