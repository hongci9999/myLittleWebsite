# 메인 위젯 섹션 리디자인 계획 (구조 강조)

**날짜:** 2026-04-01  
**관련:** [메인 위젯 섹션 UI 디자인 계획](./2026-03-31-main-widget-section-ui-design.md), [0009-design-rules](../decisions/0009-design-rules.md)

**Goal:** `/main` 위젯 섹션을 더 세련되게 보이도록, 화려함보다 **벤토 구조 리듬 + 정보 위계 + 상태 일관성**을 강화한다.

---

## 1) 방향성

- 선택된 방향: **구조 강조(B)**  
- 원칙: 미니멀/프로페셔널을 유지하고, 레이아웃과 타이포/간격으로 완성도를 올린다.
- 현재 범위: 메인 섹션에서 실제 표시 중인 `FavoriteLinksWidget` 중심 개선.

---

## 2) 접근안 비교

### 안 A. 레이아웃 중심 개선 (추천)
- 컨테이너, 그리드, span 규칙 고정
- 카드 내부는 최소 수정
- 장점: 빠르고 안정적, 기존 구조와 충돌 적음

### 안 B. 카드 패밀리 통합 동시 개선
- 안 A + 카드 내부 상태/포커스/디테일까지 동시 수정
- 장점: 체감 완성도 높음
- 단점: 수정 범위 확장

### 안 C. 시각 포인트 확장
- 안 B + 컬러/모션 강조
- 장점: 변화 체감 큼
- 단점: 0009의 절제된 톤과 충돌 가능

**이번 적용:** 안 A+B 경량 결합 (레이아웃 유지 + 링크 위젯 디테일 개선)

---

## 3) 구현 범위

1. `MainPage` / `WidgetGrid` 토큰 유지  
   - `max-w-6xl`, `px-4 sm:px-6`, `py-6 sm:py-8 md:py-10`
   - `grid-cols-1 md:grid-cols-12`, `gap-4 md:gap-6`

2. 링크 위젯 상태 표현 개선  
   - 텍스트 로딩 대신 스켈레톤(높이 유지)
   - 빈 상태 메시지 톤 통일

3. 링크 아이템 상호작용 개선  
   - `focus-visible` 링 추가
   - 행 높이/아이콘 정렬 미세 조정

---

## 4) 검증 기준

- 빌드: `client`에서 `npm run build` 성공
- 반응형: `sm`, `md`, `lg`에서 카드 간격/텍스트 줄바꿈 이상 없음
- 접근성: 키보드 탭 이동 시 링크 포커스가 시각적으로 명확함

---

## 5) 후속 후보

- 날씨/운세 위젯 재추가 시 span 규칙 재배치 (`4~6` + `12` 조합)
- 공통 `WidgetSkeleton` 컴포넌트 분리 여부 검토
