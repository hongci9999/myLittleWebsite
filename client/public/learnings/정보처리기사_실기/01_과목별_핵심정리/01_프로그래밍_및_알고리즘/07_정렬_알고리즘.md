# 정렬 알고리즘

## 핵심 요약

정렬 문제는 "어떤 알고리즘이 더 빠른가"보다도 "각 단계에서 배열이 어떻게 바뀌는가"를 읽어내는 문제가 더 자주 나온다. 따라서 정렬의 원리, 비교 방식, 교환 시점, 안정성 여부를 함께 알아야 한다.

## 왜 중요한가

- 버블/선택/삽입 정렬은 시험에서 가장 자주 다뤄지는 기본 정렬이다.
- 퀵 정렬은 분할 정복의 대표 예로 개념형 문제에 자주 등장한다.
- 정렬 문제는 코드 추적형, 비교형, 복잡도형으로 모두 출제 가능하다.

## 1. 정렬 알고리즘 비교표

| 알고리즘 | 평균 시간복잡도 | 안정성 | 핵심 특징 |
|---|---|---|---|
| 버블 정렬 | `O(n^2)` | 안정 | 인접 원소 비교 후 교환 |
| 선택 정렬 | `O(n^2)` | 불안정 | 최솟값 선택 후 앞으로 이동 |
| 삽입 정렬 | `O(n^2)` | 안정 | 이미 정렬된 부분에 삽입 |
| 퀵 정렬 | 평균 `O(n log n)` | 불안정 | 피벗 기준 분할 |

## 2. 버블 정렬

- 인접한 두 원소를 비교해 큰 값을 뒤로 보낸다.
- 1회전이 끝나면 가장 큰 값이 맨 뒤로 이동한다.

```text
[5, 3, 2]
-> [3, 5, 2]
-> [3, 2, 5]
```

## 3. 선택 정렬

- 가장 작은 값을 찾아 맨 앞으로 보낸다.
- 1회전이 끝나면 첫 번째 자리가 확정된다.

```text
[4, 2, 3, 1]
-> 최소값 1 선택
-> [1, 2, 3, 4]
```

## 4. 삽입 정렬

- 앞부분은 정렬돼 있다고 가정하고, 새 원소를 올바른 위치에 꽂아 넣는다.
- 거의 정렬된 데이터에서 효율이 좋다.

## 5. 퀵 정렬

- 피벗을 하나 정한다.
- 피벗보다 작은 값은 왼쪽, 큰 값은 오른쪽으로 분할한다.
- 각 부분을 다시 같은 방식으로 정렬한다.

## 6. 단계별 추적 예제

### 예제 1: 버블 정렬 1회전

배열: `[4, 2, 5, 1]`

1. `4`와 `2` 비교 -> 교환 -> `[2, 4, 5, 1]`
2. `4`와 `5` 비교 -> 유지 -> `[2, 4, 5, 1]`
3. `5`와 `1` 비교 -> 교환 -> `[2, 4, 1, 5]`

1회전 후 가장 큰 값 `5`가 뒤로 이동했다.

### 예제 2: 선택 정렬 1회전

배열: `[4, 2, 5, 1]`

1. 전체에서 최솟값 `1` 탐색
2. 첫 번째 원소 `4`와 교환
3. 결과: `[1, 2, 5, 4]`

### 예제 3: 삽입 정렬 흐름

배열: `[5, 3, 4]`

1. `5`는 이미 정렬
2. `3`을 앞에 삽입 -> `[3, 5, 4]`
3. `4`를 `3`과 `5` 사이에 삽입 -> `[3, 4, 5]`

## 7. 자주 틀리는 함정

### 7.1 버블 정렬과 선택 정렬 혼동

- 버블 정렬은 "인접 비교"
- 선택 정렬은 "최솟값 선택"

### 7.2 1회전 의미를 잘못 이해

- 버블 정렬 1회전 후: 가장 큰 값이 뒤로
- 선택 정렬 1회전 후: 가장 작은 값이 앞으로

### 7.3 안정성 개념 혼동

- 안정 정렬은 같은 키의 기존 순서가 유지된다.
- 버블/삽입은 안정, 선택/퀵은 일반적으로 불안정으로 본다.

## 8. 시험장에서 푸는 순서

1. 정렬 이름을 먼저 확인한다.
2. 1회전에서 어떤 값이 확정되는지 생각한다.
3. 필요하면 배열 상태를 회차별로 적는다.
4. 복잡도와 안정성을 함께 묶어서 기억한다.
5. 버블/선택/삽입 차이를 말로 설명할 수 있어야 한다.

## 한 페이지 요약

- 버블: 인접 비교, 큰 값이 뒤로 간다.
- 선택: 최솟값 선택, 작은 값이 앞으로 온다.
- 삽입: 정렬된 구간에 끼워 넣는다.
- 퀵: 피벗 기준 분할 정복이다.
- 정렬 문제는 1회전 후 배열 상태를 추적할 수 있어야 한다.
