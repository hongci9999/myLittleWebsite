# 함수, 뷰, 트랜잭션 체크포인트

## 핵심 요약

시험 직전에는 함수 이름과 뷰 특징, 트랜잭션 제어 명령만 빠르게 훑어도 점수를 지킬 수 있다. 이 문서는 자주 나오는 함수와 뷰 옵션, COMMIT/ROLLBACK 흐름만 압축 정리한다.

## 자주 나오는 함수

- LENGTH = 길이
- SUBSTRING = 부분 문자열
- UPPER / LOWER = 대소문자 변환
- ABS = 절댓값
- MOD = 나머지
- ROUND = 반올림

## 뷰 핵심

- 가상 테이블
- 복잡한 조회 단순화
- 보안성 향상
- `WITH CHECK OPTION` -> 뷰 조건 위반 갱신 방지

## 트랜잭션 핵심

- COMMIT = 변경 확정
- ROLLBACK = 변경 취소
- SAVEPOINT = 중간 복구 지점

## 함정

- 뷰를 실제 저장 테이블로 오해한다
- COMMIT 후에도 ROLLBACK 가능하다고 착각한다

## 한 페이지 요약

- 함수는 이름과 결과를 짝으로 외운다.
- 뷰는 가상 테이블, COMMIT은 확정, ROLLBACK은 취소다.
