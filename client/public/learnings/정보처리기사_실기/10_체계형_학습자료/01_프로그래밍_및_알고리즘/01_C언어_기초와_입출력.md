# C언어 기초와 입출력

## 핵심 요약

C언어는 실기에서 "짧은 코드가 실제로 어떻게 실행되는가"를 묻는 대표 언어다. 따라서 문법 자체보다도 자료형, 서식 지정자, 형 변환, 입력/출력 형식, 증감 연산이 결합될 때 결과가 어떻게 달라지는지를 정확히 이해해야 한다.

## 왜 중요한가

- 실기에서 C언어 문제는 보통 5~10줄 내외의 코드 결과 예측으로 출제된다.
- 단순 문법 암기보다 `printf`, `scanf`, 형 변환, 증감 연산이 함께 나오는 상황을 해석하는 능력이 중요하다.
- 특히 서식 지정자와 자료형 불일치, 정수 나눗셈, 문자 코드값, 전치/후치 증감은 고전적인 함정이다.

## 1. C언어의 기본 구조

- 프로그램은 보통 `main()` 함수에서 시작한다.
- 명령문은 세미콜론 `;`으로 끝난다.
- 블록은 중괄호 `{}`로 묶는다.
- 대소문자를 구분한다.

```c
#include <stdio.h>

int main() {
    printf("Hello C\n");
    return 0;
}
```

## 2. 자료형과 상수/변수

### 2.1 자료형

| 자료형 | 의미 | 대표 용도 |
|---|---|---|
| `int` | 정수 | 계산, 반복 횟수 |
| `char` | 문자 1개 | 문자/ASCII 코드 |
| `float` | 실수 | 소수 포함 계산 |
| `double` | 더 정밀한 실수 | 실수 정밀 계산 |

### 2.2 상수 표현

- 10진수: `10`
- 8진수: `012`
- 16진수: `0xA`
- 문자 상수: `'A'`
- 문자열 상수: `"ABC"`

### 2.3 변수 선언과 초기화

- 변수는 사용 전 선언해야 한다.
- 초기화하지 않으면 쓰레기값이 들어갈 수 있다.

```c
int a = 10;
char ch = 'B';
float rate = 3.5f;
```

## 3. 문자와 ASCII 코드

- `char`는 문자이면서 동시에 정수 코드값으로도 해석된다.
- `'A'`는 65, `'a'`는 97이다.
- 따라서 문자끼리의 덧셈/뺄셈이 코드값 계산으로 이어질 수 있다.

```c
char ch = 'A';
printf("%d\n", ch);      // 65
printf("%c\n", ch + 2);  // C
```

## 4. 서식 지정자와 표준 입출력

### 4.1 자주 쓰는 서식 지정자

| 서식 | 의미 |
|---|---|
| `%d` | 10진 정수 |
| `%f` | 실수 |
| `%c` | 문자 1개 |
| `%s` | 문자열 |
| `%x` | 16진수 |
| `%o` | 8진수 |

### 4.2 출력 함수

- `printf()` : 서식에 맞게 출력
- `putchar()` : 문자 1개 출력
- `puts()` : 문자열 출력 후 줄바꿈

### 4.3 입력 함수

- `scanf()` : 서식에 맞춰 입력
- `getchar()` : 문자 1개 입력

```c
int a;
scanf("%d", &a);   // 정수 a의 주소에 입력값 저장
printf("%d\n", a);
```

## 5. 형 변환과 정수 나눗셈

- `int / int` 결과는 `int`다.
- 즉, 소수점 이하는 버려진다.
- 실수 결과가 필요하면 한쪽을 실수형으로 바꿔야 한다.

```c
printf("%d\n", 5 / 2);         // 2
printf("%.1f\n", 5 / 2.0);     // 2.5
printf("%.1f\n", (float)5 / 2); // 2.5
```

## 6. 코드 추적 예제

### 예제 1: 문자와 정수의 결합

```c
int a = 3;
char ch = 'A';
printf("%d %c\n", a + 2, ch + a);
```

- `a + 2` -> `5`
- `ch + a` -> `'A' + 3` -> `65 + 3 = 68`
- ASCII 68은 `'D'`
- 최종 출력: `5 D`

### 예제 2: 정수 나눗셈 함정

```c
int a = 7, b = 2;
printf("%d\n", a / b);
printf("%.1f\n", a / b);
printf("%.1f\n", (float)a / b);
```

- 첫 줄: `7 / 2` -> `3`
- 둘째 줄: 계산은 여전히 `int / int` -> `3`, 출력만 실수 형식 -> `3.0`
- 셋째 줄: `(float)a / b` -> `3.5`

## 7. 자주 틀리는 함정

### 7.1 `scanf`에서 `&` 누락

- `scanf("%d", a);` 는 잘못된 형태다.
- 변수의 주소를 넘겨야 하므로 `scanf("%d", &a);` 가 맞다.

### 7.2 `%f`와 `%d` 혼동

- `%d`는 정수, `%f`는 실수다.
- 서식과 실제 자료형이 다르면 출력 결과가 틀어질 수 있다.

### 7.3 정수 나눗셈 후 실수 출력

- `printf("%.1f", 5 / 2);` 는 `2.5`가 아니라 `2.0`으로 생각해야 한다.
- 계산이 먼저 끝난 뒤 출력 형식이 적용되기 때문이다.

## 8. 시험장에서 푸는 순서

1. 변수 선언과 초기값을 먼저 적는다.
2. 자료형이 `int`인지 `float`인지 확인한다.
3. 문자 상수는 필요하면 ASCII 코드로 바꾼다.
4. `printf`의 서식 지정자와 실제 계산 결과를 분리해서 본다.
5. 정수 나눗셈 여부를 마지막에 한 번 더 체크한다.

## 한 페이지 요약

- C언어 기초 문제의 핵심은 자료형과 서식 지정자다.
- `char`는 문자이면서 정수 코드값으로 계산될 수 있다.
- `scanf`는 주소를 넘겨야 하므로 보통 `&변수` 형태를 쓴다.
- `int / int`는 언제나 정수 결과다.
- 출력 형식이 바뀌어도 계산 방식이 자동으로 바뀌지는 않는다.
