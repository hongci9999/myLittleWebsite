# 네트워크 프로토콜과 포트

## 핵심 요약

실기에서 프로토콜 문제는 "무슨 역할을 하는가"와 "몇 번 포트를 쓰는가"를 함께 묻는 경우가 많다. 메일, 웹, 원격 접속, 파일 전송, 이름 해석 계열로 묶어 정리하면 암기가 편하다.

## 1. 주요 프로토콜

| 프로토콜 | 역할 | 기본 포트 |
|---|---|---|
| HTTP | 웹 문서 전송 | 80 |
| HTTPS | 암호화 웹 통신 | 443 |
| FTP | 파일 전송 | 21 |
| SSH | 암호화 원격 접속 | 22 |
| Telnet | 평문 원격 접속 | 23 |
| SMTP | 메일 송신 | 25 |
| DNS | 도메인 이름 해석 | 53 |
| POP3 | 메일 수신/다운로드 | 110 |
| IMAP | 서버 동기화 메일 수신 | 143 |

## 2. 메일 프로토콜

- SMTP: 메일 보내기
- POP3: 메일을 내려받아 로컬 중심으로 사용
- IMAP: 서버와 동기화하며 관리

## 3. 원격 접속/파일 전송

- Telnet: 평문, 보안 취약
- SSH: 암호화된 안전한 접속
- FTP: 파일 업로드/다운로드
- TFTP: 단순 파일 전송, UDP 기반

## 4. 네트워크 운영 프로토콜

- DNS: 도메인 이름을 IP 주소로 변환
- DHCP: 동적 IP 할당
- SNMP: 네트워크 장비 관리
- ICMP: 오류 보고와 상태 확인

## 5. 예제

### 예제 1: 보안 원격 접속

- "보안이 필요한 원격 로그인" -> SSH

### 예제 2: 메일 송수신 구분

- 메일 전송 -> SMTP
- 메일 수신 동기화 -> IMAP

## 6. 자주 틀리는 함정

- SMTP를 메일 수신 프로토콜로 착각한다
- Telnet과 SSH 차이를 보안 관점에서 놓친다
- POP3와 IMAP의 서버 동기화 여부를 혼동한다
- DNS를 파일 전송 프로토콜로 잘못 기억한다

## 7. 시험장에서 푸는 순서

1. 웹/메일/원격접속/파일전송/이름해석 중 어느 계열인지 분류한다.
2. 역할을 먼저 떠올리고 포트를 연결한다.
3. 보안 키워드가 나오면 HTTPS, SSH를 우선 본다.

## 한 페이지 요약

- HTTP 80, HTTPS 443, FTP 21, SSH 22, Telnet 23, SMTP 25, DNS 53은 기본 암기다.
- SMTP는 송신, POP3/IMAP는 수신이다.
- SSH는 Telnet의 보안형이라고 이해하면 빠르다.
