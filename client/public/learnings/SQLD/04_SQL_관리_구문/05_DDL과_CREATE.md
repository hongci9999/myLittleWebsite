# DDL과 CREATE

## 핵심 요약

**DDL**(데이터 정의언어)은 **데이터 구조(스키마)** — DB **설계도** — 를 정의한다. **CREATE**는 **새 테이블**을 만든다. **PRIMARY KEY**는 **개체 무결성**, **FOREIGN KEY**는 **참조 무결성**을 보장한다. **NOT NULL·UNIQUE·CHECK**로 열 값을 제한한다. **CHAR**는 고정 길이·탐색에, **VARCHAR**는 가변 길이·저장 효율에 유리하다.

## 왜 중요한가

- 3과목에서 **스키마·제약·타입**은 DML·뷰의 **전제**다.
- **PK vs UNIQUE**, **FK REFERENCES** 는 시험 **빈출**.
- Oracle **DDL = 암시적 COMMIT**: [04_TCL과_트랜잭션](./04_TCL과_트랜잭션.md)

## 1. DDL (Data Definition Language)

| 항목 | 내용 |
|------|------|
| **한글** | 데이터 **정의**언어 |
| **역할** | 데이터의 **구조(스키마)** 정의 — DB **설계도** |
| **대표 구문** | **CREATE**, **ALTER**, **DROP**, **TRUNCATE**, **RENAME** … |

| 분류 | 역할 |
|------|------|
| **DDL** | **구조** 정의·변경 |
| **DML** | **데이터(행)** 조작 |
| **TCL** | 트랜잭션 제어 |

## 2. CREATE — 테이블 생성

| 항목 | 내용 |
|------|------|
| **역할** | **새 테이블** 생성 |
| **형태** | `CREATE TABLE 테이블명 ( 열정의 … )` |

```sql
CREATE TABLE 학생 (
    학번   INT          PRIMARY KEY,
    이름   VARCHAR(10)  NOT NULL,
    입학일 DATE,
    주소   VARCHAR(100) UNIQUE,
    학과   VARCHAR(10)  NOT NULL,
    성별   VARCHAR(1)   NOT NULL
             CHECK (성별 IN ('M', 'F')),
    FOREIGN KEY (학과) REFERENCES 학과리스트 (학과)
);
```

---

## 3. 열 정의 요소

| 요소 | 예 | 설명 |
|------|-----|------|
| **데이터 타입** | `INT`, `VARCHAR(10)`, `DATE` | 열에 저장할 **값의 형** |
| **PRIMARY KEY** | `학번 … PRIMARY KEY` | **기본키(PK)** — 행 **유일 식별** |
| **FOREIGN KEY** | `FOREIGN KEY (학과) REFERENCES …` | **외래키(FK)** — 다른 테이블 **참조** |
| **NOT NULL** | `이름 … NOT NULL` | **NULL 불가** |
| **UNIQUE** | `주소 … UNIQUE` | 열 값 **중복 불가** (NULL은 **허용** 가능) |
| **CHECK** | `CHECK (성별 IN ('M','F'))` | 값 **범위·조건** 제한 |

### 무결성

| 제약 | 보장하는 무결성 |
|------|-----------------|
| **PRIMARY KEY** | **개체 무결성** — 각 행을 **유일**하게 식별 |
| **FOREIGN KEY** | **참조 무결성** — 참조하는 값이 **부모 테이블에 존재** |

### PK vs UNIQUE

| 구분 | PRIMARY KEY | UNIQUE |
|------|-------------|--------|
| **중복** | 불가 | 불가 |
| **NULL** | **불가**(일반적) | **허용** (DBMS·설정) |
| **테이블당 개수** | **1개** (복합 PK 1세트) | **여러 개** 가능 |
| **역할** | 행 **식별** | **대체 후보키** 성격 |

## 4. 데이터 타입

| 유형 | 설명 |
|------|------|
| **CHAR(n)** | **고정 길이** 문자열 — n자 미만이면 **공백**으로 채움 |
| **VARCHAR(n)** | **가변 길이** 문자열 — **최대 n자**까지 저장 |
| **NUMBER** | 숫자 — **INT·FLOAT** 등으로 세분 가능(DBMS) |
| **DATE** | **날짜** (시간 포함 여부는 DBMS) |

### CHAR vs VARCHAR

| 구분 | CHAR | VARCHAR |
|------|------|---------|
| **길이** | **고정** n | **가변**, 최대 n |
| **저장** | 짧아도 n만큼 공간 | 실제 길이만큼 — **공간 효율** |
| **용도** | 코드·고정 코드값, **탐색** 유리한 경우 | 이름·주소 등 **길이 가변** |

> 교재: **VARCHAR** — 저장 공간 효율 / **CHAR** — 탐색 측면 유리.

## 5. CREATE TABLE 구조 정리

```text
CREATE TABLE 테이블명 (
    컬럼명   데이터타입  [제약조건 …],
    …
    [테이블 수준 제약: PRIMARY KEY, FOREIGN KEY, CHECK …]
);
```

| 순서(권장 이해) | 내용 |
|----------------|------|
| 1 | **열 이름** + **타입** |
| 2 | 열 **제약** (NOT NULL, UNIQUE, CHECK …) |
| 3 | **PK** (열 또는 복합) |
| 4 | **FK** — `REFERENCES 부모테이블(열)` |

## 6. 시험 포인트 / 함정

| 구분 | 내용 |
|------|------|
| DDL | **스키마(구조)** 정의 |
| CREATE | **테이블 생성** |
| PK | **개체 무결성**, NULL 불가, 테이블당 1 |
| FK | **참조 무결성**, `REFERENCES` |
| UNIQUE | 중복 불가, **NULL 가능**(PK와 차이) |
| CHECK | **IN**, 범위 조건 |
| CHAR vs VARCHAR | 고정/가변, 효율 vs 탐색 |
| 함정 | FK 열 값이 **부모에 없으면** INSERT·UPDATE 실패 |
| 함정 | Oracle **DDL 후 암시적 COMMIT** |
