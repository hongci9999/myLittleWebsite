# 객체지향 기술
## 객체지향 방법론
련실ㅇ의 개체들을 디지털 세계로 대응하여 표현
속성 - 객체에 포함되는 자료구조, 상태값 등
메소드 - 속성에 대한 연산 및 고유유수행 기능

## 구성요소
클래스
타입을 정의하고 구현하는 틀
공통된 특성을 추상화한 단위
속성과 매소드를 정의한 것

객체
클래스에 의해 구현된 대상들
고유한 속성을 가짐 클래스에서 정의한 메소드 수행 가능

인스턴스
특정 클래스에 의해 구현된 객체

메세지
객체간 통신을 위한 인터페이스

메소드
연산

## 객체지향 기술
캡슐화
문제 해결에 필요한 속성과 메소드를 하나로 묶는 것
인터페이스가단순해지고 재사용 용이

정보은닉
외부로부터 감투는 것
외부의 영햐으로부터 보호

추상화
공통된 요소들을 추출하여 상위 클래스로 구현

상속
상위 클래스의 멤버(속성과 메소드)를 하위 클래스에서 물려받도록 하는 것

다형성
상속된 여러 하위 객체들이 서로 다른 형태를 가질 수 있게 하는 성질
오버로딩- 동일한 이름의 여러 메소드 중 매개변수로 전달되는 인수의 타입과 개수를 식별하여 적절한 메소드를 호출해 주는 기능
오버라이딩-  상속받은 메소드의 내부 기능을 새롭게 정의하는 기능
둘 이상의 클래스에서 동일한 메시지에 대해 서로 다르게 반응할 수 있도록

## 생성패턴
팩토리 메소드
상위 클래스에서는 객체를 생성하기 위한 인터페이스 정의 하위 클래스에서는 어떤 클래스의 인스턴스를 생성할 것인지 결정
객체를 생성하여 반완하는 패소드를 팩토리 메소드라 하는데 이 팩토리 메소드를 오버라이딩 하여 객체를 반환하는 패턴
//알기 쉽게 설명

앱스트랙트 팩토리
관련 있는 서브클래스를 묶어서 팩토리 클래스를 만들고 조건에 따라 객체를 생성하는 패턴
객체 생성 코드가 상위클래스에 존재하여 생성된 객체를 하위 클래스가 받아서 사용
다수의 클래스를 하나의 추상 클래스로 묶어서 관리
//알기쉽게 설명

## 구조패턴
어댑터
서로 다른 인터페이스를 가진 클래스들을 함꼐 사용할 수 있도록

브릿지
복잡하게 설계된 클래스를 기능부와 구현부로 분리한 뒤 구 클래스를 연결
결합도 낮아짐 독립적으로

컴포자이트
객체들의 관계를 트리구조로 구성하여 단일 객체와 복합 객체를 동일하게 다루도록
다수의 클래스를 하나의 클래스오

데코레이터

facade

플라이웨이트

프록시
