# Python 핵심문법과 객체지향

## 핵심 요약

Python은 문법이 간결해서 쉬워 보이지만, 실기에서는 리스트 슬라이싱, 반복문, 함수, 클래스의 동작 결과를 정확히 읽어내는 문제가 자주 나온다. 따라서 "짧은 코드의 실행 결과"를 빠르게 추적하는 연습이 중요하다.

## 왜 중요한가

- Python 문제는 코드 길이는 짧지만 함축이 강해서 방심하면 틀리기 쉽다.
- 슬라이싱, 리스트/딕셔너리, 컴프리헨션은 대표 출제 포인트다.
- 객체지향 파트에서는 `self`, `__init__`, 상속, 오버라이딩이 핵심이다.

## 1. 기본 자료형

| 자료형 | 예시 |
|---|---|
| 정수 | `10` |
| 실수 | `3.14` |
| 문자열 | `"abc"` |
| 리스트 | `[1, 2, 3]` |
| 튜플 | `(1, 2, 3)` |
| 딕셔너리 | `{"a": 1, "b": 2}` |

## 2. 슬라이싱

슬라이싱은 `시작:끝:간격` 형식이다.

```python
arr = [10, 20, 30, 40, 50]
print(arr[1:4])   # [20, 30, 40]
print(arr[:3])    # [10, 20, 30]
print(arr[::2])   # [10, 30, 50]
```

- 시작 인덱스 포함
- 끝 인덱스 미포함
- 간격 생략 가능

## 3. 반복문과 함수

```python
sum_val = 0
for i in range(1, 5):
    sum_val += i
print(sum_val)  # 10
```

- `range(1, 5)` 는 1, 2, 3, 4
- 끝값은 포함되지 않는다.

함수는 `def` 로 정의한다.

```python
def add(a, b):
    return a + b
```

## 4. 리스트 컴프리헨션

```python
nums = [1, 2, 3, 4]
result = [x * 2 for x in nums if x % 2 == 0]
print(result)  # [4, 8]
```

- `for` 와 `if` 를 한 줄에 압축한 형태다.
- 가독성이 좋지만 조건 순서를 잘못 읽으면 실수하기 쉽다.

## 5. 클래스와 `self`

- `self` 는 인스턴스 자기 자신을 뜻한다.
- 메서드 정의 시 첫 번째 매개변수로 써야 한다.

```python
class Student:
    def __init__(self, name):
        self.name = name

    def show(self):
        print(self.name)
```

## 6. 상속과 오버라이딩

```python
class Parent:
    def show(self):
        print("Parent")

class Child(Parent):
    def show(self):
        print("Child")
```

`Child`는 `Parent`를 상속하고 `show()`를 재정의한다.

## 7. 코드 추적 예제

### 예제 1: 슬라이싱

```python
arr = [1, 2, 3, 4, 5, 6]
print(arr[2:5])
print(arr[:4])
print(arr[1::2])
```

- `arr[2:5]` -> `[3, 4, 5]`
- `arr[:4]` -> `[1, 2, 3, 4]`
- `arr[1::2]` -> 인덱스 1, 3, 5 -> `[2, 4, 6]`

### 예제 2: 클래스와 메서드 호출

```python
class A:
    def __init__(self):
        self.x = 10

    def show(self):
        print(self.x)

a = A()
a.show()
```

- 객체 생성 시 `__init__` 자동 호출
- `self.x = 10`
- `show()` 에서 `10` 출력

### 예제 3: 오버라이딩

```python
class Parent:
    def show(self):
        print("P")

class Child(Parent):
    def show(self):
        print("C")

obj = Child()
obj.show()
```

- 자식 클래스가 부모 메서드 재정의
- 출력: `C`

## 8. 자주 틀리는 함정

### 8.1 슬라이싱 끝 인덱스 포함으로 착각

- `arr[1:4]` 는 1,2,3까지만 포함한다.

### 8.2 `range` 끝값 포함으로 착각

- `range(1, 5)` 는 5를 포함하지 않는다.

### 8.3 `self` 생략

- 클래스 메서드 정의 시 첫 번째 인자로 `self`가 필요하다.

### 8.4 리스트와 튜플 혼동

- 리스트는 수정 가능
- 튜플은 수정 불가

## 9. 시험장에서 푸는 순서

1. 리스트와 슬라이싱 구간을 먼저 적는다.
2. `range`의 시작값과 끝값 포함 여부를 확인한다.
3. 클래스 문제는 `__init__` 호출 여부부터 본다.
4. 상속이 있으면 오버라이딩 여부를 확인한다.
5. 한 줄짜리 Python 문장은 압축을 풀어서 읽는다.

## 한 페이지 요약

- Python은 짧은 코드의 실행 결과 예측이 핵심이다.
- 슬라이싱은 시작 포함, 끝 미포함이다.
- `range`도 끝값을 포함하지 않는다.
- `self` 와 `__init__` 는 객체지향 문제의 기본 축이다.
- 컴프리헨션은 조건과 반복 순서를 풀어서 읽어야 한다.
