# 논리 개체 상세화 및 데이터 모델링

## 📌 핵심 요약
현실 세계의 데이터를 추상화하여 [[개체(Entity)]], [[속성(Attribute)]], [[관계(Relationship)]]로 표현하는 데이터 모델링 과정을 다룹니다. 특히 식별자의 분류와 논리 데이터 모델링의 정규화 과정을 통한 데이터 무결성 확보를 핵심으로 합니다.

---

## 📝 상세 정리

### 1. 논리 개체 정의
#### 1.1 개체 (Entity)
- 현실 세계의 식별 가능한 대상을 디지털화하기 위해 추상화하여 표현한 단위입니다.
- 하나의 **[[개체 타입(Entity Type)]]**에 의해 여러 개의 개체가 생성되며, 각 개체는 독립성을 가집니다.
- **개체 타입(Entity Schema):** 개체명과 개체 속성을 정의하여 명세한 것입니다.
- **개체 인스턴스(Entity Instance):** 특정 개체 타입에 의해 생성된 개체만을 호칭할 때 사용합니다.
- 명명 규칙: 약어 사용을 지양하고, 단수 명사를 사용합니다.

#### 1.2 속성 (Attribute)
- 개체 고유의 특성을 나타내며 개체의 성격을 구체화하는 가장 작은 논리적 단위입니다.
- **분류:**
    - **기본 속성(Basic Attribute):** 업무 분석을 통해 정의된 속성입니다.
    - **설계 속성(Designed Attribute):** 설계 과정에서 새로 만들거나 기존 데이터를 변형하여 정의한 속성입니다.
    - **파생 속성(Derived Attribute):** 다른 속성의 영향을 받아 계산되거나 정의되는 속성입니다 (예: 점수, 등급).
- **역할에 따른 분류:**
    - **기본키 속성(Primary Key Attribute):** 개체를 유일하게 식별할 수 있는 속성입니다.
    - **외래키 속성(Foreign Key Attribute):** 다른 개체와 관계를 가지는 속성입니다.
- **원시 속성:** 제거할 경우 데이터를 재현할 수 없는 필수 속성을 의미합니다.

#### 1.3 관계 스키마 (Relation Schema)
- 속성들의 논리적 구조를 의미하며 표(Table) 형태로 표현됩니다.
- 다수의 [[릴레이션(Relation)]]으로 분할하여 중복을 감소시키고 독립적인 관계성을 가집니다.
- **정보 무손실 원칙:** 분리된 릴레이션들을 조인하면 원래의 정보가 그대로 복원되어야 합니다.

#### 1.4 식별자 (Identifier)
- 개체를 유일하게 구분하는 속성으로 키(Key)와 같은 개념입니다. 불변성 및 존재성(Not Null)을 만족해야 합니다.
- **분류:**
    - **주 식별자(Primary Identifier):** 물리 모델의 [[기본키(PK)]]에 해당합니다.
    - **보조 식별자(Alternate Identifier):** 주 식별자 대신 개체를 식별할 수 있는 속성으로, [[고유 인덱스(Unique Index)]]로 관리됩니다.
    - **내부 식별자:** 자신의 개체 타입 내에서 스스로 생성되는 식별자입니다.
    - **외부 식별자:** 타 개체로부터 상속받은 식별자로 [[외래키(FK)]]에 해당합니다.
    - **단일 식별자:** 속성이 하나인 식별자입니다.
    - **복합 식별자(Composite Identifier):** 둘 이상의 속성으로 구성된 식별자로 [[슈퍼키(Super Key)]]의 성격을 가집니다.
    - **원조 식별자:** 가공되지 않은 원래의 식별자입니다.
    - **대리 식별자(Surrogate Identifier):** 복합 식별자를 단순화하기 위해 인위적으로 생성한 식별자입니다 (예: 일련번호).

### 2. 데이터 모델링 (Data Modeling)
기업의 정보 구조를 논리적인 데이터 모델로 변환하여 구체화하는 과정입니다.

#### 2.1 데이터 모델의 표시 요소
1. **논리적 구조 (Structure):** 개체들 간의 관계를 정의합니다.
2. **연산 (Operation):** 데이터 처리 방법을 정의합니다.
3. **제약 조건 (Constraint):** 데이터의 정확성을 유지하기 위한 규칙입니다.

#### 2.2 모델링 진행 단계
`개념 데이터 모델링` → `논리 데이터 모델링` → `물리 데이터 모델링` → `DB 구축`

- **[[개념 데이터 모델링(Conceptual Data Modeling)]]:** 상위 수준의 골격 설계 단계입니다. 엔티티 타입, 관계 도출, 주요 업무 기능 정의를 포함합니다.
- **[[논리 데이터 모델링(Logical Data Modeling)]]:** 업무 규칙을 구체적으로 표현합니다. 관계형, 계층형, 네트워크형 모델로 매핑하며 모든 데이터를 [[정규화(Normalization)]]하여 표현합니다.

---

### 3. 논리 개체의 관계 정의
#### 3.1 관계 (Relationship)의 종류
- **존재에 의한 관계:** 학교-학생과 같이 정적인 상태의 연결입니다.
- **행위에 의한 관계:** 고객-주문과 같이 동적인 이벤트에 의한 연결입니다.
- **카디널리티 (Cardinality):** $1:1$, $1:N$, $N:M$ 등의 수치로 표현하여 [[참조 무결성]]을 보장합니다.

#### 3.2 다대다($N:M$) 관계의 해소
- 다대다 관계는 논리적 불안정성을 초래하며 직접 구현이 불가능합니다.
- 반드시 **[[교차 엔티티(Intersection Entity)]]**를 도입하여 일대다($1:N$) 관계로 분해해야 합니다.

#### 3.3 종속 관계에 따른 분류
- **[[식별 관계(Identifying Relationship)]]:** 부모 개체의 주 식별자가 자식 개체의 주 식별자로 상속되는 경우입니다.
- **[[비식별 관계(Non-Identifying Relationship)]]:** 부모 개체의 주 식별자가 자식 개체의 일반 속성(외래키)으로 상속되는 경우입니다.

> [!INFO] 💡 AI 보충 설명
> **참조 무결성(Referential Integrity):** 외래키 값은 참조하는 테이블의 기본키 값과 동일하거나 NULL이어야 한다는 원칙입니다. 데이터 모델링 시 관계 정의를 통해 이 무결성을 설계 단계에서 확립합니다.

## 🎯 빈출 키워드
- **[[식별자(Identifier)]]**: 개체 내에서 인스턴스들을 유일하게 구분할 수 있는 속성입니다. 주/보조, 내부/외부, 단일/복합, 원조/대리 식별자로 세분화됩니다.
- **[[데이터 모델(Data Model)]]**: 현실 세계의 정보를 컴퓨터가 이해할 수 있도록 추상화하여 표현한 모델로, 구성 요소는 개체, 속성, 관계입니다.
- **[[관계(Relationship)]]**: 개체와 개체 사이의 논리적인 연결을 의미하며, 카디널리티(1:1, 1:N, N:M)와 참여도(필수, 선택)를 가집니다.
