# 도메인 연장 반영이 +1년이 아닌 +3개월로 계산됨

날짜: 2026-07-19

## 발생한 오류

메인 상단 도메인 만료 알림에서 관리자가 「오늘 연장 반영」을 눌렀더니, 실제 도메인 만료일(`2027-08-16`, 1년 단위 연장)과 달리 만료일이 **2026년 10월 19일**로 표시됐다.

## 원인

- `renewSiteDomainSettings`(`server/src/db/queries/site-domain.ts`)는 **"등록일=오늘, 만료일=오늘+(기존 등록~만료 기간)"** 으로 계산한다.
- 그런데 저장/폴백 기본값이 등록 `2026-05-16` → 만료 `2026-08-16`, 즉 **3개월(약 92일)** 짜리로 남아 있었다.
- 그래서 오늘(2026-07-19) 연장 시 `2026-07-19 + 92일 = 2026-10-19` 가 되어, +1년이 아니라 +3개월이 반영됐다.

## 수정 방법

실제 도메인은 만료 `2027-08-16`, **연장 시 (누른 날 기준) +1년** 모델이다. 연장 로직이 "기존 기간"을 재사용하므로, **등록~만료 기간을 365일로** 맞추면 이후 연장이 +1년으로 동작한다. 등록일을 만료 1년 전(`2026-08-16`)으로 정정.

- `server/src/config/site-domain-defaults.ts` — 폴백 값 `2026-08-16` → `2027-08-16`
- `client/src/shared/config/site-domain.ts` — 폴백 값 동일하게 수정
- `docs/plans/2026-05-19-site-domain-settings.sql` — seed INSERT 값 동기화 + 현행화 UPDATE 주석

DB의 기존 행(이미 2026-10-19 등 잘못된 값)은 Supabase SQL Editor에서 아래로 정정:

```sql
UPDATE site_domain_settings
SET registered_date = '2026-08-16',
    expiry_date     = '2027-08-16',
    updated_at      = now()
WHERE id = 1;
```

## 결과/참고

- 이후 「오늘 연장 반영」은 오늘+365일(약 +1년)로 동작.
- 관련 결정: [decisions/0019](../decisions/0019-site-domain-expiry-notice.md)
- 남은 개선점: 연장 로직이 "만료일 기준"이 아니라 "오늘 기준"으로 리셋되므로, 만료 전에 미리 누르면 그만큼 앞당겨진다. 만료일 기준 +1년이 필요하면 `renewSiteDomainSettings`를 별도 개선해야 한다.
