# Java 클래스 설계 및 변수/메서드 심화

## 📌 핵심 요약
자바 프로그래밍의 근간인 **[[클래스]]**의 상세 구조를 파악하고, 메모리 적재 시점에 따른 **[[인스턴스 변수]]**와 **[[클래스 변수]]**의 동작 차이를 완벽히 이해합니다. 또한, 객체 초기화의 핵심인 **[[생성자]]**와 다형성의 기초인 **[[오버로딩]]**을 상세 예시와 함께 정리합니다.

---

## 📝 상세 정리

### 1. 클래스(Class)의 내부 구조와 접근 제어
- **구성 요소 상세**:
    - **멤버 변수(Member Variable)**: 객체의 속성을 정의합니다. (예: 자동차의 색상, 현재 속도)
    - **멤버 메서드(Member Method)**: 객체의 행위를 정의합니다. (예: 가속하기, 브레이크 밟기)
- **접근 제한자(Access Modifier) 심화**:
    - `public`: 패키지에 관계없이 누구나 접근 가능. 오픈 API 등에 사용.
    - `protected`: **상속 관계**가 핵심. 다른 패키지라도 상속받은 자식 클래스는 접근 가능.
    - `default`: 접근 제한자를 명시하지 않았을 때 적용. **같은 패키지**라는 울타리 안에서만 허용.
    - `private`: **캡슐화**의 핵심. 오직 해당 클래스 내부의 메서드를 통해서만 변경 가능.



### 2. 변수의 종류와 메모리 동작 (Instance vs Static)
- **인스턴스 변수 (Instance Variable)**:
    - **생성 시점**: `new` 연산자로 객체가 생성될 때 Heap 메모리에 할당.
    - **특징**: 각 인스턴스마다 독립적인 값을 가짐.
    - **//상세 예시**:
      ```java
      class Card {
          String pattern; // 인스턴스 변수 (하트, 다이아 등 각자 다름)
          int number;     // 인스턴스 변수 (1~13 각자 다름)
      }
      ```
- **클래스 변수 (Static Variable)**:
    - **생성 시점**: 클래스가 메모리에 로드될 때 Data(Static) 영역에 할당.
    - **특징**: 모든 인스턴스가 하나의 저장 공간을 공유. 인스턴스 생성 없이 `클래스명.변수명`으로 접근.
    - **//상세 예시**:
      ```java
      class Card {
          static int width = 100;  // 클래스 변수 (모든 카드의 폭은 동일)
          static int height = 250; // 클래스 변수 (모든 카드의 높이는 동일)
      }
      ```

### 3. 생성자(Constructor)와 초기화
- **정의**: 객체가 생성될 때 단 한 번 호출되는 특수한 메서드.
- **규칙**: 클래스명과 동일해야 하며, 반환 타입(void 포함)이 없음.
- **기본 생성자**: 개발자가 생성자를 하나도 만들지 않으면 컴파일러가 `클래스명() {}`을 자동으로 추가함.
- **//상세 예시**:
  ```java
  class Car {
      String model;
      Car(String modelName) { // 매개변수가 있는 생성자
          this.model = modelName; // this는 현재 생성된 인스턴스 자신을 가리킴
      }
  }
  ```

### 4. 메서드 오버로딩 (Method Overloading)
- **정의**: 한 클래스 내에 이름은 같지만 매개변수의 **개수**나 **타입**이 다른 메서드를 여러 개 정의하는 것.
- **목적**: 유사한 기능을 수행하는 메서드들에게 일관된 이름을 부여하여 가독성 향상.
- **//상세 예시**:
  ```java
  class Calculator {
      int add(int a, int b) { return a + b; }
      double add(double a, double b) { return a + b; } // 타입이 다름
      int add(int a, int b, int c) { return a + b + c; } // 개수가 다름
  }
  ```

---

> [!INFO] 💡 AI 보충 설명: `this` vs `this()`
> 1. **`this`**: 인스턴스 자신을 가리키는 참조 변수입니다. 지역 변수와 멤버 변수의 이름이 같을 때 구분하기 위해 주로 사용합니다. (`this.name = name;`)
> 2. **`this()`**: 같은 클래스 내의 **다른 생성자**를 호출할 때 사용합니다. 반드시 생성자의 첫 줄에서만 호출 가능합니다.

---

## 🎯 빈출 키워드
- **[[캡슐화]] (Encapsulation)**: `private` 변수와 `getter/setter`를 통해 데이터를 보호하는 기술.
- **[[static]]**: 객체 생성 없이 공유 메모리를 사용하는 키워드. 클래스 변수와 클래스 메서드에 사용.
- **[[this]]**: 현재 실행 중인 객체의 주소를 담고 있는 숨겨진 참조 변수.
- **[[오버로딩]]**: 과적하다(Overload)는 의미처럼, 하나의 이름에 여러 함수를 담는 다형성 기법.

# Java 상속과 다형성 및 추상화 심화

## 📌 핵심 요약
기존 클래스를 확장하는 **[[상속]]**의 메커니즘을 파악하고, 부모 타입으로 자식 객체를 다루는 **[[업캐스팅]]**의 특징을 분석합니다. 또한, 강제성을 부여하는 설계도인 **[[추상 클래스]]**를 통해 객체지향의 핵심인 **[[다형성]]**을 구체화하는 방법을 학습합니다.

---

## 📝 상세 정리

### 1. 상속 (Inheritance) 심화
- **정의**: 부모 클래스(Super Class)의 멤버를 자식 클래스(Sub Class)가 물려받아 선언 없이 사용하는 것.
- **특징**:
    - `extends` 키워드 사용. Java는 **단일 상속**만 지원합니다.
    - 자식 객체 생성 시, 내부적으로 부모 객체가 먼저 생성됩니다 (`super()` 호출).
- **//상세 예시**:
  ```java
  class Animal {
      String name;
      void eat() { System.out.println("음식을 먹습니다."); }
  }
  class Dog extends Animal {
      void bark() { System.out.println("멍멍!"); }
  }
  // Dog d = new Dog();
  // d.eat(); // 부모의 메서드 사용 가능
  ```

### 2. 메서드 오버라이딩 (Method Overriding)
- **정의**: 부모로부터 물려받은 메서드의 내용을 자식 클래스에 맞게 **재정의**하는 것.
- **성립 조건**: 메서드 이름, 매개변수, 반환 타입이 부모와 완전히 동일해야 함.
- **//상세 예시**:
  ```java
  class Shape {
      void draw() { System.out.println("도형을 그립니다."); }
  }
  class Circle extends Shape {
      @Override
      void draw() { System.out.println("원형을 그립니다."); } // 내용 변경
  }
  ```

### 3. 업캐스팅(Upcasting)과 동적 바인딩
- **업캐스팅**: 자식 클래스의 객체를 부모 타입의 참조 변수에 할당하는 것.
- **핵심 원리**: 업캐스팅된 변수는 부모의 멤버만 접근 가능하지만, **오버라이딩된 메서드**가 있다면 호출 시 **자식의 메서드가 실행**됩니다. (시험 단골 주제)
- **//상세 예시**:
  ```java
  Shape s = new Circle(); // 업캐스팅
  s.draw(); // "원형을 그립니다." 출력 (자식 클래스에서 재정의된 메서드 수행)
  ```


### 4. 추상 클래스 (Abstract Class)
- **추상 메서드 (`abstract method`)**: 선언부만 있고 구현부(`{}`)가 없는 메서드. 자식 클래스에게 구현을 강제합니다.
- **추상 클래스의 특징**:
    - 하나 이상의 추상 메서드를 포함하면 반드시 추상 클래스로 선언해야 함.
    - **자체적으로 인스턴스 생성이 불가** (`new` 사용 불가).
    - 상속을 통한 확장을 목적으로 하며, 일종의 '미완성 설계도' 역할을 함.
- **//상세 예시**:
  ```java
  abstract class Player {
      abstract void play(); // 추상 메서드: 자식에서 반드시 구현해야 함
      void stop() { System.out.println("정지"); } // 일반 메서드 포함 가능
  }
  class MusicPlayer extends Player {
      void play() { System.out.println("음악 재생"); } // 강제 구현
  }
  ```

---

> [!INFO] 💡 AI 보충 설명: `super` vs `super()`
> 1. **`super`**: 자식 클래스에서 부모 클래스의 멤버(변수/메서드)에 접근할 때 사용합니다. 오버라이딩으로 가려진 부모의 메서드를 호출할 때 유용합니다. (`super.draw();`)
> 2. **`super()`**: 부모 클래스의 **생성자**를 호출합니다. 자식 클래스 생성자의 첫 줄에는 반드시 `super()`가 있어야 하며, 생략 시 컴파일러가 자동으로 추가합니다.

---

## 🎯 빈출 키워드
- **[[업캐스팅]] (Upcasting)**: 자식 객체를 부모 타입으로 형변환. 다형성 구현의 핵심 단계.
- **[[오버라이딩]] (Overriding)**: 상속받은 메서드를 자식 식에 맞게 덮어쓰기. 실행 시점에 결정됨(**동적 바인딩**).
- **[[추상 클래스]] (Abstract Class)**: 상속을 강제하고 구조를 정의하는 미완성 클래스.
- **[[super]]**: 자식 객체 내에서 부모 객체의 주소를 가리키는 참조 변수.

# Java 인터페이스, 예외 처리 및 스레드 심화

## 📌 핵심 요약
객체 간의 규격을 정의하는 **[[인터페이스]]**의 강력한 다중 상속 기능을 학습하고, 프로그램의 안정성을 보장하는 **[[예외 처리]]** 메커니즘을 마스터합니다. 또한, 시스템 효율을 극대화하는 **[[스레드]]**의 두 가지 구현 방식과 실행 원리를 상세 예시와 함께 정리합니다.

---

## 📝 상세 정리

### 1. 인터페이스 (Interface) 심화
- **정의**: 모든 메서드가 추상 메서드로 이루어진 일종의 '추상 설계도'입니다. (Java 8 이후 default/static 메서드 추가 가능)
- **특징**:
    - **다중 구현**: 클래스는 단일 상속만 가능하지만, 인터페이스는 여러 개를 동시에 구현(`implements`)할 수 있습니다.
    - **상수 필드**: 인터페이스 내 선언된 모든 변수는 자동으로 `public static final`이 됩니다.
- **//상세 예시**:
  ```java
  interface Flyable { void fly(); }
  interface Swimmable { void swim(); }

  // 다중 구현 예시
  class Duck implements Flyable, Swimmable {
      public void fly() { System.out.println("오리가 납니다."); }
      public void swim() { System.out.println("오리가 헤엄칩니다."); }
  }
  ```


### 2. 예외 처리 (Exception Handling) 심화
- **개념**: 프로그램 실행 중 발생하는 예기치 못한 상황(Runtime Error)에 대비하는 코드 구조입니다.
- **구조와 흐름**:
    1. **try**: 예외가 발생할 가능성이 있는 코드를 배치합니다.
    2. **catch**: 발생한 예외 객체를 인자로 받아 처리 로직을 수행합니다. (여러 개 사용 가능)
    3. **finally**: 예외 발생 여부와 상관없이 **무조건 실행**됩니다. 주로 DB 연결 종료, 파일 닫기 등에 사용됩니다.
- **//상세 예시**:
  ```java
  try {
      int[] arr = new int[3];
      arr[5] = 10; // ArrayIndexOutOfBoundsException 발생
  } catch (ArrayIndexOutOfBoundsException e) {
      System.out.println("인덱스 범위를 벗어났습니다.");
  } finally {
      System.out.println("시스템 리소스를 정리합니다.");
  }
  ```

### 3. 스레드 (Thread) - 병렬 처리 심화
- **정의**: 프로세스 내에서 독립적으로 실행되는 작업 단위입니다.
- **구현 방식 1: Thread 클래스 상속**:
    - `Thread` 클래스를 상속받아 `run()` 메서드를 오버라이딩합니다.
    - **//상세 예시**:
      ```java
      class MyThread extends Thread {
          public void run() { System.out.println("스레드 구동!"); }
      }
      MyThread t1 = new MyThread();
      t1.start(); // start()가 내부적으로 run()을 호출함
      ```
- **구현 방식 2: Runnable 인터페이스 구현**:
    - 다른 클래스를 이미 상속받고 있을 때 사용하는 유연한 방식입니다.
    - **//상세 예시**:
      ```java
      class MyTask implements Runnable {
          public void run() { System.out.println("인터페이스 기반 스레드!"); }
      }
      Thread t2 = new Thread(new MyTask()); // Thread 생성자에 전달
      t2.start();
      ```



### 4. 스레드의 주요 메서드 및 특징
- **start() vs run()**: 
    - `start()`: 새로운 호출 스택(Call Stack)을 생성하고 스레드를 독립적으로 실행시킵니다.
    - `run()`: 단순히 클래스에 정의된 메서드를 호출하는 것일 뿐, 새로운 스레드가 생성되지 않습니다.
- **Priority**: 스레드는 우선순위(1~10)를 가질 수 있으며 숫자가 높을수록 더 많은 실행 시간을 할당받을 가능성이 높습니다.

---

> [!INFO] 💡 AI 보충 설명: Checked vs Unchecked Exception
> 1. **Checked Exception**: 컴파일 시점에 체크되는 예외로, 반드시 `try-catch`나 `throws`로 예외 처리를 해야 합니다. (예: `IOException`, `SQLException`)
> 2. **Unchecked Exception**: 실행 시점(Runtime)에 발생하는 예외로, 컴파일러가 강제하지 않습니다. 주로 개발자의 실수로 발생합니다. (예: `NullPointerException`, `ArithmeticException`)

---

## 🎯 빈출 키워드
- **[[인터페이스]] (Interface)**: `implements`를 통한 규격 준수 및 다중 구현.
- **[[finally]]**: 예외 발생과 관계없이 자원 해제를 위해 반드시 실행되는 블록.
- **[[Runnable]]**: 다중 상속이 불가능한 자바에서 스레드를 구현하기 위한 인터페이스.
- **[[start()]]**: 스레드를 독립적인 실행 대기 상태로 만드는 핵심 메서드.
- **[[다중 상속]]**: 자바 클래스에선 불가하나, 인터페이스를 통해 간접적으로 구현 가능한 기능.
