# 정렬 알고리즘 (Sorting Algorithm)

## 핵심 요약
- 데이터를 특정 기준(오름차순 또는 내림차순)에 따라 나열하는 알고리즘으로, 탐색 성능 향상의 기초가 됩니다.
- 정보처리기사 시험에서는 각 정렬 방식의 **작동 원리**, **시간 복잡도($O(n^2)$ vs $O(n \log n)$)**, 그리고 **C/Java 코드 구현** 문제가 자주 출제됩니다.

## 상세 정리

### 1. 정렬의 기본 방향
- **오름차순(Ascending):** 값이 작거나 앞선 순서대로 나열.
- **내림차순(Descending):** 값이 크거나 뒤에 있는 순서대로 나열.

### 2. 주요 정렬 알고리즘별 특징

#### [[선택 정렬]] (Selection Sort)
- **원리:** 기준 위치의 데이터를 나머지 전체 데이터와 비교하여 가장 작은(최소값) 데이터를 찾아 해당 위치와 교환합니다.
- **특징:** - 첫 데이터가 최소값이 되면, 다음 회전에서는 해당 위치를 제외하고 정렬 대상으로 재지정합니다.
    - 시작되는 위치($i$)가 회전마다 1씩 증가합니다.
- **복잡도:** $O(n^2)$

#### [[버블 정렬]] (Bubble Sort)
- **원리:** 인접한 두 데이터를 비교하여 정해진 기준에 따라 위치를 교환합니다.
- **특징:**
    - 한 회전이 끝나면 가장 큰 데이터가 맨 뒤(마지막)로 이동합니다.
    - 매 회전마다 정렬이 완료된 마지막 위치를 제외하므로, 반복 종료 위치가 점차 감소합니다.
- **복잡도:** $O(n^2)$

#### [[삽입 정렬]] (Insertion Sort)
- **원리:** 이미 정렬된 앞부분의 요소들과 비교하여 자신의 위치를 찾아 **삽입**하는 방식입니다.
- **특징:**
    - 두 번째 데이터부터 시작하여 앞쪽 데이터들과 비교합니다.
    - 데이터가 삽입될 때, 기존에 있던 데이터들은 오른쪽으로 밀려납니다(Shift).
- **복잡도:** $O(n^2)$

#### [[쉘 정렬]] (Shell Sort)
- **원리:** 삽입 정렬의 단점(데이터 이동 거리)을 보완한 방식입니다.
- **특징:** - 일정한 **간격(Gap)**을 정해 그룹별로 삽입 정렬을 수행하고, 점차 간격을 줄여가며 최종 정렬합니다.
    - 보편적인 간격은 $n/2$이며, 삽입 정렬보다 효율적입니다.

#### [[힙 정렬]] (Heap Sort)
- **원리:** **완전 이진 트리(Complete Binary Tree)**를 기반으로 하는 힙(Heap) 자료구조를 이용합니다.
- **특징:**
    - 자식 노드와 부모 노드의 크기를 비교하여 자료를 교환합니다.
    - 최대값이나 최소값을 빠르게 찾기에 유리하며, 하향식 힙 생성 방식을 사용합니다.
- **복잡도:** $O(n \log n)$

#### [[병합 정렬]] (Merge Sort, 이진 병합)
- **원리:** 데이터를 더 이상 쪼갤 수 없을 때까지 나눈 뒤, 다시 합치면서 정렬하는 **분할 정복(Divide and Conquer)** 방식입니다.
- **특징:** 두 데이터를 한 쌍으로 병합하고, 정렬된 두 그룹을 다시 한 쌍으로 합칩니다.
- **복잡도:** $O(n \log n)$

#### [[퀵 정렬]] (Quick Sort)
- **원리:** **기준값(Pivot)**을 설정하고 이를 기준으로 작은 값은 왼쪽, 큰 값은 오른쪽 배열로 분할합니다.
- **특징:**
    - 분할 정복 알고리즘의 대표 사례이며, **재귀(순환 호출)**를 이용해 정렬을 수행합니다.
    - 평균적인 정렬 속도가 가장 빠릅니다.
- **복잡도:** 평균 $O(n \log n)$, 최악 $O(n^2)$

#### [[버킷 정렬]] (Bucket Sort)
- **원리:** 데이터 범위를 균등하게 나눈 여러 개의 '버킷'을 생성하여 분산 배치한 후 정렬합니다.
- **특징:** 데이터의 값 범위(분포)를 사전에 파악할 수 있을 때 매우 효율적입니다.

## 3. 빈출 내용 정리
- **@[[선택 정렬]], [[버블 정렬]], [[삽입 정렬]]**: 이 세 가지는 $O(n^2)$ 알고리즘으로, 실제 C언어나 Java 코드로 빈칸 채우기 문제가 자주 나옵니다.
- **회전별 중간 과정**: 특히 버블 정렬과 선택 정렬의 경우, 1회전 또는 2회전 수행 후의 배열 상태를 묻는 문제가 핵심입니다.
- **분할 정복 알고리즘**: 퀵 정렬과 병합 정렬이 이에 해당하며, 대량의 데이터를 정렬할 때 효율적이라는 점이 자주 출제됩니다.
